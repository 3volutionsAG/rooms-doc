Knowledge Base (Deutsch) : Notifikationen standortbezogen erfassen & bearbeiten

Created by Former user, last modified by Former user (Deleted) on Jul 24, 2013

Artikel Nummer

Gültig ab Release

4.2

Verwandte Seiten

Einleitung

Damit im späteren Buchungsprozess Notifikationen (siehe auch Notifikationsmatrix) per Mail verschickt werden (z.B. neue Buchung, Löschen einer Buchung etc.), müssen die Vorlagen dazu erstellt werden. Nebst den standortbezogenen Vorlagen, sind ebenfalls auch Benutzergruppen spezifische Notifikationsvorlagen (Notifikationen Benutzergruppen bezogen erfassen und bearbeiten) vorhanden.

Vorbedingungen

Initial ist ein Standort erfasst

Benötigte Rechte

  • Dar Sektion Einstellungen nutzen
  • Darf Standorte anzeigen/editieren
  • Darf Standorte erstellen/löschen
  • Darf Notifikationen verwalten

Vorgehen / Beschrieb

Navigation zu >Einstellungen>Basisdaten für Ressourcen>Standorte
Mittels Klick auf die Bezeichnung eines Haupt oder Substandortes (siehe auch Standorte erfassen & bearbeiten) erscheinen die Icons zum Hinzufügen von Substandorten, der Bearbeitung, Ansicht und Löschung. Über das Bearbeiten-Symbol und Wechsel in das Registers "Notifikationen" öffnet das modale Popup zur Auswahl der gewünschten Notifikationsvorlage. Die Felder und deren Funktionen sind er unten stehenden Tabelle zu entnehmen:

 

Die Notifikationsvorlagen Reservation - Neu, Reservation - Anpassung, Rerservation - Annullierung, Reservation - Reminder, Reservation - Löschung können zusätzlich pro Ressourcenart separat konfiguriert werden.

Kann eine Notifikationsvorlage nicht mehr ausgewählt werden, da diese bereits hinzugefügt bzw. erfasst wurde, erscheint anstelle des Hand-Cursors ein "Stopsymbol"

 

Beim Aufbereiten des Notifikationsversandes wird im Buchungsprozess geprüft, ob auf dem Standort der gebuchten Ressource oder einem übergeordneten Standort eine entsprechende Notifikationsvorlage vorhanden ist (z.B. Buchung Arbeitsplatz wird eine Suche nach Vorlage Reservation - Neu (Arbeitsplatz) ausgeführt). Falls nein wird die Notifikation Reservation - Neu verschickt.

 

Damit die Bearbeitung des E-Mailbody benutzerfreundlicher wird, kann die Box maximiert und wieder minimiert werden (ab 4.2.4).

Die Notifikationsplatzhalter können nicht nur zur Konfiguration des E-Mailbodys verwendet werden, sondern ebenfalls für den Titel (Subject)

Feld

Funktion

Standardeinstellung

Eigenschaften

Regeln

Notfikationstyp

Bezeichnet die Vorlage z.B. Vorlage für das Bestätigungsmail bei einer neuen Buchung

 

Jeder Notifikationsvorlagen Typ kann nur einmal verwendet werden

Benachrichtigung im CC an

Erfassung von gültigen E-Mail Adressen, damit beim Versenden der Notifikation  ein CC an die entsprechenden Adressaten verschickt wird.

 

 

  • leer = keine CC Benachrichtigung
  • Mehrere E-Mail Adressen werden mittels Semikolon separiert

Sprache

Es stehen alle im System aktivierten Sprachen zur Verfügung

 

 

 

Titel (Subject)

Eingabe eines Sprechenden Titels (Subject) für die Notifikation

 

 

E-Mail Body

Erfassung des Inhalts der E-Mailnachricht. Diese kann mehrsprachig erfasst werden, dazu wird jeweils im Dropdown die gewünschte Sprache gewählt und danach der E-Mail Body eingerichtet. Für die Einstellung des Inhalts stehen unterschiedliche Platzhalter zur Verfügung siehe hierzu Notifikationsplatzhalter.

 

 

  • Wurde kein E-Mail-Server eingetragen (siehe Globale System Parameter konfigurieren), erscheint dieses Feld nicht. Klick auf einen Platzhalter fügt diesen an der aktuellen Cursor Position ein. Ist kein Cursor positioniert wird der Platzhalter am Anfang des Body Textes eingefügt
  • Es ist immer nur eine Platzahlter Sektion gleichzeitig geöffnet.
  • Die Platzhalter übernehmen die Formatierung des Body Editors

= Pflichtfelder
= Daten mehrsprachig erfassen

Nach Erfassung alles optionalen und Pflichtdaten, wird der Vorgang über den Button Speichern abgeschlossen.
