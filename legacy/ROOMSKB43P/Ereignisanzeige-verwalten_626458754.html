Knowledge Base (Deutsch) : Ereignisanzeige verwalten

Created by Former user, last modified by Former user on Apr 29, 2014

Artikel Nummer

Gültig ab Release

4.2

Verwandte Seiten

Einleitung

ROOMS rapportiert alle Probleme in eine interne Logdatei. Diese Logdatei hilft bei der Suche der Ursache.

Vorbedingungen

Ein Fehler wird ausgelöst oder ein Trace (Überwachung) wird initiert.

Benötigte Rechte

  • Darf das Eventlog verwalten

Vorgehen / Beschrieb

Benutzer navigiert auf Einstellungen >System >Ereignisanzeige
Benutzer erfasst optional Suchkriterien und klickt Button Finden

Ein Suchresultat über alle verfügbaren Ereignisse werden in einer Smartlist dargestellt (Spalten: Zeitstempel, Ereignistyp, Meldung, Quelle, Applikationskontext, Benutzername, Servername, ProzessID, ThreadID).

Eine oder mehrere Ereignisse können mit Klick auf Button Entfernen gelöscht werden.

Klick auf "Zeitstempel" zeigt den Inhalt im Anzeigemodus (Viewmode)

Über folgende Werte kann gesucht werden, bzw. werden im Detail angezeigt.

  • Source
    • sucht/zeigt Resultate über einen bestimmten Bereich - z.B. nur den ROOMS Service
  • Event Typ
  • Verfügbare Werte
    • Critical => Fehler auf übergeordneter Ebene, zB konnten die einzelnen Prozesse des WindowsServices gar nicht angestossen werden.
    • Error => Fehler
    • Information => Wie der Name sagt, zB Export gestartet oder Export erfolgreich.
    • Warning => Fehler ohne Auswirkungen oder „verdächtiges“ Ereignis, zB konnte bei der BackSync ein Exchange-Appointment nicht gefunden werden (könnte sein, dass es noch gar nicht auf dem Server ist, muss also kein Fehler sein) oder eine Person hat mehrere zugeteilte OEs (ist zwar ein Fehler, wir können die Ausführung aber fortsetzen).
    • Verbose => Hier kommen zT auch Dev-Informationen wie zB „DataContext initialisiert“, die fürs Debugging hilfreich sein können.
  • Jeder Typ schliesst die übergeordneten immer ein, dh bei Verbose kommt ALLES.
  • Mehrfachselektion erlaubt
  • Zeitstempel
    • Standardwert: nicht selektiert => alle werden angezeigt
    • Standardwert bei Selektion: der aktuelle Tag
  • Context
    • sucht in/zeigt eine bestimmten Code Basis
  • Message
    • schränkt sie Suche auf einen bestimmten Fehlertext ein, bzw. zeigt einen bestimmten Fehlertext
  • User Domain Name
    • sucht in einer bestimmten user domäne, zeigt die user domäne
  • Maschine Name
    • sucht über einen bestimmten Server, zeigt den Server
  • Stack Trace
    • gesamter Fehlerverlauf - wo ist er entstanden
  • Exception
    • Fehlerhafte Sequenz
  • Daten
    • bewusste Datenausgabe, Zusatzinformationen
