Knowledge Base (Deutsch) : Slots definieren & publizieren

Created by Former user, last modified on Jul 05, 2013

Artikel Nummer

Gültig ab Release

4.2

Verwandte Seiten

Einleitung

Mit Slots werden fix definierte Zeiträume zur Buchung angeboten. Damit diese Funktionalität im späteren Buchungsprozess genutzt werden kann, müssen diese Slots definiert und auch publiziert werden.

Vorbedingungen

Mindestens 1 Ressource muss erstellt worden sein mit dem Flag "Nur Slotbuchungen zulässig" (siehe z.B. Ressourcenart Raum erfassen & bearbeiten)

Benötigte Rechte

  • Darf Slot administrieren
  • Darf Sektion Kalender nutzen

Vorgehen / Beschrieb

Slot erfassen

Navigation zu >Kalender

Das Action Popup öffnet und mit dem Button Detailreservation erscheint die Eingabemaske zur Definition des Slots. Die Felder und deren Funktionen sind der unten stehenden Tabelle zu entnehmen. Es werden nur die für die Definition zwingenden Felder erläutert, für alle anderen siehe Handbücher 4.0 - 4.2.

Status

Mittels Statusdropdown wird definiert, ob der Slot definiert oder publiziert ist.

 

  • Slot defniert = der Slot defniert aber noch nicht sicht- und buchbar
  • Slot publiziert = der Slot ist definiert und auch sicht- und buchbar

Zeilennummer im Kalender

Die Zeilennummer gibt an, an welcher Stelle im Kalender auf der entsprechenden Ressource der Slot angezeigt werden soll.

0

 

Wert 0 = die oberste Zeile (nicht die erste, da diese wie bereits erwähnt nur für die Erstellung eines Slots genutzt werden kann)

= Pflichtfelder
= Daten mehrsprachig erfassen

Slot defnieren

Mittels Button Reservation abschliessen werden die getätigten Einstellung gespeichert und die Summarypage wird angezeigt. Nach nochmaliger Kontrolle der Daten wird der Vorgang mit Klick auf den Button Reservation bestätigen abgeschlossen und der Slot ist definiert.


Slot publizieren

Navigation zu >Kalender
Mit dem Mauszeiger wird über den Slot gefahren, damit das Plus-Symbol für die Bearbeitung erscheint. Klick darauf öffnet das Action Popup.



Im Action Popup wird mittels des vorher definierten Workflows (Workflow erfassen & bearbeiten) der Slot vom Status "definiert" auf den Status "publiziert" gesetzt. Ab diesen Zeitpunkt ist er sicht- und buchbar.


Derselbe Vorgang kann auch über den Button Bearbeiten erfolgen. Es wird im Detail mittels Dropdown Status der gewünschte Status ausgewählt und über den Button Reservation abschliessen und danach Reservation bestätigen übernommen.



Um denselben Slot über einen grösseren Zeithorizont zur Verfügung zu stellen, kann dieser Slot als Serie angelegt werden (siehe Handbücher 4.0 - 4.2)

Weitere Regeln

  • Der Kalender wird zweizeilig aufgebeaut.
  • Die erste Zeile ist für die Erstellung eines Slots und für Drag&Drop
  • die zweite Zeile zeigt den effektiven Publikationszeitpunkt
  • Pro zeitgleichliegenden Slot wird eine zusätzliche Zeile hinzugefügt
  • Die Annullationsfunktion und auch die Terminierungsfunktionen Checkout und Freigeben stehen nicht zur Verfügung
  • Fixes Equipment und virtuelle Räume können einer als Slot buchbaren Ressource nur zugeordnet werden wenn sie ebenfalls nur als Slot buchbar sind. Ressourcen die nur als Slot buchbar sind können normalen Ressourcen nicht zugeordnet werden.
  • Mobiles Equipent (d.h. Equipment das erst bei der Buchung durch den Benutzer hinzugefügt wird) muss nicht als Slot buchbar sein. Es gelten die normalen Regeln für Reservationen.
  • Farbwechsel zwischen den Status "Slot definiert" und "Slot publiziert", die Farben werden auf dem Workflow vergeben (Workflow erfassen & bearbeiten)

Navigation zu >Kalender
