Wieso wird kein Equipment angezeigt, dass einem Raum fix zugewiesen werden kann?
Former user (Deleted) posted on Aug 16, 2013

Q: Beim Erstellen eines neuen Raumes wird im Register "Fixes Equipment" über den Button "Equipment hinzufügen" eine leere Liste angezeigt, wieso ist die Auswahl leer?

A: Damit beim Erstellen eines Raumes neben der Equipment Schnellerfassung (siehe Ressourcenart Raum erfassen & bearbeiten Fixes Equipment hinzufügen) auch über den Button "Equipment hinzufügen" eine Auswahl gelistet wird, muss zwingend der Raum gespeichert werden, damit die Berechtigungen übernommen werden und es muss initial auf dem selben Standort ein entsprechendes Equipment erfasst worden sein. Verhalten Filter
Former user (Deleted) posted on Aug 16, 2013

Q: Wie ist das Grundverhalten von ROOMS in Bezug auf die Filtereinstellungen (was passiert wenn die Filtereinstellungen gelöscht werden auf dem eigenen Profil)?

A: ROOMS hat eine Standardeinstellung der Filtersettings, die vom Benutzer beeinflusst werden kann. Unten stehend finden Sie die Ausführungen zum Standard.

Konfiguration: Default-Filtersetting (enthält eine Person-Id oder 0)

Person Settings: StayAtCurrentDate (Rücksprung auf gewähltes / aktuelles Datum), Standart Standort, Ressourcetypen für Plan/Kalender und Reservieren, Standard Kapazität

Erster Start Applikation bzw. nach Löschen der Filtersettings:

  • Wenn ein DefaultFiltersetting definiert ist, wird der Filter übernommen. Sonst wird der ein Reset des Filters gemacht.
  • Reset des Filters bedeutet laden der Standardwerte von den Person Settings (siehe Tabelle).
  • Weil ein DefaultFiltersetting einer Person zugeordnet sein muss, geht das nicht zusammen mit einem globalen Filtersetting.

Warum werden Personen mit Doppelnamen nicht gefunden?
Former user (Deleted) posted on Aug 16, 2013

Q: Die Person mit Vornamen "Peter Alexander" und Nachnamen "Müller" wird nicht gelistet mit dem Suchbegriff "Peter". Wie verhält sich die Suche generell in ROOMS?

A: Der Vorname Peter Alexander wird in der Datenbank in ein einziges Feld gespeichert. Wird eine Suchanfrage formuliert und kein Platzhalter (wildcard; "*") verwendet, muss der Wert der Suchanfrage genau mit dem gespeicherten Wert übereinstimmen:

SuchanfrageGespeicherter WertWird gefunden?
PeterPeter AlexanderNein
Peter AlexanderPeter AlexanderJa
Peter*Peter AlexanderJa
*ander*Peter AlexanderJa
