Knowledge Base (Deutsch) : Fixe Ressourcen der Person zuweisen

Created by Former user, last modified by Former user on Aug 29, 2014

Artikel Nummer

Gültig ab Release

4.2

Verwandte Seiten

Einleitung

Das zuweisen von fixen Ressourcen dient der Langzeit Buchung (z.B. dauerhafter Arbeitsplatz oder Parkplatz).

Eine Langzeitbuchung kann während einer Abwesenheit freigegeben werden. Dies ist in KB Fixe Ressourcen temporär freigeben beschrieben

Vorbedingungen

Person ist im System vorhanden und im Bearbeiten Modus geöffnet (siehe KB Person bearbeiten)

Benötigte Rechte

Keine

Vorgehen / Beschrieb

Benutzer navigiert auf Register "Zugewiesene Ressourcen"

Falls bereits vorhanden, werden in einer Liste die bereits fix zugewiesenen Ressourcen aufgelistet. Folgende Spalten sind verfügbar:

  • Bezeichnung (Ressourcen Bezeichnung in der Sprache des angemeldeten Benutzers)
  • Name (Eindeutige Kennung der Ressource)
  • Ressourcenart (Art der Ressource: Arbeitsplatz, Fahrzeug, etc.)
  • Sortierung (Sortierung der zugewiesenen Ressourcen)
  • Autom. Freigabe (Automatische Freigabe bei Buchungen auf die zugewiesene Ressource)
  • Standort Hierarchisch (Standort der Ressource)
  • Letzte Buchung (bis wie weit in die Zukunft ist die Ressource fix gebucht. Anmerkung: dieser Wert wird täglich automatisch verlängert gemäss Produkt Konfiguration  - siehe KB Konfigurationsparameter)
  • Status (aktiv oder inaktiv)


 

 

Klick auf Button "Entfernen" erlaubt eine oder mehrere Ressourcen aus der Zuweisung zu entfernen. Dabei werden auch alle bereits erstellten Buchungen gelöscht.

Klick auf Button "Ressource hinzufügen" öffnet ein modales Popup. In einer Liste werden alle verfügbaren Ressourcen, welche in der Zukunft noch keine Buchung aufweisen, aufgelistet. Eine oder mehrere Ressourcen können mittels Selektion und Klick auf Button "Ressourcen hinzufügen" der Person zugewiesen werden.

 

Es stehen folgende Spalten zur Ansicht zur Verfügung:

  • RessourcenID (Identitätsname der Ressource)
  • Bezeichnung (Ressourcen Bezeichnung in der Sprache des angemeldeten Benutzers)
  • Ressourcenart (Art der Ressource: Arbeitsplatz, Fahrzeug, etc.)
  • Standort Hierarchisch (Standort der Ressource)
  • Status (aktiv oder inaktiv)
  • Verantwortliche Person (Vorname und Nachname der verantwortlichen Person)
  • Die zugewiesene Ressourcen werden erst gespeichert (und entsprechende Buchungen erstellt), wenn die Person gespeichert wird.
  • Die zugewiesenen Ressourcen sind im Anzeigemodus "Zugewiesene Ressourcen Informationen" ersichtlich.
