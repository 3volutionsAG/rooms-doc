Einleitung

Fehlerseiten werden per Produktstandard zweisprachig (Deutsch und English) und aus unterschiedlichen Gründen ausgegeben.

Vorbedingungen

Fehler in der Applikation, Bedienungsfehler, Error auf Grund von Validierungsregeln etc.

Vorgehen / Beschrieb

401
Falls ein Benutzer eine URL aufruft und keine Berechtigung hat. Dies ist eine zweisprachige Statische Seite:


404
Falls der Benutzer eine Fehleingabe in der URL macht oder die Seite aus einem anderen Grund nicht gefunden wird, soll ein „Seite nicht gefunden – 404 Fehler" ausgegeben werden. Dieser wird 3 Sekunden angezeigt und dann automatisch auf die Startseite des Benutzers weiter navigiert. Als Alternative steht das Home Symbol (Haus) zur Verfügung.


500
Im Fall eines Server Errors (500) wird nachstehende Seite aufgerufen. Die Seite loggt den Error in die dafür vorgesehene Tabelle der ROOMS Datenbank und zeigt dem Benutzer zwei Möglichkeiten:

  1. Navigation auf die zuletzt besuchte Seite
  2. Navigation auf seine Home Seite (Haus Symbol)

    Anmerkung Entwicklung:
    500, 401 und 404 Fehler, die innerhalb der Applikation auftreten, werden "sauber" von einer internen Seite behandelt (500er sind meist "normale" Programmierfehler, 401er gibt's wegen fehlenden Funktionsrechten und 404er wegen ungültigen Links). -> 500er wirds ab und an geben, doch die anderen beiden sind eher unwahrscheinlich bzw. können eigentlich nur auftreten, wenn der Benutzer manuell die Url modifiziert.

    401 Fehler, die ausserhalb der Applikation auftreten (unbekanntes Login), werden auf die gleiche interne Seite gelinkt.

    404 Fehler kann es ausserhalb der Applikation gar nicht geben, weil das MVC Framework jeden Request behandelt.

    -500 Fehler, die ausserhalb der Applikation auftreten (Konfigurationsfehler), werden auf die gleiche interne Seite gelinkt; allerdings ist das Verhalten hier unberechenbar: Ist die Applikation komplett unerreichbar (falsche Konfiguration oder so), dann werden auch unsere Fehlerseiten nicht erreicht -> es kommt die Meldung, dass der Fehler nicht angezeigt werden kann bzw. ein 404er!

    Und schliesslich: Der IIS6 übernimmt unsere Seiten, der IIS7 gibt aber seine eigenen Warnungen aus.

Die aufgeführten Errors werden ins Log (siehe Häufigste Fehler im Log) geschrieben.
