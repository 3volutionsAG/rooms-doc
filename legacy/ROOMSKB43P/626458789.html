Einleitung

Die verfügbaren Rechte können einer Rolle zugeordnet werden. Rechte können mehrere Rollen zugeordnet werden.

Es wird unterschieden zwischen Standortbezogenen Rechten und globalen Rechte.

Die Rollen mit Ihren Rechten werden immer über Standorte oder Ressourcen zugeordnet (siehe KB Standorte erfassen & bearbeiten und Ressourcenart Raum erfassen & bearbeiten)

Vorbedingungen

Rechte wurden durch die Entwicklung implementiert.

Benötigte Rechte

  • Darf Rollen verwalten

Vorgehen / Beschrieb

Benutzer navigert auf >Einstellungen >Sicherheitsdaten für Personen >Rollen.

Suchresultat wird entsprechend den Filterkriterien in der Smartlist angezeigt (Bezeichnung der Rolle):

Mit Selektion einer oder mehreren Rollen und Klick auf Entfernen können die Rollen gelöscht werden.

Klick auf Link Bezeichnung zeigt die Rolle mit allen Rechten im Anzeigemodus (Viewmode)

Klick auf Button Bearbeiten oder in der Smartlist auf Bearbeiten Icon oder Button Neu wechselt in den Bearbeiten Modus:

Benutzer befindet sich auf Register "Stammdaten"

Benutzer klickt auf Register "Rechte". Es werden alle Rechte in einer Smartlist angezeigt (Bezeichnung, Typ, Gültigkeit, Dauer, Einheit)

Die verfügbaren Rechte werden in KB Rechte detailliert beschrieben


Mit Selektion eines oder mehreren Rechte und Klick auf Entfernen können Rechte von der Rolle entfernt werden.

Benutzer klickt auf Button Hinzufügen, um ein neues Recht der Rolle hinzuzufügen.

Ein Modales Popup öffnet sich und zeigt alle der Rolle noch nicht zugweisenen Rechte in einer Smartlist (Bezeichnung, Typ):

Einheit der Berechnung:
Dropdown (Tage / Stunden):

  • Tage= Recht gilt ab Anzahl Tage, wobei ein Tag um 00:00 Startet. Beispiel: Aktuelles Datum/Zeit= 05.03.2010 14:00; Eingabe: ab 1 Tage, hat zur Folge das der Raum erst am 6.3.2010 ab 00:01 gebucht werden kann
  • Stunden= Recht gilt ab sofort + Anzahl Stunden. Beispiel: Aktuelles Datum/Zeit= 05.03.2010 14:00, Eingabe: ab 24 Stunden, hat zur Folge das der Raum erst am 6.3.2010 ab 14:01 gebucht werden kann
