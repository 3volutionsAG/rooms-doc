Einleitung

Im System werden drei Arten von Status unterschieden: der Zustand einer Ressource, der Systemstatus und frei definierbare Status auf Buchungen. Diese Status dienen im Buchungsprozess beispielsweise dazu, dass ein Raum zuerst nur provisorisch besetzt und erst zu einem späteren Zeitpunkt definitv besetzt ist.

Vorgehen / Beschrieb

Status erfassen

Navigation zu >Einstellungen>Basisdaten für Ressourcen>Status

In der Smartlist sind alle vom System fix definierten Haupt-Status (unveränderlich) aufgelistet. Mit Klick auf einen Haupt-Status erscheinen die Icons zur Bearbeitung, Erfassung und dem Viewmode.

Folgende Haupt Reservation-Status sind verfügbar (System/Workflow Status)

  1. Besetzt (System)--> Es wird eine Systembuchung erstellt, die Ressource wird dadurch für den entsprechenden Zeitraum geblockt
  2. Besetzt Provisorisch--> Eine Buchung ist provisorsisch und bleibt in diesem Zustand, bis der Status von einer berechtigten Person geändert wird
  3. Besetzt Temporär--> Eine Buchung ist temporär und wird diese nicht innerhalb einer auf der Ressource eingellten Gültigkeitsdauer (sihe z.B. Ressourcenart Arbeitsplatz erfassen & bearbeiten) in einen endgültigen Status gesetzt wird, löscht das System die Buchung
  4. Besetzt Definitiv--> Eine Buchung ist definitv
  5. Annulliert--> Eine getätigte Buchung wird durch einen Benutzer mit den erforderlichen Rechten annulliert
  6. Slot definiert => Prozessschritt nur in Zusammenhang mit Slots möglich (siehe Slots definieren & publizieren)
  7. Slot publiziert => Prozessschrit nur in Zusammenhang mit Slots möglich (siehe Slots definieren & publizieren)

Status bearbeiten

Navigation zu >Einstellungen>Basisdaten für Ressourcen>Status
In der Smartlist werden alle bereits erfassten Status angezeigt. Mittels Klick auf die Bezeichnung und erscheinen die Icons zur Bearbeitung, Ansicht und Löschung. Über das Ansichts-Symbol wird der Status im Anzeigemodus (Viewmode) dargestellt.


Mittels Button Editieren oder über das Editieren-Symbol wechselt das System in den Bearbeiten Modus. Es können alle Felder verändert werden.

Status löschen

Navigation zu >Einstellungen>Basisdaten für Ressourcen>Status
In der Smartlist werden alle bereits erfassten Status angezeigt. Mittels Klick auf die Bezeichnung und erscheinen die Icons zur Bearbeitung, Ansicht und Löschung. Über das Löschen-Symbol erscheint eine Sicherheitsabfrage, ob der Status wirklich entfernt werden soll. Quittieren dieser Meldung entfernt den Status. Im Action Message wird die Anzahl der entfernten Einträge ausgegeben.



Weitere Regeln zur Löschung:

  • Nur auf Substatus möglich
  • Löscht den aktuellen Status und sämtliche Konfigurationen (Verbindung zu Ressourcentabelle)
  • Hat ein Status eine Abhängigkeit zu einer Ressource, werden die entsprechenden Felder auf der Ressource auf Null gesetzt (StatusId, AlarmStatuswechsel)
  • Hat ein Status eine Abhängigkeit zu einem Workflow-Ereignis (Workflow erfassen & bearbeiten), wird das Ereignis gelöscht.
  • Hat ein Status eine Abhängigkeit zu einer Buchung kann der Status nicht gelöscht werden.
