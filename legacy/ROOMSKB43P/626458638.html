Knowledge Base (Deutsch) : Register Serieninformationen - Erstellung & Änderung Serienbuchung

Created by Former user, last modified by Former user on Jul 29, 2013

Artikel Nummer

Gültig ab Release

4.3

Verwandte Seiten

Einleitung

Die Erstellung und Änderung einer Serie auf Basis einer Einzelbuchung. Bei aktivierter Synchronisation wird die Serie auch ins Outlook übertragen

Vorbedingungen

Achtung: Die Erstellung der Serie kann begrenzt sein. Es kann sein, dass per Konfiguration die maximale Anzahl Serie-Wiederholungen beschränkt ist oder eine Beschränkung auf Zeit vorliegt.
Details für Administratoren können dem KB Artikel Benutzergruppe verwalten entnommen werden. 

Benötigte Rechte

Benutzer: Benötigt Rechte um die gewählte Ressource zu buchen/bearbeiten und auch den Bereich "Serieninformation" zu bearbeiten

Administratoren: Finden einen detaillierten Beschrieb im KB Artikel Rechte.

Vorgehen / Beschrieb

Nach der Navigation auf das Register "Serie" zeigt sich augenfällig ein Warnhinweis. Achtung: nur sichtbar bei aktivierter Synchronisation

Mit Mausklick auf den Warnhinweis verschwindet dieser.

 

 

Im obersten Bereich der Serienerfassung wird der Serie-Start angezeigt und die Möglichkeit die Serie nach einer bestimmten Zeit (Eingabe eines Datums) oder nach bestimmten Anzahl Iterationen (Eingabe einer Zahl) zu beenden.

ROOMS unterstützt keine unbegrenzten Serien - eine Serie muss immer per Datum oder Anzahl Iterationen begrenzt werden

Periodizität

Danach besteht - analog Outlook - die Möglichkeit die Periodizität wie folgt festzulegen:

TäglichWöchentlichMonatlich

Alle X Tage

Eingabe von 1 bucht täglich (1 = Standardwert)
Eingabe einer beliebigen Ganzzahl > 0 möglich 

Wiederholungen alle X Wochen

Eingabe von 1 bucht jede Woche (1 = Standardwert)
Eingabe einer beliebigen Ganzzahl > 0 möglich

Checkboxen für alle 7 Wochentagen können selektiert werden.
Standardmässig ist der Wochentag der Startbuchung selektiert und kann nicht verändert werden. 

Tag X jeden Y Monat

Bucht an einem bestimmten Tag im Monat - Beispielsweise immer am 25 jeden zweiten Monat

Eingabe von 1 bei Y bucht monatlich.  Eingabe einer beliebigen Ganzzahl > 0 möglich
X kann einen Wert zwischen 1 und 31 beinhalten.

An allen Wochentagen

Bucht täglich, aber Wochenenden werden ignoriert

 

Am ERSTEN MONTAG jedes X Monats

Es stehen folgende Werte zur Verfügung: Ersten, zweiten, dritten, vierten, Letzten

Es stehen folgende Tage zur Verfügung: Montag, Dienstag, Mittwoch, Donnerstag, Freitag, Samstag, Sonntag

Eingabe von 1 bei X bucht monatlich.  Eingabe einer beliebigen Ganzzahl > 0 möglich.

Änderung von Serien

Bei Änderungen von Serienpattern (z.B. von Wöchentlich auf Monatlich) wird automatisch die Serie beendet (per letzter Iteration) und eine neue Serie erstellt per aktuell gewählter Buchung.

Beispiel
Es besteht eine tägliche Serie mit 10 Iterationen jeden Tag:

14.7
15.7
16.7
17.7
18.7
19.7
20.7
21.7
22.7
23.7 

Der Benutzer wählt die Buchung vom 17.7 und navigiert ins Detail der Buchung auf das Register "Serieninformationen" und wählt statt "alle 1 Tage" neu "alle 2 Tage".
Es entstehen nun zwei Serien. Die erste vom 14.7. bis 16.7, die zweite ab 17.7 - alle zwei Tage. 

Klick auf "Erstellen"

Nachdem "Erstellen" geklickt wurde, wird überprüft ob eine Serienbeschränk festgelegt worden ist. Falls ja, werden nur die Anzahl der erlaubten Iterationen erstellt.

Während einer kurzen Wartezeit für den Benutzer, erstellt das System für jede Iteration eine Buchung, sofern möglich. Falls die Ressource bereits besetzt ist oder ein anderes Problem vorliegt, wird dies angezeigt. Wir unterscheiden die folgenden Zustände:

    • Grün: alles konnte analog der Hauptbuchung erstellt werden (auch Equipment und Dienstleistungen)
    • Orange: Buchung konnte erstellt werden, aber 
      • es steht nicht alles Equipment zur Verfügung ODER
      • es stehen nicht alle Dienstleistungen zur Verfügung ODER
      • Reservation ist an einem Feiertag
    • Rot: Buchung konnte nicht erstellt werden, weil 
      • Ressource besetzt
      • Ressource gesperrt

Nebst dem Prüfstatus werden noch die Spalten Ressourcenbezeichnung, Von und Bis Datum angezeigt. Zusätzlich steht der Button "Alternative wählen" zur Verfügung 

Klick auf "Alternative wählen"

In einem neuen Fenster besteht die Möglichkeit die Buchung zeitlich (Datum und Zeit) zu verschieben oder in eine andere Ressource zu buchen.

Es werden dabei nur freie Ressourcen der gleichen Art (z.B. Räume) angezeigt

Klick auf "Übernehmen" überträgt die Änderung auf die Serienbuchung.

 

Button "Löschen"

Der Button "Löschen erlaubt zuvor selektierte Termine (Buchungen) aus der Serie zu entfernen.
Beispielsweise, weil diese an einem Feiertag liegen und sowieso nicht genutzt werden.

Button "Ergänzen"

Falls genügend Berechtigung vorhanden, können auch zusätzliche Termine erfasst werden. Diese müssen nicht dem Schema der eingestellten Serienbuchung entsprechen, werden aber als "Ausnahmen" behandelt.
Bei Klick auf Button "Ergänzen" öffnet sich dasselbe Fenster wie bei "Alternative wählen" - es stehen die gleichen Möglichkeiten zur Auswahl zur Verfügung.

 

Änderung auf Serie übernehmen

  • nur sichtbar beim Editieren einer Buchung (nicht bei Neuerstellungen)
  • übernimmt bei Klick alle Änderungen der aktuellen Reservation (Hauptbuchung) auf alle künftigen Serie Items
  • Bei Bedarf wird die Serie aufgesplittet: Eine Aufteilung erfolgt, wenn mindestens eine vorhergehende Reservation existiert und Zeit, Pattern, Ressource, Titel, Organisator, Teilnehmer oder Body-Text geändert und "Auf nachfolgende..." geklickt wird.
  • Ausnahmen (Exeptions) innerhalb einer Serie (andere Ressource oder andere Zeit) unterliegen folgenden Zusatzregeln:
    • Übernahme Catering erfolgt Additiv
    • Bestuhlung wird geändert
    • Teilnehmer Änderungen erfolgen immer auf der gesamten Serie
    • Ressourcenänderungen werden auf die gesamte Serie übernommen
    • Allgemeine Änderungen werden auf die ganze Serie übernommen
