Knowledge Base (Deutsch) : Workflow erfassen & bearbeiten

Created by Former user, last modified by Former user (Deleted) on Jul 31, 2013

Artikel Nummer

Gültig ab Release

4.2

Verwandte Seiten

Einleitung

Bereits definierte Status werden zu einem Workflow zusammengefasst und einer Ressource zugewiesen. Dies dient im späteren Buchungsprozess der Einhaltung von unterschiedlichen Prozessschritten d.h. einer Reihenfolge von Ereignissen (Statuswechsel).

Vorbedingungen

  • Status wurden initial erstellt
  • Benutzergruppen wurden definiert (Benutzergruppe verwalten)

Benötigte Rechte

  • Darf Workflows verwalten
  • Darf Sektion Einstellungen nutzen

Vorgehen / Beschrieb

Workflow erfassen

Navigation zu >Einstellungen>Basisdaten für Ressourcen>Workflows



Mittels Button Erstellen öffnet sich die Eingabemaske im Register "Workflow". Das Feld und dessen Funktion ist der untenstehenden Tabelle zu entnehmen.

Sprechende Bezeichnung des Workflows


 



Im Register "Ereignisse" werden die einzelnen Schritte des Workflows die sog. Ereignisse definiert. Über den Button Hinzufügen öffnet ein Modales Popup zur Einstellung des Ereignisses. Die Felder und deren Funktionen sind der untenstehenden Tabelle zu entnehmen.



Feld

Funktion

Standardeinstellung

Eigenschaften

Regeln

Wechsel von Status

Dropdown zur Auswahl des Ausgangsstatus d.h. aus welchem Status erfolgt ein Wechsel

Neu

Neu = Zustand einer unbebuchten Ressource

Wechsel in Status

Dropdown zur Auswahl des Zielstatus d.h. in welchen Status wird gewechselt. In der Abbildung ist ein Beispiel eines Statuswechsels im Buchungsprozess visualisiert.

 

Wird der Status Temporär ausgewählt muss auf der Ressource im Register "Spezifische Daten" (siehe z.B. Ressourcenart Arbeitsplatz erfassen & bearbeiten) die "Gültigkeit für temporäre Reservationen" eingetragen sein.

Notifikationsgruppe

Dropdown zur Auswahl einer Notifikationsgruppe, die eine E-Mail erhält, sobald im Buchungsprozess ein Statuswechsel erfolgt

leer


  • leer = keine Notifikationen
  • Alle Mitglieder der ausgewählten Gruppe erhalten eine Notifikation beim Statuswechsel (dazu muss die korrekte Notifikationsvorlage Reservation - Statuswechsel, siehe Notifikationen Benutzergruppen bezogen erfassen und bearbeiten, aktiviert sein)
  • Notifikation nur an Ersteller und/oder Organisator kann nur eingestellt werden, wenn eine Gruppe ohne Mitglieder erstellt und ausgewählt wird. Anders kann dies derzeit nicht umgangen werden

Nachricht an Ersteller

Selektieren der Checkbox ermöglicht für den späteren Buchungsprozess die Notifizierung des Buchungserstellers, wenn ein Statuswechsel erfolgt (dazu muss die Notifikationsvorlage Reservation - Anpassung aktiviert sein Notifikationen standortbezogen erfassen & bearbeiten)

nicht selektiert

 
  • nicht selektiert = keine Notifikation an Ersteller
  • Erscheint erst, sobald eine Notifikationsgruppe ausgewählt wurde

Nachricht an Organisator

Selektieren der Checkbox ermöglicht für den späteren Buchungsprozess die Notifizierung des Buchungsorganisators, wenn ein Statuswechsel erfolgt (dazu muss die Notifikationsvorlage Reservation - Anpassung aktiviert sein Notifikationen standortbezogen erfassen & bearbeiten)

nicht selektiert

 
  • nicht selektiert = keine Notifikation an Organisator
  • Erscheint erst, sobald eine Notifikationsgruppe ausgewählt wurde

Sortierung

Angabe eines alphanummerischen Wertes zur Definition der Reihenfolge der Ereignisse während des Buchungsprozesses. So kann beispielsweise gesteuert werden, ob der Prozess im Status Besetzt provisorisch oder Besetzt definitv gestartet werden soll.

0

 

0 = keine Sortierung, System folgt der logischen Reihenfolge der Ereignisse
Wird eine Sortierung vorgenommen, wird gemäss Reihenfolge der entsprechende Status als Basis für die Schnellreservation verwendet. 

= Pflichtfelder
= Daten mehrsprachig erfassen

Doppelte Prozessschritte sind nicht zu empfehlen aber möglich (muss organisatorisch verhindert werden). Dies bedeutet, dass z.b. 2x von „Neu“ in „Besetzt definitv“ definiert werden kann.



Nach Eingabe aller Pflicht- und der gewünschten optionalen Daten wird das Ereignis mittels Button Übernehmen dem Workflow angehängt. Mittels Button Speichern wird der Workflow gesichert und in der Smartlist angezeigt.

Worfkflow bearbeiten

Navigation zu >Einstellungen>Basisdaten für Ressourcen>Workflows
In der Smartlist werden alle bereits erfassten Workflows angezeigt. Mittels Klick auf die Bezeichnung wird der Workflow im Anzeigemodus (Viewmode) dargestellt.


Über den Button Bearbeiten oder das Editieren-Symbol wechselt das System in den Bearbeiten Modus. Die Bezeichnung, sowie die Ereignisse können geändert werden.

Workflow löschen

Navigation zu >Einstellungen>Basisdaten für Ressourcen>Workflows
In der Smartlist wird das Suchresultat bzw. alle bereits erfassten Workflows angezeigt. Durch Selektieren der Checkbox (Mehrfachselektion möglich) und Bestätigung über den Button Löschen erscheint eine Sicherheitsabfrage, ob der Workflow wirklich entfernt werden soll. Quittieren dieser Meldung entfernt den Workflow. Im Action Message wird die Anzahl der entfernten Einträge ausgegeben.
