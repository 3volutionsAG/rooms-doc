Verwandte Seiten

Einleitung

Ein Buchungsticker wird als zusätzlicher Container zur Verfügung gestellt, der je nach Layout irgendwo im Kopfbereich der Applikation platziert werden kann. Der Buchungsticker erlaubt einige wichtige Verwaltungsfunktionen für die laufenden Buchungen des angemeldeten Benutzers.

Vorbedingungen

Aktuell laufende Buchung des angemeldeten Benutzer vorhanden.

Vorgehen / Beschrieb

Der Buchungsticker kann pro Mandant global ausgeschaltet werden, indem keine Platzierung im Kopfbereich erfolgt. Wird er platziert, ist er für alle Benutzer auf allen Ansichten verfügbar.

  • Weist der aktuelle Benutzer keine laufende Buchungen auf, entspricht die Darstellung derjenigen eines nicht selektierten Subpanels und enthält einen Standard-Text. Es ist keine Interaktion möglich.
  • Der Buchungsticker enthält beim Laden einer Seite die aktuellen Buchungsdaten. Zusätzlich führt der Ticker automatisch einen regelmässigen Refresh zusammen mit dem Action Message durch.
  • Der Buchungsticker zeigt nur Buchunen mit Status "definitv gebucht" an.



Weist der aktuelle Benutzer laufende Reservationen auf, entspricht die Darstellung derjenigen eines selektierten Sub-Panels. Die Reservationen werden nach folgenden Kriterien in der angegebenen Reihenfolge sortiert:

  • Ressourcentyp: Deskshare / Raum / Parkplatz / Fahrzeug / Equipment / Person
  • Reservationsdauer: kürzer / länger
  • Reservationsbeginn: früher / später
  • Reservations-ID: kleiner / grösser


Der Ressourcenname der ersten Buchung gemäss genannter Sortierung wird direkt im Buchungsticker angezeigt. Hinter dem Namen werden einige Icons für Aktionen mit dieser Buchungen aufgelistet. Mit einem Klick auf den Ticker irgendwo neben den Icons werden die restlichen laufenden Buchungen bzw. die Ressourcennamen inkl. Icons eingeblendet. (Es wird bewusst auf Mouse-Over Funktionalität verzichtet, da sich diese bisher immer negativ auf die Usability ausgewirkt hat.) Ein Klick auf eines der Icons führt die entsprechende Aktion aus und schliesst die Einblendung (Popup). Jeder Klick ausserhalb des Popups schliesst das Popup ebenfalls.

Folgende Icons werden angezeigt:

  • View: Absprung in den Ansichtsmodus der Buchung.
  • Edit (wenn es sich um keine zugeordnete Ressource handelt): Absprung in den Edit-Modus der Reservation.
  • Freigaben bearbeiten (wenn es sich um eine zugeordnete Ressource handelt): Öffnen des Bearbeitungsfensters für die Freigaben (siehe vorangehendes Kapitel).
  • Checkout (wenn das Funktionsrecht vorhanden ist): Sofortiges Beenden der Buchung (exkl. Nachlaufzeit).
  • Freigabe (wenn das Funktionsrecht vorhanden ist): Sofortiges Beenden der Buchung (inkl. Nachlaufzeit).


Die Breite des Reservationstickers richtet sich nach dem breitesten angezeigten Text (Ressourcennamen) und ist daher flexibel.
