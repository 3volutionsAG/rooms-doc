Einleitung

Reservationsdaten aus Exchange Ressourcen können nach dem Export aus Outlook in ROOMS wieder zu der entsprechenden Ressource importiert werden.

Das Datenaustauschformat ist CSV.

Vorbedingungen

Es können nur CSV Dateien verarbeitet werden. Die CSV wird automatisch durch einen Kalender Export aus Outlook generiert

Format CSV Datei

  • Fileformat: CSV
  • Delimiter: Semikolon (;)
  • Feldmarker: Textfelder sind in Anführungs- und Schlusszeichen möglich
  • Reihenfolge ist zwingend einzuhalten
  • Header wird ignoriert - Spaltenbezeichnung ist dementsprechend irrelevant

Vorgehen / Beschrieb

Benutzer navigiert auf >Einstellungen >System >Outlook Importe und Klick Button Finden

Eingabefelder für Upload werden angezeigt:

Klick auf Button Upload prüft Datei.

  • Im Erfolgsfall wird Datei für den Import in den Speicher geladen und eine Ausgabe im Actionpanel erfolgt
  • Ist die Datei nicht konsistent wird eine Fehlermeldugn im Actionpanel ausgegeben.

Klick auf Button Import starten startet den Import Prozess. Reservationen werden temporär importiert. Das System weist im Actionpanel jede Reservation aus, welche an einem nicht verfügbaren Zeitpunkt liegt.

Reservationen werden in einer Smartlist angezeigt (Spalten: Reservationstitel, Organisator, Von, Bis, Anzahl Personen, Privat). Die CSV Datei kann im Feld Besprechungsplanung (Organisator) folgende möglichen Werte haben, welche beim Import an Hand der vorhanden Personen interpretiert werden

  • Vorname Nachname
  • Nachname Vorname
  • E-Mail
  • Personalnummer
  • Konnte keine Auflösung stattfinden wird der Organisator angezeigt und ein Icon Bearbeiten angezeigt, über welches in einem Modales Popup der Zeitraum und Organisator edtiert werden kann.

Wichtig: Wird der Organisator nicht geändert, wird die aktuell angemeldete Person (Importeur) als Organisator verwendet.

Reservationen mit einem Zeit-Konflikt haben auch zusätzlich das Icon Bearbeiten.

Nur bereinigte Ressourcen werden importiert.

Klick auf Button Speichern, speichert die Reservationen.

Die Speicherung wird im Actionpanel in folgender Form bestätigt: Es wurden (0) Reservationen gespeichert.

  • Jede nicht gespeicherte Reservation auf Grund eines Konflikts wird im Actionpanel mit folgender Meldung ausgegeben: Die Ressource (Ressource.Bezeichnung) ist zwischen (Import.Reservation.Von) und (Import.Reservation.bis) nicht verüfgbar

  • Jede nicht gespeicherte Reservation bleibt in der Liste enthalten und kann erneut mutiert (Icon Bearbeiten) und danach importiert werden

    Teilnehmer werden analog des Organisators verifiziert und falls im System gefunden, auch importiert
