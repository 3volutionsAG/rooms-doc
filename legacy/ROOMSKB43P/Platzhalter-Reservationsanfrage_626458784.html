Einleitung

Die Reservationsanfrage (Request Formular) wird den Benutzern zur Verfügung gestellt, welche nicht direkt im System buchen können/dürfen.

Die Reservationsanfrage kann dann direkt zu einem Event verarbeitet werden. Um dies zu gewährleisten müssen die Eingabefelder im System bekannt sein. Deshalb existieren Platzhalter von bekannten Feldern und Platzhalter für frei definierbare Felder.

Vorbedingungen/Grundregeln

  • Es steht eine Reihe von Platzhaltern zur Verfügung, welche zur Laufzeit mit dem entsprechenden Text abgefüllt werden.
  • Die Werte werden immer in der aktuellen Benutzersprache abgefüllt
  • Auf Klick eines Platzhalter wird dieser an der aktuellen Stelle im Editor platziert
  • Das Requestformular kann mit HTML formatiert werden

Benötigte Rechte

  • Darf Sektion Listen nutzen
  • Darf Anlässe als Kalender sehen
  • Darf Anlässe buchen

Anlassbezogene Platzhalter.
Bei der Generierung eines Anlasses werden diese Felder übernommen.

Platzhalter Beispiel an Hand von Titel

<RoomsTextBox labelText="Views_Reservationsanfrage_Edit_Titel" name="Reservationsanfrage.Titel" hilfeText="Views_Reservationsanfrage_Edit_Titel_HilfeText" renderMode="FieldSet" />

Views_Reservationsanfrage_Edit_Titel = Beschriftung des Feldes (optional). Es kann in den Übersetzungen ein sinnvoller Text in allen vorhandenenen Sprachen erfasst werden.
Views_Reservationsanfrage_Edit_Titel_HilfeText = Hilfetext des Feldes (optional). Es kann in den Übersetzungen ein sinnvoller Text in allen vorhandenenen Sprachen erfasst werden.
