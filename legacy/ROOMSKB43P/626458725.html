Einleitung

Fix zugeordnete Ressourcen (auch bekannt als "long term" Ressourcen können für einen bestimmten Zeitraum freigegeben werden.

Die freigegebenen Zeiträume stehen dann allen berechtigten Personen zur freien Buchung zur Verfügung.

Vorbedingungen

Benutzer befindet sich im Anzeigemodus einer Person (siehe KB Person suchen & finden)

Fixe Ressource wurde vorgängig der Person zugeordnet (siehe KB Fixe Ressourcen der Person zuweisen)

Benötigte Rechte

keine

Vorgehen / Beschrieb

Im Anzeigemodus (Viewmode) werden alle fix hinzugefügten Ressourcen aufgelistet.


Klick auf "Freigabe" Icon öffnet Modales Popup um eine Freigabe zu tätigen oder eine Freigabe aufzuheben:

Für die Freigaben gelten folgende generelle Regeln:

  • Es sind nur Buchungen/Freigaben pro Tag möglich; Halbtage können nicht über die Freigaben verwaltet werden.
  • Alle Anpassungen werden sofort wirksam; es werden keine temporären Blocker erstellt und die Änderungen können nicht über einen Abbrechen-Button rückgängig gemacht werden.
  • Freigaben erfolgen in Form von Reservationslöschungen (und nicht Annullationen).
  • Anpassungen setzen die üblichen Datenberechtigungen voraus.
  • Beim Speichern werden noch keine Buchungen erstellt! Erst nach dem Speichern wird ein Service-Prozess die neue Zuweisung entdecken und asynchron für einen konfigurierbaren Zeitraum im Voraus tägliche Buchungen erstellen. Standard 1 Jahr (Anmerkung: dieser Wert kann in der globalen Konfiguration - KB Konfigurationsparameter - geändert werden)
  • Das Datum der letzten Buchung wird auf der Zuweisung hinterlegt; die Buchungen im Anschluss an das hinterlegte Datum werden danach laufend nachgeführt, so dass zu jedem Zeitpunkt der konfigurierte Zeitraum im Voraus gebucht ist. Eine zusätzliche Validierungsregel verhindert, dass zugewiesene Ressourcen zu einem Zeitpunkt nach dem hinterlegten Datum gebucht werden können. Die Reservationen erhalten den Standard-Titel für Schnellreservationen. Die täglichen Buchungen dauern von 0:00 bis 23.59 des jeweiligen Tages gemäss der Zeitzone der entsprechenden Ressource (und nicht ausgehend vom Benutzer).
  • Falls eine Ressource nicht gespeichert werden kann, wird pro Ressource eine Notifikation verschickt: Template Nr. 25 - Zugewiesene Ressource - Fehlermeldungen Reservation 
  • Nach dem Entfernen wird der Service Prozess analog alle Buchungen in der Zukunft löschen (und nicht annullieren).

Freigaben Serien

Freigaben können auch mittels Intervalle (analog einer Serie) erstellt werden. Folgende Freigaben stehen zur Verfügung

  • Täglich
    • Feld „Tag(e)“ muss Ganzzahl beinhalten
    • Standardwert: alle 1 Tag
  • Wöchentlich
    • Feld „Wiederholen“ muss eine Ganzzahl beinhalten
    • Min. eine Checkbox der Wochentage muss selektiert sein
    • Min. der Tag der Seriestart (z.B. Mittwoch) muss selektiert sein
    • Standardwert: Wiederholung alle 1 Woche am #Selektion des Wochentags#
  • Monatlich
    • Option „Tag“ muss Ganzzahl beinhalten – Monat muss Ganzzahl beinhalten
    • Option „Am“, Eintrag aus Listen und „Monat“ muss Ganzzahl beinhalten.

    • gibt die Buchung gemäss Serien Pattern Einstellungen frei (löscht die entsprechenden Buchungen)

    • Buchungen welche bei künftiger Erstellung freigegeben werden sollen => im Moment wo der Service neue Buchungen erstellt, müssen diese Freigaben berückscihtigt werden (z.B. jeden Freitag)

Automatische Freigabe (ab 4.2.3):

Im Anzeigemodus (Viewmode) werden alle fix hinzugefügten Ressourcen aufgelistet. Die Checkbox "Autom. Freigabe" wird als Spalte angezeigt, sofern (siehe Abbildung 2) ausgewählt.

 

Standardmässig ist die Checkbox nicht selektiert. Bei selektierter Checkbox "Autom. Freigabe" gelten folgende Regeln:

  • Beim Speichern einer Reservation wird geprüft, ob zum Zeitpunkt der Reservation bereits eine oder mehrere Reservationen von zugewiesenen Ressourcen vom selben Ressourcentyp mit aktivierter "Automatisch freigeben" Checkbox vorliegen. Ist dies der Fall, werden alle diese Reservationen entfernt/gelöscht (analog wie wenn der Benutzer beim Verwalten der zugewiesenen Ressourcen im Kalender auf einen Tag klickt)
  • Eine entsprechende Warnung wird  im Action Message ausgegeben ("Achtung: Die zugewiesene Ressource ABC wurde am 21.10.2012 automatisch freigegeben.").
  • Der Service prüft analog beim Erstellen neuer Reservationen für die zugewiesenen Ressourcen, ob zeitgleich bereits eine Reservation für denselben Ressourcentyp vorliegt, wenn die Checkbox "Automatisch freigeben" der Ressource aktiviert ist. Ist dies der Fall, dürfen keine neuen Reservationen erstellt werden.
