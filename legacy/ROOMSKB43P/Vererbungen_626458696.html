Grundsätzliches:

  • Prüfungen innerhalb des Buchungsprozesses erfolgen nur auf die Ressourcen - nie auf die hierarchische Baumstruktur – diese dient nur der vereinfachten Administration
  • Nachträglich hinzugefügte Ressourcen erben automatisch immer von der Baumstruktur (Feiertage, Sperrzeiten und Rechte)

Folgende Regeln gelten:

  • Hinzufügen: Wird auf eine Node a, b, c, d oder e hinzugefügt wird dies an alle Nodes und an die angehängten Ressourcen weitergegeben.Wird auf einer Ressource a oder b hinzugefügt hat dies nur für diese Ressource Gültigkeit – das hinzufügen eines gleichen Eintrags auf Baumebene hat keinen Einfluss
  • Entfernen: Wird auf einem Node a, b oder c entfernt, erfolgt die Löschung über sämtliche Childnodes bis hin zur Ressource. Die Entfernung kann auch nur auf der Ressource erfolgen
  • Update: Erfolgt auf einem Node bei a, b, c, d oder e eine Änderung, wird diese überall dort vererbt, wo der Pfad noch Bestand hat bis hin zur Ressourcen.
  • Änderungen: Änderungen werden nur auf Nodes übertragen, welche noch dem Ursprungsnode entsprechen.
  • Vererbung: Die Vererbung erfolgt in jedem Fall nur abwärts zur Ressource. Nie gegen oben – auch nicht bei einem Update 

Berechtigungen

  • Eine Berechtigung kann pro Gruppe nur einmal auf einem Standort oder Ressource existieren. Dies wird technsich sichergestellt.
  • Bei fixem Equipment werden immer alle Berechtigungen entfernt.
