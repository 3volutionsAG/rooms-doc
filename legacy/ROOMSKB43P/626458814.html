Knowledge Base (Deutsch) : Standorte erfassen & bearbeiten

Created by Former user, last modified by Former user (Deleted) on Jul 31, 2013

Artikel Nummer

Gültig ab Release

4.2

Verwandte Seiten

Einleitung

Als Basis für die Arbeit mit ROOMS müssen nebst der Personen die Standorte erfasst werden. Diese können unterschiedlichen Benutzerkreisen zur Verfügung gestellt werden. Ebenso können Pläne, Sperrzeiten, Feiertage, Notifikationen und Reservationsanfragen standortbezogen verwaltet werden.

Vorbedingungen

Keine

Benötigte Rechte

  • Darf Feiertage verwalten
  • Darf Ressourcentypen mit Sperrzeiten verwalten
  • Darf Standorte anzeigen/editieren
  • Darf Standorte erstellen/löschen
  • Darf Sektion Einstellungen nutzen
  • Darf Standorte mit Plänen verwalten (siehe Pläne verwalten)
  • Darf Plan sehen
  • Darf Notifikationen verwalten (siehe Notifikationen standortbezogen erfassen & bearbeiten)

Plan

Im Register "Plan" werden für die spätere Navigation und den Buchungsprozess die Verknüpfungen auf Standorte und Ressourcen erstellt. Siehe dazu den Artikel Pläne verwalten.

Falls beim Hochladen eines Plans Probleme auftreten sollten wie z.B. ein Timeout, dann können folgende temporäre Einstellungen für den Zeitraum des Hochladens getätigt werden:

  • Auf dem Webserver im file Web.config den Wert <httpRuntime executionTimeout="3600"/> setzen. Dies muss nach dem Hochladen wieder auf die ursprüngliche Konfiguration zurück gesetzt werden!
  • Unter >Einstellungen>System>Globale Parameter den Wert KonfigurationIdMaxUploadFileSize erhöhen auf 10 MB.

Berechtigungen hinzufügen

Im Register "Berechtigungen" werden die Benutzergruppen mit den gewünschten Rollen bzw. Berechtigungen (Rechte & Rolle verwalten) hinzugefügt. Klick auf den Button Hinzufügen öffnet das modale Popup zur Auswahl der gewünschten Berechtigungen bzw. Rollen. Selektieren einer oder mehrerer Rollen und Auswahl der Benutzergruppe über die Intelibox. Selektieren der Checkbox "Vererben" vererbt (Vererbungen) die Rechte bis auf Ressourcenebene.


Der Vorgang wird mit Klick auf den Button Speichern abgeschlossen. Erst nach dem Speichern des gesamten Standortes greifen die Rechte.

Berechtigungen entfernen

Im Register "Berechtigungen" sind alle bereits definierten Berechtigungen aufgeführt. Durch Selektieren einer oder mehrerer Checkboxen und anschliessender Bestätigung über den Button Entfernen erscheint eine Sicherheitsabfrage, ob die Berechtigung wirklich entfernt werden soll. Quittieren dieser Meldung entfernt alle selektierten Berechtigungen. Im Action Message wird die Anzahl der entfernten Einträge ausgegeben.


Sperrzeiten hinzufügen

Im Register "Sperrzeiten" erfolgt die Definition von Zeiträumem, in denen der Standort im Buchungsprozess nicht verfügbar ist. Mittels Button Hinzufügen öffnet das modale Popup zur Erstellung einer Sperrzeit. Der Titel der Sperrzeit muss mehrsprachig erfasst werden. Die Festlegung des gesperrten Zeitraumes erfolgt über die Datum und Zeit Felder. Der Vorgang wird über den Button Speichern abgeschlossen und die Sperrzeit wird hinzugefügt. Selektieren der Checkbox "Vererben" vererbt (Vererbungen) die Sperrzeit bis auf Ressourcenebene.


Zusätzlich können Sperrzeiten auch als Serien hinzugefügt werden. Mittels Button Sperrzeitserie hinzufügen öffnet das Modales Popup zur Erstellung der Sperrzeitserie. Die Felder und der Funktionen sind der unten stehenden Tabelle zu entnehmen.

Sperrzeiten entfernen

Im Register "Sperrzeiten" sind alle bereits definierten Sperrzeiten aufgeführt. Durch Selektieren einer oder mehrerer Checkboxen und anschliessender Bestätigung über den Button Entfernen erscheint eine Sicherheitsabfrage, ob die Sperrzeit wirklich entfernt werden soll. Quittieren dieser Meldung entfernt alle selektierten Sperrzeiten. Im Action Message wird die Anzahl der entfernten Einträge ausgegeben.

Feiertage hinzufügen

Im Register "Feiertage" werden die Feiertage (siehe Feiertage importieren, erfassen & bearbeiten) für den betreffenden Standort ausgewählt. Im späteren Buchungsprozess sind die Feiertage visualisiert (siehe Farbgebung im Kalender) und nur buchbar mit dem Recht an Feiertagen buchen zu dürfen. Klick auf den Button Hinzufügen öffnet das modale Popup zur Auswahl der gültigen Feiertage. Selektieren einer oder mehrerer Checkboxen und anschliessende Bestätigung über den Button Hinzufügen listet die gewählten Feiertage für diesen Standort auf. Wird die Checkbox "Rekursiv" selektiert, werden die Feiertage bis auf Ressourcenebene vererbt (Vererbungen).

Feiertage entfernen

Im Register "Feiertage" sind alle bereits definierten Feiertage aufgeführt. Durch Selektieren einer oder mehrerer Checkboxen und anschliessender Bestätigung über den Button Entfernen erscheint eine Sicherheitsabfrage, ob die Feier wirklich entfernt werden soll. Quittieren dieser Meldung entfernt alle selektierten Feiertage vom Standort. Im Action Message wird die Anzahl der entfernten Einträge ausgegeben.

Notifikationen hinzufügen

Im Register "Notifikationen" werden die Notifikationsvorlagen erstellt und ausgewählt, damit im späteren Buchungsprozess die gewünschten E-Mails versandt werden. Die Funktionalität ist den KB Artikel Notifikationen standortbezogen erfassen & bearbeiten zu entnehmen.

Notifikationen entfernen

Im Register "Notifikationen" sind alle bereits definierten Notifikationsvorlagen aufgeführt. Durch Selektieren einer oder mehrerer Checkboxen und anschliessender Bestätigung über den Button Entfernen erscheint eine Sicherheitsabfrage, ob die Notifikationsvorlage wirklich entfernt werden soll. Quittieren dieser Meldung entfernt alle selektierten Notifikationsvorlagen vom Standort. Im Action Message wird die Anzahl der entfernten Einträge ausgegeben.

Reservationsanfragen hinzufügen

Im Register "Reservationsanfragen" werden die Reservationsanfragen bzw. der Anlasstyp hinzugefügt, damit im späteren Buchungsprozess diese Anfragen erfasst werden können. Klick auf den Button Hinzufügen öffnet das modale Popup zur Auswahl der gewünschten Anlasstypen. Selektieren einer oder mehrerer Checkboxen und anschliessender Bestätigung über den Button Hinzufügen wird der gewünschte Anlasstyp und somit das hinterlegte Reservationsanfrageformular auf dem Standort verfügbar gemacht. Detailierte Informationen über die Anlasstypen und die Konfiguration der Reservationsanfrage sind dem KB Anlasstypen erfassen & bearbeiten zu entnehmen.


Reservationsnfragen entfernen

Im Register "Reservationsanfragen" sind alle bereits definierten Reservationsanfragen bzw. Anlasstypen aufgeführt. Durch Selektieren einer oder mehrerer Checkboxen und anschliessender Bestätigung über den Button Entfernen erscheint eine Sicherheitsabfrage, ob der Anlasstyp wirklich entfernt werden soll. Quittieren dieser Meldung entfernt alle selektierten Anlasstypen vom Standort. Im Action Message wird die Anzahl der entfernten Einträge ausgegeben.


Hauptstandort speichern

Nach Eingabe aller Pflichtdaten und der gewünschten optionalen Daten wird der Standort mittels Button Speichern gesichert und erscheint in der Standortliste.


Hauptstandort bearbeiten

Navigation zu >Einstellungen>Basisdaten für Ressourcen>Standorte

In der Smartlist werden alle bereits erfassten Standorte angezeigt. Mittels Klick auf die Bezeichnung erscheinen die Icons zum Hinzufügen von Substandorten, der Bearbeitung, Ansicht und Löschung. Über das Ansichts-Symbol wird der Standort im Anzeigemodus (Viewmode) dargestellt. Das Anlegen, Bearbeiten und Löschen von Substandorten kann nach derselben Anleitung wie bei den Hauptstandorten erfolgen.


Über den Button Bearbeiten oder das Editieren-Symbol wechselt das System in den Bearbeiten Modus. In allen, bereits vorgängig in diesem Artikel beschriebenen Registern, können Anpassungen vorgenommen werden. Die Änderungen werden mit dem Button Speichern übernommen.

Recht: Darf Standorte erstellen/löschen übergeht die loakelen Rechte und gibt einen globalen Zugriff auf die Bearbeitung der Standorte.

Hauptstandort löschen

Navigation zu >Einstellungen>Basisdaten für Ressourcen>Standorte
In der Smartlist wird das Suchresultat bzw. alle bereits erfassten Standorte in der Baumstruktur angezeigt. Mittels Klick auf die Bezeichnung und erscheinen die Icons zum Hinzufügen von Substandorten, der Bearbeitung, Ansicht und Löschung. Über das Löschen-Symbol erscheint eine Sicherheitsabfrage, ob der Status wirklich entfernt werden soll. Quittieren dieser Meldung entfernt den Status. Im Action Message wird die Anzahl der entfernten Einträge ausgegeben.


Standorte können nur dann gelöscht werden

  • wenn keine Abhängigkeiten zu Ressourcen mehr bestehen, resp. keiner der Sub-Standorte noch eine Abhängigkeit zu einem Ressourcen mehr aufweist
  • wenn der Benutzer das „Standort Bearbeiten“ Recht hat

Weitere Löschregeln

  • Falls einem Standort (z.B. Bern) diverse Substandorte (z.B. Laupenstrasse\ 1.Stock) zugewiesen sind, aber kein Standort mehr über Ressourcen verfügt, wird der Standort inkl. allen Substandorten gelöscht (sofern man auf dem ganzen Standortbaum die nötigen Berechtigungen besitzt)
  • Beim Löschen werden die Abhängigkeiten zu den Berechtigungen gelöscht
  • Alle anderen Abhängigkeiten (Sperrzeiten, Feiertage, etc.) werden gelöscht
