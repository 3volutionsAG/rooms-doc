Use Cases : 105.001 Ressourcen verwalten

Use Case Überblick


ID

105.001

Owner

Lukas Läderach

Rev-Number

2.7

Priorität

1

Ziel & Beschreibung

  1. Suchen und Finden einer bestimmten Ressource und
  2. Bei entsprechender Berechtigung (Zugriffs und Standortabhängig) die Bearbeitung der Ressource (Add, Edit und Delete)

Notwendige Rechte

  • Darf Sektion Einstellungen nutzen ODER Darf Sektion Listen nutzen
  • Darf Ressourcetyp Raum verwalten
  • Darf Ressourcetyp Equipment verwalten
  • Darf Ressourcetyp Arbeitsplatz verwalten
  • Darf Ressourcetyp Parkplatz verwalten
  • Darf Ressourcetyp Automobil verwalten
  • Darf Ressource bearbeiten (standortabhäniges Recht)
  • Darf Reservation mit automatischer Verlängerung editieren

Bedingungen

  • Panels „Stammdaten“ und „Spezifische Daten“ müssen umgesetzt werden

Hauptszenario in Schritten

  1. Nutzer navigiert auf >Einstellungen >Ressourcen
  2. Nutzer gibt im Sidepanel die Suchkriterien ein (optional) und klickt „Anzeigen"
  3. System liefert im Detailbereich eine Liste mit Suchresultaten - es kann ein oder mehrere Einträge gelöscht werden
  4. Nutzer kann einen Eintrag anklicken, um die Details anzuzeigen (View Modus)
  5. Oder (bei entsprechender Berechtigung) über Edit das Detail zu bearbeiten

Beziehungen von/zu UC


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Regeln

Favoritenliste (siehe auch UC 230.001 Liste erstellen und (als Favorit) speichern)

  • Wird eine Liste als Favorit definiert wird diese standardmässig angezeigt.
  • Ansonsten ist die Liste leer.
  • Nach Wechsel eines Eintrages, werden Sidepanel und Inhaltsbereich automatisch neu geladen
    (Achtung: Bevor Liste gespeichert wird, muss "Finden" Button geklickt werden, um alle Kriterien zu speichdern)

Name/Bezeichnung

  • Combi Suchfeld: sucht alle Ressourcen in den Feldern Name (ID) und Bezeichung.
  • Nach Bezeichnung wird in der Sprache des angemeldeten Benutzers gesucht

Ressourcenart

  • Standard "alle" - Suche über alle Ressourcenarten
  • Es werden nur Ressourcenarten angezeigt (und gesucht), für welche der Benutzer auch über die entsprechende Funktionsrechte verfügt

Equipmenttyp

  • Nur bei Ressourcenart Equipment sichtbar
  • Dropdown mit WertenIst ein Equipment zu einem Raum mitgebucht worden, zeigt der View Link (siehe Schritt 3 - Feld: Reservationstitel) und Edit Icon (siehe Schritt 3) immer auf die Raumbuchung
    • Alle => zeigt alle Equipment Reservationen
    • Mobil auf Raum => zeigt alle Equipment Reservationen, welche zu einem Raum gebucht wurden
    • Fix auf Raum => zeigt alle Equipment Reservationen, welche als fixes Equipment zu einem Raum gebucht wurden
    • Mobil alle => zeigt alle Equipment Reservationen, welche mobil sind (ob zum Raum gebucht wurde oder nicht)
    • Mobile ohne Raum => zeigt alle Equipment Reservationen, welche ohne Raum gebucht wurden

Gliederungen (siehe auch UC 104.001 Gliederungen verwalten)

  • Es werden alle Gliederungstypen der gewählten "Ressourcenart" angezeigt
  • Bei Ressourcenart "alle" werden alle Gliederungstypen aller Ressourcenarten aufgelistet
  • Es werden nur die Gliederungen (eines Gliederungstyps) angezeigt für welche der Benutzer auch Rechte hat

Ressourcenpicker

  • Der Ressourcenpicker zeigt alle Ressourcen - unabhängig davon was im Feld "Ressourcenart" selektiert worden ist

Ressourcenart
Nur Bezeichnungsfeld (nicht editierbar)

Aktiv/Inaktiv

  • Standard = inaktiv
  • Nur aktive Ressourcen erscheinen grundsätzlich in den Buchungssuchen (zusätzlich relevant ist das entsprechende Standortbezogene Recht
  • Wird eine Ressource auf inaktiv gesetzt, werden alle zugehörigen Reservationen mit Exchange Synchronisation von der Synchronisation abgekoppelt.

ist Öffentlich
Filterkrierium in der Suche (Reservieren) - hat keine weitere Funktionalität


Checkin aktiviert

  • aktiviert die Checkin Funktionalität (auf Türschild)
  • ob ein Checkin getätigt werden muss definiert die Benutzergruppe (siehe 109.003 Benutzergruppen verwalten)


Checkin in Min vorher erlaubt

  • Definiert wie viel Minuten vor Buchungsbeginn bereits eingecheckt werden kann
  • wird eingeblendet, sobald die Checkin aktiviert Checkbox aktiv ist
  • Standardwert = leer


No Show Delay

 

  • Definiert nach wie vielen Minuten ohne Checkin der Raum wieder freigegeben wird
  • Standardwert = 15 Minuten


Checkin Regeln

  • Prüfung ob es sich um eine Buchung einer Ressource mit aktiver Checkbox: Checkin Einzelbuchungen handelt
  • Prüfung ob Organisator in einer Benutzergruppe ist, die ein Checkin erfordert (siehe Regeln unter "Benutzergruppe")
  • Prüfung ob bei der Buchung nicht bereits das Flag "Bestätigt" gesetzt wurde
  • Prüfung ob die aktuelle Zeit > Buchungsstart (ohne Vorlauf) + No-show Delay


Tarifkategorie (aus UC 105.004 Tarife verwalten )

  • Es muss keine Tarifkategorie gewählt werden. Falls keine Tarikfategorie gewählt wurde, erfolgt keine Preisberechnung auf der Ressource
  • Standard = leer (keine Selektion)

Info URL

  • Übersetzungseintrag „Alias für InfoURL“ muss erstellt werden
  • Ist der Übersetzungseintrag nicht übersetzt (leer), werden die effektiven URL’s der Ressourcen angezeigt
  • Ist die Übersetzung nicht leer wird der Text der Übersetzung in der entsprechenden Benutzersprache übernommen
  • Die Implementation erfolgt im Sidepanel (Reservationsdetail) und im View Mode der Ressource

Verantwortliche Person

  • Intelibox mit allen aktiven verfügbaren Personen
  • rein infomratives Feld - keine weitere Funktionalität

Minimale Reservationsdauer

  • Standardwert 0
  • 0 = keine Minimaldauer festgelegt
  • Eingabe in Minuten
  • Falls die Minimalreservationsdauer unterschritten wird, erfolgt im Reservationsprozess die Ausgabe einer Fehlermeldung (siehe auch UC 201.000 Ressourcen reservieren)

Maximale Reservationsdauer

  • Standardwert 0
  • 0 = keine Minimaldauer festgelegt
  • Eingabe in Minuten
  • Falls die Minimalreservationsdauer unterschritten wird, erfolgt im Reservationsprozess die Ausgabe einer Fehlermeldung (siehe auch UC 201.000 Ressourcen reservieren )

Vorlaufdauer

  • Standardwert: 0
  • 0 = keine Vorlaufzeit wird berechnet
  • Eingabe in Minuten
  • Bei der Berechnung im Reservationsprozess wird der grösste Wert aller Vorlaufzeiten (Ressourcen, Services, Bestuhlungen) verwendet (kein Kummulatives verhalten)

Nachlaufdauer

  • Standardwert: 0
  • 0 = keine Vorlaufzeit wird berechnet
  • Eingabe in Minuten
  • Bei der Berechnung im Reservationsprozess wird der grösste Wert aller Vorlaufzeiten (Ressourcen, Services, Bestuhlungen) verwendet (kein Kummulatives verhalten)

Priorisierung

  • Angabe eines Sortiermerkmal (Alphanummerischer Wert)
  • Wird in Listen (UC 220.001 Reservationen in Liste anzeigen ) und unter Reservieren (UC 201.000 Ressourcen reservieren) als zusätzliche Spalte angezeigt
  • Kalender (UC 213.001 Reservationen im Kalender suchen und buchen) wird automatisch nach dieser Spalte sortiert (aufsteigend)

Automatische Verlängerung (siehe auch UC 201.001 Raum reservieren)

  • Aktivierung der automatischen Verlängerung erfolgt durch die Eingabe eines Wert grösser 0  ( >0 ). Ist der Wert 0, dann ist die automatische Verlängerung ausgeschalten
  • Prüfung erfolgt auf Ganzzahlen: Feldvalidierung (siehe 000.001 Globale Funktionen)
  • Weitere Regeln:
    • Das Ausführungsdatum ist Reservationsende - 5 Minuten.
    • Eine automatische Verlängerung löst keine E-Mail Notifikationen aus
    • Synchronierte Reservationen werden im Outlook nachgetragen OHNE MeetingUpdate für die Teilnehmer.
    • Eine Reservation mit automatischer Verlängerung endet bei Checkout, Konflikt mit der nächsten Reservation oder am Ende des Kalenderteages (23:59)
    • Konfliktsituation: Ausgabe im Actionpanel des Erstellers und Organisators. E-Mail an Ersteller und Organisator (Mailtemplate: Reservation - Konflikt beim Verlängern siehe UC 107.001 Notifikationen verwalten)

nur Slotbuchungen zulässig

Spezifische Daten

PlanLabel

Konfigurationsdetails in 000.003 Planlabel

Reminder für provisorische Buchungen in Minuten

  • Verschickt einen zusätzlichen Reminder auf allen provisorischen Buchungen auf dieser Ressource in Minuten vorher (sofern Wert > 0 ist)
  • Standardwert = leer
  • Die Notifikation wird an dieselbe Benutzerguppe verschickt, welche bereits beim Statuswechsel (in Status "prov. Buchung") informiert wurde. Diese muss also vorgängig als Workflow konfiguriert werden (siehe UC 108.001 Workflows verwalten)
  • Es wird das Notifikationstemplate No. "30: Provisorische Reservation Reminder" verwendet - siehe Notifikationsmatrix
  • Ist das Notifikationstemplate nicht verfügbar wird keine Notifikation verschickt (siehe UC 107.001 Notifikationen verwalten)
  • Enthält die Reservation Dienstleistungen mit Vorlaufzeiten wird diese zum Remindervorlauf dazu gezählt

Alarmierung in Status

  • Erscheint nur bei Ressourcenart "Raum"
  • Dropdown zeigt alle erfassten Status (unabhängig vom Status der Ressouce) - siehe UC 108.002 Status verwalten
  • Es muss der Status erfasst werden, bei welchem das Alarmierungspopup angezeigt werden soll (siehe UC xxx ) in Kombination mit dem "Dauer nach Statuswechsel" (nächster Punkt)

Dauer nach Statuswechsel

  • Erscheint nur bei Ressourcenart "Raum"
  • Angabe in Minuten
  • 0 oder leer = deaktiviert
  • Löst Event "Alarmierungspopup" (siehe UC 214.002 Alarmierung Überfälligkeit Organisator & Warnung Folgereservationen) nach Ablauf der erfassten Minutenzahl aus, wenn der unter "Alarmierung in Status" definierte Status eintrifft.

Alarmierung bei Folgereservation

  • Erscheint nur bei Ressourcenart "Raum"
  • Angabe in Minuten
  • Die Minuten beziehen sich auf den Reservationsstart ohne Vorlauf
  • 0 oder leer = deaktiviert
  • Löst Event "Alarmierungspopup" (siehe UC 214.002 Alarmierung Überfälligkeit Organisator & Warnung Folgereservationen) nach Ablauf der erfassten Minutenzahl aus, falls zum aktuellen Zeitpunkt eine Reservation besteht und sich auf der Ressource einen Reservation in Zukunft befindet
  • Beispiel Reservation startet um 11:00, Alarmierung beträgt 60. Eine Vorgängige Reservation dauert von 08:00 bis 10:30 - so erscheint das Popup zwischen 10:00 und 10:30

Gliederungen (aus 104.001 Gliederungen verwalten )
In Abhängigkeit zur der Ressource zugeordnete Ressourcenart, werden die Gliederungen aufgelistet. Dies in Form von Dropdown (für Singleselektionen) und Multiselektionsfeld (für die Zuweisung von mehreren Gliederungen). Siehe Abbildung:

Telefonnummer
Telefonnummer des Raumes. Wird auch im View Modus angezeigt. Anzeige im Reservationsdetail (Sidepanel und Summary Page) nur wenn Inhalt vorhanden ist, ansonsten ist Nummer UND Label nicht sichtbar.

Gültigkeit für temporäre Buchungen

  • Sobald man eine Minutenanzahl bei „Temporäre Buchungen löschen nach (Minuten)“ einfügt, überprüft der Service alle Reservationen, welche auf der entsprechenden Ressource aufbauen.
  • Es könne maximal 9999  Minuten eingegeben werden.
  • Ist der Wert 0 oder leer ist die Funktion deaktivert
  • Wird z.B. die Zahl 4 in der Ressource hinterlegt, heisst dies, dass der Benutzer ab dem Zeitpunkt, wo er die Ressource Temporär reserviert hat, 4 Minuten Zeit hat, diese auf Definitiv umzustellen, da die Buchung sonst gelöscht wird.
  • Um temporäre Reservationen zu erstellen ist das Standortabhängie Recht "Darf Reservationen temporär bearbeiten" notwendig (siehe auch Rechte)

Reminder Notifikationszeit in Minuten

  • Für alle Reservationen auf dieser Ressource wird eine E-Mail Notifikationen xy Minuten vor Reservationsbegin an den Organisator verschickt
  • Anzahl in Minuten – wann eine globale Notifikation ausgelöst werden soll
  • leer = ausgeschalten
  • Reminder Template muss vorhanden sein, bzw. konfiguriert werden: "Reservation - Reminder (global)" - siehe auch 107.001 Notifikationen verwalten)
