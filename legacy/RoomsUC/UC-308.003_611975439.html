Name

Texte in mehreren Sprachen ausgeben

Ziel

Applikationstexte und Stammdaten können mehrsprachig angezeigt werden

Abhängigkeiten zu anderen UC

309.002



Grobbeschrieb

Die Applikation muss eine pro Instanz konfigurierbare Anzahl Sprachen unterstützen. Die unterstützten Sprachen ergeben sich aus den unterstützten Kulturen (TwoLetterIsoLanguageName, z.B. „de“).

Alle Applikationstexte werden in Ressourcen-Files abgelegt. Für die Ressourcen wird mittels Custom-Tool eine Designer-Klasse für den typisierten Zugriff erstellt. Die Designer-Klasse verwendet einen Übersetzungsmanager zum Laden der Ressourcen. Der Manager kann konfiguriert werden und die Übersetzungen zusätzlich mit Datenbankeinträgen abgleichen. Für die Datenbankeinträge wird eine spezielle Tabelle (Translating) erstellt.

Für die Stammdaten wird eine generische Tabelle (Text) erstellt, in der die sprachabhängigen Texte abgelegt werden. Der Zugriff wird weitgehend über die Funktionalität des OR-Mappers abgedeckt.
