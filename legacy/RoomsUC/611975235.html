Use Case Details

Schritt 1 - Nutzer navigiert auf Tab "Door Sign" & klickt Türschild aktivieren

Controls

Erfolgsfall

  • Intelibox: Konfigurationsbenutzer wählen
  • Button: Türschild aktivieren

Fehlerfall

  • Label: Es sind keine Lizenzen mehr verfügbar, um dieses Türschild zu aktivieren. Bitte deaktivieren Sie ein anderes Türschild oder erwerben weitere Lizenzen.

Regeln

  1. Lizenzierung wird geprüft
  2. Falls genügend Lizenzen verfügbar, wird folgendes Control eingeblendet
    1. Intelibox: Konfigurationsbenutzer wählen 
  3. Ist eine Person selektiert, wird folgendes Control eingeblendet
    1. Button: Türschild aktivieren
  4. Klick auf "Türschild aktivieren" führt zu Schritt 2
  5. Falls nicht genügend Lizenzen verfügbar wird Fehlermeldung angezeigt: s sind keine Lizenzen mehr verfügbar, um dieses Türschild zu aktivieren. Bitte deaktivieren Sie ein anderes Türschild oder erwerben weitere Lizenzen.

Schritt 2 - Konfigurationsfelder Türschild werden angezeigt

 

Controls

 

Inhaltsbereich

  • Dropdown: Thema
  • Dropdown: Sprache
  • Zahlenpicker: Wartezeit in Sekunden
  • Zahlenpicker: Refresh Interval (in Minuten)
  • Checkbox: Titel anzeigen
  • Checkbox: Organisator anzeigen
  • Checkbox: Anonym Checkin
  • Dropdown: Checkout Modus
  • Bild: QR Code
  • Intelibox: Konfigurationsbenutzer


  • Button: Türschild deaktivieren

 

Regeln

Generell

  • Durch die Aktivierung wird ein QR-Code erstellt und in der Datenbank abgelegt
  • Es wird eine Door Sign-Lizenz abgebucht
  • Jede Änderung eines Wertes hat zur Folge, dass der Zeitstempel aktualisiert wird, damit das Türschild weiss ob die Konfiguration aktualisiert werden muss.

 

Field
Description & Rules
ThemaDas Türschild kann in schwarz oder weiss angezeigt werden durch entsprechende Wahl in der Dropdown
Sprache
  • alle im Web aktivierten Sprachen sind verfügbar
  • Standard = Sprache des angemeldeten Benutzers
Wartezeit in Sekunden

Die Wartezeit beschreibt die Zeit, welche das Türschild ohne Interaktion wartet, bevor es wieder in den "View Mode" wechselt

  • Wert grösser 0 erforderlich
  • Standard = 10
  • max. 30
  • min. 5 
Refresh Interval (in Minuten)

Interval in Minuten, bei welchem das Türschild die Konfiguration prüft und neue Buchungsdaten abruft.

  • Wert grösser 0 erforderlich
  • Standard = 5
  • max. 1440
  • min. 1
Titel anzeigen

Soll der Titel auf dem Türschild angezeigt werden

  • Standard = ja
Organisator anzeigen

Soll der Organisator auf dem Türschild angezeigt werden

  • Standard = ja
Anonym Login

Darf ein Benutzer ohne sich anzumelden ein Checkin machen.

  • Standard = nein
  • Aktiviert wird die Checkin Funktion auf der Ressource (siehe dazu UC 105.001 Ressourcen verwalten)
Checkout Modus

Wie erfolgt das Checkout der Buchung. Zwei Werte

  • Freigeben
  • Terminieren

Details unter UC 201.018 Reservation Freigeben oder Terminieren

QR Code
  • QR Code beinhaltet die Encodierte URL für den Webservice Zugriff auf die selektierte Ressource
  • Zusätzlich beinhaltet der QR-Code den "Service User" der verwendet wird um Buchungen abzufragen (View Mode)
Konfigurationsbenutzer
  • Person, mit welcher sich das Türschild sich gegenüber dem Webserver authentifiziert. 
  • Die Person benötigt ein Logon vom Typ "QR Code".
  • Wenn die Person ändert wird ein neuer QR Code generiert. Falls der Person keinen "QR Code" Logontyp hinterlegt ist, wird eine ValidationMessage ausgegeben: "Dem Konfigurationsbenutzer muss ein QR Code hinterlegt sein"

Schritt 3 - Türschild deaktivieren

 

Controls

Inhaltsbereich

  • Button: Türschild deaktivieren

Regeln

Türschild deaktivieren

  • QR-Code wird gelöscht
  • Raum wird freigegeben
  • Lizenz wird freigegeben
  • Daten bleiben im Hintergrund gespeichert. Bei erneuter Aktivierung stehen Daten zur Verfügung.
  • "Keine Konfiguration gefunden. Es können keine Daten abgerufen werden" - "No configuration found! No data available"
