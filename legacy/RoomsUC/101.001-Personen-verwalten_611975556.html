Use Cases : 101.001 Personen verwalten

Ziel & Beschreibung

Finden einer Person (intern oder extern) und Ausgabe der Detailinformationen zur Person. Bei entsprechender Berechtigung: Neuerfassung, Editieren, Löschen (inaktiv)

Rechte

  • Darf Sektion Listen nutzen ODER Darf Sektion Einstellungen nutzen
  • Darf Person erstellen
  • Darf Person sehen
  • Darf Person editieren intern
  • Darf Person editieren extern
  • Darf Kostenträger anpassen
  • Darf Kostenträger ergänzen
  • Darf Stellvertreter editieren.
  • Darf zugewiesene Ressourcen verwalten

Bedingungen

  • Tab „Login“ muss auf Grund 101.003 Logon verwalten (Login verwalten eingefügt werden)
  • Für Schritt 7 bis 8 muss der globale Parameter "Reservationen für zugewiesene Ressourcen" in 000.004 Konfigurationsparameter

Hauptszenario in Schritten

  1. Nutzer klickt auf Profil Link (eigner Name)
  2. Nutzer navigiert auf Menüpunkt >Einstellungen >Personen oder >Listen >Personen
  3. Nutzer gibt Suchkriterien ein (optional) und klickt Button „Finden"
  4. System liefert Suchresultat den Kriterien entsprechend - ein oder mehrere Personen können gelöscht werden
  5. Nutzer kann ein Rekord anklicken, um zu den Detailinformationen (View Modus) zu sehen
  6. Nutzer kann (bei entsprechender Berechtigung) Person editieren
  7. System zeigt Liste aller zugewiesenen Ressourcen - eine oder mehrere können entfernt werden
  8. Dem Benutzer kann eine neue Ressource zugewiesen werden
  9. Zugewiesene Ressource kann freigegeben werden

Regeln

Aktive Buchungen

  • Alle aktiven Buchungen (Zeitzone Benutzer), bei welchen die Person in der Liste als Organisator oder Teilnehmer eingetragen ist, werden kommasepariert aufgelistet.
  • Angezeigt wird: Ressourcen Bezeichnung (in der Sprache des angemeldeten Benutzers)
  • Ressourcen Bezeichnung ist verlinkt mit dem Detail der Buchung (View Mode)
  • Der Link verfügt über ein Tooltipp (identisch zum Kalender), dieser wird per Ajax geladen - Code Kalender/Map konsolidieren.
  • Ist die Ressource mit einem Plan verknüpft wird, wird zusätzlich das Plan Icon angezeigt (sofern Rechte auf dem Plan; falls eine Mehrfachzuweisung besteht, wird die Erste in der Liste gewählt)
  • Das Berechtigungskonzept wird berücksichtigt (ohne Leserechte werden keine Reservationen angezeigt)
  • Private Buchungen: ohne das Recht "darf Private Buchungen von dritten lesen" wird statt des Links auf der Ressourcen Bezeichnung nur der Text ausgegeben. Plan-Icon bleibt bestehen (sofern Rechte auf dem Plan)

 

Aktive Raumeinheiten

  • Voraussetzung: Recht vorhanden: Darf Raumeinheiten sehen
  • Alle der Person zugewiesenen Raumeinheiten werden angezeigt
  • Es wird die Bezeichnung in der Sprache des angemeldeten Benutzers angezeigt
  • Klick auf Bezeichnung wechselt auf den View Mode der Raumeinheit
  • Es wird das Raumeinheit Icon angezeigt, sofern mit einem Plan verlinkt
  • Klick wechselt auf Plan mit max. Zoom (Analog Funktion "Aktive Reservationen")

Speichern

  • Beim Speichern werden noch keine Buchungen erstellt! Erst nach dem Speichern wird ein Service-Prozess die neue Zuweisung entdecken und asynchron für einen konfigurierbaren Zeitraum im Voraus tägliche Buchungen erstellen. Standard 1 Jahr: (gemäss Anzahl Vorgaben aus globale Parameter "Reservationen für zugewiesene Ressourcen" in Konfigurationsparameter / Configuration parameter)
  • Das Datum der letzten Buchung wird auf der Zuweisung hinterlegt; die Buchungen im Anschluss an das hinterlegte Datum werden danach laufend nachgeführt, so dass zu jedem Zeitpunkt der konfigurierte Zeitraum im Voraus gebucht ist. Eine zusätzliche Validierungsregel verhindert, dass zugewiesene Ressourcen zu einem Zeitpunkt nach dem hinterlegten Datum gebucht werden können. Die Reservationen erhalten den Standard-Titel für Schnellreservationen. Die täglichen Buchungen dauern von 0:00 bis 23.59 des jeweiligen Tages gemäss der Zeitzone der entsprechenden Ressource (und nicht ausgehend vom Benutzer).
  • Falls eine Ressource nicht gespeichert werden kann, wird pro Ressource eine Notifikation verschickt: Template Nr. 25 - Zugewiesene Ressource - Fehlermeldungen Reservation (siehe auch Notifikationsmatrix)
  • Nach dem Entfernen wird der Service Prozess analog alle Buchungen in der Zukunft löschen (und nicht annullieren).

Eine Person ist aktiv, sobald sie einer gültigen Organisationseinheit/Firma zugeordnet ist – Regeln:

  1. Zuordnung muss vorhanden sein
  2. Gültigkeit muss innerhalb des aktuellen Datums des Reservationsprozesses sein.

Wird gesteuert über das intern, extern Flag auf der Organisationseinheit/Firma

Autom. Freigabe

  • Standardwert: nicht selektiert
  • Beim Speichern einer Reservation wird neu geprüft, ob zum Zeitpunkt der Reservation bereits eine oder mehrere Reservationen von zugewiesenen Ressourcen vom selben Ressourcentyp mit aktivierter "Automatisch freigeben" Checkbox vorliegen. Ist dies der Fall, werden alle diese Reservationen entfernt/gelöscht (analog wie wenn der Benutzer beim Verwalten der zugewiesenen Ressourcen im Kalender auf einen Tag klickt)
  • Eine entsprechende Warnung wird  im ActionPanel ausgegeben ("Achtung: Die zugewiesene Ressource ABC wurde am 21.10.2012 automatisch freigegeben.").
  • Der Service muss beim Erstellen neuer Reservationen für die zugewiesenen Ressourcen analog prüfen, ob zeitgleich bereits eine Reservation für denselben Ressourcentyp vorliegt, wenn die Checkbox "Automatisch freigeben" der Ressource aktiviert ist. Ist dies der Fall, dürfen keine neuen Reservationen erstellt werden - auch hier soll eine entsprechende Meldung im ActionPanel (persistent) ausgegeben werden.

Entfernen

  1. Mehrfachselektionen möglich
  2. Klick auf Löschen
  3. Warnhinweis
    1. Bestätigungsmeldung im Actionpanel (inkl. Anzahl der gelöschten Einträge) ODER
    2. Fehlermeldung im Actionpanel

Schritt 9 - Zugewiesene Ressource kann freigegeben werden

Controls

Modales Popup: Freigeben bearbeiten

Bereich "Kalender"

  • Label: Der Kalender gibt die vorhandenen Buchungen der Ressource wieder. Durch Klicken auf eines der Kalenderfelder wird die Ressource freigegeben oder gebucht.
  • Kalendercontrol (embedded)

Bereich "Freigeben/Buchen"

  • Label: Definieren Sie das Intervall für individuelle Freigaben oder Buchungen oder für die zukünftigen automatischen Buchungen.
  • Label: Letzte Buchung
  • Kalenderpicker: Zeitraum
  • Täglich
    • Option: Label: alle + Feld: Tage + Label: Tag/e
    • Option: Label: an allen Wochentagen
  • Wöchentlich
    • Label: Wiederholen alle + Feld: Wochen + Label: Wochen am:
    • Checkbox: Montag
    • Checkbox: Dienstag
    • Checkbox: Mittwoch
    • Checkbox: Donnerstag
    • Checkbox: Freitag
    • Checkbox: Samstag
    • Checkbox: Sonntag
  • Monatlich
    • Option Tag
      • Feld: Anzahl
      • Label: jeden
      • Feld: Anzahl
      • Label: Monat/e
    • Option am
      • Dropdown mit Werten: ersten, zweiten, dritten, vierten, letzten
      • Dropdown mit Werten: Montag, Dienstag, Mittwoch, Donnerstag, Freitag, Samstag, Sonntag
      • Label: jedes
      • Feld: Monat
      • Label: Monat/e
  • Label: Wählen Sie die Aktion, welche mit dem definierten Intervall ausgeführt wird.
  • Option: Freigeben
  • Option: Buchen
  • Option: Zukünftig freigeben

Buttonbereich

  • Übernehmen
  • Schliessen

Regeln

Generell

  • Es sind nur Buchungen/Freigaben pro Tag möglich; Halbtage können nicht über die Freigaben verwaltet werden.
  • Alle Anpassungen werden sofort wirksam; es werden keine temporären Blocker erstellt und die Änderungen können nicht über einen Abbrechen-Button rückgängig gemacht werden.
  • Freigaben erfolgen in Form von Reservationslöschungen (und nicht Annullationen).
  • Anpassungen setzen die üblichen Datenberechtigungen voraus.

Kalender

  • Kalender zeigt den aktuellen Monat
  • es werden alle Buchungen (blau) und Freigaben (weiss) eingefärbt
  • Buchungen in Vergangenheit werden ausgegraut
  • Klick auf eine Buchung löscht die Buchung => Freigabe der Ressource für diesen Tag
  • Klick auf einen freien Tag, bucht die Ressource

Zeitraum

  • zwei Kalenderpicker (von/bis) über welche ein Zeitraum selektiert werden kann

Freigabe Serien

  • Täglich
    • Feld „Tag(e)“ muss Ganzzahl beinhalten
    • Standardwert: alle 1 Tag
  • Wöchentlich
    • Feld „Wiederholen“ muss eine Ganzzahl beinhalten
    • Min. eine Checkbox der Wochentage muss selektiert sein
    • Min. der Tag der Seriestart (z.B. Mittwoch) muss selektiert sein
    • Standardwert: Wiederholung alle 1 Woche am #Selektion des Wochentags#
  • Monatlich
    • Option „Tag“ muss Ganzzahl beinhalten – Monat muss Ganzzahl beinhalten
    • Option „Am“, Eintrag aus Listen und „Monat“ muss Ganzzahl beinhalten.

Aktionen

  • Freigeben
    • gibt die Buchung gemäss Serien Pattern Einstellungen frei (löscht die entsprechenden Buchungen)
    • nicht vorhandene Buchungen werden ignoriert
  • Buchen
    • bucht die freigegebenen Buchungen gemäss Serienpattern als Einzel-Tages-Buchungen
    • bereits vorhandene Buchungen werden ignoriert (ist an einem Tag bereits eine Buchung vorhanden wird diese nicht erneut gebucht).
    • Bei Fehler: Ausgabe im Actionpanel
  • Künftig freigeben:
    • Buchungen welche bei künftiger Erstellung freigegeben werden sollen => im Moment wo der Service neue Buchungen erstellt, müssen diese Freigaben berückscihtigt werden (z.B. jeden Freitag)
