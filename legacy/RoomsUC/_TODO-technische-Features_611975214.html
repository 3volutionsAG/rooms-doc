CS Spez:

Every cost field in the entire application will be adjusted to hold not only a numeric value but also a dropdown offering the selection of a currency. The currency list will consist of the currency of each culture configured in the application sorted alphabetically. When entering a new cost value the currency will be preselected according to the culture selected in the user profile.

The submitted values will be stored with the currency information and each calculation throughout the application will take into account the currency information but only values from the same currency will be calculated together. This means that no currency conversion will be performed and therefore no conversion rates need to be maintained. Whenever values from different currencies should be calculated together (e.g. when showing a total) a replacement text such as “m.c.i.” (Multiple currencies involved) will be displayed instead.


The text displayed (as an example “m.c.i.” is used) can be replaced in the translation settings by any content desired but the space for the text is very limited so for layout reasons only a short text should be used.
