Use Cases : 120.001 Anlasstyp verwalten

Created by Former user (Deleted), last modified on Apr 11, 2014

Use Case Überblick

 

ID

120.001

Owner

Tim Bänziger

Rev-Number

1.1

Priorität

3

Ziel & Beschreibung

Request Formulare können auf Basis von Anlasstypen verwaltet werden

Rechte

  • Darf Anlasstypen verwalten
  • Darf Reservationsanfragen administrieren

Bedingungen

–

Hauptszenario in Schritten

  1. Nutzer navigiert auf Einstellungen >Stammdaten für Anlässe>Anlasstypen verwalten
  2. Nutzer gibt Suchkriterien ein (optional) und klickt Button „Finden“
  3. Ein Suchresultat über alle Anlasstypen wird angezeigt – ein oder mehrere Anlasstypen können gelöscht werden
  4. Nutzer kann einen Eintrag anklicken, um die Details anzuzeigen (View Modus)
  5. Nutzer editiert oder erstellt einen Anlasstyp (Stammdaten)
  6. Nutzer editert Request Formular
  7. Liste aller zugefügten Benutzergruppen wird angezeigt - ein oder mehrere Benutzergruppen können entfernt werden
  8. Benutzergruppe kann zu Anlasstyp hinzugefügt werden
  9. Standortverknüpfung (aus UC 103.001 Standorte verwalten)
    1. Liste aller Anlasstypen wird angezeigt - eine oder mehrere Anlasstypen können entfernt werden
    2. Ein Anlasstyp kann zum Standort hinzugefügt werden

Beziehungen zu UC


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Schritt 5 - Nutzer editiert oder erstellt einen Anlasstyp

Controls

Inhaltsbereich – Tab „Stammdaten“

  • Feld: Bezeichnung
  • Textarea: Zusatz
  • Checkbox: Anfragen für Jobmanager
  • Intelibox: Gruppe für Notifikationen
  • Sonderzimmer


  • Button: Speichern
  • Button: Abbrechen

Regeln

Bezeichnung

  • Erfassung muss mehrsprachig erfolgen

Zusatz

  • Erfassung kann mehrsprachig erfolgen
  • Zusatz hat keine Funktionalität und wird ausser im Bereich der "Anlasstyp verwaltung" nicht visualisiert (dient zu Export Zwecken)

Als Job erstellen

  • Wenn diese Checkbox selektiert ist und ein resp. mehrere Reservierungsformulare dazu hinterlegt sind, dann:
    • wird beim ausfüllen und speichern eines entsprechenden Reservierungsformulars ein Eintrag in die Kommunikationstabelle (Tabelle: Lektion) geschrieben. Im Jobmanager findet sich dies dann als Job zur Weiterverarbeitung wieder.

 

Abbrechen-Button

  • geht zurück zum Ursprungsort

Button Speichern

  • Das Actionpanel bestätigt die Speicherung.
  • Rücksprung auf den Ursprungsort

Schritt 6 - Nutzer editert Request Formular

Controls

Inhaltsbereich – Tab „Reservationsanfrage“

  • Dropdown: Benutzergruppe für Notifikation
  • Textarea: Detail der Reservationsanfrage
  • Controlliste: ROOMS Fieldset Controls** Titel** Anlasstyp** Bemerkungen** Datum** Standort* Raum Bezeichnung** Priorität
    • Anzahl Teilnehmer
    • Anzahl Räume
    • Minimale Personenzahl
    • Verantwortlicher
    • Ersteller
    • Organisator
    • Kostenstellenpicker
    • Catering Picker
  • Controlliste: Andere Controls
    • Custom Textbox
    •  Custom Dropdown

Button Bereich

  • Button: Speichern
  • Button: Abbrechen

Regeln

Generell

  • Darf Reservationsanfragen administrieren

Benutzergruppe

  • Benutzergruppe, welche notifiziert wird nach Versenden des Request Formulars (siehe UC 215.001 Reservationsanfrage erfassen) 
  • Notifikationstemplate: 11 - ReservationsanfrageNeu (siehe Notifikationsmatrix) muss bei Benutzergruppe hinterlegt worden sein

Textarea

  • Freie HTML und JavaScript Eingabe für Gestaltung des Request Formulars
  • Platzierung der Controls aus den Controllisten möglich
  • Beginn und Ende des Blocks müssen die HTML Tags <customHtml> und </customHtml> verwenden

Titel

  • wird im Request Formular als Eingabefeld dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • MUSS Feld
  • wird in das Feld "Event Name" des Anlass übernommen - bei der automatischen Anlass Erstellung (siehe 215.001 Reservationsanfrage erfassen)

Anlasstyp

  • wird im Request Formular als Label dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • wird in das Feld "Anlasstyp" des Anlass übernommen - bei der automatischen Anlass Erstellung (siehe 215.001 Reservationsanfrage erfassen)

Bemerkungen

  • wird im Request Formular als Textarea dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • wird in das Feld "Beschreibung" des Anlass übernommen - bei der automatischen Anlass Erstellung (siehe 215.001 Reservationsanfrage erfassen)

Datum

  • wird im Request Formular als Datumpicker dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • MUSS Feld
  • Standardwert: Aktueller Tag
  • wird in das Feld "Datum" des Anlass übernommen - bei der automatischen Anlass Erstellung (siehe 215.001 Reservationsanfrage erfassen)

Standort

  • wird im Request Formular als Standortpicker dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • MUSS Feld
  • wird in das Feld "Standort" des Anlass übernommen - bei der automatischen Anlass Erstellung (siehe 215.001 Reservationsanfrage erfassen)

Raum Bezeichnung

  • wird im Request Formular als Eingabefeld dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)

Priorität

  • wird im Request Formular als Dropdown dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • Beinhaltet die Werte
    • Termin
    • Standort
  • MUSS Feld

Anzahl Teilnehmer

  • wird im Request Formular als Eingabefeld dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • Validierung auf Zahl
  • Standartwert Kapazität aus dem Profil

Anzahl Räume

  • wird im Request Formular als Eingabefeld dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • Validierung auf Zahl
  • Default: 1

Minimale Personenzahl

  • wird im Request Formular als Eingabefeld dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • Validierung auf Zahl
  • Standartwert Kapazität aus dem Profil

Verantwortlicher

  • wird im Request Formular als Intelibox Personen dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • MUSS Feld
  • im Fehlerfall wird eine Feldbezogene Fehlermeldung ausgegeben UND die angemeldete Person automatisch eingetragen
  • wird in das Feld "Verantwortlicher" des Anlass übernommen - bei der automatischen Anlass Erstellung (siehe 215.001 Reservationsanfrage erfassen)

Ersteller

  • wird im Request Formular als Label dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • keine Änderungen möglich
  • wird in das Feld "Ersteller" des Anlass übernommen - bei der automatischen Anlass Erstellung (siehe 215.001 Reservationsanfrage erfassen)

Organisator

  • wird im Request Formular als Intelibox Personen dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • MUSS Feld
  • im Fehlerfall wird eine Feldbezogene Fehlermeldung ausgegeben UND die angemeldete Person automatisch eingetragen
  • wird in das Feld "Organisator" des Anlass übernommen - bei der automatischen Anlass Erstellung (siehe 215.001 Reservationsanfrage erfassen)

Kostenstellenpicker

  • wird im Request Formular als Kostenstellenpicker dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • wird in das Feld "Kostenstelle" des Anlass übernommen - bei der automatischen Anlass Erstellung (siehe 215.001 Reservationsanfrage erfassen)

Catering Picker

  • wird im Request Formular als Catering Picker dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • wird in das Feld "Catering" des Anlass übernommen - bei der automatischen Anlass Erstellung (siehe 215.001 Reservationsanfrage erfassen)

Status

  • wird im Request Formular als Dropdown dargestellt (siehe UC 215.001 Reservationsanfrage erfassen)
  • Folgende Status stehen zur Verfügung
    • Neu (bei neuer Reservationsanfrage wird dieser Status automatisch gesetzt - kann nicht geändert werden)
    • in Verarbeitung (manuelle Änderung möglich - benötigt das Recht: Darf Reservationsanfragen bearbeiten)
    • Bestätigung erwartet (manuelle Änderung möglich - benötigt das Recht: Darf Reservationsanfragen bearbeiten)
    • Gebucht (manuelle Änderung möglich - benötigt das Recht: Darf Reservationsanfragen bearbeiten)

Control Aufbau
Beispiele:

  • <RoomsTextBox labelText="Views_Reservationsanfrage_Edit_Titel" name="Reservationsanfrage.Titel" hilfeText="Views_Reservationsanfrage_Edit_Titel_HilfeText" renderMode="FieldSet" />
  • <RoomsLabel labelText="Views_Reservationsanfrage_Edit_Anlasstyp" name="AnlasstypBezeichnung" renderMode="FieldSet" />
  • <RoomsTextArea labelText="Views_Reservationsanfrage_Edit_Kommentar" name="Reservationsanfrage.Kommentar" hilfeText="Views_Reservationsanfrage_Edit_Kommentar_HilfeText" renderMode="FieldSet" />
  • <RoomsTextBox labelText="Label Text To Replace" name="RequestFormular.UniqueLableName" />
  • <RoomsDropDown options="Value1, Value2, Value3" labelText="Label Text To Replace" name="RequestFormular.UniqueDropDownNameName" renderMode="FieldSet" />

Details:

  • rot = Control Typ
  • blau = Bezeichnungsfeld des Controls
    • kann in den Übersetzungen sprechend bezeichnet und in alle aktivierten Sprachen übersetzt werden
    • Label kann geändert werden, muss aber per Script durch GARAIO MItarbeiter in die Übersetzungen nachgetragen werden
  • violett = Eindeutiger Name des Controls
    • Muss eindeutig sein
    • Bei Mehrfachverwendung Namen ändern
    • Originalname dient für das Übernahme-Mapping bei der automatischen Anlasserstellung (siehe UC 215.001 Reservationsanfrage erfassen)
  • grün =HIlfetext des Controls
    • Optinal
    • kann in den Übersetzungen sprechend bezeichnet und in alle aktivierten Sprachen übersetzt werden
    • Label kann geändert werden, muss aber per Script durch GARAIO MItarbeiter in die Übersetzungen nachgetragen werden
  • orange = Darstellungsmodus
    • Fieldset = Control wird mit Label zusammen dargestellt
  • hellblau = Optionen der Custom Dropdown
    • Keine Übersetzungen
    • müssen Kommasepariert sein
  • Die Custom Controls können auf Zahlen, URL, E-Mail, Datum und Text validiert werden
