Use Cases : 400.004 Terminals verwalten

Created by Former user (Deleted), last modified on Aug 14, 2012

Ziel & Beschreibung

Terminals, wie elektronische Türschilder oder SPS Steuerungen können administriert werden

Rechte

  • Darf Terminals verwalten

Bedingungen

Terminaltypen müssen vorgängig implementiert werden.
Aktuell implementierte Typen:

  • SPS Steuerung

Hauptszenario in Schritten

  1. Nutzer navigiert auf Einstellungen >Erweiterte Basisdaten für Ressourcen >Terminals
  2. Nutzer gibt Suchkriterien ein (optional) und klickt Button „Finden“
  3. Ein Suchresultat über alle Terminals wird angezeigt – ein oder mehrere Terminals können gelöscht werden
  4. Nutzer kann einen Eintrag anklicken, um die Details anzuzeigen (View Modus)
  5. Nutzer editiert oder erstellt ein Terminal
  6. Liste der zugefügten Ressourcen wird angezeigt - eine oder mehrere Ressourcen können entfernt werden
  7. Ressource kann hinzugefügt werden

Schritt 7 - Ressource kann hinzugefügt werden

Controls

Modales Popup  „Ressourcen“

Standard Spalten:

  • Name
  • Bezeichnung
  • Standort hierarchisch

Optionale Spalten:

  • Ressourcenart
  • Status

Button Bereich

  • Feld: Identifikationsnummer
  • Button: Speichern
  • Button: Abbrechen

Regeln

Generell

  • alle verfügbaren Terminals werden gelistet
  • Die Smartlist verfügt über Spaltensuchfelder
  • Die Anzeige aller Spalten erfolgt in der Sprache des angemeldeten Benutzers

Identifikation

  • Eindeutige Kennung einer Terminal-Raum Kombination

Abbrechen-Button

  • schliesst Popup

Button Speichern

  • Mehrfachselektion erlaubt
  • Actionpopup wird geschlossen
  • Refresh Smartlist
  • Bestätigungsmeldung im Actionpanel, inkl. der Anzahl der zugefügten Einträge
