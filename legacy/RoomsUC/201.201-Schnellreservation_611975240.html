Ziel & Beschreibung

Die Schnellbuchung von einer oder mehreren Ressourcen, über Reservieren, Plan oder Kalender

Rechte

  • Darf Schnellreservation durchführen
  • DarfMassenmutationenVornehmen

Bedingungen


  • Nutzer befindet sich unter
    • Reservieren ODER
    • Kalender ODER
    • Plan ODER
  • Nutzer kommt vom Anlass und hat
    • Button "Ressource aus Liste dazu reservieren"
    • Button "Ressource aus Kalender dazu reservieren"

Hauptszenario in Schritten


Schnellreservation Einzeln
1. Nutzer selektiert die zu reservierenden Ressourcen und klickt auf Schnellreservation oder Anlass Schnellreservation
2. Actionpopup wird angezeigt und Nutzer gibt zusätzliche Informationen ein
3. Nutzer schliesst Reservation ab 
Schnellreservation Multiple
1. Nutzer selektiert eine oder mehrere zu reservierenden Ressourcen und klickt auf Schnellreservation oder Anlass Schnellreservation
2. Actionpopup wird angezeigt und Nutzer gibt zusätzliche Informationen ein
3. Nutzer schliesst Reservation ab

Use Case Details

Schnellreservation Einzeln

Schritt 1 - Nutzer selektiert die zu reservierenden Ressourcen und klickt auf Schnellreservation oder Anlass Schnellreservation

Controls

Benutzer befindet sich in einer der folgenden Ansichten

  • Reservieren - Schritte 1, 2 und 3 des UC 201.000 Ressourcen reservierensind ausgeführt worden
    • Button: Schnellreservation - auf jeder freien Ressource, SOFERN nicht deaktiviert (siehe auch 105.001 Ressourcen verwalten)
  • Plan
    • Klick auf eine freie Ressource zur Selektion
    • Klick auf ein Status Icon
  • Kalender
    • Klick auf freien Zeitpunkt im Kalender ohne vorgängiger Checkbox Selektion

Regeln

Rechte

  • Um Schnellreservationen auszuführen wird das folgende Recht benötigt: Darf Schnellreservation durchführen (Recht 112)

Bilder

Schritt 2 - Actionpopup wird angezeigt und Nutzer gibt zusätzliche Informationen ein

Controls

Inhaltsbereich (Hintergrund) gemäss Navigationsherkunft:

  • Reservieren - Liste (siehe UC 201.000 Ressourcen reservieren)
  • Kalender (siehe UC 213.001 Reservationen im Kalender suchen und buchen)
  • Plan (siehe UC 214.001 Reservationen im Plan suchen und buchen)

Action Popup

  • Datumpicker: Datum/Zeit von bis
  • Intelibox: Organisator
  • Textfeld: Titel
  • Intelibox: Verantwortlicher
  • Textarea: Bemerkungen
  • Checkbox: mit Outlook synchronisieren
  • Checkbox: E-Mail verschicken
  • Button: Reservieren

Regeln

Datum/Zeit von bis

  • Datum/Zeit von bis
  • Standardwert Datum/Zeit von: gemäss Selektion im Picker oder Kalender Selektion
  • Standardwert Datum/Zeit bis: gemäss Profil Standardwert Schnellreservation (siehe UC 101.001 Personen verwalten)

Organisator

  • Wird nur editierbar angezeigt mit Recht "Darf Reservationen mit Organisator buchen"
  • Intelibox über alle Personen
  • Standard = angemeldete Person
  • Klick auf Icon öffnet sich View Modus der Person als modales Popup.
  • Tooltip mit Zusatzinformationen auf Label (siehe dazu 000.001 Globale Funktionen und Konfigurationsparameter / Configuration parameter)


Titel

  • Anzeige gemäss globaler Konfiguration: SchnellreservationTitelEditierbar = True/False
  • Validierung auf MUSS Feld
  • Defaultwert = gemäss Schnellreservation (z.B. "Gebucht für PERSONENNAME")

Verantwortlicher

  • Wird nur angezeigt mit Recht "Darf Kontaktperson auf Reservation setzen"
  • Intelibox über alle Personen
  • Standard = leer (nicht abgefüllt)
  • Klick auf Icon öffnet sich View Modus der Person als modales Popup.
  • Tooltip mit Zusatzinformationen auf Label (siehe dazu 000.001 Globale Funktionen und Konfigurationsparameter / Configuration parameter)
  • Ist ein Verantworlicher vorhanden, werden die E-Mail Notifikationen (sofern im Profil aktiviert) an den Verantworlichen geschickt und NICHT mehr an den Organisator

Bemerkungen

  • Anzeige gemäss globaler Konfiguration: SchnellreservationBemerkungenEditierbar = True/False
  • freie Eingabe von Bemerkungen
  • entspricht dem Feld "Bemerkungen" in der Detailreservation

mit Outlook synchronisieren

  • wird nur angezeigt, wenn die Outlook Sync im Profil des Benutzers aktiviert ist
  • nur bei Ressourcenart "Raum" verfügbar
  • Standardwert = selektiert

E-Mail verschicken

  • Standardwert = selektiert
  • Selektion = E-Mails werden gemäss Profileinstellungen des Erstellers, Organisators, bzw. Verantwortlicher verschickt.
  • Nicht selektiert = E-Mail Versand wird unterdrückt


Zusätzliche Daten

  • Wurde die Schnellbuchung aus dem Kalender ausgeführt und es wurde in der "Free/Busy" Teilnehmer Erfassung Personen ausgewählt, werden diese automatisch mit eingeladen (gespeichert als Teilnehmer)
  • siehe auch UC 213.001 Reservationen im Kalender suchen und buchen

Reservieren

  • SOFERN nicht deaktiviert (siehe auch 105.001 Ressourcen verwalten)
  • siehe Schritt 3


Bilder

Schritt 3 - Nutzer schliesst Reservation ab

Controls

siehe Schritt 2

Regeln

Ablauf

  1. Reservation wird gespeichert
    1. der Titel wird automatisch vergeben - siehe hierzu den entsprechenden Eintrag „Schnellreservation“ im unter 000.001 Globale Funktionen bezüglich Handling der Standard-Werte
    2. Kostenträger werden gemäss Profil gesetzt
    3. Status (Workflow) wird gemäss der Ressource hinterlegtem Workflow (siehe UC 108.001 Workflows verwalten) und den Berechtigungen (siehe UC 109.005 Berechtigungen erteilen und entfernen) gesetzt
  2. Fehlerfall
    • Ausgabe der Fehlermeldung im Actionpanel
    • Buchung wird entfernt
  3. Erfolgsfall
    1. Bestättigungsmeldung im Actionpanel (Reservation wurde erstellt)
    2. Buchung wird im Kalender visualisiert
    3. Buchung wird im Plan visualisiert
  4. Rücksprung:
    1. Buchung aus Event => Rücksprung zum ursprünglichen Event (Startpunkt)
    2. Direkt Buchung => Kein Rücksprung

Schritt 3 - Nutzer schliesst Reservation ab

Controls

Inhaltsbereich gemäss Navigationsherkunft:

  • Reservieren - Liste (siehe UC 201.000 Ressourcen reservieren)
  • Kalender (siehe UC 213.001 Reservationen im Kalender suchen und buchen)
  • Plan (sihe UC 214.001 Reservationen im Plan suchen und buchen)

Regeln

Ablauf

  1. Anlass wird gespeichert
    1. Anlasstyp wird übernommen
    2. der Titel wird automatisch vergeben - siehe hierzu den entsprechenden Eintrag „Schnellreservation“ im unter |||\000.001 Globale Funktionen bezüglich Handling der Standard-Werte
    3. Bemerkungen werden auf Anlass gespeichert
    4. Buchungsklassifikationen müssen übernommen werden und auf alle Buchungen übertragen werden
      1. Bedingung: die Klassifikationen können auf den Reservationen auch gebucht werden. Falls nicht, werden diese nicht gesetzt - es wird keine Fehlermeldung oder Warning ausgegeben. 
    5. Organisator wird übernommen
  2. Reservation wird gespeichert
    1. der Titel wird automatisch vergeben - siehe hierzu den entsprechenden Eintrag „Schnellreservation“ im unter |||\000.001 Globale Funktionen bezüglich Handling der Standard-Werte
    2. Kostenträger werden gemäss Profil gesetzt
    3. Status (Workflow) wird gemäss der Ressource hinterlegtem Workflow (siehe UC108.001 Workflows verwalten und den Berechtigungen (siehe UC 109.005 Berechtigungen erteilen und entfernen) gesetzt
  3. Fehlerfall auf Ebene Anlass
    • Ausgabe der Fehlermeldung im Actionpanel
    • Buchung wird entfernt
  4. Fehlerfall auf Einzelreservation
    • Fehlerhafte Reservation wird rot markiert (Datum Fehler)
    • Schnellreservation kann nicht abgeschlossen werden
    • Es wird keine Reservation gespeichert.
  5. Erfolgsfall
    1. Bestättigungsmeldung im Actionpanel (Reservation wurde erstellt)
    2. Buchung wird im Kalender visualisiert
    3. Buchung wird im Plan visualisiert
  6. Rücksprung:
    1. Buchung aus Event => Rücksprung zum ursprünglichen Event (Startpunkt)
    2. Direkt Buchung => Kein Rücksprung
