Use Cases : 201.004 Serienbuchungen erstellen

Ziel & Beschreibung

Erstellung und Verwaltung einer Serienreservation auf einer Ressource

Rechte

  • Darf Reservationen als Serie buchen
  • Darf Reservationen als Serie ergänzen

Bedingungen

Hauptreservation wurde erstellt 201.000 Ressourcen reservieren

  1. Grundsätzlich kann auf jeder Ressourcenart eine Serie erstellt werden.
  2. Datenberechtigungen müssen überprüft werden (bei Alternativen)
  3. Tab „Serieninformation“ muss an zweiter Stelle erstellt werden
  4. Serie kann pro Benutzergruppe maximiert werden (z.B. 50 Iterationen) - siehe UC 109.003 Benutzergruppen verwalten. Die Maximierung erfolgt auf Iterationen oder Laufdauer unterteilt in
    1. Monate
    2. Wochen
    3. Tage
  5. Serie-Modifikation: Gleiches Hauptszenario, aber die Serieninformationen können nicht mehr verändert werden (locked) nur noch das Datum.
    1. Wird das Datum verlängert, werden die verfügbaren buchbaren Ressourcen der Serie hinzugefügt
    2. Wird das Datum verkürzt, werden die bereits gebuchten Reservationen (welche in der Zukunft liegen) annulliert.
  6. „Reservation abschliessen“ speichert die verschiedenen Reservationen definitiv.
  7. Wird ein Serieeintrag zeitlich verschoben oder auf eine andere Ressource verschoben, bleibt diese Teil der Serie, wird aber als Exeption behandelt

Hauptszenario in Schritten

  1. Benutzer navigiert auf Tab „Serienbuchung"
  2. Benutzer erfasst Serienkriterien und klickt „Erstellen"
  3. System liefert Resultat
  4. Benutzer kann Alternativen wählen oder Einzelbuchung hinzufügen
  5. Benuzter kann Serie verlängern
  6. Serie wird abgeschlossen
  7. Summarypage wird angezeigt

    • Option Tag

      • Dropdown mit Werten: Montag, Dienstag, Mittwoch, Donnerstag, Freitag, Samstag, Sonntag

      • Feld: Monat

Regeln

Kopfbereich

  • Option "Enden am" ist selektiert
  • Datumpicker Default: heute() + 30 (Tage)
  • Option "Enden nach" per Standard leer
  • Klick in das Feld "Enden nach" aktiviert, Option "Enden nach"

Muss Felder/Feldvalidierungen bei

  • Täglich
    • Feld „Tag(e)“ muss Ganzzahl beinhalten
    • Standardwert: alle 1 Tag
  • Wöchentlich
    • Feld „Wiederholen“ muss eine Ganzzahl beinhalten
    • Min. eine Checkbox der Wochentage muss selektiert sein
    • Min. der Tag der Seriestart (z.B. Mittwoch) muss selektiert sein. Diese Selektion kann nicht entfernt werden
    • Standardwert: Wiederholung alle 1 Woche am #Selektion des Wochentags#
  • Monatlich
    • Option „Tag“ muss Ganzzahl beinhalten – Monat muss Ganzzahl beinhalten
    • Option „Am“, Eintrag aus Listen und „Monat“ muss Ganzzahl beinhalten.

Schritt 3 - System liefert Resultat

Controls

Smartliste im Inhaltsbereich

  • Checkbox für Mehrfachselektion
  • Icon: Prüfstatus
  • Ressourcenbezeichnung (in der Sprache des angemeldeten Benutzers)
  • Icon "Alternative"
  • Von (Datum/Zeit der Buchung)
  • Bis (Datum/Zeit der Buchung)

Buttonbereich

  • Ergänzen
  • Löschen
  • Stornieren (Annulieren)
  • Checkbox: Vorhergehende Reservationen anzeigen

Regeln

Generell nach Klick auf "Erstellen"

  • Ausgabe des Hinweistextes: Serie wird erstellt
  • Es werden "Blocker" Systemreservationen erstellt
  • Seriebeschränkung:
    • Pro Benutzergruppe kann eine Beschränkung der Serie hinterlegt werden (wurde in 109.003 Benutzergruppen verwalten nachgeführt)
    • Die Beschränkung kan unterschiedlich für Tage, Wochen und Monate erstellt werden
    • Die Beschränkung ist unabhängig vom gewählten Modell der Seriebeschränkung (nach "Iteratioinen" oder nach "Laufdauer" - siehe Konfigurationsparameter / Configuration parameter)
    • Wert muss eine Ganzzahl grösser 0 sein.
    • Nach Klick auf Erstellen werden nur soviele Serie-Items erstellt, wie es die Beschränkung auf der gewählten Option zulässt (z.B. für 30 Tage)

Generell Smartlist

  • Smartlist kann nicht sortiert werden
  • Es steht kein Paging zur Verfügung => es werden immer alle Serieneinträge angezeigt
  • Es können keine zusätzlichen Spalten gewählt werden oder die Reihenfolge geändert werden

Mehrfachselektion

  • Mehrfachselektion zur Entfernung eines Serientermins
  • Aktueller Serieneintrag hat keine Checkbox (bei Neureservation ist dies immer der Erste)
  • Serieneinträge in Vergangenheit können nicht mehr entfernt werden

Prüfstatus
Es gibt 3 Zustände einer möglichen Reservation

  • Grün: alles konnte analog der Sourcereservation gebucht werden (auch Equipment und Dienstleistungen)
  • Orange: Basisreservation konnte gebucht werden, aber (Ausgabe nachstehender Fehlermeldungen)
    • es steht nicht alles Equipment zur Verfügung
    • es stehen nicht alle Dienstleistungen zur Verfügung
    • Reservation ist an einem Feiertag
  • Rot: Basisreservation konnte nicht erstellt werden, weil (Fehlermeldungen)
    • Ressource besetzt
    • Ressource gesperrt

Im Tooltipp wird der jeweilige Zustand in Textform asugegeben

Bezeichnung

  • Im Viewmodus (Summary) ist die Bezeichnung ein Link, welcher erlaubt direkt in die gewählte Reservation abzuspringen. Dabei werden auf der Hauptreservation keine Änderungen gespeichert. Der Benutzer wird gewarnt (Sicherheitsabfrage)

Alternative

  • Serieneinträge in Vergangenheit können nicht mehr edtiert werden => Icon wird ausgeblendet
  • Alternativen können in jedem Prüfstatus gesucht werden (auch bei "grün")
  • Es öffnet sich ein modales Popup => siehe Schritt 4

Von

  • Ausgabe von Datum/Zeit
  • Format: dd.mm.yyyy HH:MM

Bis

  • Ausgabe von Datum/Zeit
  • Format: dd.mm.yyyy HH:MM

Serie ergänzen

  • Sichtbar, wenn Recht "Darf Reservationen als Serie ergänzen" vorhanden ist.
  • Eine einzelne Reservation kann hinzugefügt werden
  • öffnet modales Popup (siehe Schritt 4)

Löschen

  • Warnhinweis: Sind Sie sicher, dass der Serieneintrag entfernt werden soll?
  • Löschung des Serieneintrags (unwiederruflich)
  • Aktualisierung der Smartlist
  • Bestätigungsmeldung im Actionpanel

Stornieren

  • Nicht sichtbar bei Neubuchung
  • Warnhinweis: Sind Sie sicher, dass der Serieneintrag entfernt werden soll?
  • Annullation des Serieneintrags
  • Aktualisierung der Smartlist
  • Bestätigungsmeldung im Actionpanel

Vorhergehende Reservationen anzeigen

  • zeigt alle vorhergehenden Reservationen: solche in Vergangenheit (Reservationsdatum < heute) und solche zwischen dem aktuellen Zeitpunkt und dem Zeitpunkt der aktuell bearbeitenden Reservation

Schritt 4 - Benutzer kann Alternativen wählen oder Einzelbuchung hinzufügen

Controls

Modales Popup

  • Titel: Alternative wählen
  • Datumpicker: Datum/Zeit
  • Ressourcenpicker: Ressource

Button Bereich

  • Button „Speichern“
  • Button „Abbrechen“

Regeln

Generell

  • "Vorhergehende Reservationen" können nicht mehr editiert werden (Alternativen gebucht werden)
  • „Alternativen suchen“ und „Serie ergänzen“ haben die gleiche Funktion:
    • Eine Reservation auf derselben Ressourcen, wie die Source-Ressource aber zu einem anderen Datum und Zeit zu erstellen UND/ODER
    • Eine Reservation zur selben Datum/Zeit zu erstellen, aber an einem anderen Standort/Ressource

Datum/Zeit Picker

  • siehe 000.001 Globale Funktionen

Standort Ressourcen Picker

  • Zeigt nur verfügbare Ressourcen
  • Zeigt nur Ressourcen der gleichen Art (z.B. nur Räume)
  • Zeigt bei Ressourcenart "Raum" nur Räume mit gleicher Kapazität

Button „Speichern“

  • Prüfung ob die Regeln der Seriebeschränkung noch eingehalten werden
  • Fügt gewählte Reservation zu Liste hinzu und erstellt eine Systemreservation
  • Refresh der Smartlist
    • Anpassung Ressourcen Bezeichnung
    • Anpassung Datum/Zeit
    • Anpassung Prüfstatus
  • Bestättigungsmeldung im Actionpanel

Button „Abbrechen“

  • Schliesst Popup und wechselt auf Smartlist

Schritt 5 - Benuzter kann Serie verlängern

Controls

Kopfbereich

  • Label: Serie Start
  • Label: Start Datum und Start Zeit (aus Feld Datum/Zeit von Tab "Allgemein" - siehe UC 201.000 Ressourcen reservieren)
  • Optionen
    • Enden am + Datumpicker
    • Enden nach + Feld: Iterationen + Label: Iterationen

Button Bereich

  • Button „Ergänzen“

Regeln

Generell

Datum/Zeit Picker

  • Option "Enden am" ist selektiert ODER Option "Enden nach" ist selektiert
  • Standard wird das gewählte Datum oder die Anzahl der Eingegebenen Iterationen angezeigt
  • Durch Eingabe
    • eines weiter in Zukunft liegenden Datum kann die Serie verlängert werden
    • einer höheren Anzahl an Iterationen kann die Serie verlängert werden

Button „Ergänzen“

  • Ergänzt die Serie mit Systemblocker gemäss dem auf der Ursprungsreservation definierten Serienpattern (z.B. jeden zweiten Mittwoch)
  • Die neuen Einträge werden immer ergänzt. Es werden NIE Lücken in der bestehenden Serie mit neuen Reservationen gefüllt
  • Die neuen Einträge sind Teil der Serie
  • Speicherung erfolgt bei Speicherung des Reservationseintrages (siehe Schritt 6)

Schritt 6 - Serie wird abgeschlossen

Controls

Button Bereich der Hauptreservation

  • Checkbox: Auf nachfolgende Serieneinträge übernehmen

Regeln

Änderung auf Serie übernehmen

  • nur sichtbar im Editer Mode (nicht bei Neuerstellungen)
  • übernimmt bei Klick alle Änderungen der aktuellen Reservation (Masterreservation) auf alle künftigen Serie Items
  • Bei Bedarf wird die Serie aufgeteilt: ine Aufteilung erfolgt, wenn mindestens eine vorhergehende Reservation existiert und Zeit, Pattern, Ressource, Titel, Organisator (darf neu verändert werden!), Teilnehmer oder Body-Text geändert und "Auf nachfolgende..." geklickt wird. Dabei erhält der Benutzer beim Klick auf "Bestätigen" (vor Wechsel auf Summary-Page) eine MessageBox eingeblendet, die ihn über die Aufteilung informiert ("Sie haben Änderungen an der Serie vorgenommen, die eine Aufteilung der Serie erfordern. Alle Serieneinträge ab dem XX.XX.XXXX werden dazu zu einer neuen Serie zusammengefasst. Die Teilnehmer der Serie erhalten für diese Termine zuerst einzelne Absagen und danach für die neue Serie eine einzelne neue Einladung. Wollen Sie die Serie wie beschrieben aufteilen?").
  • Ausnahmen (Exeptions) innerhalb einer Serie (andere Ressource oder andere Zeit) unterliegen folgenden Zusatzregeln:
    • Übernahme Catering erfolgt Additiv
    • Bestuhlung wird geändert
    • Teilnehmer Änderungen erfolgen immer auf der gesamten Serie
    • Ressourcenänderungen werden auf die gesamte Serie übernommen
    • Allgmeine Änderungen werden auf die ganze Serie übernommen

Schritt 7: Summary Page wird angezeigt

Controls

Summarypage

Panel "Serieninformationen" unterhalb der "allgemeinen Informationen" mit Spalten

  • Prüfstatus
  • Bezeichnung
  • Reservation Beginn
  • Reservation Ende

Button Bereich

  • Checkbox "Serie zum Anlass hinzufügen"

Regeln

Prüfstatus

  • aus Schritt 3

Bezeichnung

  • Bezeichnung der Ressource in der Sprache des angemeldeten Benutzers

Reservation Beginn

  • Ausgabe von Datum/Zeit
  • Format: dd.mm.yyyy HH:MM

Reservation Ende

  • Ausgabe von Datum/Zeit
  • Format: dd.mm.yyyy HH:MM

Serie zum Anlass hinzufügen

  • Wurde die Serie aus einem Anlass erstellt (Klick im Anlass auf "Ressourcen zu Anlass reservieren" (Liste und Kalender) und dann ins Detail einer neuen Reservation gewechselt und eine Serie erstellt), wird Checkobox eingeblendet ODER
  • wurde eine Einzelreservation, welche bereits zu einem Anlass gehört, nachträglich in eine Serie gewandelt, wird die Checkbox eingeblendet
  • Standardwert: Checkbox = selektiert
  • Wenn geklickt werden alle erstellten Buchungen mit dem Anlass verknüpft
