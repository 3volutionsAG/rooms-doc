Use Cases : 101.010 Benutzer Selbstregistration

Ziel & Beschreibung

Ein Benutzer kann sich am System selbtständig registrieren und über Forms Authentication anmelden

Rechte

 

Bedingungen

  • Globale System Einstellung "Darf Forms-Benutzer erstellen" muss aktiviert worden sein (Einstellung: TRUE) siehe 000.004 Konfigurationsparameter
  • Empfänger für Mail Notifikation muss global hinterlegt werden: Vorlagebenutzer für Anmeldeformular (siehe 000.004 Konfigurationsparameter)

Hauptszenario in Schritten

  1. Nutzer gibt URL von ROOMS ein - Anmeldefenster wird angezeigt
  2. Nutzer gibt Registrationsformular Daten ein

Beziehungen von/zu UC


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Use Case Details

Schritt 1 - Nutzer gibt URL von ROOMS ein - Anmeldefenster wird angezeigt

Controls

Inhaltsbereich

  • Feld: Benutzername
  • Feld: Passwort
  • Button: Anmelden
  • Link: Kein Zugang? Klicken Sie auf Account erstellen...

Regeln

Passwort

  • eingabe erfolgt anonymisiert (nur Punkte)

Anmelden

  • Verifiziert Benutzername und Passwort
  • Falls OK => Zutritt aufs System gemäss Standard Einstellungen des Benutzers (siehe UC 101.001 Personen verwalten)
  • Falls NOK => Feldbezogene Fehlermeldung

Kein Zugang? Klicken Sie auf Account erstellen...

  • siehe Schritt 2

Bilder

 

Schritt 2 - Nutzer gibt Registrationsformular Daten ein

Controls

Sidepanel

  • Feld: Benutzername
  • Feld: Passwort
  • Feld: Passwort erneut eingeben
  • Feld: Firma
  • Dropdown: Geschlecht
  • Feld: Vorname
  • Feld: Name
  • Feld: E-Mail
  • Feld: Adresse 1
  • Feld: Adresse 2
  • Feld: PLZ
  • Feld: Ort
  • Feld: Land
  • Button: Registrieren

Regeln

Geschlecht

Dropdown mit den folgenden Werten:

  • keine Angabe
  • männlich
  • Weiblich

 


Registrieren
Fehler:

  • Ausgabe einer Feldbezogenen Fehlermeldung:
    • Rahmen Control wird rot eingefärbt
    • Fehlerdreieck rechts des Controls positioniert
    • Mouseover gibt sprechende Fehlermeldung aus
  • Fehlerhafte Felder werden geleert
  • Restliche Fehler bleiben bestehen

Erfolgreich:

  • Benutzer wird automatisch im System erstellt
  • Benutzer wird Standardbenutzergruppe zugeordnet (siehe Konfigurationsparameter / Configuration parameter)
  • Benutzer wird automatisch der Firma/Oe zugeordnet
  • E-Mail Notifikation wird an Verantwortliche Person verschickt (siehe Bedingungen in diesem UC)

  • Benutzer erhält Zugang zum System
