Use Cases : 107.001 Notifikationen verwalten

Ziel & Beschreibung

Die Erfassung und Verwaltung von Notifikationsvorlagen für die verschiedenen Anwendungszwecken.
Die Aktivierung der Notifikationsvorlagen an den richtigen Stellen (Standorte oder Benutzergruppen)
Das Versenden der Notifikation

Rechte

  • Darf Notifikationen verwalten

Bedingungen

Folgende Punkte werden in anderen Use Cases beschrieben

  • Benutzer kann Reservationsnotifikationen unterdrücken (201.000 Ressourcen reservieren )
  • Benutzer reserviert Dienstleistungen (201.003 Angebote hinzufügen, entfernen und  UC 211.001 Anlass erstellen )
  • Notifikation unterdrücken bei Synchronisationsfehler
  • Änderungen auf einer Ressource (UC 107.003 Vorkonfiguriertes Mail versenden )
  • Details sind dem Notifikationsmatrix zu entnehmen 000.006 Notifikationsmatrix
  • Folgende UC müssen vorgängig ausgeführt worden sein
    • 103.001 Standorte verwalten (Schritte 1, 2 und 4)
    • 109.003 Benutzergruppen verwalten (Schritte 1, 2, 3 und 5)

Hauptszenario in Schritten

  1. Navigation auf Tab "Notifikationen"
  2. Liste aller verfügbaren Notifikationen wird angezeigt - eine oder mehrere Notifikationen können gelöscht werden
  3. Klick auf Hinzufügen (oder Edit Icon) kann eine Notifikation neu erstellt, bzw. bearbeitet werden 
  4. Notifikationen werden versendet

Regeln

Hinzufügen/Edit Icon

  • öffnet modeles Popup (siehe Schritt 3)

Entfernen

  • Es kann/können ein oder mehrere Notifikationen selektiert werden
  • Sicherheitsabfrage
  • Löscht die entsprechende( n ) Notifikation(en)
  • Bestätigungsmeldung inkl. Der Anzahl der Entfernungen im Actionpanel
  • Smartlist wird neu geladen

Notifikationen auf Standort

 

Schritt 3 - Klick auf Hinzufügen (oder Edit Icon) kann eine Notifikation neu erstellt, bzw. bearbeitet werden

Controls

Modales Popup

  • Dropdown: Notifkationstyp (siehe Notifikationsmatrix)
  • Feld: Benachrichtigung im CC an
  • Dropdown: Sprache
  • Feld: Titel (Subject)
  • Editor/Textarea: E-Mail Body
  • Textarea: SMS Body (Text)
  • Platzhalter Liste (siehe Notifikationsplatzhalter)

Button Bereich

  • Speichern
  • Abbrechen

Regeln

Notifikationstyp

  • Jeder Notifikationstyp darf nur einmal erfasst werden.
  • Die verfügbaren Notifikationstemplates pro Typ "standortspezifische" oder "gruppenspezifische"  und deren Funktionsverhalten ist in Notifikationsmatrix beschrieben

Benachrichtigung im CC an

  • Erfassung von gültigen E-Mail Adressen, welche beim Versenden der Notifikation  im CC an die entsprechenden Adressaten verschickt werden
  • Mehrere E-Mail Adressen werden mittels semikolon separiert

Sprache

  • Es stehen alle aktivierten Sprachen zur Verfügung

Titel (Subject)

  • Der Titel muss mehrsprachig erfasst werden

E-Mail Body

  • Wurde kein E-Mail-Server eingetragen, erscheint dieses Feld nicht.
  • Es steht ein Texteditor inkl. Formatierungen zur Verfügung
  • E-Mail Body kann mehrsprachig abgefüllt werden
  • Ist der E-Mail Body leer wird keine Notifikation verschickt
  • Es stehen alle Formatierungen gemäss Editor zur Verfügung

SMS Body

  • Wurde kein SMS-Gateway eingetragen, erscheint dieses Feld nicht.
  • Normale Textarea
  • Kann Mehrsprachig erfasst werden
  • Ist die Textarea leer wird keine SMS verschickt

Platzhalter Liste

  • Es steht eine Reihe von Platzhaltern zur Verfügung, welche zur Laufzeit mit dem entsprechenden Text abgefüllt werden.
  • Platzhalter können im Titel (Subject) und im E-Mail Body verwendet werden
  • Die Werte werden immer in der aktuellen Benutzersprache abgefüllt
  • Auf Klick eines Platzhalter wird dieser an der aktuellen Stelle im Editor platziert
  • Die verfügbaren Platzhalter sind in Notifikationsplatzhalter ersichtlich
  • Klick auf einen Platzhalter fügt diesen an der aktuellen Cursor Position ein. Ist kein Cursor positioniert wird der Platzhalter am Anfang des Body Textes eingefügt
  • Es ist immer nur eine Platzahlter Sektion gleichzeitig geöffnet.
  • Die Platzhalter übernehmen die Formatierung des Body Editors

Speichern

  • Popup wurde geschlossen
  • Template Konfiguration wird gespeichert
  • Bestätigungsmeldung im Action Panel
  • Refresh Smartlist

Abbrechen

  • Popup wird geschlossen

Schritt 4 – Notifikationen versenden

Controls

keine

Regeln

Das Regelwerk ist detailliert im Notifikationsmatrix beschrieben: Notifikationsmatrix

Gruppenspezifische Vorlagen

  • Muss durch Hinzufügen auf der Benutzergruppe aktiviert werden
  • Wird veschickt sobald eine Aktion des entsprechenden Template Types ausgelöst wird

Standortspezifische Vorlagen

  • Muss durch Hinzufügen auf dem Standort aktiviert werden
  • Auf Grund des Standorts der Ressource wird beim Auslösen der Aktion das entsprechende Template gesucht und verschickt
  • Wird auf dem zugehörigen Standort kein Template gefunden, wird auf dem übergeordneten Standort gesucht, wird wieder keines gefunden erfolgt die Suche wiederum auf dem übergeordeneten, usw.
  • Wird bis zum Root Standort kein entsprechendens Notifikationstemplate gefunden, wird nichts verschickt
  • Bei der Aufbereitung zum Versand einer Notifikation wird zusätzlich hierarchisch (gleichen oder übergeordneten Standort)  geprüft ob ein Template der entsprechenden Ressourcenart vorhanden ist. D.h. beim Versand der Buchung eines Arbeitsplatz wird geprüft ob hierarchisch das Notifikationstemplate 01b vorhanden ist. Falls
    • ja wird dieses verschickt
    • nein, wird die Prüfung nochmals durchgeführt auf das Haupttemplate (in diesem Fall 01)
      • ist dieses vorhanden, wird die Notifikation gemäss Haupttemplate verschickt => diese Regel existiert bereits
      • ist dieses nicht vorhanden wird keine Notifikation verschickt => diese Regel existiert bereits
