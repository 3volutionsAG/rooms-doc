Use Cases : 305.001 Outlook Plugin Scheduling Assistant

Created by Former user (Deleted), last modified on May 31, 2014

Use Case Überblick

 

ID

305.001

Owner

Former user (Deleted)

Rev-Number

1.0

Priorität

1

Ziel & Beschreibung

Benutzer sollen aus dem Outlook einen Raum reservieren können.

Rechte

  • Darf Reservationen für Ressourcetyp Arbeitsplatz buchen
  • Darf Reservationen für Ressourcetyp Equipment buchen   
  • Darf Reservationen für Ressourcetyp Fahrzeug buchen
  • Darf Reservationen für Ressourcetyp Parkplatz buchen
  • Darf Reservationen für Ressourcetyp Raum buchen
  • Darf Reservationen mit Vor-/Nachlaufzeit editieren (21) 

Bedingungen

Die Übersetzungen müssen lokal gespeichert werden. Dabei werden beliebige Sprachen unterstützt werden. Initial wird das Plugin zweisprachig (DE/EN) ausgeliefert

Hauptszenario in Schritten

  1. Benutzer erstellt Kalendereintrag (Appointement oder Meeting Request)
  2. Outlook Plugin wird geladen (inkl. Sprache)
  3. Benutzer erfasst (optional) Kalenderdaten und wechselt auf "Besprechungsplanung"
  4. Benutzer klickt ROOMS Sidepanel - verfügbare Ressourcenarten werden angezeigt
  5. System zeigt ROOMS Scheduling Planung inkl. Filterkriterien
  6. Benutzer erfasst Suchkriterien (optional) und klickt Finden
  7. System liefert Suchresultat über alle Ressourcen gemäss den Filterkriterien
  8. Spezialfall Serie
  9. Benutzer klickt auf "Abschliessen"
  10. Benutzer klickt „Save & Close" oder „Senden"
  11. System aktualisiert über Exchange die ROOMS Buchung
  12. Benutzer editiert Kalendereintrag
  13. Benutzer löscht Kalendereintrag

Beziehungen zu UC


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Schritt 3 - Benutzer erfasst (optional) Kalenderdaten und wechselt auf "Besprechungsplanung"

Controls

Sidepanel

Regeln

Benutzer kann Outlook Kalenderdaten erfassen, wie:

  1. Teilnehmer
  2. Bemerkungen (Body)
  3. Titel (Subject)
  4. Serien Informationen (Serien Informationen müssen zeitlich oder iterativ begrenzt sein, ansonsten meldet das Plugin einen Fehler).
  5. Im Tab "Besprechungsplanung" (Scheduling Assistant) können auch gleich Teilnehmer (interne/externe) erfasst werden, um deren Verfügbarkeit zu checken

Schritt 4 - Benutzer klickt ROOMS Sidepanel - verfügbare Ressourcenarten werden angezeigt

Controls

Buttons

  • Raum
  • Parkplatz
  • Arbeitsplatz
  • Fahrzeug
  • Equipment

Regeln

Es müssen zwei oder mehrere Ressourcenarten zur Buchung erlaubt sein (entsprechendes Ressourcen Recht vorhanden):

  • Trifft beides NICHT zu, wird folgender Text mehrsprachig ausgegeben: Sie verfügen nicht über die notwendigen Berechtigungen, um diese Funktion zu nutzen
  • Trifft der Fall zu, dass nur eine Ressourcenart aktiviert ist, dann wird der Benutzer automatisch auf Such- und Resultatseite weitergeleitet (weiter auf Schritt 5)
  • Klick auf einen Button, wechselt zu Schritt 5

 

Fehlerfall - Meldung wird ausgegeben

  • Offline: es ist keine Verbindung zu ROOMS vorhanden
  • Andere Verbindungsprobleme: es ist keine Verbindung zu ROOMS vorhanden

Schritt 6 - Benutzer erfasst Suchkriterien (optional) und klickt Finden

Controls

Plugin geöffnet:

  • Label: Raum reservieren – Eingabe der Suchkritieren
  • Intelibox: Standort
  • Picker: Gliederung
  • Picker: Buchungsklassifikationen
  • Checkbox: Suche ohne Substandorte
  • Checkbox: meine favorisierten Ressourcen
  • Zahlenpicker: neue Vorlaufzeit
  • Zahlenpicker: neue Nachlaufzeit

Button Bereich

  • Button: Finden
  • Button: Zurück

 

Sofern das Recht "Darf Reservationen mit Vor-/Nachlaufzeit editieren" (21) vorhanden ist wird im Filter Bereich zusätzlich die freie Eingabe einer Vorlaufzeit und Nachlaufzeit eingeblendet.

Die Angabe erfolgt in Minuten. Die eingegebene Zeiten übersteuern die in der Listenansicht errechnete Zeiten und werden effektiv für die Suche übernommen (0 oder Minuswerte werden nicht berücksichtigt. Eingabe von Buchstaben führen zu einem Validierungsfehler)

 

Regeln

Generell

  • Impliziter Filter auf die in Schritt 4 gewählte Ressourcenart

Standort

  • alle verfügbaren und berechtigte Standorte werden angezeigt

 

Gliederungen

  • Nur Ressourcenart Raum
  • Nur wenn Flag „sichtbar“ anzeigen
  • Gemäss interner Reihenfolge wird die erste Gliederung angezeigt
  • die restlichen Gliederungen können über den Link "mehr" aufgerufen werden

Buchungsklassifikationstypen (siehe UC 104.002 Buchungsklassifikationen verwalten)

  • werden in der Listenansicht angezeigt, sofern
    • der angemeldete Benutzer in einer der Buchungsklassifikationstypen zugeordneten Benutzergruppe zugeteilt ist
    • die Ressourcenart gewählt wurde, welche der Buchungsklassifikationstyp zugeordnet wurde - bei keiner Auswahl werden alle Buchungsklassifikationstypen
    • die Checkbox "sichtbar in Listen" auf dem Buchungsklassifikationstypen selektiert worden ist
  • Die Buchungsklassifikationstypen werden in Form von Dropdown oder Multiselektoren (sofern Checkbox "Mehrfachselektionen erlaubt" angeklickt worden ist)
  • Bei Mehrfachselektionen werden die angeklickten Buchungsklassifikationen bei der Suche mit einer AND Verknüpfung berücksichtigt.

 

Ressourcen Name/Bezeichnung

  • Suche nach über die ID der Ressourcen und über die Bezeichnung in der Benutzersprache des angemeldten Benutzers

Suche ohne Substandorte

  • Begrenzt die Suche auf den  aktuell gewählten Standort oder die aktuelle gewählten Standorte. Die jeweiligen Substandorte werden nicht berücksichtigt.

meine favorisierten Ressourcen

  • zeigt nur Ressourcen auf welchen die angemeldete Person jemals als Owner mindestens eine Buchung hatte (angemeldete Person = Organisator)
  • Berücksichtigt werden alle Ressourcenarten
  • Berückscihtigt werden Vergangenheit, Gegenwart und Zukunft

 

Neue Vorlaufzeit

  • Standardwert = 0
  • Wird nur eingeblendet mit Recht:  "Darf Reservationen mit Vor-/Nachlaufzeit editieren" (21)"
  • Eingabe erfolgt in Minuten
  • Die eingegebene Zeiten übersteuern die errechnete Zeiten und werden beim Systemblocker hinzugezählt
  • 0 oder Minuswerte werden nicht berücksichtigt
  • Eingabe von Buchstaben führen zu einem Validierungsfehler)

 

Neue Nachlaufzeit

  • Standardwert = 0
  • Wird nur eingeblendet mit Recht:  "Darf Reservationen mit Vor-/Nachlaufzeit editieren" (21)"
  • Eingabe erfolgt in Minuten
  • Die eingegebene Zeiten übersteuern die errechnete Zeiten und werden beim Systemblocker hinzugezählt
  • 0 oder Minuswerte werden nicht berücksichtigt
  • Eingabe von Buchstaben führen zu einem Validierungsfehler)

 

Finden

  • Suchdaten (Datum/Zeit, Gliederungen, Klassifikationen, Anzahl Personen, Standort) werden an Webservice von ROOMS übermittelt
  • Siehe Schritt 6

Schritt 7 - System liefert Suchresultat über alle Ressourcen gemäss den Filterkriterien

Controls

Scheduling geöffnet – Kalender (Definition, Navigation und Aufbau des Kalender in UC 213.001 Reservationen im Kalender suchen und buchen)

  • Bereich "Teilnehmer hinzufügen"
    • Bereits hinzugefügte Teilnehmer:
      • Vorname
      • Name
      • Zeitzone
      • Entfernen Icon
      • Free/Busy Informationen pro Person
    • Person Intelibox
  • Bereich "Ressourcen"
    • Farbcode
    • Raum Bezeichnung mit MouseOver Tooltipp der Raumeigenschaften
    • Optionsbutton für Selektion
    • Kalender mit allen getätigten Buchungen
  • Zeitraum-Highlighter (bestehend aus grüner (Start) und roter (Ende) Linie
  • Button "Abschliessen"
  • Buttoon "Zurück"

 

Regeln

Generell

  • Es werden nur Ressourcen der gewählten Ressourcenart angezeigt

Bereits hinzugefügte Teilnehmer

  • Icon "Entfernen" entfernt die selektierte Person aus der Liste
  • Free/Busy Informationen werden gemäss den vom Benutzer im Outlook definierten Berechtigungen des Benutzers geladen.
    Dabei ausschliesslich die Einstellungen des Standard (Default) Users:

 

Intelibox Person

  • Funktkionsweise der Intelibox ist in Inteliboxen & Multiselektoren beschrieben
  • Nach Auswahl einer Person, werden die Free/Busy Daten geladen gemäss Berechtigungen des Users (siehe oben)

Raum-Bezeichnung

  • Wird in der Sprache des Benutzers ausgegeben
  • MouseOver mit Ressourcen Informationen (Tooltipp mit Informationen zur Ressource, Konfiguration unter Konfigurationsparameter)

Optionsbutton zur Selektion

  • Selektion der Option
  • Modales Popup mit Meldung, dass die selektierte Ressource blockiert wird
  • Dabei wird in ROOMS (Webservice) die Ressource blockiert (Gebucht mit Status 1 - Systemreservation)
    • Titel (Wenn Titel leer wird „Gebucht für Organisator“ ins ROOMS übernommen)
    • Datum/Zeit von (werden durch den Highlighter definiert)
    • Datum/Zeit bis (werden durch den Highlighter definiert)
    • Vorlaufzeit von (wird berechnet oder übersteuert durch Feld "neue Vorlaufzeit" (siehe Schritt 6)
    • Nachlaufzeit bis  (wird berechnet oder übersteuert durch Feld "neue Nachlaufzeit" (siehe Schritt 6)
    • Mitreserviertes Equipment (Ressourcen) 
    • Anzahl Personen
  • Danach verschwindet das modale Popup automatisch
  • Subject von Outlook wird mit Ressourcen.Bezeichnung abgefüllt
  • Es kann nur eine Ressource selektiert werden
  • Wird nach einer Selektion eine andere Ressource selektiert, wird die erste wieder freigegeben (und der Blocker freigegeben)

 

Zeitrraum Highlighter

  • Grüne Linie = Startzeit
  • Rote Linie = Endzeit
  • Mouseover über eine Linie erlaubt die Linie zu verschieben => Zeitraum zu verkleinern oder zu vergrössern. Die Verschiebungszeiten werden in einem Tooltipp echtzeit angezeigt
  • Mouse zwischen den beiden Linien erlaubt beide Linien parallel zu verschieben
  • Der definierte Zeitraum wir als Buchungszeitraum bei Selektion des Optionsbuttons (siehe oben) übernommen.

Button Abschliessen

  • siehe Schritt 9

Kein Suchresultat

  • Bei keinem Treffer wird Meldung ausgegeben.

Schritt 11 – System aktualisiert über Exchange die ROOMS Buchung

Controls

Kein Inhalt

Regeln

Über die (ehem.) EventSink Komponente (Exchange) wird die Systemreservation in eine Definitive Reservation gewandelt.

Dabei werden folgende Aktualisierungen und Aktionen vorgenommen:

  1. System Reservation in Definitive Reservation
  2. System Reservation von mitreservierten Ressourcen (Equipment) in definitive Reservation
  3. Weitere Aktualisierungen:
    1. Teilnehmer
    2. Private Flag
    3. Anzahl Personen
    4. Titel
    5. Catering
    6. Klassifikationen
    7. Kostenträger
    8. Serieninformationen
  4. Meeting Request wird an Teilnehmer verschickt
    1. Verlinkung der Outlook Eintrag mit ROOMS Buchung
  5. Bestätigungsmail von ROOMS wird gemäss Notifikationseinstellungen der Benutzer, Dienstleister, etc. verschickt.

Fehlerfall
Sync. Error Mail an Benutzer

Schritt 12 – Benutzer editiert Kalendereintrag

Controls

Outlook Kalender Controls

Sidepanel

Regel

Mittels Öffnen des Kalendereintrags (Outlook)

Sidepanel

  • Bezeichnung: Reservation Detail
  • Klick auf Sidepanel zeigt Buchung im View Mode (Summary)
  • Folgende Buttons werden angezeigt:
    • Bearbeiten: öffnet die Webapplikation und zeigt die aktuelle Raumbuchung
    • Annullieren: storniert die aktuelle Raumbuchung - es kann eine neue Raumbuchung zum aktuellen Kalendereintrag gebucht werden - siehe UC 201.019 Reservation stornieren (annullieren)
    • Löschen: löscht die aktuelle Raumbuchung (nur verfügbar mit entsprechenden Rechten) - siehe UC 201.010 Buchungen löschen

Outlook Controls

  • Änderungen mit Relevanz auf die Synchronisation: werden Datum/Zeit, Titel, Privateflag oder Teilnehmer geändert wird alles nach Speichern über Outlook/Exchange => EventSink synchronisiert
  • Änderungen ohne Relevanz auf die Synchronisation: werden Location oder andere Werte geändert wird nichts Synchronisiert.
  • Serienbuchung: diese kann nicht geändert werden. System informiert, dass in diesem Fall alle Reservation annulliert und neu erstellt werden müssen.

Bilder

 

Mittels Drag & Drop + Maus-Ziehen

Regeln

 

Chached Mode

  • die Änderungen werden in Outlook vorgenommen
  • Exchange gibt die Informationen an ROOMS weiter
  • kann die Buchung zeitlich nicht verschoben werden (Konflikt) wird die Buchung über Exchange wieder zurück gesetzt
  • der Benutzer erhält ein Synchronisationsfehler Mail
  • im Erfolgsfall werden die Änderungsnotifikationen und Meeting Request Updates verschickt

Online Mode

  • die Änderungen werden in Outlook vorgenommen und direkt an den ROOMS Webservice weitergegeben
  • kann die Buchung zeitlich nicht verschoben werden (Konflikt) wird die Buchung direkt durch das Outlook Plugin zurückgesetzt
  • im Erfolgsfall werden die Änderungsnotifikationen und Meeting Request Updates verschickt

 

Schritt 13 – Benutzer löscht Kalendereintrag

Controls

Sidepanel

  • Im Kalendereintrag
  • Oder über rechte Maustaste

Regeln

Löschung

  • Outlook übermittelt über Exchange den Löschauftrag an ROOMS
  • Raumreservation wird annulliert
  • Stornierungsrequest an alle Teilnehmer
  • Annullationsmail an Organisator/Ersteller

Anhang

Anhang - Regeln für das Buchen in Kalender von Dritten

  • Buchen in Kalender von Dritten:
    • Bei der Sync muss sichergestellt werden, dass auch der Ersteller übermittelt wird
