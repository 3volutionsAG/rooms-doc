Use Cases : 105.002 Bestuhlung verwalten

Ziel & Beschreibung

Die Erfassung und Verwaltung von Bestuhlungsoptionen und die Zuweisung auf der Ressource. Eine „Bestuhlungsoption“ beinhaltet alle Felder der Tabelle „Bestuhlung“. Eine „Bestuhlungsvariante“ beinhaltet zusätzlich das Mapping zu der Ressource inkl. der Felder in der Mapping-Tabelle „RessourceBestuhlung“.

Rechte

  • Darf Ressourcetypen mit Bestuhlungen verwalten

Bedingungen

–

Hauptszenario in Schritten

  1. Nutzer navigiert auf Einstellungen >Basisdaten für Ressourcen >Bestuhlungen
  2. Nutzer gibt Suchkriterien ein (optional) und klickt Button „Suchen"
  3. Ein Suchresultat über alle erfassten Bestuhlungsoptionen wird angezeigt - ein oder mehrere Bestuhlungen können gelöscht werden
  4. Nutzer kann einen Eintrag anklicken, um die Details anzuzeigen (View Modus)
  5. Nutzer editiert oder erstellt eine Bestuhlung

Beziehungen zu UC


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

 

Use Case Details

Schritt 1 - Nutzer navigiert auf Einstellungen >Basisdaten für Ressourcen >Bestuhlungen

Controls

Sidepanel

Navigation öffnet sich entsprechend

Regeln

Funktionsrechte müssen implementiert werden:

  • Darf Ressourcetypen mit Bestuhlungen verwalten

 

Schritt 2 - Nutzer gibt Suchkriterien ein (optional) und klickt Button „Suchen“

Controls

Sidepanel

  • Bezeichnung
  • Kapazität
  • Button „Anzeigen“
  • Button „Zurücksetzen

Regeln

Anzahl Personen
Wird ein Wert beim Feld „Anzahl Personen“ angegeben, werden alle Bestuhlungsoptionen angezeigt, wo der angegebene Wert im Bereich zwischen Min- und Max-Anzahl Personen liegt

Bilder

 

Schritt 3 - Ein Suchresultat über alle erfassten Bestuhlungen wird angezeigt – ein oder mehrere Bestuhlungen können gelöscht werden

Controls

Smartlist mit Spalten
Standard Spalten:

  • Checkbox (für Mehrfachselektion)
  • Bezeichnung (View Link)
  • Bearbeiten Icon
  • MinPersonen
  • MaxPersonen

Optional Spalten:

  • Vorlaufdauer
  • Nachlaufdauer
  • Bildvorschau
  • Button: Entfernen
  • Button: Neu

Regeln

Bezeichnung
Die Bezeichnug wird in der Sprache des Benutzers angezeigt

Bild
Klick Thumbnail öffnet das zugewiesene Bild als Zoom im modalen Popup

Löschen

  1. Mehrfachselektionen möglich
  2. Klick auf Löschen
  3. Warnhinweis
    1. Bestätigungsmeldung im Actionpanel (inkl. Anzahl der gelöschten Einträge) ODER
    2. Fehlermeldung im Actionpanel

WICHTIG: Bestuhlungen können nur gelöscht werden, wenn keine Verlinkung zu einer Ressource mehr besteht. Ansonsten Fehlerausgabe im Actionpanel

Schritt 4 - Nutzer kann einen Eintrag anklicken, um die Details anzuzeigen (View Modus)

Controls

Inhaltsbereich – TAB „Bestuhlung“

  • Bezeichnung
  • Max. Personenzahl
  • Min. Personenzahl
  • Vorlaufzeit (Minuten)
  • Nachlaufzeit (Minuten)
  • Bildvorschau (Icon)
  • Button: Bearbeiten
  • Button: Zurück
  • Button: Drucken

Regeln

Der View Modus wird in der Sprache des angemeldeten Benutzers angezeigt

Bildvorschau
Klick auf Bild öffnet das Zoom (im modalen Popup)

Button Bearbeiten

Wird nur angezeigt, wenn das Funktionsrecht „Darf Ressourcetypen mit Bestuhlungen verwalten" vorhanden ist

Schritt 5 - Nutzer editiert oder erstellt eine Bestuhlung

Controls

Inhaltsbereich – TAB „Stammdaten“

  • Bezeichnung
  • Max. Personenzahl
  • Min. Personenzahl
  • Vorlaufzeit (Minuten)
  • Nachlaufzeit (Minuten)
  • Datei-/Verzeichnis-Browser (für Bild)
  • Upload-Button (für Bild)
  • Zugewiesenes Bild (Icon)
  • Button: Speichern
  • Button: Abbrechen

Regeln

Bezeichnung
Erfassung muss Mehrsprachig erfolgen

Max. Person

  • Defaultwert: 0
  • wird im Reservationsprozess dazu verwendet, um auf Grund der gewünschten Kapazität den passende Raumgrössen zu finden (siehe auch UC 201.001 Raum reservieren)

Min. Person

  • Defaultwert: 0
  • wird im Reservationsprozess dazu verwendet, um auf Grund der gewünschten Kapazität den passende Raumgrössen zu finden (siehe auch UC 201.001 Raum reservieren)

Vorlaufdauer

  • Standardwert = 0
  • Angabe in Minuten
  • Wird im Reservationsprozess für die Berechnung der Vorlaufdauer verwendet (siehe auch 201.000 Ressourcen reservieren)

Nachlaufdauer

  • Standardwert = 0
  • Angabe in Minuten
  • Wird im Reservationsprozess für die Berechnung der Vorlaufdauer verwendet (siehe auch 201.000 Ressourcen reservieren)

Abbrechen-Button
geht zurück auf die Suchresultatliste

Zugewiesenes Bild

  • Es kann nur ein Bild zugewiesen werden
  • Es wird das Thumbnail angezeigt
  • Klick auf Bild öffnet das Zoom (im modalen Popup)

Button Speichern
Das Actionpanel bestätigt die Speicherung.
