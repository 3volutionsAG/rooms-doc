Use Cases : 105.005 Sperrzeiten verwalten

Ziel & Beschreibung

Die Erfassung und Verwaltung von standortbezogenen Sperrzeiten und von Ressourcenbezogenen Sperrzeiten.
Sperrzeiten auf den Standorten werden vererbt.
Das Grundverahlten von Vererbungen wird in 000.001 Globale Funktionen beschrieben

Rechte

  • Funktionsrecht „Darf Ressourcetypen mit Sperrzeiten verwalten“ muss implementiert werden

Bedingungen

Standortbezogene Sperrzeiten:

  • UC 103.001 Standorte verwalten muss in den Schritten 1, 2 und 3 ausgeführt worden sein 
    Ressourcenbezogene Sperrzeiten
  • UC 105.001 Ressourcen verwalten muss in den Schritten 1, 2, 3 und 5 ausgeführt worden sein 
    Weitere Bedingungen
    # Sperrzeiten auf den Standorten werden bis auf die Ressource vererbt und dort gespeichert# Wird eine Ressource an einem Standort neu hinzugefügt erbt die Ressource automatisch die Sperrzeiten des Standortes# Wird auf einer Ressource eine Sperrzeit hinzugefügt, hat dies keinen Einfluss auf den Standort# Wird nach Pt. 3 auf dem übergreifenden Standort eine der Ressourcen-Sperrzeit identische Sperrzeit erfasst, wird diese auch auf die Ressource vererbt (ist also doppelt vorhanden)“# Eine Sperrzeit wirkt sich so aus, dass die Ressourcen nicht buchbar sind (ausgegraut) – z.B. im Kalender, Plan, Suchresultat, etc.

Hauptszenario in Schritten

  1. Navigation in den Bereich der Sperrzeiten (Tab Sperrzeiten)
  2. Liste mit allen erfassten Sperrzeiten wird angezeigt - eine oder mehrere Sperrzeiten können gelöscht werden
  3. Sperrzeit kann erfasst werden
  4. Ein oder mehrere Sperrzeiten können editiert werden
  5. Sperrzeit als Serie erstellen

Schritt 2 - Liste mit allen erfassten Sperrzeiten wird angezeigt – eine oder mehrere Sperrzeiten können gelöscht werden

Controls

Liste mit Spalten

  • Checkbox zur Selektion
  • Bezeichnung
  • Beginn
  • Ende
  • Vererben

Button Bereich

  • Button „Hinzufügen
  • Button „Entfernen“
  • Button "Serie hinzufügen"
  • Button "Bearbeiten"

Regeln

Eine Sperrzeit kann nicht editiert werden – nur Hinzufügen und Entfernen

Smartlist

  • Jede Spalte verfügt oberhalb des Spaltentitels über ein Suchfeld
  • Mehrfachselektion möglich

Serie hinzufügen

  • Modales Popup öffnet sich
  • siehe Schritt 5

Bearbeiten

  • eine oder mehrere Sperrzeiten müssen selektiert worden sein
  • Modales Popup öffnet sich
  • siehe Schritt 4

Entfernen

  1. Sicherheitsabfrage
  2. entfernt die Sperrzeit (bei Standorten: Löschweitergabe über Vererbung bis zur Ressource)
  3. Bestätigungsmeldung im Actionpanel inkl. Anz. der gelöschten Sperrzeiten

Schritt 4 - Ein oder mehrere Sperrzeiten können editiert werden

Controls

Modales Popup mit

  • Checkbox UND Feld: Bezeichnung (Mehrsprachig)
  • Checkbox UND Datumpicker: Start Datum
  • Checkbox UND Zeitpicker: Start Zeit
  • Checkbox UND Datumpicker: End Zeit
  • Checkbox UND Zeitpicker: End Zeit

Button Bereich

  • Button Speichern
  • Button Abbrechen

Regeln

Checkboxen

  • Aktiviert ein Feld für eine Eingabe
  • Wird das Feld nicht aktiviert, bleiben die Werte der einzelnen Sperrzeiten erhalten
  • Wird das Feld aktiviert, wird der neue Wert auf alle Sperrzeiten übernommen

Bezeichnung

  • Die Bezeichnung der Sperrzeit muss in allen verfügbaren Sprachen erfasst werden

Stardatum

  • Datumpicker (siehe auch 000.001 Globale Funktionen)
  • Standardwert bei Selektion: aktuelles Datum

Startzeit

  • Zeitpicker (siehe auch 000.001 Globale Funktionen)
  • Standardwert bei Selektion: 00:00

Enddatum

  • Datumpicker (siehe auch 000.001 Globale Funktionen)
  • Standardwert bei Selektion: aktuelles Datum

Endzeit

  • Zeitpicker (siehe auch 000.001 Globale Funktionen)
  • Standardwert bei Selektion: 23:59

Speichern

  • Feldvalidierungen
  • Zurück auf die vorherige Ansicht (Popup wird geschlossen)
  • Ausgabe im Actionpanel: "Die Änderungen werden nun übernommen. Sie werden  wieder informiert, sobald der Vorgang abgeschlossen ist."
  • Aktionseinträge für Bearbeitung werden in Action Tabelle geschrieben
  • Asynchrone Abarbeitung (durch den Service) des Auftrags
  • Ausgabe im Actionpanel: "Die Änderungen von X  Einträgen vom Typ Sperrzeiten  sind abgeschlossen (a / b / c wurden  angepasst; d / e / f konnten nicht angepasst werden)."

Abbrechen

  • Abbrechen entfernt die Locks
  • Zurück auf die vorherige Ansicht

Schritt 5 - Sperrzeit als Serie definieren

Controls

Modales Popup mit

Kopfbereich

  • Feld: Name
  • Label: Serie Start
  • Label: Start Datum und Start Zeit
  • Optionen
    • Enden am + Datumpicker
    • Enden nach + Feld: Iterationen + Label: Iterationen

Pattern Bereich

  • Täglich
    • Option: Label: alle + Feld: Tage + Label: Tag/e
    • Option: Label: an allen Wochentagen
  • Wöchentlich
    • Label: Wiederholen alle + Feld: Wochen + Label: Wochen am:
    • Checkbox: Montag
    • Checkbox: Dienstag
    • Checkbox: Mittwoch
    • Checkbox: Donnerstag
    • Checkbox: Freitag
    • Checkbox: Samstag
    • Checkbox: Sonntag
  • Monatlich
    • Option Tag
      • Feld: Anzahl
      • Label: jeden
      • Feld: Anzahl
      • Label: Monat/e
    • Option am
      • Dropdown mit Werten: ersten, zweiten, dritten, vierten, letzten
      • Dropdown mit Werten: Montag, Dienstag, Mittwoch, Donnerstag, Freitag, Samstag, Sonntag
      • Label: jedes
      • Feld: Monat
      • Label: Monat/e
  • Checkbox: ist Rekursiv

Button Bereich

  • Button Speichern
  • Button Abbrechen

Regeln

Kopfbereich

  • Name: wird mehrsprachig erfasst
  • Option "Enden am" ist selektiert
  • Datumpicker Default: heute() + 30 (Tage)
  • Option "Enden nach" per Standard leer
  • Klick in das Feld "Enden nach" aktiviert, Option "Enden nach"

Muss Felder/Feldvalidierungen bei

  • Täglich
    • Feld „Tag(e)“ muss Ganzzahl beinhalten
    • Standardwert: alle 1 Tag
  • Wöchentlich
    • Feld „Wiederholen“ muss eine Ganzzahl beinhalten
    • Min. eine Checkbox der Wochentage muss selektiert sein
    • Min. der Tag der Seriestart (z.B. Mittwoch) muss selektiert sein
    • Standardwert: Wiederholung alle 1 Woche am #Selektion des Wochentags#
  • Monatlich
    • Option „Tag“ muss Ganzzahl beinhalten – Monat muss Ganzzahl beinhalten
    • Option „Am“, Eintrag aus Listen und „Monat“ muss Ganzzahl beinhalten.

 Speichern

  • Feldvalidierungen
  • Zurück auf die vorherige Ansicht (Popup wird geschlossen)
  • Ausgabe im Actionpanel: "Die Änderungen werden nun übernommen. Sie werden  wieder informiert, sobald der Vorgang abgeschlossen ist."
  • Aktionseinträge für Bearbeitung werden in Action Tabelle geschrieben
  • Asynchrone Abarbeitung (durch den Service) des Auftrags.
  • Dabei werden auch Dupletten und Überlappende Einträge validiert - diese werden nicht gespeichert
  • Ausgabe im Actionpanel: "Die Änderungen von X  Einträgen vom Typ Sperrzeiten  sind abgeschlossen (a / b / c wurden  angepasst; d / e / f konnten nicht angepasst werden)."

Abbrechen

  • Zurück auf die vorherige Ansicht (Popup wird geschlossen)
