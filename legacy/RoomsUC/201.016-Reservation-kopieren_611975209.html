Use Cases : 201.016 Reservation kopieren

Ziel & Beschreibung

Von einer bestehenden Reservation kann eine Kopie erstellt werden.
Die Kopie muss dann entweder in einer anderen Ressource und/oder zu einem anderen Zeitpunkt erstellt werden

Rechte

–

Bedingungen

  1. Eine bestehende Reservation ist vorhanden
  2. Der Benutzer hat min. "Read" Rechte oder es handelt sich um seine eigene Reservation (ich bin Ersteller, Organisator oder Verantwortlicher)
  3. Der Benutzer befindet sich im View Mode

Hauptszenario in Schritten

  1. Benutzer klickt kopieren
  2. Kopie wird erstellt - Benutzer änder Zeitraum oder Ressource

Beziehungen zu UC

201.000 Ressourcen reservieren


Legende

  • Rote Schrift = MUSS Felder

Use Case Details

Schritt 1 - Benutzer klickt Button kopieren

Controls

Buttonbereich im Viewmode einer Reservation

  • Button: Kopieren

Regeln

Kopieren:

  • Warnhinweis:Soll die Reservation kopiert werden?
  • Erstellung einer identsichen Kopie der Reservation, als Systemreservation
  • Wechsel in Edit-Mode der Kopie

Kopier Regeln

  • Kopfdaten werden 100% übernommen, inkl. der Ressourcenspezifischen Kopfdaten (wie Anzahl Personen, etc.)
  • Zusätzliche Ressourcen werden analog Originalbuchung mitgebucht – falls nicht möglich Warning im Actionpanel
  • Bestuhlung wird übernommen, falls dieselbe Ressource gewählt wurde.
    • Ansonsten wird unter der Berücksichtigung der maximalen Kapazität die entsprechende Bestuhlung übernommen.
    • Falls diese immer noch zu hoch, wird die Personenzahl auf die maximale Kapazität zurückgesetzt und eine entsprechende Warnung ausgegeben
  • Teilnehmer werden übernommen, ohne Verfügbarkeitscheck
  • Dienstleistungen werden übernommen, sofern derselbe Raum, bzw. dieselbe Angebotszonen zur Verfügung stehen. Ansonsten wird Warning im Actionpanel ausgegeben und KEINE Dienstleistung übernommen.
  • Serienbuchungen können nicht gesamthaft kopiert werden (einzelne Buchungen einer Serie können kopiert werden, sind aber nicht mehr Teil der Serie)

Schritt 2 - Kopie wird erstellt - Benutzer änder Zeitraum oder Ressource

Controls

gemäss UC 201.000 Ressourcen reservieren

Regeln

  • Benutzer muss Datum/Zeit ODER Ressource ändern um die Kopie speichern zu können
  • Die Speicherung der Ressource erfolgt analog des "normalen" Speicherprozesses - siehe UC 201.000 Ressourcen reservieren und 201.001 Raum reservieren
