Ziel & Beschreibung

Die Konfiguration einer Ressource für Slot Buchungen. Slot Erstellung und Slot Buchung

Rechte

  • Darf Slot administrieren

Terminiologie

  • Slot-Ressource: Ressource, die nur über Slots gebucht werden kann.
  • Slot: Reservation im Zustand Slot definiert/publiziert
  • Slot-Buchung: Kopie eines Slots in einem "normalen" Workflow-Zustand.

Bedingungen

  • UC 105.001 Ressourcen verwalten wurde ausgeführt und Checkbox "nur Slot Buchungen zulässig" wurde selektiert
  • Slot Reservationen stehen nur im Kalender und Plan zur Verfügung* Wird ein Slot gebucht, wird ein Clone erstellt und der Clone erhält die verfügbaren Stati gemäss Workflow (z.B. definitiv gebucht)
  • Werden zusätzliche Ressourcen mitgebucht, wird der Slot als auf diese Ressource übertragen. Während dieser Zeit sind keine weiteren Reservationen möglich, allerdings ist bei der zusätzlichen Ressource ein freies Buchen möglich.

Hauptszenario in Schritten

  1. Benutzer navigiert auf Kalender - Kalender zeigt Slot Buchbarkeit
  2. Nutzer definiert Slot
  3. Nutzer publiziert Slot
  4. Benutzer bucht Slot

Regeln

Slotansicht Kalender

  • Der Kalender wird zweizeilig aufgebeaut.
  • Die erste Zeile ist für die Erstellung eines Slots und für Drag&Drop
  • die zweite Zeile zeigt den effektiven Publikationszeitpunkt
  • Pro zeitgleichliegenden Slot wird eine zusätzliche Zeile hinzugefügt
 

 

Schritt 2 - Benutzer definiert Slot

Controls

Action Popup

  • Controls analog Einzelschnellreservation: siehe UC 201.201 Schnellreservation
  • Zahlenpicker: Zeilennummer Kalender

Regeln

Generell

  • Recht muss vorhanden sein: Darf Slot administrieren
  • Slot-Definitionen können nur bearbeitet oder gelöscht werden. Die Annulationsfunktion steht nicht zur Verfügung.

Zeilennummer Kalender

  • Es können mehrere Slots zum gleichen Zeitpunkt definiert und publiziert werden.
  • Zeilennummer gibt die Reihenfolge im Kalender (von oben nach unten, wobei 0 die oberste ist) an
  • Standardwert: 0

Workflow-Regeln:

  • Wird eine neue Reservation auf einer Slot-Ressource erstellt, stehen nur die beiden Slot-Status zur Auswahl.
  • Wird eine Reservation auf einer Slot-Ressource editiert, die in einem Slot-Status ist, stehen nur die beiden Slot-Status zur Auswahl.

Reservieren

  • definiert Slot analog einer Einzelschnellreservation - siehe UC 201.201 Schnellreservation

Detailreservation

  • Slot wird im Detail definiert und konfiguriert, gemäss UC 201.000 Ressourcen reservieren und 201.001 Raum reservieren

Speichern Spezialfall Slot

  • Alle Einstellungen des Reservationsdetail stehen grundsätzlich zur Verfügung, machen aber nicht grundsätzlich Sinn. So empfehlen wir beispielsweise das Teilnehmer Tab für alle Benutzergruppen auszublenden.

Schritt 4 - Nutzer bucht Slot

Controls

Sidepanel

  • Feld 1
  • Feld 2
  • Button „Finden“
  • Button „Zurücksetzen

Regeln

Slot im Kalender buchen

  • Ein publizierter Slot hat die Eigenschaft, dass er im Kalender nicht mehr verschoben werden kann. Auch nicht nachdem ein Kunde eine Buchung gemacht hat.

Slot im Plan buchen

  • Ist nur ein Slot auf der Ressource (zum gewählten Zeitpunkt) publiziert, verhält sich das System wie bis anhin: Action Popup und Reservation (Schnellbuchung oder Detailreservation). Sind mehrere Slots auf der Ressource publiziert und verfügbar, kann der Benutzer einen Slot im Actionpopup aus einer Dropdown selektieren und dann die Detail oder Schnellreservation ausführen.

Clone erstellen

  1. Der Clone hat folgende Regeln und Eigenschaften
    • Wird der Clone erstellt wird der angemeldete Benutzer (Kunde) als Organisator übernommen
    • Datum/Zeit und Vorlaufzeiten sind nicht änderbar – aber sichtbar
    • Ressource/Standort sind nicht änderbar – aber sichtbar
    • Serieninformationen sind nicht sichtbar
    • Der ursprüngliche Titel wird gelöscht und das Feld leer gelassen, so dass der Kunde einen Titel eingeben muss.
    • Der Status ändert auf einen Workflow Status gemäss Bereichtigungen des Benutzers – z.B. provisorisch Buchen
    • Ein Clone ist nie Teil einer Serie: ist der Slot Teil einer Serie, wir danach ein Slot gebucht, wird immer eine Einzelreservation gebucht (Nie eine Serie)
  2. Buchungsregeln*:*
    1. Sind über den gleichen Zeitraum zwei oder mehrere Slots auf einer Ressource verfügbar und wird einer der Slots gebucht (wird ein Clone erstellt), sind alle Slots nicht mehr buchbar und werden nicht mehr angezeigt (Kalender, Plan, Reservieren)
    2. Wird eine Slot Buchung (Clone) gelöscht oder annulliert, werden alle Slots dieses Zeitpunkts wieder sichtbar

Sperrzeiten, Feiertage und Wochenende

  • Bei dem Erstellen einer Slot-Reservation müssen diese Regeln geprüft werden. Benutzer mit entsprechenden Berechtigungen können diese Regeln übersteuern. (Gleich wie normale Reservationen.)
  • Bei der Buchung eines Slots dürfen diese Regeln nicht geprüft werden. Ein publizierter Slot ist mit der entsprechenden Berechtigung immer buchbar.

Funktionen und Rechte beim Definieren und Buchen

  • Grundsatz: Es werden die Funktionen zur Verfügung gestellt, die in dem Kontext sinnvoll sind und für die der Benutzer die nötigen Funktionsrechte und das entsprechende Datenrecht hat.
  • Die Checkout-Funktionen stehen bei Slot-Reservationen nicht zur Verfügung.
  • Slot-Buchung verfügen über die Funktionen Bearbeiten, Annulieren und Löschen.

Untergeordnete Ressourcen

  • Fixes Equipment und virtuelle Räume können einer als Slot buchbaren Ressource nur zugeordnet werden wenn sie ebenfalls nur als Slot buchbar sind. Ressourcen die nur als Slot buchbar sind können normalen Ressourcen nicht zugeordnet werden.
  • Mobiles Equipent (d.h. Equipment das erst bei der Buchung durch den Benutzer hinzugefügt wird) muss nicht als Slot buchbar sein. Es gelten die normalen Regeln für Reservationen.

Workflow Logik

  • Wird ein Slot gebucht, wird anhand des auf der Ressource hinterlegten Workflows der nächstmögliche Status berechnet (bestehende Logik).
  • Wird eine Reservation auf einer Slot-Ressource editiert, die nicht in einem Slot-Status ist, wird anhand des auf der Ressource hinterlegten Workflows der nächstmögliche Status berechnet (bestehende Logik).

Reservationsdeatil

  • Zeitänderungen auf Slot-Buchungen dürfen nicht möglich sein. Zusätzliche Felder (gem. Spez S. 8) im Reservationsdetail dürfen nicht editiert werden (readonly setzen).
