Ziel & Beschreibung

Die Raum-/Fläche Einheit" kann genutzt werden, um eine beliebige Fläche (Openspace, Büro, etc.) zu definieren und dieser Fläche Personen zuzuweisen

Rechte

    • Darf Raum Einheit sehen
    • Darf Raum Einheit bearbeiten

Bedingungen

Sichtbar unter

  • Einstellungen
  • Listen

Hauptszenario in Schritten

  1. Nutzer navigiert auf Einstellungen >Raumeinheiten ODER Listen >Raumeinheiten
  2. Nutzer gibt Suchkriterien ein (optional) und klickt Button „Finden"
  3. Ein Suchresultat über alle Raumeinheiten wird angezeigt - ein oder mehrere Raumeinheiten können gelöscht werden
  4. Nutzer kann einen Eintrag anklicken, um die Details anzuzeigen (View Modus)
  5. Nutzer editiert oder erstellt eine Raumeinheit
  6. Liste mit allen der Raumeinheit zugeordneten Personen wird angezeigt - eine oder mehrere können gelöscht werden
  7. Person kann der Raumeinheit zugeordnet werden

Beziehungen zu UC


Legende

  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Schritt 3 - Ein Suchresultat über alle Raumeinheiten wird angezeigt – ein oder mehrere Raumeinheiten können gelöscht werden

Controls

Inhaltsbereich

  • Bezeichnung (nicht Name wie im Screenshot) mit Link auf View Mode
  • Edit Button
  • Beschreibung
  • In Plan zeigen (wird nur angezeigt, sofern eine Planverknüpfung vorhanden)
  • Standort
  • Anzahl Arbeitsplätze
  • Fläche (M2)
  • Höhe
  • Gewichtung
  • Typ
  • Büro Kollegen (Auflistung aller der Raumeinheit zugeordneten Personen; Verlinkung auf View Mode des Personenprofils)
  • Button: Entfernen
  • Button: Hinzufügen


Regeln


Bezeichnung

  • wird angezeigt in der Sprache des angemeldeten Benutzers
  • Link auf View Mode - siehe Schritt 4

Edit Button

  • siehe Schritt 5

Beschreibung

  • wird angezeigt in der Sprache des angemeldeten Benutzers

In Plan Anzeigen

  • Inhalt wird nur angezeigt, sofern Verknüpfung auf Plan erfolgt ist

Standort

  • Standort wird hierarchisch ausgegeben
  • wird angezeigt in der Sprache des angemeldeten Benutzers


Büro Kollegen

  • Auflistung aller der Raumeinheit zugeordneten Personen
  • Verlinkung auf View Mode des Personenprofils


Entfernen

  • Sicherheitsabfrage
  • Löschung der Referenz Person / Raumeinheit
  • Löschung der Raumeinheit
  • Bestätigungsmeldung in Form einer ActionMessage
  • Aktualisierung der Liste

Hinzufügen

  • siehe Schritt 5

Schritt 5 - Nutzer editiert oder erstellt ein eine Raumeinheit

Controls

Inhaltsbereich

Zwei Tabs

  • Stammdaten
  • Personen (siehe Schritt 6)

Stammdaten

  • Feld: Bezeichnung
  • Textarea: Beschreibung
  • Standortpicker: Standort
  • Kostenträgerpicker: Kostenträger
  • Zahlenpicker: Anzahl Arbeitsplätze (Kapazität)
  • Feld: Fläche (M2)
  • Feld: Höhe 
  • Feld: Gewichtung 
  • Dropdown: Typ
  • Button „Speichern“
  • Button „Abbrechen



Regeln


Bezeichnung

  • Mehrsprachige Erfassung möglich
  • Standard = leer

Beschreibung

  • Mehrsprachige Erfassung möglich
  • Standard = leer

Standort

  • Standard = leer

Kostenträger

  • Standard = leer

Anzahl Arbeitsplätze (Kapazität)

  • Standard = leer
  • Validierung auf Ganzzahlen

Fläche (m2)

  • Standard = leer
  • Validierung auf Dezimalzahlen

Gewichtung

  • Standard = leer
  • Validierung auf Dezimalzahlen

Typ

  • Einsprachig
  • Werte aus Liste (z.B. Archiv, Schulung, Material, etc.)
  • Das Editieren der Liste muss über ein SQL Script oder XML erfolgen

Speichern

  • speichert alle Werte
  • zurück auf Schritt 4 oder 3

Abbrechen

  • zurück auf Schritt 4 oder 3

Schritt 7 - Person kann der Raumeinheit zugeordnet werden

Controls

Tab "Personen"

Smartlist mit ROOMS Personen mit folgenden Spalten:

  • Checkbox für Mehrfachselektion
  • Nachname
  • Vorname
  • E-Mail
  • Personalnummer
  • Initialen

Zusätzliche Felder gültig für alle selektierten Einträge:

  • Beginn (kann leer sein)
  • Ende (kann leer sein)

Buttons:

  • Hinzufügen
  • Abbrechen


Regeln

Generell

  • Die Smartlist kann gefiltert (gesucht) werden.
  • eine Person kann mehrmals der gleichen Raum Einheit zugeordnet werden (auch zeitlich überschneidend)
  • eine Person kann in verschiedenen Raum Einheiten vorkommen (auch zeitlich überschneidend)

Gültig von  

  • Standard = leer
  • Validierung auf Datum 

Gültig bis  

  • Standard = leer
  • Validierung auf Datum

Hinzufügen

  • Einträge werden der Liste hinzugefügt (siehe Personen)
  • Modales Fenster wird geschlossen
  • Bestätigung als ActionMessage

Abbrechen

  • Modales Fenster wird geschlossen (ohne Speicherung)
