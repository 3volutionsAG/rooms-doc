Use Cases : 105.006 Feiertage importieren und verwalten

Created by Former user, last modified by Former user (Deleted) on Oct 13, 2014

ID

105.006

Owner

Former user (Deleted)

Rev-Number

2.0

Priorität

Ziel & Beschreibung

Die Erfassung und Verwaltung von Feiertagen, bzw. das Importieren von Feiertagen.
WICHTIG: Es wird zwischen einer Feiertag (nur Bezeichnung) = z.B. Weihnachten, Silvester UND Feiertage-Daten (inkl. Datumzuordnung) = z.B. Weihnachten 25.12.2009, Silvester 31.12.2009 unterschieden

Rechte

  • Darf Feiertage verwalten

Bedingungen

–

Hauptszenario in Schritten

  1. Nutzer navigiert auf >Einstellungen >Erweiterte Basisdaten für Ressourcen >Feiertage
  2. Nutzer gibt Suchkriterien ein (optional) und klickt Button „Finden"
  3. Ein Suchresultat über alle erfassten Feiern wird angezeigt - ein oder mehrere Feiertage können gelöscht werden
  4. Nutzer kann einen Feiertag anklicken, um in den View Modus zu wechseln
  5. Nutzer kann Stammdaten eines Feiertag editieren (Tab "Feier")
  6. Liste der Feiertage-Daten wird angezeigt - Nutzer kann ein oder mehrere Daten löschen
  7. Nutzer kann neue Feiertag-Daten erfassen
  8. Nutzer kann Feiertage mit Feiertag-Daten importieren

Beziehungen zu UC


Legende

  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Schritt 7 – Nutzer kann neue Feiertag-Daten erfassen

  • Datumpicker: Datum (Von, Bis)
  • Zeitselektor: Von
  • Zeitselektor: Bis

Button Bereich

  • Button „Speichern“
  • Button „Abbrechen“

Regeln

Datumpicker

Standardwert: heutiges Datum

Zeitselektor
Von: Standardwert: 00:00
Bis: Standardwert: 23:59
Diese Start- und Endzeiten entsprechen einem Tag.

Speichern

  • Modales Popup wird geschlossen
  • Smartlist wird aktualisiert um neues Datum
  • Bestätigungsmeldung im Actionpanel

Abbrechen

  • Modales Popup wird geschlossen

Schritt 8 - Nutzer kann Feiertage mit Feiertag Daten importieren

Controls

Tab „Importieren“

  • Import von Datei (Verzeichnis-/Datei-Browser-Control)
  • Upload Button
  • Dropdown: Sprache
  • Importierte Daten

Butten Bereich

  • Speichern
  • Abbrechen

Regeln

Import von Datei

  • Das entsprechende File mit den Feiertagen muss mittels Browser-Control ausgewählt werden
  • Die Datei muss über eine gültige Formatierung aufweisen (siehe Datei Format)

Sprache

  • Die Dropdown-Sprache beinhaltet die entsprechenden Sprachen, wovon die Benutzersprache vordefiniert ist

Upload

  • Nach Klick auf Upload werden die Feiertage als Liste zur Vorschau angezeigt (siehe Abbildung)
  • Gespeichert werden diese erst, wenn der „Speichern“ geklickt wird

Datei Format

  • Da das zu importierende File jeweils nur eine Sprache beinhaltet, wird beim ersten Import (wenn noch keine Feiertage im ROOMS PRO bestehen) die Bezeichnung für jede verfügbare Sprache (im ROOMS PRO) in der Sprache des importierten Files übernommen (z.B. in Deutsch).
  • Der Nutzer kann nun nach und nach die Files in den anderen Sprachen importieren. Sollte beim erneuten Import z.B. mit einem Englischen-File ein weiterer Feiertag hinzugekommen sein, wird dieser für sämtliche ROOMS PRO-Sprache wieder in Englisch dargestellt
  • Unterstützt werden die Dateien TXT und CSV werden (mittels Delimiter Untertrennung)
  • Es reichen die Angaben Zeit, <Delimiter> und Bezeichnung - eine Überschriftszeile darf nicht vorhanden sein (siehe Abbildung)
  • Die Delimiter Komma (,) und (Semikolon) ( ; ) werden unterstützt.
  • Das Datum soll mindestens in den Formaten „12.04.2008“, „2008/04/12“ und „2008-04-12“ unterstützt werden
  • Es spielt keine Rolle, ob das Datum und die Bezeichnung in Hochkommas abgespeichert sind oder nicht (Beispiel:
    2008/12/25;Weihnachten ODER „2008/12/25“;“Weihnachten“)
  • Als Hauptreferenz für den Import gilt http://www.feiertagskalender.ch/ (siehe Abbildung)
  • Die Datei ist ROOMS Komfort anzupassen (erste Spalte = Datum, zweite Spalte = Feiertag Bezeichnung, etc.)
