Use Cases : 104.002 Buchungsklassifikationen verwalten

Ziel & Beschreibung

Buchungsklassifikationen können erfasst und verwaltet werden

Rechte
(000.002 Rechte)

  • Darf Klassifikationen verwalten

Bedingungen

–

Hauptszenario in Schritten

  1. Nutzer navigiert >Einstellungen >Erweiterte Basisdaten für Ressourcen >Klassifikationen und klickt Button "Finden"
  2. Nutzer kann durch den Klassifikationen-Baum navigieren und die weiteren Aktionen ausführen
  3. Nutzer wechselt in Viewmodus einer Klassifikation oder eines Klassifikationstypen
  4. Nutzer kann Klassifikationstypen erfassen und diese editieren
  5. Nutzer kann neue Klassifikation erfassen und diese editieren
  6. Liste mit allen dem Buchungsklassifikationstypen zugeordneten Ressourcen wird angezeigt - eine oder mehrere Ressourcen können entfernt werden
  7. Ressourcen können zu den Buchungsklassifikationstypen hinzugefügt werden
  8. Liste mit allen dem Buchungsklassifikationstypen zugeordneten Benutzergruppen wird angezeigt - eine oder mehrere Benutzergruppen können entfernt werden
  9. Benutzergruppen können einem Buchungsklassifikationstypen hinzugefügt werden
  10. Nutzer kann Klassifikationen löschen

Schritt 2 - Nutzer kann durch den Buchungsklassifikationen-Baum navigieren und die weiteren Aktionen ausführen

Controls

Inhaltsbereich

  • Buchungsklassifikationen-Baum
    • Neu Icon
    • View Icon
    • Editieren Icon
    • Löschen Icon

Button Bereich

  • Button „Klassifikation erstellen“

Regeln

Die Baumstruktur ist zweistufig. Je nachdem an welche Stelle im Baum navigiert wird, kann auf den jeweiligen Bereichen mehr oder weniger Aktionen durchführen (auch auf Grund der Anzahl der angezeigten Icons ersichtlich).

Auf der Hierarchiestufe „Buchungsklassifikationtypen“ stehen die Buttons „Neuerfassen“, „Anzeigen“, „Editieren“ und „Löschen“ zur Verfügung.

Klick auf „Neu“-Button möglich

  • Man gelangt ins Detail einer neuen Klassifikation, um diese zu erfassen (siehe Schritt 5)

Klick auf „View“ Button möglich

  • Details der Buchunsklassifikation werden nicht editierbar angezeigt (siehe Schritt 3)

Klick auf „Editieren Button möglich

  • Man gelangt ins Detail des Buchungsklassifikationstypen und kann diesen editieren (siehe Schritt 4)

Klick auf „Löschen“ Button möglich

  • Löscht den entsprechenden Buchungsklassifikationstyp (Siehe Schritt 8)

Auf der Hierarchiestufe „Buchungsklassifikationen“ stehen die Buttons „Anzeigen“, „Editieren“ und „Löschen“ zur Verfügung.

Klick auf „Anzeigen“ Button möglich

  • Details der Klassifikation werden nicht editierbar angezeigt (siehe Schritt 3)

Klick auf „Editieren" Button möglich

  • Man gelangt ins Detail der gewählten Klassifikation und kann diese editieren (siehe Schritt 5)

Klick auf „Löschen“ Button möglich

  • Löscht die entsprechende Klassifikation (Siehe Schritt 8)

Buchungsklassifikation Typ erstellen
Benutzer kann einen neuen Buchungsklassifikationstypen erstellen (siehe Schritt 4)

Schritt 4 - Nutzer kann Klassifikationstypen erfassen und diese editieren

Controls

Inhaltsbereich

Tab „Stammdaten“

Felder:

  • Textfeld: Bezeichnung
  • Textfeld: Code
  • Multiselektor: Ressourcentyp
  • Eingabefeld Sortierung
  • Checkbox: Mehrfachselektion erlaubt (ja/nein)
  • Checkbox: Sichtbar in Listen (ja/nein)
  • Checkbox: Sichtbar in Outlook (ja/nein)
  • Dropdown oder Multiselektor: Standardwert(e)
  • Multiselektion: Anlass Typ

Tab "Ressourcen"

  • Liste aller zugeordneten Ressourcen (siehe Schritt 6)

Tab "Gruppen"

  • Liste aller zugeordneten Benutzergruppen (siehe Schritt 8)

Buttons

  • Speichern
  • Abbrechen

Regeln

Bezeichnung

  • Erfassung muss Mehrsprachig erfolgen

Ressourcentyp

  • Es können 1-n Ressourcenarten (Sitzungszimmer, Equipment, etc.) ausgewählt werden, womit definiert wird, bei welchen Ressourcenarten der Buchungsklassifikationstyp im Reservationsdetail zur Verfügung steht
  • Es sind nur diejenigen Ressourcenarten wählbar, auf denen das entsprechendes Funktionsrecht vorhanden ist.

Sortierung

  • Mittels Sortierung wird definiert, an welcher Stelle resp. in welcher Reihenfolge dieser Buchungsklassifikationstyp gegenüber den anderen Buchungsklassifikationstypen erscheinen soll (Relevanz im Detail der Reservation)

Mehrfachselektion erlaubt (ja/nein)

  • Wird im Reservation dargestellt als Multiselektor für Mehrfachselektion oder als Dropdown zur Einzelselektion

Sichtbar in Listen

  • Eine zusätzliche Checkbox „sichtbar in Listen“ definiert, ob der Klassifikationstyp auch als Filterkriterium in den Listen sichtbar ist  (Sidepanel)

Sichtbar im Outlook

  • Eine zusätzliche Checkbox „sichtbar in Outlook“ definiert, ob der Klassifikationstyp auch als Filterkriterium im Outlook verfügbar ist  (Bereich Suchkriterien)

Standardwert(e)

  • Wenn "Mehrfachselektion erlaubt" aktiviert ist, erscheint ein Multiselektor. Kein, ein oder mehrere Werte können als Standardwerte angegeben werden (Anmerkung: Standardwerte werden im Reservationsprozess und im Sidepanel vorselektiert sein)
  • Wenn "Mehrfachselektion erlaubt" nicht aktiviert ist, wird eine Dropdown eingeblendet. Es kann ein Wert als Standardwert selektiert werden (Anmerkung: der Standardwert wird im Reservationsprozess und im Sidepanel vorselektiert sein)

Anlass Typ

  • Multiselektionsmöglichkeit für Anlasstyp
  • Wird ein oder mehrere Anlasstypen selektiert, werden die Buchungsklassifikationen bei Auswahl des entsprechenden Anlasstyps auch zur Verfügung gestellt (siehe auch 211.001 Anlass erstellen und 201.030 Massenmutation auf Buchung)

Speichern

  • Speichert die Daten
  • Bestätigungsmeldung im Actionpanel
  • Rücksprung auf Buchungsklassifikationen Baum

Abbrechen

  • Rücksprung auf Buchungsklassifikationen Baum

Buchungsklassifikationen werden in verschiedenen Bereichen verfügbar gemacht.

Buchungsklassifikation im Reservationsdetail
Nachstehende Abbildung wie die Buchungsklassifikationen im Reservationsdetail dargestellt werden. Details in UC 201.000 Ressourcen reservieren

Buchungsklassifikationen als Suchparameter im Sidepanel bei Reservieren (UC 201.000 Ressourcen reservieren) und Listen (UC 217.001 Ansicht im Listenformat anzeigen)

Buchungsklassifikationen im Kalender (UC 213.001 Reservation in der Kalenderansicht verwalten)

  • Diese wird dann im Kalender visualisiert (siehe UC 213.001 Reservationen im Kalender suchen und buchen)

  • Bestätigungsmeldung im Actionpanel
  • Aktualisierung Baumstruktur
  • Rücksprung auf Baumstruktur

Abbrechen

  • Rücksprung auf Buchungsklassifikationen-Baum

Schritt 6 - Liste mit allen des Buchungsklassifikationstypen zugeordneten Ressourcen wird angezeigt – eine oder mehrere Ressourcen können entfernt werden

Controls

Inhaltsbereich – Tab „Ressourcen“

Smartlist mit Spalten
Default:

  • Selektionscheckbox
  • Name (ID)
  • Bearbeiten Icon
  • Bezeichnung
  • Ressourcenart
  • Standort

Optional:

  • Verantwortliche Person
  • Status

Button Bereich

  • Button: Ressource hinzufügen
  • Button: Entfernen

Regeln

Smartlist

  • Jede Spalte verfügt oberhalb des Spaltentitels über ein Suchfeld, falls aktiviert (siehe 000.001 Globale Funktionen).
  • (Buchungklassifikationtyp steht nur bei den zugefügten Ressourcen zur Auswahl)

Entfernen
Es kann/können ein oder mehrere Ressourcen selektiert werden. Klick auf „Entfernen“ bewirkt folgendes:

  • Sicherheitsabfrage
  • Entfernt die Verlinkung zum Buchungsklassifikationstypen
  • Aktualisierung der Smartlist
  • Bestättigungsmeldung inkl. Der Anzahl der Entfernungen im Actionpanel

„Ressourcen hinzufügen“

  • Ein modaler Dialog wird geöffnet und man kann eine/mehrere Ressource( n ) dem Buchungsklassifikationstypen hinzufügen (siehe Schritt 7)

Bilder

 

Schritt 7 - Ressourcen können zur Buchungsklassifikationstyp hinzugefügt werden

Controls

Modaler Dialog- Smartlist mit Spalten
Default:

  • Selektionscheckbox
  • Name (ID)
  • Bearbeiten Icon
  • Bezeichnung
  • Ressourcenart
  • Standort

Optionale Spalten

  • Status
  • Verantwortliche Person

Buttons

  • Ressourcen hinzufügen
  • Entfernen

Regeln

Smartlist

  • Jede Spalte verfügt oberhalb des Spaltentitels über ein Suchfeld, falls aktiviert (globale Dokumentation).
  • Es werden nur Ressourcen angezeigt, welche noch nicht hinzugefügt worden sind.

Ressourcen hinzufügen

  • Mehrfachselektion erlaubt
  • Regeln:
    1. Funktionsrecht für den entsprechenden Ressourcentypen
    2. Zuordnung des Ressourcen auf den Buchungsklassifikationstypen
  • Ressourcen werden der Liste (siehe Schritt 6) hinzugefügt
  • Popup wird geschlossen
  • Smartlist wird aktualisiert
  • Bestätigungsmeldung im Actionpanel über die Anzahl der hinzugefügten Ressourcen

Bilder

 

Schritt 8 - Liste mit allen Buchungklassifikationstypen zugeordneten Benutzergruppen wird angezeigt – eine oder mehrere Benutzergruppen können entfernt werden

Controls

Inhaltsbereich – Tab „Gruppe“ mit Smartlist


Default:

  • Checkbox zur Mehrfachselektion
  • Bezeichnung

Optional:

  • Button: Gruppe hinzufügen
  • Button: Entfernen

Regeln

Smartlist

  • Jede Spalte verfügt oberhalb des Spaltentitels über ein Suchfeld, falls aktiviert (globale Dokumentation).
  • Buchungklassifikationstyp steht nur für die Benutzergruppen zur Auswahl, welche vorgängig zugeordnet wurden.

Entfernen
Es kann/können ein oder mehrere Ressourcen selektiert werden. Klick auf „Entfernen“ bewirkt folgendes:

  • Sicherheitsabfrage
  • Entfernt die Verlinkung zum Buchungsklassifkationsypen
  • Aktualisierung Smartlist
  • Bestätigungsmeldung inkl. der Anzahl der Entfernungen im Actionpanel

„Ressourcen hinzufügen“

  • Ein modaler Dialog wird geöffnet und eine/mehrere Benutzergruppen können dem Buchungsklassifikationstype hinzugefügt werden (siehe Schritt 9)

Schritt 10 - Nutzer kann (bei entsprechender Berechtigung) Buchungsklassifikationen oder Buchungsklassifikationstypen löschen

Controls

Inhaltsbereich

  • Buchungsklassifikationen-Baum
    • Bearbeiten Icon
    • View Icon
    • Neu Icon
    • Löschen Icon

Regeln

Löschen

  • Klick auf Löschen, Warnhinweis erscheint
  • Bei erfolgreicher Löschung erscheint Bestätigungsmeldung im Actionpanel
  • Aktualisierung der Baumstruktur
  • Löschregeln:
    • Löschung eines Klassikfikationstyps nur möglich, wenn keine Ressource mehr verknüpft und keine anghängten Klassifikationen mehr vorhanden sind. Ansonsten Warnmeldung im Actionpanel
    • Löschung einer Klassifikation nur möglich, wenn keine Verbindung mehr zu einer Buchung vorhanden ist. Ansonsten Warnmeldung im Actionpanel.
