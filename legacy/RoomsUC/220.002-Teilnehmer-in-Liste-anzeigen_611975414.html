Use Cases : 220.002 Teilnehmer in Liste anzeigen

Ziel & Beschreibung

Teilnehmer werden in der Liste angezeigt

Rechte

  • Darf Teilnehmerliste sehen

Bedingungen

 

Hauptszenario in Schritten

  1. Nutzer navigiert auf >Listen >Teilnehmer
  2. Nutzer gibt Suchkriterien ein (optional) und klickt Button „Finden"
  3. Ein Suchresultat über alle Teilnehmer wird angezeigt
  4. Walk-in Teilnehmer wird erfasst

Beziehungen zu UC

 

 


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Use Case Details

Schritt 1 - Nutzer navigiert auf >Listen >Teilnehmer

Controls

Sidepanel

Navigation öffnet sich entsprechend

Regeln

Rechte müssen implementiert werden:

  • Darf Teilnehmerliste sehen
 

Schritt 2 - Nutzer gibt Suchkriterien ein (optional) und klickt Button „Finden“

Controls im Sidepanel

Generell (dies wird nicht als Subpanel abgebildet)

  • Favoritenliste
  • Datumpicker: Datum/Zeit von/bis
  • Standortpicker: Standort
  • Feld: Reservationstitel
  • Feld: Vorname
  • Feld: Nachname
  • Button „Finden“
  • Button „Zurücksetzen

Regeln

Favoritenliste (siehe auch UC 230.001 Liste erstellen und (als Favorit) speichern)

  • Wird eine Liste als Favorit definiert wird diese standardmässig angezeigt.
  • Ansonsten ist die Liste leer.
  • Nach Wechsel eines Eintrages, werden Sidepanel und Inhaltsbereich automatisch neu geladen
    (Achtung: Bevor Liste gespeichert wird, muss "Finden" Button geklickt werden, um alle Kriterien zu speichdern)

Datum/Zeit von/bis

  • Standard = nicht selektiert
  • Standard Datum = aktueller Tag
  • Standard Zeit = 00:00 bis 23:59 (= 1 Tag)

Standort

  • Als Multipler Picker verfügbar

Titel

  • Reservations- oder Anlasstitel
  • Wildcardsuche (*) möglich

Vorname

  • Wildcardsuche (*) möglich

Nachname

  • Wildcardsuche (*) möglich

Finden

  • sucht mittels "AND" Verknüpfung alle Teilnehmer von Reservationen und Anlässe
  • siehe Schritt 3

Zurücksetzen

  • setzt die Suchmaske auf den Standardwert zurück

Bilder

 

Schritt 3 - Ein Suchresultat über alle Teilnehmer wird angezeigt

Controls

Smartlist

Standard Spalten

  • Name
  • Beginn
  • Ende
  • Titel
  • Organisator
  • Ressource
  • Standort
  • VIP

Optionale Spalten

  • Mail
  • Kommentar
  • Ersteller
  • Firma
  • Standort Hierarchisch

Button Bereich

  • Button: Neuer Teilnehmer

Regeln

Name

  • Name des Teilnehmers
  • Format: Vorname Nachname
  • siehe UC 201.001 Raum reservieren für Reservationen
  • siehe UC 211.001 Anlass erstellen für Anlässe

Beginn

  • Start der Reservation oder Anlass
  • Vorlaufzeiten werden nicht berücksichtigt
  • Format: dd.mm.yyyy HH.MM
  • siehe UC 201.000 Ressourcen reservieren für Reservationen
  • siehe UC 211.001 Anlass erstellen für Anlässe

Ende

  • Ende der Reservation oder Anlass
  • Nachlaufzeiten werden nicht berücksichtigt
  • Format: dd.mm.yyyy HH.MM
  • siehe UC 201.000 Ressourcen reservieren für Reservationen
  • siehe UC 211.001 Anlass erstellen für Anlässe

Titel

  • Titel der Reservation oder des Anlasses
  • Nur bei Reservation: Link auf View Mode der Reservation
  • siehe UC 201.000 Ressourcen reservieren für Reservationen
  • siehe UC 211.001 Anlass erstellen für Anlässe

Organisator

  • Organisator der Reservation oder Anlass
  • Format: Vorname Nachname
  • siehe UC 201.000 Ressourcen reservieren für Reservationen
  • siehe UC 211.001 Anlass erstellen für Anlässe

Ressource

  • Ressource Bezeichnung
  • wird in der Sprache des angemeldeten Benutzers ausgegeben
  • nur bei Raum Reservationen verfügbar

Standort

  • Standort Bezeichnung
  • wird in der Sprache des angemeldten Benutzers ausgegeben

VIP

  • Werte: ja/nein
  • siehe UC 201.000 Ressourcen reservieren für Reservationen
  • siehe UC 211.001 Anlass erstellen für Anlässe

Mail

  • E-Mail Adresse des Teilnehmers
  • kann leer sein
  • siehe UC 201.001 Raum reservieren für Reservationen
  • siehe UC 211.001 Anlass erstellen für Anlässe

Kommentar

  • Kommentar auf dem Teilnehmers
  • kann leer sein
  • siehe UC 201.001 Raum reservieren für Reservationen
  • siehe UC 211.001 Anlass erstellen für Anlässe

Ersteller

  • Ersteller der Reservation oder Anlass
  • Format: Vorname Nachname
  • siehe UC 201.000 Ressourcen reservieren für Reservationen
  • siehe UC 211.001 Anlass erstellen für Anlässe

Firma

  • Firma/OE des Organisators
  • wird in der Sprache des angemeldten Benutzers ausgegeben

Standort Hierarchisch

  • Standort Bezeichnung in hierarchischem Aufbau
  • wird in der Sprache des angemeldten Benutzers ausgegeben

Neuer Teilnehmer

  • Spontan Teilnehmer kann erfasst werden - ohne einer Reservation oder eines Anlasses zuzordnen
  • Klick = Anlass wird erstellt mit einem Teilnehmer
  • siehe Schritt 4

Bilder

 

Schritt 4 - Walk-in Teilnehmer wird erfasst

Controls

Inhaltsbereich

  • Feld: Vorname
  • Feld: Nachname
  • Feld: Mail
  • Feld: Betreff
  • Feld: Kommentar
  • Intelibox: Organisator
  • Datumpicker: Zeit
  • Standortpicker: Standort
  • Dropdown: Anlasstyp
  • Checkbox: VIP

Button Bereich

  • Button: Speichern
  • Button: Spechern & Neu
  • Button: Abbrechen

Regeln

Vorname

  • Vorname des Teilnehmers
  • Vorname und Nachname ODER E-Mail müssen erfasst werden

Nachname

  • Nachname des Teilnehmers
  • Vorname und Nachname ODER E-Mail müssen erfasst werden

Mail

  • E-Mail des Teilnehmers
  • Vorname und Nachname ODER E-Mail müssen erfasst werden

Betreff

  • Bezeichnung/Grund des Teilnehmer
  • Default: Spontaner Teilnehmer (kann ¨übersetzt werden - siehe UC 400.006 Übersetzungen verwalten)

Kommentar

  • zusätzlicher Kommentar zum Teilnehmer

Organisator

  • Verantwortliche Person (Organisator)
  • Standard: angemeldeter Benutzer

Zeit

  • Datum/Zeitpicker
  • Standard Datum: aktueller Tag
  • Standard Zeit von: aktuelle Stunde und aktuelle Minute
  • Standard Zeit bis: aktuelle Stunde + 1 und aktuelle Minute

Standort

  • Standort des Besuches (des Teilnehmers)
  • Default: Standard-Standort aus dem Profil des angemeldeten Benutzers

Anlasstyp

  • Dropdown aus UC 120.001 Anlasstyp verwalten
  • Standard: erster Wert in Liste

VIP

  • VIP Teilnehmer?
  • a/nein

Speichern

  • Speichert den Teilnehmer
  • Wechsel auf Smartlist
  • Smartlist wird aktualisiert
  • siehe Schritt 2

Speichern & Neu

  • Speichert den Teilnehmer
  • Fokus bleibt auf der aktuellen Ansicht
  • Felder werden geleert
  • Neuer Teilnehmer kann erfasst werden

Abbrechen

  • Wechsel auf Smartlist
  • siehe Schritt 2
