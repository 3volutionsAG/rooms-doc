Use Cases : 215.002 Bestehende Reservationsanfrage suchen und bearbeiten

Created by Former user (Deleted), last modified on May 30, 2012

ID

215.001

Owner

﻿Former user (Deleted)

Rev-Number

Priorität

2

Ziel & Beschreibung

Bestehnde Reservationsanfragen suchen, bearbeiten und daraus einen Anlass erstellen.

Rechte

  • Darf Sektion Listen nutzen
  • Darf Reservationsanfragen bearbeiten

Bedingungen

Vorgängig müssen Anlass Status und Anlass Typen erfasst worden sein (siehe UC 120.001 Anlasstyp verwalten und 120.002 Anlassstatus verwalte)

Hauptszenario in Schritten

  1. Benutzer navigiert auf >Listen >Listen - Reservationsanfragen
  2. Benutzer gibt Suchkriterien ein (optional)
  3. Liste mit allen Reservationsanfragen wird angezeigt
  4. Benutzer navigiert in View Mode und kann Anfrage zu einem neuen Anlass übertragen

Use Case Details

Schritt 1 - Benutzer navigiert auf >Listen >Listen - Reservationsanfragen

Controls

Navigation öffnet sich entsprechend

Regeln

Menü nur sichtbar mit Recht:

  • Darf Reservationsanfragen bearbeiten

Bilder

Schritt 2 - Benutzer gibt Suchkriterien ein (optional) und klickt auf Finden

Controls im Sidepanel

Generell (wird nicht als Subpanel angezeigt)

  • Datumpicker: Datum/Zeit
  • Intelibox: Standort

Anfragespezifische Kriterien

  • Feld: Titel
  • Dropdown: Anlasstyp
  • Multiselektor: Status
  • Dropdown: Ist Anlass vorhanden

Personenspezifische Kriterien

  • Intelibox: Verantwortlicher
  • Intelibox: Organisator
  • Intelibox: Ersteller

Buttons

  • Finden
  • Zurücksetzen

Regeln

Generell

  • alle Suchkriterien stehen in einer AND Verknüpfung zueinander

Status

  • Folgende Status stehen zur Verfügung
    • Neu
    • in Verarbeitung
    • Bestätigung erwartet
    • Gebucht
  • Es kann ein oder mehrere Status angeklickt und danach gesucht werden
  • Die Status stehen in einer ODER Suche zu einander

Datum/Zeit

  • Defaultwert: Datum = heute; Standardzeiten gemäss globale Einstellungen (analog Anlasssuche)

Anlasstyp

  • Zeigt alle Anlasstypen (Request Formulare) für welche der angemeldete Benutzer Berechtigung hat (siehe UC 120.001 Anlasstyp verwalten)
  • Recht notwendig: Darf Reservationsanfragen bearbeiten
  • Standard: alle

Verantwortlicher, Organisator, Ersteller

  • Intelibox Person
  • Standardwert = leer

Ist Anlass vorhanden

  • alle
  • ja
  • nein

Bilder

Schritt 3 – Liste mit allen Reservationsanfragen gemäss Suchkriterien wird angezeigt

Controls

Standardspalten

  • Titel (mit Link)
  • Edit Icon
  • Von
  • Bis
  • Organisator
  • Ersteller
  • Anlass vorhanden?

Optionale Spalten

  • Bemerkungen
  • Anlasstyp
  • Standort
  • Status
  • Priorität
  • Anzahl Teilnehmer
  • Verantwortlicher

Button Bereich

  • Löschen

Regeln

Generell

  • Die Inhalte der Smartlist wird in der Sprache des angemeldeten Benutzers angezeigt

Edit Icon (Bearbeiten)

  • Weiterleitung auf UC 215.001 Reservationsanfrage erfassen - Schritt 3

Anlass vorhanden

  • Sofern die Reservationsanfrage bereits mit einem neuen Anlass „verknüpft“ ist (Daten übertragen), dann wird hier ein „JA“ ausgegeben (siehe Schritt 8)

Löschen

  • Mehrfachselektion erlaubt
  • Sicherheitshinweis
  • Reservationsanfrage wird gelöscht und aus Smartlist entfernt - Aktualisierung der Smartlist
  • Bestätigungsmeldung im Actionpanel

Bilder

Schritt 4 - Benutzer navigiert in View Mode und kann Anfrage in einen neuen Anlass übertragen

Controls

Generell

  • Controls gemäss Konfiguration aus UC 120.001 Anlasstyp verwalten (Tab "Requestform") werden dargestellt
  • Der View Mode wird in der Sprache des angemeldeten Benutzers angezeigt

Buttons

  • Bearbeiten
  • Zurück
  • Anlass erstellen
  • Drucken

Regeln

Bearbeiten

  • Weiterleitung auf UC 215.001 Reservationsanfrage erfassen - Schritt 3

Anlass erstellen

  • Es wird ein neuer Anlass erstellt
  • Sämtliche Daten der Reservationsanfrage werden in die Stammdaten des Anlasses kopiert (siehe UC 215.001 Reservationsanfrage erfassen)
  • Die unstrukturierten Controls (Custom Controls) werden nicht auf den Anlass übernommen
  • Der Fokus ist auf dem Anlass (Bearbeiten Modus)
  • Die Reservationsanfrage wird als separates Tab im Viewmodus des Anlass (an dritter Stelle) dargestellt (siehe UC 211.001 Anlass erstellen)
