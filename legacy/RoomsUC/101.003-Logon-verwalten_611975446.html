Use Cases : 101.003 Logon verwalten

Ziel & Beschreibung

Ein Benutzer kann mehrere Logins beinhalten (z.B. verschiedene Windows Accounts, welche aber auf dasselbe (User-) Profil gemappt sein müssen). Dieser UC basiert auf dem UC 101.001 – Personen verwalten.

Rechte

  • Darf Person editieren intern
  • Darf Person editieren extern

Bedingungen

Alternativ können/werden Logins mit den Personen zusammen über den Benutzerdaten Import eingelesen

Hauptszenario in Schritten

  1. Nutzer navigiert auf Tab „Logons"
  2. Eine Liste mit allen erfassten „Logons" wird angezeigt - ein oder mehrere Logons können gelöscht werden
  3. Nutzer Logon editieren oder neues Logon zuordnen

Beziehungen von/zu UC


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Controls

Keine zusätzlichen Regeln

Bilder

 

Schritt 2 - Eine Liste mit allen erfassten „Logins“ wird angezeigt - ein oder mehrere Logons können gelöscht werden

Controls

Smartlist mit folgenden Spalten

  • Logonname
  • Bearbeiten Icon
  • Logontyp

Button Bereich:

  • Button „Erstellen“
  • Button „Entfernen“

Regeln

Entfernen

  1. Sicherheitsabfrage
  2. Logon wird gelöscht
  3. Smartlist wird aktualisiert
  4. Bestätigungsmeldung inkl. Anzahl der gelöschten Logons im Actionpanel

Erstellen/Bearbeiten Icon

  • es öffnet sich ein modales Popup

Bilder

 

Schritt 3 - Nutzer Logon editieren oder neues Logon zuordnen

Controls

Inhaltsbereich modales Popup

  • Feld: Logonname
  • Dropdown Logontyp:
    • Forms Credentials
      • Passwort (optional)
      • Passwort Bestätigung (optional)
    • Windows NT Credentials
    • (Custom Servervariable)
    • QR Code
      • Button: QR Code erstellen
      • Bild: QR Code
    • PIN
      • Button: PIN erstellen
      • Label: PIN

Button Bereich

  • Button Speichern
  • Button Abbrechen

Regeln

Logonname

  • Eindeutige Benutzerkennung
  • nur bei Forms Credential, Windows Credential & Custom Servervariable verfügbar

Logontyp

  • Alle implementierten Logons stehen zur Auswahl zur Verfügung.

Passwort

  • Passwort erscheint nur bei Logontyp „Forms Credentials“ – dann ist dieser obligatorisch

 

QR Code

Field
Description & Roules
Logontyp
  • Neuer Typ "QR Code" implementieren
Code erstellen
  • Button wird aktiv nach Wahl des Logontyps "QR Code"
  • Klick auf Button führt zu QR
QR Code
  • enthält den generierten und verschlüsselten QR Code des Benutzers


PIN

Field
Description & Roules
Logontyp
  • Dieser wird nur angezeigt wenn in der globalen Konfiguration von ROOMS der Wert "Darf PIN Authentisierung nutzen" den Wert True (1) hat
  • Default: False (0)
Code erstellen
  • Button wird aktiv nach Wahl des Logontyps "PIN"
  • Klick auf Button führt zu PIN
PIN
  • Enthält den generierten im System eindeutigen 4-6 stelligen PIN.
