Use Cases : 104.001 Gliederungen verwalten

Created by Former user, last modified by Former user (Deleted) on Oct 14, 2014

Use Case Überblick

 

ID

104.001

Owner

Former user (Deleted)

Rev-Number

2.2

Priorität

2

Ziel & Beschreibung

Die vollständige Verwaltung von Gliederungen, im Groben:

  • Die Gliederungen werden in jeweils nur zwei Hierarchiestufen aufgeteilt (Hauptgliederung und Subgliederung) sowie die Initialstufe (welche auch als „Gliederungen“ bezeichnet wird)
  • Es werden keine Gliederungen standardmässig von GARAIO zur Verfügung gestellt
  • Der Kunde kann selber ein Set von Gliederungen erstellen und diese als Filter-und/oder Berechtigungs-Kriterien verwenden.

Rechte

  • Darf Gliederungen verwalten

Bedingungen

–

Hauptszenario in Schritten

  1. Nutzer navigiert auf >Einstellungen >Erweiterte Basisdaten >Gliederungen
  2. Nutzer kann durch den Gliederungen-Baum navigieren und die weiteren Aktionen ausführen
  3. Nutzer wechselt in Viewmodus einer Gliederung oder Gliederungstypen
  4. Nutzer kann Gliederungstypen erfassen und diese editieren
  5. Nutzer kann neue Gliederungen erfassen und diese editieren
  6. Liste mit allen der Gliederung zugeordneten Ressourcen wird angezeigt - eine oder mehrere Ressourcen können entfernt werden
  7. Ressourcen können zu der Gliederung hinzugefügt werden
  8. Liste mit allen der Gliederung zugeordneten Benutzergruppen wird angezeigt - eine oder mehrere Benutzergruppen können entfernt werden
  9. Benutzergruppen können einer Gliederung hinzugefügt werden
  10. Nutzer kann Gliederungen löschen

Beziehungen von/zu UC


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Schritt 2 - Nutzer kann durch den Gliederungen-Baum navigieren und die weiteren Aktionen ausführen

Controls

Inhaltsbereich

  • Gliederungen-Baum
    • Bearbeiten Icon
    • View Icon
    • Neu Icon
    • Löschen Icon
  • Button „Gliederungstyp erstellen“

Regeln

Der Tree (Gliederungsbaum) ist zweistufig. Je nachdem wo das man sich im Tree befindet, kann man auf den jeweiligen Nodes mehr oder weniger Aktionen durchführen (auch an der Anzahl Icons ersichtlich).

Auf der Hierarchiestufe „Gliederungstypen“ stehen die Buttons „Neuerfassen“, „Anzeigen“, „Editieren“ und „Löschen“ zur Verfügung.

Klick auf „Neuerfassen“-Button möglich

  • Man gelangt ins Detail und kann einen neue Subgliederung erfassen (siehe Schritt 5)

Klick auf „View“ Button möglich

  • Details der Hauptgliederung werden nicht editierbar angezeigt (siehe Schritt 3)

Klick auf „Edit Button möglich

  • Man gelangt ins Detail des Gliederungstyps und kann diese editieren (siehe Schritt 4)

Klick auf „Delete“ Button möglich

  • Löscht die entsprechende Gliederung (Siehe Schritt 8)

Auf der Hierarchiestufe „Gliederung“ stehen die Buttons „Anzeigen“, „Editieren“ und „Löschen“ zur Verfügung.

Klick auf „View“ Button möglich

  • Details der Hauptgliederung werden nicht editierbar angezeigt (siehe Schritt 3)

Klick auf „Edit Button möglich

  • Man gelangt ins Detail der Subgliederung und kann diese editieren (siehe Schritt 5)

Klick auf „Delete“ Button möglich (falls entsprechende Berechtigungen vorhanden)

  • Löscht die entsprechende Subgliederung (Siehe Schritt 8)

Gliederungstyp erstellen
Benutzer kann einen neuen Gliederungstyp erstellen (siehe Schritt 4)

Schritt 4 - Nutzer kann neue Hauptgliederungen erfassen oder bestehende editieren

Controls

Inhaltsbereich TAB/Register „Stammdaten“

EINGABEBEREICH - Felder

  • Textfeld: Bezeichnung
  • Rich-Intelibox (Multiselektdropdown): Ressourcenart
  • Aggregiert für
  • Eingabefeld Sortierung
  • Checkbox: sichtbar (ja/nein)
  • Checkbox: Sonderzimmer (ja/nein)

Buttons

  • Speichern
  • Abbrechen

Regeln

Bezeichnung wird in der Benutzersprache des angemeldeten Benutzers geladen (über Buch-Icon) kann das Feld übersetzt werden.

Sortierung
Mittels Sortierung wird definiert, an welcher Stelle resp. in welcher Reihenfolge dieser Gliederungstyp gegenüber den anderen Gliederungstypen erscheinen soll (Relevanz im Sidepanel)

Ressourcenart

  • Es können 1-n Ressourcenarten (Sitzungszimmer, Equipment, etc.) ausgewählt werden, womit definiert wird, bei welchen Ressourcenarten das Feld dargestellt werden und als Filterkriterium dienen soll.
  • Es sind nur diejenigen Ressourcenarten wählbar, auf denen das entsprechendes Funktionsrecht vorhanden ist.

Aggregiert für

Wahl einer Ressourcenart für welche diese Gliederungstyp zusammengezogen wird. Wird als nummerische Eingabefelder dargestellt, um dem Benutzer eine Zahleneingabe zu ermöglichen.

(Beispiel: Gliederungstyp "Equipmenttyp" von der Ressourcenart "Equipment" wird auf dem "Raum" aggregiert. Somit stehen dem Benutzer im Reservationsprozess alle Gliederungen - Beamer, Flipchart, etc. - auf dem Raum zur Verfügung. Dies erlaubt die Suche nach einem freien Sitzungszimmer mit Beamer)

Sichtbar
Eine zusätzliche Checkbox „sichtbar“ definiert, ob das Feld dann effektiv als Filterkriterium-Feld in den Listen sichtbar dargestellt wird oder nicht sichtbar (nur für Berechtigungszwecke) ist (über alle ausgewählten Ressourcenarten)

 

Sonderzimmer

  • Checkbox mit Werten ja/nein (ja = selektiert)
  • Relevant bei der Nutzung des Jobmanagers (s. UC 500.001 Massenverarbeitung mit Jobmanager)

 

Speichern

  • Speichert die Daten
  • Bestätigungsmeldung im Actionpanel
  • Rücksprung auf Gliederungsbaum

Abbrechen

  • Rücksprung auf Gliederungsbaum
    ----Bemerkungen (betrifft andere UC)
    Das neu erstellte und als sichtbar gekennzeichneter Gliederungstyp, taucht in folgenden Masken wie folgt auf (in diesem Beispiel gehen wir davon aus, dass nur die Ressourcenart „Sitzungszimmer“ bei der Erstellung der Hauptgliederung „Raumtyp“ definiert wurde):
  1. Sidepanel: Generell sofern in einem Sidepanel die Ressourcenart „Sitzungszimmer“ gewählt wurde, muss der Gliederungstyp (z.B. „Raumtyp“) als Multiselekt Intelibox innerhalb des „Ressourcenspezifischen Suchfilters“ dargestellt werden.
  2. Sidepanel: Bei mehreren selbst erstellten Hauptgliederungen für die Ressourcenart „Raum“, werden diese in der Reihenfolge gemäss Sortierung innerhalb des „Ressourcenspezifischen Suchfilters“ dargestellt werden.
  3. Sidepanel: Die neu erstellten Felder / Hauptgliederungen müssen (sofern die bereits erwähnten Bedingungen erfüllt sind) unter den Menüpunkten „Reservieren“, „Listen“ und bei „Einstellungen“ im Sidepanel als Filterkriterium dargestellt werden
  4. Sidepanel: Handelt es sich um eine aggregierten Gliederungstyp, wird der Gliederungstyp nur bei Selektion der Ressourcenart, welche den Eintrag "aggregiert für" hat, angezeigt. 
  5. Ressource: Als Selektionskriterium bei der Ressource im Tab "spezifische Daten" zur Auswahl. Aggregierge Gliederungen: gleiche Regel wie unter Punkt 4

Gliederung in Suche
Nachstehende Abbildung wie diese im Sidepanel dargestellt werden. Die Gliederungen werden im Subpanel „ressourcenspezfische Filter“ in Abhängigkeit zur gewählten Ressourcenart geladen.

Schritt 6 - Liste mit allen der Gliederung zugeordneten Ressourcen wird angezeigt – eine oder mehrere Ressourcen können entfernt werden

Controls

Inhaltsbereich – TAB „Ressourcen“

Liste/Grid mit Spalten
Default:

  • Selektionscheckbox
  • Name (ID)
  • Bearbeiten Icon
  • Bezeichnung
  • Ressourcenart
  • Standort

Optional:

  • Verantwortliche Person
  • Status
  • Button: Ressource hinzufügen
  • Button: Entfernen

Regeln

Smartlist
Jede Spalte verfügt oberhalb des Spaltentitels über ein Suchfeld, falls aktiviert (globale Dokumentation).

Entfernen
Es kann/können ein oder mehrere Ressourcen selektiert werden. Klick auf „Entfernen“ bewirkt folgendes:

  • Sicherheitsabfrage
  • Entfernt die Verlinkung zur Gliederung
  • Bestättigungsmeldung inkl. Der Anzahl der Entfernungen im Actionpanel

„Ressourcen hinzufügen“

  • Ein modaler Dialog wird geöffnet und man kann eine/mehrere Ressource( n ) der Gliederung hinzufügen (siehe Schritt 7)

Schritt 7 - Ressourcen können zu der Gliederung hinzugefügt werden

Controls

Modaler Dialog

Liste/Grid mit Spalten
Default:

  • Selektionscheckbox
  • Name (ID)
  • Bearbeiten Icon
  • Bezeichnung
  • Ressourcenart
  • Standort

Optionale Spalten

  • Status
  • Verantwortliche Person

Buttons

  • Ressourcen hinzufügen
  • Entfernen

Regeln

Smartlist
Jede Spalte verfügt oberhalb des Spaltentitels über ein Suchfeld, falls aktiviert (globale Dokumentation).

Ressourcen hinzufügen

  • Mehrfachselektion erlaubt
  • Regeln:
    1. Funktionsrecht für den entsprechenden Ressourcentypen
    2. Zuordnung des Ressourcentypen auf den Gliederungstypen
  • Ressourcen werden der Liste (siehe Schritt 6) hinzugefügt
  • Popup wird geschlossen
  • Bestätigungsmeldung im Actionpanel über die Anzahl der hinzugefügten Ressourcen

Alternative über 105.001 Ressourcen verwalten
Alternativ können Gliederungen auch direkt auf der Ressource zugeordnet werden. Unter „spezifische Daten“ werden alle Gliederungen in Abhängigkeit der Ressourcenart und in Form von Dropdown oder Checkboxfields aufgelistet.

Durch Auswahl eines Eintrages (optional) und/oder Klick von einer oder mehreren Checkboxen (optional) kann die Ressource den Gliederungen zugeordnet werden.

(Handelt es sich um eine aggregierten Gliederungstyp, wird der Gliederungstyp nur bei Selektion der Ressourcenart, welche den Eintrag "aggregiert für" hat, angezeigt)

Schritt 10 - Nutzer kann (bei entsprechender Berechtigung) Gliederungen löschen

Controls

Inhaltsbereich

  • Gliederungen-Baum
    • Bearbeiten Icon
    • View Icon
    • Neu Icon
    • Löschen Icon

Regeln

Löschen

  • Klick auf Löschen, Warnhinweis erscheint
  • Bei erfolgreicher Löschung erscheint Bestätigungsmeldung im Actionpanel
  • Gliederungstypen können nur gelöscht werden, wenn unterhalb keine Gliederungen mehr bestehen – ansonsten erscheint Fehlermeldung im Actionpanel
  • Gliederungen mit zugeordneten Ressourcen können nicht gelöscht werden – es erscheint eine Fehlermeldung im Actionpanel
