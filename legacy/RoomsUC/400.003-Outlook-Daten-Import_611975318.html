Ziel & Beschreibung

CSV Exporte eines Outlook Kalenders in ROOMS importieren

Rechte

  • Darf Outlook Kalender Daten importieren

Bedingungen

Pro Raum wurde eine CSV Datei wurde aus dem Outlook per Export erstellt und manuell bereinigt (z.B. Absagen gelöscht)

Hauptszenario in Schritten

  1. Nutzer navigiert auf Einstellungen >System >Outlook Import und klickt Button „Finden"
  2. Erfassungsmaske für Outlook Import wird angezeigt - eine CSV Datei wird ins System geladen
  3. Liste aller Reservationen wird angezeigt (Vorschau)
  4. Buchungen können mutiert werden
  5. Reservationen werden im System gespeichert

Schritt 2 - Erfassungsmaske für Outlook Import wird angezeigt - eine CSV Datei wird ins System geladen

Controls

Sidepanel

  • Dateipicker: Outlook Exportdatei
  • Ressourcenpicker: Zielressource
  • Feld: Trennzeichen
  • Feld: Feldbegrenzerzeichen
  • Feld: Maskierungszeichen
  • Personen-Picker: Default-Organisator
  • Feld: Spaltenindex Titel
  • Feld: Spaltenindex Beginn-Datum
  • Feld: Spaltenindex Beginn-Zeit
  • Feld: Spaltenindex End-Datum
  • Feld: Spaltenindex End-Zeit
  • Feld: Spaltenindex Organisator
  • Feld: Spaltenindex erforderliche Teilnehmer
  • Feld: Spaltenindex optionale Teilnehmer
  • Feld: Spaltenindex Bemerkungen
  • Feld: Spaltenindex Ist Privat

 

Button Bereich

  • Button: Import starten

Regeln

Generell

  • es können nur CSV Dateien verarbeitet werden
  • die CSV wird automatisch durch einen Kalender Export aus Outlook generiert
  • das Format der CSV Datei und ein Besipiel ist im Anhang zu finden

Outlook Exportdatei

  • Dateipicker besteht aus
    • Feld für Anzeige
    • Button: Browse zur Dateiselektion
  • Nach Selektion erfolgt ein automatischer Upload:
    • Actionpanel bestätigt Upload ODER
    • Fehlermeldung im Actionpanel

Trennzeichen

  • Standardwert: , (Komma)
  • Alternativen: ; (Semikolon), \t (Tabulator)

Feldbegrenzerzeichen

  • Standardwert: "
  • Alternativen: ' (Hochkomma)

Maskierungszeichen

  • Standardwert: "" 
  • Beispiel

Default-Organisator:

  • Standard: der angemeldete Benutzer
  • Wird verwendet, falls eine Person aus der Importdatei nicht in ROOMS gefunden wird.

Spalten Indexe

  • definiert die Reihenfolge der Spalten in der Importdatei
  • Beispiel: an welcher Stelle steht der Titel?

Import starten

  • Reservationen werden temporär importiert
  • Das System weist im Actionpanel jede Reservation aus, welche an einem nicht verfügbaren Zeitpunkt liegt: "Die Ressource (Ressource.Bezeichnung) ist zwischen (Import.Reservation.Von) und (Import.Reservation.bis) nicht verfügbar"
  • siehe Schritt 3

Schritt 3 - Liste aller Reservationen wird angezeigt (Vorschau)

Controls

Liste

  • Status Icon
  • Reservationstitel
  • Organisator
  • Icon: Bearbeiten
  • Von
  • Bis
  • Anzahl Personen
  • Privat

Button Bereich

  • Button: Speichern
  • Button: Abbrechen

Regeln

Status Icon

  • Es werden folgende Stati ausgegeben:
    • rot = Buchung kann nicht verarbeitet werden
    • grün = Buchung kann verarbeitet werden

Organisator

  • wird aufgelöst auf Grund der folgenden Parameter:
    • Vorname Nachname
    • Nachname Vorname
    • E-Mail
    • Personalnummer
  • Konnte keine Auflösung stattfinden wird der Organisator angezeigt und ein Edit Icon dargestellt

Bearbeiten

  • Modales Popup öffnet sich
  • siehe Schritt 4

Von

  • Reservationsstart Datum & Zeit
  • Format: dd.mm.yyyy HH:MM
  • Keine Vorlaufzeiten möglich
  • Ist zur aktuellen

Bis

  • Reservationsende Datum & Zeit
  • Format: dd.mm.yyyy HH:MM
  • Keine Vorlaufzeiten möglich

Speichern

  • siehe Schritt 5

Abbrechen

  • Wechselt zurück auf Schritt 3

Bilder

Reservationen Liste

Actionpanel Meldung für Reservationen, welche an einem nicht verfügbaren Zeitpunkt liegen

 

Schritt 4 - Buchungen können mutiert werden

Controls

Modales Popup

  • Label: Titel
  • Intelibox: Organisator
  • Datumpicker: Datum/Zeit

Button Bereich

  • Übernehmen
  • Abbrrechen

Regeln

Organisator

  • Organisator kann manuell gewechselt werden
  • Ersteller bleibt in diesem Fall der CurrentUser (= der Importteur)

Datum/Zeit

  • Datum/Zeit kann angepasst werden
  • nur verfügbar auf der aktuellen Ressource

Übernehmen

  • Verfügbarkeit wird geprüft
  • Bei Fehler: Ausgabe der Fehlermeldung unterhalb des Datums
  • Modales Popup wird geschlossen

Abbrechen

  • schliesst das Modale Popup
  • Wechsel zu Schritt 3

Bilder

 

Schritt 5 - Reservationen werden im System gespeichert

Controls

Inhaltsbereich

  • gemäss Schritt 3

Regeln

Generell

  • Bestätigungmeldung im Actionpanel: Es wurden (0) Reservationen gespeichert.
  • Nicht espeicherte Buchung werden entfernt
  • Buchungen wurden erstellt:
    • für den Ersteller wird der gleiche Wert wie der Organisator
    • Teilnehmer werden erstellt - auflösbare Teilnehmer werden mit ROOMS Benutzer verbunden
    • Nicht gefundene Teilnehmer werden als "externe Teilnehmer" hinzugefügt

  • Header wird ignoriert - Spaltenbezeichnung ist dementsprechend irrelevant
