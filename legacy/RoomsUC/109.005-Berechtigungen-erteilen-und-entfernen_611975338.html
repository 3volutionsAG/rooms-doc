Use Cases : 109.005 Berechtigungen erteilen und entfernen

Created by Former user (Deleted), last modified on Jul 16, 2013

Use Case Überblick

 

ID

109.005

Owner

Tim Bänziger

Rev-Number

1.0

Priorität

1

Ziel & Beschreibung

Berechtigungen können auf Standorte und Ressourcen durch Zuweisung von Benutzergruppen zu Rollen vergeben werden.

Rechte

  • Darf Standort bearbeiten
  • Darf Standorte anzeigen/editieren
  • Darf Berechtigungen für Ressourcen verwalten
  • Darf Berechtigungen für Standorte verwalten
  • Darf Sektion Einstellungen nutzen

Bedingungen

UC 105.001 Ressourcen verwalten wurde ausgeführt ODER
UC 103.001 Standorte verwalten wurde ausgeführt

Hauptszenario in Schritten

  1. Navigation auf Tab "Berechtigungen"
  2. Nutzer kann Liste aller berechtigen Benutzergruppen - Rollen Zuweisungen sehen - und eine oder mehrere löschen
  3. Nutzer kann neue Berechtigungen vergeben

Beziehungen zu UC

105.001 Ressourcen verwalten
103.001 Standorte verwalten


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Use Case Details

Schritt 1 - Navigation auf Tab "Berechtigungen"

Controls

keine

Regeln

Eines der folgenden Rechte notwendig:

  • Darf Standorte bearbeiten
  • Darf Standorte anzeigen/editieren
  • Darf Berechtigungen für Ressourcen verwalten
  • Darf Berechtigungen für Standorte verwalten

Bilder

 

Schritt 2 - Nutzer kann Liste aller berechtigen Benutzergruppen - Rollen Zuweisungen sehen - und eine oder mehrere entfernen

Controls

Smartlist

  • Checkbox zur Mehrfachselektion
  • Benutzergruppe
  • Rolle
  • Geerbt (ja/nein)
  • Vererbt (ja/nein)

Button Bereich

  • Entfernen
  • Hinzufügen
  • Speichern
  • Abbrechen

Regeln

Smartlist
Die Smartlist verfügt über ein Suchfeld oberhalb der Spalten Gruppe und und Rolle

Button „Entfernen“

  • Mehrfachselektion erlaubt
  • Löscht – nach Sicherheitsabfrage – einen  oder mehrere Einträge
  • Bestätigungsmeldung oder Fehlermeldung als Action Message

Button „Hinzufügen“
Öffnet modales Popup zur Konfiguration eines neuen Ereignisses. (siehe Schritt 3)

Bilder

 

Schritt 3 - Nutzer kann neue Berechtigung vergeben

Controls

Modales Popup

Smartlist

  • Checkbox zur Mehrfachselektion
  • Bezeichnung "Rolle"

Weitere Felder

  • Intelibox "Gruppe"
  • Checkbox "Vererben"

Button Bereich

  • Speichern
  • Abbrechen

Regeln

Vererben

  • Checkbox steht nur auf den Standorten zur Verfügung
  • Falls selektiert wird Eintrag bis auf die Ressource vererbt (gemäss Vererbungskonzept - siehe auch 000.001 Globale Funktionen )

Speichern

  • Eintrag wird hinzugefügt - Modales Popup wird geschlossen
  • Nochmal Speichern schliesst den Bearbeitungskontext des Standortes / Ressource - Ausgabe Action Message: Eintrag wurde gespeichert.

 

Abbrechen

  • Popup wird geschlossen
  • Vergebene Berechtigungen werden nicht gespeichert
