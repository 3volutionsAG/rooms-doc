Use Cases : 201.013 Equipment reservieren

Created by Former user, last modified on Jun 22, 2011

Use Case Überblick


ID

201.013

Owner

Tim Bänziger

Rev-Number

1.1

Ziel & Beschreibung

  1. Das Suchen und Finden von einem oder mehreren, den Kriterien entsprechenden, verfügbaren Räumen (Ressourcenart „Equipment“).
  2. Die Reservation der selektierten Equipment Ressource ( n )
  3. Das Editieren einer Reservation (Schritte 4,5 und 6)

Datenrechte

–

Funktionsrechte

  • Funktionsrechte von SUC 201.001 gelten

Alternativszenario

Ein oder mehrere Equipments können über Schnellreservation gebucht werden (AS 201.001)

Ausnahmeszenario

Kein Suchresultat gefunden:

  • Globale Fehlermeldung wird ausgegeben (Fehlermeldung und Verhalten in SUC 201.001 dokumentiert)

Bedingungen

  • Defaults müssen geladen werden (PF 302)

Hauptszenario in Schritten

  1. Navigation auf „Reservieren“ – Wahl der Ressourcenart „Equipment“ im Sidepanel
  2. Subpanel „Ressourcenspezifische Kriterien“ wird geöffnet. Nutzer kann zusätzliche Filterkriterien angeben (optional) und „Suchen“ klicken.
  3. System liefert Suchresultat (zusätzlich zu SUC 201.001) der verfügbaren Ressourcen
  4. Nutzer navigiert ins Detail, erfasst Equipment spezifische Daten und schliesst Reservation ab
  5. Summary Page wird angezeigt (Optional)

Beziehungen zu UC

201.001 Raum reservieren


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt



Use Case Details

Schritt 1 - Navigation auf „Reservieren“ – Wahl der Ressourcenart „Equipment“ im Sidepanel

Controls

Sidepanel

  • Intelibox „Ressourcenart“

Regeln

Benutzer muss Resourcenart „Equipment“ wählen (entsprechendes Funktionsrecht aus dem SUC 201.001 muss vorhanden sein)

Schritt 2 - Subpanel „Ressourcenspezifische Kriterien“ wird geöffnet. Nutzer kann zusätzliche Filterkriterien angeben (optional) und „Suchen“ klicken.

Controls

Sidepanel – Subpanel (ressourcenspezifische Suchfilter)

  • Multiselektionsbox „Equipment“ (zeigt alle Equipmenttypen aus den Gliederungen)
  • Gliederungen (restliche)

Regeln

Subpanel Ressourcenspezifischen Kriterien ist per Default geöffnet und wird nach Wahl der Ressourcenart „Equipment“ neu geladen

Grundsätzlich werden als ressourcenspezifische Filterkriterien nur Gliederungen angezeigt. Zum einen ein Spezialfall der Gliederungen (Equipmenttyp) und zum anderen die restlichen Gliederungen (siehe dazu 104.001 Gliederungen verwalten )

Gliederungen Equipmenttyp

  • Beim Equipment handelt es sich um einen speziellen Gliederungstyp. Dieses wird zwar in einer Multiselektionsbox angezeigt (Vergleiche dazu 104.001 Gliederungen verwalten ), muss aber statt Checkboxen Eingabefelder für Ganzzahlen haben. Standardwert ist 0.
  • Es werden nur die Equipmenttypen angezeigt, bei welchen der angemeldete Benutzer auch min. Leserechte hat
  • Darstellung der Liste, sollte noch mit dem Softwareentwickler besprochen werden. Ev. sollte die Liste zwei-spaltig dargestellt werden (wäre allerdings nur bei einem Popup möglich), bei manchen Kunden hat es bis 70 Equipmenttypen.
  • Mehrfachselektionen möglich

Suche
Es wird nur nach mobilem Equipment (nicht einer anderen Ressource zugeordnet) gesucht

Gelbmarkierter Bereich wurde für Release 4.1 zurückgestellt: Datenrechte auf den Gliederungen

Schritt 4 - Nutzer navigiert ins Detail, erfasst Equipment spezifische Daten und schliesst Reservation ab

Controls

Es gibt keine Equipment spezifische Daten

Regeln

Es gibt keine Equipment spezifische Daten

Folgende Tabs dürfen nicht sichtbar sein, auch wenn entsprechende Funktionsrechte vorhanden:

  • Dienstleistungen
  • Bestuhlung
  • Teilnehmer
  • Equipment

Folgende Tabs sind auf Grund der Funktionsrechte sichtbar:

  • Serieninformationen
  • Equipment (zusätzliche Ressource)

Folgende Felder im Detailbereich dürfen nicht sichtbar sein (gegenüber 201.001 Raum reservieren ):

  • Kapazität (Anzahl Personen
