Use Cases : 201.011 SOT Parkplatz reservieren

Created by Former user, last modified on Jun 22, 2011

Use Case Überblick


ID

201.011

Owner

Tim Bänziger

Rev-Number

1.1

Ziel & Beschreibung

  1. Das Suchen und Finden von einem oder mehreren, den Kriterien entsprechenden, verfügbaren Kurzzeit Parkplätzen (Ressourcenart „Parkplatz“).
  2. Die Reservation der selektierten Parkplatz Ressource ( n )
  3. Das Editieren einer Reservation (Schritte 4 und 5)

Datenrechte

–

Funktionsrechte

  • Funktionsrechte von SUC 201.001 gelten

Alternativszenario

Ein oder mehrere Parkplätze können über Schnellreservation gebucht werden (AS 201.001)

Ausnahmeszenario

Kein Suchresultat gefunden:

  • Globale Fehlermeldung wird ausgegeben (Fehlermeldung und Verhalten in SUC 201.001 dokumentiert)

Bedingungen

  • Defaults müssen geladen werden (PF 302)
  • Es werden nur SOT Parkplätze, welche verfügbar sind angzeigt

Hauptszenario in Schritten

  1. Navigation auf „Reservieren“ – Wahl der Ressourcenart „Parkplatz“ im Sidepanel
  2. Subpanel „Ressourcenspezifische Kriterien“ wird geöffnet. Nutzer kann zusätzliche Filterkriterien angeben (optional) und „Suchen“ klicken.
  3. System liefert Suchresultat (zusätzlich zu SUC 201.001) der verfügbaren Parkplatz Ressourcen
  4. Nutzer navigiert ins Detail, erfasst Parkplatz spezifische Daten und schliesst Reservation ab
  5. Summary Page wird angezeigt (Optional)

Beziehungen zu UC

201.001 Raum reservieren


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt



Use Case Details

Schritt 1 - Navigation auf „Reservieren“ – Wahl der Ressourcenart „Parkplatz“ im Sidepanel

Controls

Sidepanel

  • Intelibox „Ressourcenart“

Regeln

Benutzer muss Resourcenart „Parkplatz“ wählen (entsprechendes Funktionsrecht aus dem SUC 201.001 muss vorhanden sein)

Schritt 2 - Subpanel „Ressourcenspezifische Kriterien“ wird geöffnet. Nutzer kann zusätzliche Filterkriterien angeben (optional) und „Suchen“ klicken.

Controls

Sidepanel – Subpanel (ressourcenspezifische Suchfilter)

  • Gliederungen (Parkplatzspezifische)

Regeln

Subpanel Ressourcenspezifischen Kriterien ist per Default geöffnet und wird nach Wahl der Ressourcenart „Parkplatz“ neu geladen

Grundsätzlich werden als ressourcenspezifische Filterkriterien nur Gliederungen angezeigt (siehe dazu UC 104.001).

Es ist davon auszugehen, dass zwei Gliederungen erfasst werden (damit sich der Entwickler ein Bild machen kann):

  • Parkplatztyp
    • Behindertenparkplatz
    • Normal
    • Frauenparkplatz
  • Parkplatzart
    • Garage
    • Aussenparkplatz
  • Es werden nur die Gliederungen angezeigt, bei welchen der angemeldete Benutzer auch min. Leserechte hat
  • Darstellung der Gliederungen gemäss 104.001 Gliederungen verwalten (Singleselektion in der Dropdown und Multiselektion in Form von Checkboxlisten)

Suche
Es wird nur nach verfügbaren Parkplätzen vom Typ SOT gesucht UND Freigegebenen Parkplätze vom Typ LOT ( 201.010 Buchungen löschen )

Gelbmarkierter Bereich wurde für Release 4.1 zurückgestellt: Datenrechte auf den Gliederungen

Schritt 4 - Nutzer navigiert ins Detail, erfasst Parkplatz spezifische Daten und schliesst Reservation ab

Controls

Es gibt keine Parkplatz spezifische Daten

Regeln

Es gibt keine Parkplatz spezifische Daten

Folgende Tabs dürfen nicht sichtbar sein, auch wenn entsprechende Funktionsrechte vorhanden:

  • Dienstleistungen
  • Bestuhlung
  • Teilnehmer
  • Equipment

Folgende Tabs sind auf Grund der Funktionsrechte sichtbar:

  • Serieninformationen
  • Equipment (zusätzliche Ressource)

Folgende Felder im Detailbereich dürfen nicht sichtbar sein (gegenüber 201.001 Raum reservieren ):

  • Kapazität (Anzahl Personen
