Use Cases : 201.031 Massenstornierung von Buchungen

Ziel & Beschreibung

Zwei oder mehrere Buchungen können im gleichen Prozessschritt storniert werden

Rechte

  • DarfMassenmutationenVornehmen

Bedingungen

  • Es bestehen Reservationen
  • Benutzer befindet sich im Plan, Kalender oder unter Listen

Hauptszenario in Schritten

  1. Nutzer selektiert zwei oder mehr Reservationen
  2. Nutzer klickt Button „Stornieren“

Beziehungen zu UC




Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Use Case Details

Schritt 1 - Nutzer selektiert zwei oder mehr Reservationen

Controls

  • Listenansicht: Benutzer hat bei zwei oder mehreren Reservationen die Checkbox in der Smartlist selektiert
  • Planansicht: Benutzer selektiert mittels Mauslkick mehrere Hotspots. Wird ein Hotspot selektiert, bleibt es eingeblendet; ausserdem wird die Transparenz der Polygon-Füllung entfernt, d.h. der Polygon wird hervorgehoben.
  • Kalender: Benutzer hat bei zwei oder mehreren Reservataionen die Checkbox selektiert. Box erscheint beim Mouse-Over, sofern folgendes Recht vorhanden ist: DarfMassenmutationenVornehmen

Regeln

Rechte müssen implementiert werden:

  • DarfMassenmutationenVornehmen

Bilder

Schritt 2 - Nutzer klickt Button „Stornieren“

Controls

Inhaltsbereich

  • Button „Stornieren“

Regeln

Stornieren

  • erscheint nur mit dem Recht "DarfMassenmutationenVornehmen"
  • Button wird im Buttonbereich des Inhaltsbereich angezeigt (Höhe "auto-refresh" Icon)

Ablauf:

  1. Klick auf Button prüft für jede selektierte Reservation, ob der Benutzer Annullations-Rechte besitzt (schliesst auch Prüfung auf Vergangenheit usw. mit ein -> RechtRules) und ob der Eintrag noch nicht gelockt ist:
    • Trifft dies auf keinen Eintrag zu, wird im ActionPanel die Meldung "Sie können zur Zeit keine der selektierte(n) Reservation(en) annullieren." ausgegeben; der Benutzer bleibt auf der aktuellen Ansicht, die Checkboxen sind wieder deaktiviert.
    • Trifft dies auf genau einen Eintrag zu, wird der Einzel-Annullationsprozess (Popup mit Kosten usw.) gestartet (ohne zusätzliche Meldung); wurden mehrere Einträge selektiert, wird vorgängig ein Popup mit der Meldung "Sie annullieren 1 Reservation; die andere(n) X Reservation(en) können zur Zeit von Ihnen nicht annulliert werden.".
  2. Hat der Benutzer für mehr als eine Reservation die benötigten Rechte, wird ein Ja/Nein-Dialog angezeigt. Die Frage lautet:
    • Wenn alle selektierten Reservationen annulliert werden können und keine Kosten entstehen: "Wollen Sie die selektierten X Reservationen wirklich annullieren?"
    • Wenn alle selektierten Reservationen annulliert werden können und Kosten entstehen: "Für die Annullationen entstehen Annullationskosten. Wollen Sie die selektierten X Reservationen wirklich annullieren?"
    • Wenn nicht alle selektierten Reservationen annulliert werden können und keine Kosten entstehen: "Sie können zur Zeit Y der selektierten Reservationen nicht annullieren. Wollen Sie die selektierten X Reservationen wirklich annullieren?"
    • Wenn nicht alle selektierten Reservationen annulliert werden können und Kosten entstehen: "Sie können zur Zeit Y der selektierten Reservationen nicht annullieren. Für die Annullationen entstehen Annullationskosten. Wollen Sie die selektierten X Reservationen wirklich annullieren?"
  3. Klickt der Benutzer Nein, passiert nichts (Checkboxen bleiben selektiert).
  4. Klickt der Benutzer Ja, wird die Seite neu geladen (Checkboxen wieder deselektiert)
  5. die selektierten Reservationen werden gelockt und ein neuer Aktionseintrag mit allen relevanten Angaben für die Annullationen wird erstellt
  6. Ausgabe im ActionPanel: "Die Änderungen werden nun übernommen. Sie werden wieder informiert, sobald der Vorgang abgeschlossen ist."
  7. Asynchrone Abarbeitung (durch den Service) des Auftrags
  8. Ausgabe im Actionpanel: "Die Änderungen von X Einträgen vom Typ Reservation sind abgeschlossen (a / b / c wurden angepasst; d / e / f konnten nicht angepasst werden)."
