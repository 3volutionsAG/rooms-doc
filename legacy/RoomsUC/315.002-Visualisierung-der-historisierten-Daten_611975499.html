Use Cases : 315.002 Visualisierung der historisierten Daten

Ziel & Beschreibung

Nachvollziehbarkeit der Änderung von wichtigen und relevanten Daten

Rechte

  • Darf historisierte Daten sehen

Bedingungen

Nutzer befindet sich im View Mode des gewünschten Records

Hauptszenario in Schritten

  1. Benutzer klickt im Viewmodus das „History“ Tab
  2. History Daten werden gruppiert nach Änderung angezeigt

Beziehungen zu UC

 


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

 

Use Case Details

Schritt 1 - Benutzer klickt im Viewmodus das „History“ Tab

Controls

Sidepanel

  • Tab „History“ muss implementiert werden

Regeln

  • Nur sichtbar für Benutzer mit Funktionsrecht „Darf historisierte Daten sehen“
 

Schritt 2 - History Daten werden gruppiert nach Änderung angezeigt

Controls

Inhaltsbereich
Es wird eine gruppierte Liste angezeigt:

Gruppierung

  • Datum/Zeit
  • Art (Änderung, Löschung, Erstellung)
  • Modifiziert durch: Vorname + Name

Details

  • Feld (wenn möglich ein sprechender Feldnamen)
  • Alter Wert
  • Neuer Wert

Regeln

Gruppierungen

  • Die Gruppierungen können auf Link Klick aufgeklappt werden, um die Details anzuzeigen
  • Die Gruppierung sind chronologisch absteigend nach Datum sortiert (aktuellstes Datum zuoberst)
  • Ein Paging ist nicht zwingend notwendig

Details

  • Ausgebene Werte müssen für den Benutzer sprechend sein - keine ID's
