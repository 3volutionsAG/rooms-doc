Name

Concurrency-Locking auf allen Elementen sowie neuen Objekten

Ziel

Ein Datensatz soll nur von einem Benutzer gleichzeitig bearbeitet werden können.

Abhängigkeiten zu anderen UC

–



Wichtige Anmerkungen

Jede Liste (Grid) beinhaltet immer das Personalisieren von Spalten, Sortierung und Paging sowie optionale Checkboxen für Mehrfachselektion.

Jede Detailansicht beinhaltet immer einen Readonly-Modus, einen Editier-Modus für Einzelmutationen und einen Editier-Modus für Mehrfachmutationen.

Diese Funktionalitäten müssen deshalb beim Beschreiben der Use Cases nicht aufgeführt werden.

Zudem müssen Validierungsregeln zum aktuellen Zeitpunkt nicht beschrieben werden.

Grobbeschrieb

Eine Framework-Komponente übernimmt als Teil des O/R-Mappers das Sperren von Datensätzen; zum Bearbeiten muss ein Checkout/Checkin erfolgen.

Jede Detailansicht eines bestehenden Objekts befindet sich zuerst im Readonly-Modus; der Benutzer muss explizit auf Editieren klicken, was ein Checkout auslöst. Beim Speichern oder Wegnavigieren erfolgt ein Checkin.

Von dieser Sperrung nicht betroffen sind Massenmutationen.
