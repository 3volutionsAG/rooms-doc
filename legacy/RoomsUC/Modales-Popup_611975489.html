Use Cases : Modales Popup

Created by Former user, last modified by Former user on Jan 09, 2012

Innerhalb eines Tabs (im Inhaltsbereich) wird der Inhalt teilweise in Smartlists angezeigt

Es gibt zwei Gründe um aus dieser Ansicht in ein modales, transparentes Popup abzuspringen:

  1. Einzelner Record wird geändert oder hinzugefügt (Edit oder Add Modus)
  2. Ein Mapping von verschiedenen Records findet statt (z.B. mehrere Benutzer der aktuellen OE zuordnen). Innerhalb dieses Popups gibt auch Suchfunktionen Suchfelder über dem Grid 

Modales Popup Darstellung

  • Der ganze Applikationsbereich wird mit einem transparenten Schwarz eingefärbt
  • Der schwarze Bereich ist nicht klickbar
  • Auf der Fläche des Inhaltsbereich hat das modale Popup einen editerbaren Bereich (weisse Darstellung)
  • In diesem Bereich werden die editerbaren Controls aufgelistet
  • Wird das Browserfenster in der Grösse geändert, ändert sich auch der editerbare Bereich
  • Der Editierbare Bereich ist scrollbar
  • Fehlermeldungen und Hilfetexte müssen auch im modalen Dialog angezeigt werden
  • Das modale Popup schliesst sich auf Knopfdruck (Abbrechen oder Funktionsknopf)
