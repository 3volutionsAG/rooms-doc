Use Cases : 212.001 Video Conference erstellen

Created by Former user (Deleted), last modified on Jun 04, 2014

ID

212.001

Owner

Former user (Deleted)

Rev-Number

1.0

Priorität

Ziel & Beschreibung

Eine Video Konferenz kann benutzergesteuert erstellt werden. Im Hintergrund wird ein Event erstellt.

Erstellte Video Konferenzen können dann unter den Events gesucht und bearbeitet werden (siehe auch 211.002 Anlass suchen und finden

Video Konferenzen haben (im Gegensatz zu Events) kein Catering Bereich

Rechte

    • Darf Anlässe als Kalender sehen
    • Darf Anlässe buchen
    • Darf Anlässe mit Reservationen buchen
    • Darf Anlässe mit Teilnehmernotifikationen editieren
    • Darf Teilnehmer editieren
    • Darf VideoConferencing Buchen sehen

Bedingungen

Benutzer befindet sich unter Experten Suche (RESERVIEREN) oder KALENDER

VC Buchung mit oder ohne TMS (kann global konfiguriert werden: Eingabe der "TMS ROOT URL" (siehe 000.004 Konfigurationsparameter)

Hauptszenario in Schritten

  1. Nutzer navigiert auf Panel Video Conferencing
  2. Nutzer gibt Host-, Child-Location und (optional) weitere Suchkriterien ein und klickt Button „Anzeigen"
  3. Kalenderansicht wird angezeigt
  4. Buchungsinformationen & VC wird angezeigt
  5. System validiert Eingaben und Verbindung

Beziehungen zu UC


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem künftigen Release umgesetzt

Schritt 2 - Nutzer gibt Host-, Child-Location und (optional) weitere Suchkriterien ein und klickt Button „Anzeigen“

Controls Sidepanel

Generell

  • Dropdown: Gespeicherte Listen (siehe UC 230.001 Liste erstellen und (als Favorit) speichern)
  • Standortpicker: Host location
  • Button: Standorte hinzufügen
  • Standortpicker (N): Child location
  • Button: Teilnehmer

Erweiterte Kriterien

  • Feld: Ressourcen Name/Bezeichnung
  • Checkbox: Suche ohne Substandorte
  • Checkbox: meine favorisierten Ressourcen
  • Zahlenpicker: Kapazität (Anzahl Personen)
  • Checkbox: Max. Kapazität berücksichtigen (Max. Bestuhlung)

Kapazität Child Locations

  • Pro Standort werden folgende Felder angezeigt
    • Label: Standort
    • Zahlenpicker: Kapazität (Anzahl Personen)
    • Checkbox: Max. Kapazität berücksichtigen (Max. Bestuhlung)

Gliederungen

  • Ressourcenspezifische Kriterien der Gliederungen (siehe UC 104.001 Gliederungen verwalten und 201.000 Ressourcen reservieren) (D)

Klassifikationen

  • Ressourcenspezifische Kriterien der Klassifikationen (siehe UC 104.002 Buchungsklassifikationen verwalten und 201.000 Ressourcen reservieren) (C)



  • Button „Anzeigen“

 

Regeln

Generell

Favoritenliste (siehe auch UC 230.001 Liste erstellen und (als Favorit) speichern)

  • Wird eine Liste als Favorit definiert wird diese standardmässig angezeigt.
  • Ansonsten ist die Liste leer.
  • Nach Wechsel eines Eintrages, werden Sidepanel und Inhaltsbereich automatisch neu geladen
    (Achtung: Bevor Liste gespeichert wird, muss "Finden" Button geklickt werden, um alle Kriterien zu speichdern)


Host Location (Definition Standortpicker in 000.001 Globale Funktionen)

  • Die Möglichkeit einer Standort Selektion
  • Im Popup des Standort-Pickerwerden nur Standorte angezeigt für welche der Benutzer das standortabhängige Recht "Darf Standorthierarchie sehen (Reservieren)" hat
  • Damit Standorte im Intelibox Modus des Pickers erscheinen ist eines der folgenden Standortabhängigen Rechte notwendig:
    • Darf Reservationen lesen
    • Darf prov. Reservationen bearbeiten
    • Darf def. Reservationen bearbeiten
  • Nach Selektion eines Standorts erscheint der Button "Standorte hinzufügen"

Standorte hinzufügen

  • erscheint nur wenn ein Host Standort ausgewählt wurde
  • Durch klicken wird ein neuer Child Standort hinzugefügt

Child Standort (Definition Standortpicker in 000.001 Globale Funktionen)

  • Die Möglichkeit einer oder mehrerer Standort Selektion
  • Im Popup des Standort-Pickerwerden nur Standorte angezeigt für welche der Benutzer das standortabhängige Recht "Darf Standorthierarchie sehen (Reservieren)" hat
  • Damit Standorte im Intelibox Modus des Pickers erscheinen ist eines der folgenden Standortabhängigen Rechte notwendig:
    • Darf Reservationen lesen
    • Darf prov. Reservationen bearbeiten
    • Darf def. Reservationen bearbeiten
  • Nach Selektion eines Standorts erscheint ein weiteres  Child Standort-Picker Feld

Teilnehmer

  • zeigt im Kalenderbereich (siehe Schritt 3) die Teilnehmer
  • erneutes Klicken, blendet die Teilnehmer wieder aus

Erweiterte Kriterien

  • Panel standardmässig geöffnet

Kapazität (Anzahl Personen)

  • Standard ist leer - wird nicht vom Profil übernommen. 
  • Leer = es werden alle Räume angezeigt 
  • Manuell erfasster Wert kann in Filter gespeichert werden. Auch der Wert max. Kapazität (berückscihtigt alle Bestuhlungen einer Ressource) 
  • Berechnung erfolgt nur auf Ressourcenart "Raum" - d.h. alle anderen Ressourcenarten werden immer angezeigt 
  • Berechnung wirkt sich auf Ressourcenbox aus.

Max. Kapazität berücksichtigen (Max. Bestuhlung)

  •  Berücksichtigt alle Bestuhlungen einer Ressource) 
  • Der Wert wird in "meinen Listen" gespeichert
  • Berechnung erfolgt nur auf Ressourcenart "Raum" - d.h. alle anderen Ressourcenarten werden immer angezeigt 
  • Berechnung wirkt sich auf Ressourcenbox aus.
     

Ressourcen Name/Bezeichnung

  • Suche nach über die ID der Ressourcen und über die Bezeichnung in der Benutzersprache des angemeldten Benutzers

Suche ohne Substandorte

  • Begrenzt die Suche auf den  aktuell gewählten Standort oder die aktuelle gewählten Standorte. Die jeweiligen Substandorte werden nicht berücksichtigt.

meine favorisierten Ressourcen

  • zeigt nur Ressourcen auf welchen die angemeldete Person jemals als Owner mindestens eine Buchung hatte (angemeldete Person = Organisator)
  • Berücksichtigt werden alle Ressourcenarten
  • Berücksichtigt werden Vergangenheit, Gegenwart und Zukunft

Kapazitäten Child Standort

  • Nur sichtbar sofern ein Child Standort selektiert wurde
  • Panel standardmässig geöffnet

Pro Child Standort werden folgende Felder angezeigt und für die Suche berücksichtigt:

Kapazität (Anzahl Personen)

  • Standard ist leer - wird nicht vom Profil übernommen. 
  • Leer = es werden alle Räume angezeigt 
  • Manuell erfasster Wert kann in Filter gespeichert werden. Auch der Wert max. Kapazität (berückscihtigt alle Bestuhlungen einer Ressource) 
  • Berechnung erfolgt nur auf Ressourcenart "Raum" - d.h. alle anderen Ressourcenarten werden immer angezeigt 
  • Berechnung wirkt sich auf Ressourcenbox aus.

Max. Kapazität berücksichtigen (Max. Bestuhlung)

  •  Berücksichtigt alle Bestuhlungen einer Ressource) 
  • Der Wert wird in "meinen Listen" gespeichert
  • Berechnung erfolgt nur auf Ressourcenart "Raum" - d.h. alle anderen Ressourcenarten werden immer angezeigt 
  • Berechnung wirkt sich auf Ressourcenbox aus.

 

Gliederungen (siehe auch UC 104.001 Gliederungen verwalten)

  • ganzer Bereich wird nur eingeblendet, wenn Gliederungen vorhanden sind
  • es werden alle verfügbaren Gliederungstypen pro Ressourcenart angezeigt, sofern
  • Gliederungstyp als "sichtbar" (Checkbox) markiert wurde
  • Die einzelnen Gliederungen sind sichtbar, sofern sich der angemeldete Benutzer in einer Benutzergruppe befindet, die der entsprechenden Gliederung zugeordnet worden ist

Klassifikationen (siehe UC 104.002 Buchungsklassifikationen verwalten - Schritt 4 ff.)

  • ganzer Bereich wird nur eingeblendet, wenn Klassifikationen vorhanden sind
  • Buchungsklassifikationstypen erscheinen als Dropdown oder Multiselektoren (sofern als solche aktiviert), falls
    • der angemeldete Benutzer sich in einer der Buchungsklassifikationstypen zugewiesenen Benutzergruppe befindet UND
    • der Buchungsklassifikationstyp der gewählten Ressourcenart zugewiesen worden ist UND
    • die Checkbox "sichtbar in Listen" aktiviert worden ist
  • Bei der Suche gelten die Checkboxen als AND Verknüpfungen untereinander und in Bezug auf alle anderen gewählten Kriterien

Schritt 3 - Kalenderansicht wird angezeigt

Controls Kalender

Kalenderkopf

  • Link: Legende
  • Icon: Teilnehmer anzeigen
  • Navigationselement Vor und Zurück (mit Pfeilen über dem Kalender)
  • Slider: Zoom
  • Button: Angepasst
  • Heute Button
  • Datum Picker

Kalenderbereich

  • Ressourcenliste (mit Tooltip) wird vertikal angezeigt
  • Checkbox VC Raum Selektion 

Tagesansicht gruppiert nach Standorten

  • 24 Stunden Ansicht
  • Zentrierung auf 12:00 (bei kleiner Auflösung wichtig)
  • Navigation über ziehen der Zeitlinie
  • Anzeige Zeit im Halbstunden Takt
  • Zeit Highlighter

Button: Weiter zu Stammdaten erfassen

 

Regeln

Generell

  • Zeitlinie bleibt immer Fix - Scrolling erfolgt unterhalb
  • sind mehrere Standorte selektiert (siehe Schritt 2), werden diese als Blöcke untereinander dargestellt
  • Bei den Blöcken wird der jeweilige Standort in Form der Hierarchie als Titel dargestellt
  • ganz oben im Kalender werden die Teilnehmer dargestellt

Ressourcenliste

  • Von oben nach unten werden die Ressourcen Bezeichnung gemäss Priorität aufgelistet (siehe Feld "Priorisierung" aus UC )
  • Anzeige der Ressourcenbezeichnung in der Sprache des angemeldeten Benutzers
  • Mouse-Over: Ausgabe des Tooltipps - Konfigurierbar global (siehe )
  • Checkboxen für die Selektion für Schnellreservation (siehe UC 201.201 Schnellreservation)

Legende

  • Klick auf Link öffnet ein Popup
  • Es werden die wichtigsten Zusatände angezeigt:
    • Farbgebung Sperrzeiten (fix)
    • Farbgebung Feiertage (fix)
    • Farbgebung Meine Reservationen (aus Profil - siehe auch UC 101.001 Personen verwalten)
    • Farbgebung Status (beliebige gemäss Definition - siehe auch UC 108.002 Status verwalten)
    • Farbgebung Buchungsklassifikationen (siehe auch UC 104.002 Buchungsklassifikationen verwalten)
    • Icons Serie und Anlass.
  • Bei kleiner Auflösung ist die Legende Scrollbar

Slider Zoom

  • Standard = Default Mode (siehe "Angepasst")
  • Minus: reduziert die Kalenderansicht in zentel Schritten (1/10)
  • Plus: maximiert die Kalenderansicht in zentel Schritten (1/10)
  • Vergrössern kann also zur Folge haben, dass nicht mehr die vollen 24h angezeigt werden und horizontal (auf der Zeitlinie) gescrollt werden muss
  • Verkleiner kann also zur Folge haben, dass Leerbereiche rechts vom Kalender entstehen
  • Diese Einstellung wird in den persönlichen Filter gespeichert
  • Die Einstellung wird in Prozenten gespeichert. So dass, wenn sich der Benutzer mit einer anderen Auflösung der Kalender öffnet dieser trotzdem gezoomt dargestellt wird.

Angepasst

  • Passt den Kalender gemäss Bildschirmauflösung an die optimale Grösse an => ganzer Kalender ist sichtbar und füllt den ganzen Inhaltsbereich
  • Diese Einstellungen wird in den persönlichen Favoriten gespeichert
  • Wird dieser Mode in den persönlichen Favoriten gespeichert, wird immer der Angepasst Modus verwendet, unabhängig davon mit welcher Auflösung der Benutzer das nächste Mal die Kalenderansicht aufruft

Navigationselement

  • der Zeitbalken  kann mittels Maus-Ziehen nach links oder nach rechts innerhalb der gewählten Zeitperiode (Tag, Woche, Arbeitswoche) verschoben werden
  • Automatisches Nachladen der Reservationen auf den Ressourcen

Mehrzeilige Ansicht

  • Erhöht den Kalender um zwei zusätzlichen Zeilen pro Ressource
  • Zusätzlich werden Von und Bis Zeit angezeigt
  • Zusätzlich wird der Kommentar angezeigt

Heute Button

  • Klick auf Heute Button wechselt unabhängig vom gewählten Kalender auf das aktuelle Datum (oder eben aktuelle Kalenderwoche)
  • Automatisches Nachladen der Reservationen auf den Ressourcen

Datum Picker

  • Eingae eines Datums über Eingabefeld oder Kalender-Popup
  • Wechsel auf das gewählte Datum (automatisch nach Eingabe)
  • Automatisches Nachladen der Reservationen auf den Ressourcen

Intelibox Person

  • Funktionalität in UC Inteliboxen & Multiselektoren
  • Auswahl einer Person durch tippen
  • Ist die Person gewählt, werden auf dem Server die "Free/Busy" Daten abgefragt
    • Sind keine Free/Busy Daten vorhanden wird die Kalenderlinie grau dargestellt
    • Sind Free/Busy Daten vorhanden werden diese in rot dargestellt 
    • Achtung: Die verfügbaren Daten sind von der Kalender freigabe abhängig, siehe Blog post: http://exchangeserverpro.com/show-full-freebusy-exchange-2010-room-resource-mailboxes/ Es wird nur der "Default" Benutzer berücksichtigt. Explizite Freigaben von Benutzern werden nicht berücksichtigt
  • Free/Busy Einstellungen werden in den Favoriten gespeichert (siehe Schritt 5)
  • Alle ausgewählten Personen werden in Schritt 4 übernommen
  • Mittels Klick auf das Entfernen Icon  wird die Person deselektiert und die Free/Busy Daten dieses Teilnehmers gelöscht 

Zeit Highlighter

  • Mit dem Zeit highleighter kann der gewünschte Zeitpunkt selektiert werden
  • Die grüne Linie definiert den Startzeitpunkt
  • Die rote Linie definiert den Endzeitpunkt
  • Klick mit der Maus zwischen die beiden Linien bewegt beide Linien. Ein Tooltipp zeigt die neuen exakten Start- und Endzeiten 
  • Klick auf die grüne Linie verschiebt den Startzeitpunkt. Ein Tooltipp zeigt die neuen exakten Start- und Endzeiten 
  • Klick auf die rote Linie verschiebt den Endzeitpunkt. Ein Tooltipp zeigt die neuen exakten Start- und Endzeiten 
  • Die selektierten Start- und Endzeiten werden automatisch auf alle Buchungen übernommen

 

VC Raum Selektion

  • ein oder mehrere Räume können selektiert werden

 

Weiter zu Stammdaten erfassen

  • Button ist ausgeblendet
  • es muss min. eine Host Location und ein weiterer Raum ODER ein Teilnehmer selekiert werden, um den Button einzublenden

Titel

Anzahl Personen

  • wird übernommen aus den Eingaben auf den Child Locations im Sidepanel

Kommentar

  • wird automatisch auf alle Buchungen auf Child Locations übertragen

Organisator

  • per Standard wird der aktuell angemeldete Benutzer für alle Buchungen verwendet
  • Intelibox ist beschrieben unter Inteliboxen & Multiselektoren

Kostenträger

  • per Standard wird der Kostenträger des Organisators übernommen

Klassifikationen

  • Beschrieben unter 104.002 Buchungsklassifikationen verwalten
  • Es werden die Buchungsklassifikationen aus Schritt 2 auf alle Buchungen übernommen
  • Validierung erfolgt auf jeder Location (stehen die Klassifikationen auch zur Verfügung) und gibt allenfalls eine Warnung aus

Audio Konferenz Lync ID #

  • Standard = leer
  • Prüfung auf Zahl

Dropdown: Präsentation

  • Werte: ja, nein

  • Eine Checkbox "Linq User" identifiziert den Teilnehmer als Linq Teilnehmer
  • Teilnehmer können mittels Klick auf das "Entfernen" Icon wieder gelöscht werden

Schritt 5 - System validiert Eingaben und Verbindung

Controls

Modales Popup

    • Log Bereich
    • Button: Wiederholen
    • Button: Abbrechen

 

Regeln

Log Bereich

  • Es wird ein Log über alle Aktivitäten ausgegeben
  • Für jede Location (Buchung) wird folgendes geprüft:
    • Titel vorhanden
    • Organisator vorhanden
    • Anzahl Personen entspricht der Standard Bestuhlung
  • Es erfolgt die Verbindungsprüfung zu TMS (nur wenn TMS aktiviert)
  • Ist die Verbindung vorhanden wird versucht ein TMS Eintrag zu erstellen.

 

Fehlerfall

  • Ist die Validierung der Ressourcen fehlgeschlagen kann dies wiederholt werden (siehe Button "Wiederholen")
  • Ist die Verbindung und/oder Erstellung zur TMS fehlgeschlagen, kann der Benutzer eine Wiederholung versuchen (siehe Button "Wiederholen")
  • Oder er bricht den Prozess ab (zurück zu Schritt 4)
  • In beiden Fällen (TMS ist nicht verfügbar oder ein Fehler bei der Erstellung) wird eine E-mail Notifikation an das Helpdesk geschickt (Sinnes gemäss: The video conferencing system seems to be unreachable. Please contact your helpdesk)
    Dabei wird keine Buchung in ROOMS gespeichert (kein Event, keine Ressourcen, keine TMS).

Wiederholen

  • Nur sichtbar, wenn Validierung fehlgeschlagen oder TMS Buchung fehlgeschlagen
  • Bei TMS Buchung ist nur eine Wiederholung möglich, danach steht nur noch der Cancel Button zur Verfügung

Abbrechen

  • zurück zu Schritt 4


Erfolgsfall

  • Kann die TMS Conference erstellt werden, werden die notwendigen Coference Werte (conferenceid) in den Event geschrieben (nur bei aktivierter TMS)
  • Event wird erstellt: 
    • Title = Title der Buchung der Host Location
    • Location = host location
    • Organisator = Organisator der host location
    • Date/Time = Date/Time of the first (begin) to the last (end) booking
    • EventType = empty (may have to be changed)
  • Buchungen werden erstellt gemäss den Eingaben
  • Weiter zu Schritt 5 des UC 211.001 Anlass erstellen
