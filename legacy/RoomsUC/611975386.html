Use Cases : Actionpanel (neu ActionMessage)

Created by Former user, last modified by Former user on Jun 26, 2013

ActionMessage

Die ActionMessage zeigt die verschiedenen Systemrückmeldungen für den Benutzer. Grundsätzlich unterscheiden wir zwischen folgenden Messages:

Message

Art*

Beschrieb

Verhalten

Warning

temporäre Meldung

Warnings sollen den Benutzer auf einen bestimmten Systemzustand aufmerksam machen. Diese Meldungen sind aber für den Benutzer in erster Linie informativ. Beispiel: Sie buchen an einem Feiertag.

Warnings verschwinden, sobald die Seite das nächste Mal geladen wird. Also z.B. wenn der Benutzer weiter navigiert

Error

temporäre Meldung

Errors sind Meldungen, welche auf eine Fehlmanipulation oder ein Fehlverhalten des Systems aufmerksam gemacht. z.B. Datensatz kann nicht bearbeitet werden, weil gerade Benutzer „xy" diesen bearbeitet.

Errors verschwinden, sobald die Seite das nächste Mal geladen wird. Also z.B. wenn der Benutzer weiter navigiert

Status Normal

temporäre Meldung

Normale Status Meldungen sind als Ergänzung zu den ersten beiden Statusmeldungen um erfolgreich ausgeführte Systeminteratkionen anzuzeigen. Z.B. Der Datensatz wurde erfolgreich gelöscht.

Normale Status Meldungen verschwinden, sobald die Seite das nächste Mal geladen wird. Also z.B. wenn der Benutzer weiter navigiert

Status Aufgabe

persistente Meldung

Bei Statusmeldungen von Aufgaben (oder auch System) geht es um Meldungen, wenn das System eine Aufgabe ausgeführt hat. Z.B. die Generierung eines Reports oder einer Statistik.

Status Meldungen von Aufgaben verschwinden erst, wenn der Benutzer die Meldung quittiert hat, also die Meldung explizit weggeklickt hat.

* temporäre Meldung verschwinden bei Seiten-Wechsel, persistente Meldungen verschwinden erst auf Benutzer-Interaktion

Darstellung

  • pro ausgegebene Meldung erscheint ein kleines Popup Fenster am rechten Browser Rand
  • Ein Icon (Warning, Status, Error) zeigt die Art der Meldung.
  • Beim Mouse-Over erscheint ein Schliessen Icon (X) mit welchem die Meldung weggeklickt werden kann
  • Im Profil kann der Benutzer definieren nach wievielen Sekunden die temporären Meldungen verschwinden sollen oder ob diese manuell weggeklickt werden müssen (siehe auch 101.001 Personen verwalten)
