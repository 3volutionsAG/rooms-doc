Use Cases : 201.019 Reservation stornieren (annullieren)

Created by Former user (Deleted), last modified on May 30, 2014

Use Case Überblick

 

ID

Rev-Number

Priorität

Ziel & Beschreibung

Eine Reservation kann aus der Weboberfläche oder Outlook Plugin storniert werden.
Eine Reservation wird auf "no-Show" gesetzt

Rechte

  • Darf eigene annullieren
  • Darf Reservationen und Serien annullieren (fremde)
  • Darf Reservationen in Vergangenheit buchen
  • Darf Reservationen als No Show kennzeichnen

Bedingungen

Reservationen sind vorhanden (in Listen, Plan oder Kalender)
Benutzer hat vorgängig

  • Reservation im Viewmode geöffnet
  • Actionpopup geöffnet
  • Reservationsticker geöffnet
  • Listenansicht selektiert
  • ViewMode im Outlook selektiert

 

Hauptszenario in Schritten

  1. Reservation wird storniert (annulliert)
  2. Reservation wird "no show" gesetzt

Beziehungen zu UC

Use Case Details

Schritt 1 - Reservation wird storniert

Controls

View, Listenbereich oder Action Popup

  • Button: „Annullieren“

 

Regeln

Generell

  • Rechte
    • Bei eigenen Reservationen (ich bin Organisator, Ersteller oder Verantwortlicher) können immer storniert (annulliert werden)
    • Bei Reservationen Dritter (ich bin weder Organisator, Ersteller oder Verantworlicher) muss das standortabhängige Recht: Darf Reservationen und Serien annullieren (fremde)
  • Der Annullierungsbutton erscheint bei laufenden und/oder zukünftigen Reservationen
  • Für Reservationen in Vergangenheit ist zusätzlich das Recht "Darf Reservationen in Vergangenheit buchen" notwendig

Klick auf Button

  • Popup mit Warnhinweis: Wollen Sie die Reservation wirklich annullieren? Die Annullierungskosten betragen CHF x.xx (die Berechnung der Annullationskosten findet sich in UC 105.004 Tarife verwalten)
    • Verhalten bei einer aktiven Buchung: Es fallen nur die Reservations-Kosten gemäss Kostenmodell an.
    • Verhalten bei einer zukünftigen Buchung: Es fallen die Annullations-Kosten gemäss Kostenmodell an.
    • Checkbox "Annullationsmails versenden"
      • Standardmässig selektiert
      • Nicht selektiert unterdrückt den Versand einer Notifikation an Ersteller und Organisator/Verantwortlicher
  • bei Klick auf „Annullieren“, wird die aktive (laufende) oder zukünftige Buchung auf den Zeitpunkt, an welchem der Button betätigt wurde, storniert (annulliert)
  • In den Kalender-Ansichten ist die Buchung nicht mehr sichtbar.
  • In der Listenansicht ist die Buchung nur noch sichtbar, sofern der entsprechende Status "annulliert" gewählt wurde
  • Nach der Annullation, wird die Buchung immer noch in den Reports (Auswertungen & Statistiken) geführt.
  • Notifikation (ReservationAnnullierung) wird an Ersteller, Organisator und Verantwortlichen verschickt (siehe Notifikationsmatrix)
  • Notifikation (BestellungDelete) wird an Dienstleister verschickt (siehe Notifikationsmatrix und UC 106.002 Angebotszonen verwalten)
  • Notifikation (EquipmentNeuOrUpdate) wird an Dienstleister verschickt (siehe Notifikationsmatrix und UC 106.002 Angebotszonen verwalten)
  • die Synchronisation wird "abgehängt" - die Reservation wird nie wieder synchronisiert
  • Serieninformation
    Vergangenheit und laufende Buchung bleibt bestehen. Künftige Buchungen werden gelöscht. Annullationsmodell muss „greifen“. Bestätigungsmeldung an Organisator/Ersteller (1 E-Mail für die ganze Serie)

Stornierte Buchungen bearbeiten

  • Grundsätzlich können stornierte Buchungen bearbeitet werden (z.b. um nachträglich Annullationskosten zu mutieren)
  • Nicht möglich: Änderung des Status (Reservation bleibt annulliert)
  • Nach Änderung:
    • es werden KEINE Notifikationen verschickt
    • es wird keine Synchronisation vorgenommen

Schritt 2 - Reservation wird "no-show" gesetzt

Controls

  • Button "no-show" auf Listen, Actionpopup und Reservationsdetail (View Mode)

Regeln

Rechte

  • Button wird nur sichtbar mit Recht: Darf Reservationen als No Show kennzeichnen

Klick auf Button

  1. Reservation wird annulliert (gemäss Schritt 1)
  2. Zusätzlicher Text im Warnhinweis: Wollen Sie die Reservation wirklich als No Show kennzeichnen? Die Reservation wird unwiderruflich annulliert, und allfällige Annullationskosten werden angerechnet
  3. Reservation wird zusätzlich als "no-Show" gekenzeichnet
    1. "No-Show" sind in der Listenansicht über den Status Filter "No-Show" auffindbar (siehe auch UC 220.001 Reservationen in Liste anzeigen)

Stornierte Buchungen bearbeiten

  • Grundsätzlich können stornierte Buchungen bearbeitet werden (z.b. um nachträglich Annullationskosten zu mutieren)
  • Nicht möglich: Änderung des Status (Reservation bleibt annulliert)
  • Nach Änderung:
    • es werden KEINE Notifikationen verschickt
    • es wird keine Synchronisation vorgenommen
