Use Cases : 107.003 Vorkonfiguriertes Mail versenden

Ziel & Beschreibung

Nach Änderung wichtiger Stammdaten einer Ressource sollen die betroffenen (Ersteller und Organisatoren von künftigen Reservationen) benachrichtigt werden.

Rechte

–

Bedingungen

  1. Ressource erfasst (UC 105.001 Ressourcen verwalten ) und Änderung auf Ressource vorgenommen
  2. Notifikationen erfasst (UC 107.001 Notifikationen verwalten )

Hauptszenario in Schritten

  1. Klick Checkbox „beim Speichern Notifikation an Betroffene senden“
  2. Ressourcen Speicherung abschliessen

Beziehungen zu UC

Bildet Abschluss von 105.001 Ressourcen verwalten
107.001 Notifikationen verwalten

Use Case Details

Schritt 1 - Klick Checkbox „beim Speichern Notifikation an Betroffene senden“

Controls

Inhaltsbereich --Button Bereich

  • Checkbox „beim Speichern Notifikation an Betroffene senden“

Regeln

Checkbox ist standardmässig „unklicked“

Bilder

Schritt 2 - Ressourcen Speicherung abschliessen

Regeln

Notifikationsregeln

  • Das Notifikationstemplate ist konfiguriert (siehe Notifikationsmatrix)
  • Für jeden Ersteller und Organisator einer Reservation wird ein E-Mail bereitgestellt (Eintrag in Notifikationstabelle)
  • Die Profile der Ersteller und Organisator bezüglich Notifikation werden berücksichtigt (also es erhalten nur diejenigen eine E-Mail, welche dies auch im Profil aktiviert haben)
  • Organisatoren/Ersteller von Reservationen innerhalb einer Serie erhalten nur eine E-Mail
  • Das Mail wird erst bei klick auf „Speichern“ ausgelöst.
  • Das E-Mail wird nur ausgelöst, wenn
    • Der Body im Notifikationstemplate nicht leer ist
    • Eines der definierten Felder geändert wird (Details in 000.006 Notifikationsmatri)
