Ziel & Beschreibung

Die Zuweisung von Feiertagen auf Standorte (UC 103.001 Standorte verwalten) & Ressourcen (105.001 Ressourcen verwalten).
Use Cases müssen in den Schritten 1, 2 und 5 ausgeführt worden sein

Ein Feiertag wirkt sich so aus, dass die Ressourcen zwar buchbar sind, diese aber durch zu definierende Symbole, Farben und Meldungen z.B. im Kalender, Plan, Suchresultat, etc. erkennbar sind.
Im Reservationsprozess wird ein Warnhinweis ausgegeben

Rechte

  1. „Darf Feiertage verwalten“ muss implementiert werden
  2. „Darf Reservationen an Feiertagen buchen“ muss implementiert werden

Bedingungen

  1. Feiertage auf den Standorten werden bis auf die Ressource vererbt und dort gespeichert
  2. Wird eine Ressource an einem Standort neu hinzugefügt erbt die Ressource automatisch den Feiertag des Standortes

Hauptszenario in Schritten

  1. Klick auf Tab „Feiertage" bei Standorten oder Ressourcen
  2. Liste mit allen dem Standort/der Ressource zugeordneten Feiertagen wird angezeigt - ein oder mehrere Feiertage können entfernt werden
  3. Feiertag(e) kann/können dem Standort/der Ressource zugefügt werden

Beziehungen zu UC

Use Case Details

Schritt 1 – Klick auf Tab „Feiertage“ bei Standorten oder Ressourcen

Controls

Sidepanel
Keine Veränderung

Regeln

Sofern Funktionsrecht „Darf Feiertage verwalten“ vorhanden ist

Bilder

 

Schritt 2 - Liste mit allen dem Standort, bzw. Ressource zugeordneten Feiertagen wird angezeigt – ein oder mehrere Feiertage können entfernt werden

Controls

Smartlist

  • Selektionscheckbox
  • Bezeichnung der Feier
  • Ist Rekursiv

 

Button Bereich

  • Button „Hinzufügen"
  • Button "Entfernen"

Regeln

Smartlist
Jede Spalte verfügt oberhalb des Titels über ein Suchfeld, falls aktiviert (siehe 000.001 Globale Funktionen)

Bezeichnung

  • Wird in der Sprache des Benutzers ausgegeben

Ist Rekursiv

  • ob ein Feiertag vererbt wurde

Hinzufügen

  • Siehe Schritt 3

 

Entfernen

  1. Mehrfachselektion erlaubt
  2. Sicherheitsabfrage
  3. Entfernt die Feiertage
  4. Auf dem Standort wird die Vererbung bis zur Ressource entfernt (rekursiv). Vererbungsregeln sind in 000.001 Globale Funktionen dokumentiert
  5. Bestätigungsmeldung im Actionpanel

Bilder

 

Schritt 3 – Feiertag(e) kann/können dem Standort/der Ressource zugefügt werden

Controls

Modaler Dialog

Smartlist

  • Selektionscheckbox
  • Bezeichnung der Feier (Mehrsprachig)

Button Bereich

  • Checkbox „Rekursiv
  • Button „Abbrechen“
  • Button „Hinzufügen"

Regeln

Smartlist
Jede Spalte verfügt oberhalb des Titels über ein Suchfeld, falls aktiviert

Bezeichnung
Wird in der Sprache des Benutzers ausgegeben

Rekursiv

  • Ist diese Checkbox angeklickt wird der Feiertag auf die unteren Standorte bis hin zur Ressource vererbt
  • Standardwert = angeklickt
  • nur Verfügbar bei Zuordnung auf Standort

Hinzufügen

  • Mehrfachselektion erlaubt
  • Popup wird geschlossen
  • Smartlist wird aktualisiert
  • Ausgabe im Actionpanel, inkl. Anzahl der Hinzugefügten Einträge

Abbrechen

  • Modales Popup wird geschlossen
