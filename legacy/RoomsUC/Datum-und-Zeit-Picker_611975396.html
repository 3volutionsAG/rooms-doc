Use Cases : Datum und Zeit Picker

Created by Former user, last modified by Former user on Jan 09, 2012

Der Datum/Zeit Picker besteht aus folgenden einzelnen Controls:

  • Checkbox für „manuelle Zeitangabe"
  • Feld "Datum von"
  • Feld "Zeit von"
  • Feld "Zeit bis"
  • Label "Datum bis" sofern unterschiedlich zu Datum von.



    Checkbox für manuelle Zeitangabe
    Wird diese Checkbox geklickt werden die anderen 4 Felder entsperrt und der Benutzer kann Datum/Zeit ändern. Ist die Checkbox nicht geklickt wird das aktuelle Datum/Zeit verwendet.Die Checkbox hat ein Tooltip: Hier Klicken um NICHT aktuelles Datum/Zeit zu verwenden (manuelle Eingabe). Der Standardwert (Checkbox geklickt oder nicht kann durch jeden Benutzer in seinem Profil festgelegt werden) Datum von
    Validierung auf korrekte Eingabe eines gültigen Datums in der aktuellen Benutzersprache des Users. Mittels Pfeiltasten kann ein Datum vor und zurück gewechselt werden.
    Mittels Klick öffnet sich Popup für Kalender Erfassung.

    Zeit von, Zeit bis
    Prüfung auf Eingabe einer gültigen Zeit – zwei korrekte Varianten möglich:
  • 1800 wird automatisch auf 18:00 gewechselt
  • 18:00

    Weitere Prüfung auf von Datum KLEINER als bis Datum
    Mittels Pfeiltasten kann in einem GLOBAL vordefinierten Intervall vor und zurück navigiert werden.

    Popup
    Das Popup erlaubt die Eingabe mittels Kalender. Folgende Controls sind darin enthalten:
  • Kalender "Datum von"
  • Kalender "Datum bis"
  • Zeit "von"
  • Zeit "bis"

    Kalender „Datum von"
  • Selektion erfolgt durch Anklicken eines Datums.
  • Es kann nur ein Datum >= (grösser, gleich) dem aktuellen Datum selektiert werden. Ausnahme: Funktionsrecht "Darf in Vergangenheit reservieren" ist vorhanden
  • Heute Button (nicht visualisiert) springt auf heutiges Datum
  • Mittels Pfeiltasten kann innerhalb des Kalenders jeweils einen Monat vor und zurück geblättert werden Zeit von, Zeit bisPrüfung auf Eingabe einer gültigen Zeit – zwei korrekte Varianten möglich:
  • 1800 wird automatisch auf 18:00 gewechselt
  • 18:00 Weitere Prüfung auf von Datum KLEINER als bis DatumMittels Pfeiltasten kann in einem GLOBAL vordefinierten Intervall vor und zurück navigiert werden. Kalender „Datum bis"
  • Wird nur angezeigt, wenn Funktionsrecht „darf Bis-Datum editieren" vorhanden ist, ansonsten wird Popup um die Grösse des Kalenders verkleinert.
  • Ansonsten gleiche Funktionalität wie Kalender „Datum von", PLUS
    • Es können nur Daten angeklickt werden, welche >= (grösser, gleich) dem „Datum von" sind

Ist ein „Bis Datum" >= (grösser, gleich) dem „Von Datum" selektiert worden, wird ein Label oberhalb der Bis Zeit erstellt (Inhalt: Bis Datum in Kurzform --siehe auch Screenshot)

      Weitere Regeln
      Nachstehend weitere Regeln zu Datum/Zeit unterteilt in zwei Prioritäten:
      Release 4.0 (Prio 1) und 4.1 (Prio 2)
      Release 4.0 (Prio 1):
  • im gleichen Tab Datum/Zeit beibehalten (persistieren)
  • Zeit Sprung über Pfeile ist immer 15 Minuten
  • Tab „Reservieren" → Von Zeit immer auf die nächste Viertelstunde genau (auch bei aktuelles Datum/Zeit verwenden). Beispiel: aktuell ist es 17:34 dann wird in der von Zeit 17:45 eingetragen. Der vorgeschlagene Zeitrange kann so belassen werden.
  • Tab „Plan" → wurde bereits in Schritt 2 des AS_201.001 spezifiziert.
  • Tab „Liste" → ist bereits OK umgesetzt

    Release 4.1 (Prio 2):
  • Tab „Reservieren Ressourcenart "Arbeitsplatz" auf Minute genau - ansonsten immer auf 15 Minuten genau die nächste Stunde anzeigen
  • Tab „Kalender" → behält grundsätzlich das aktuelle Datum. Wird im Kalenderpicker einen Tag gewählt und zwischen den Kalender gesprungen, dann bleibt das Datum bestehen. Der gewählte Tag sollte in den Wochenansichten markiert sein.
