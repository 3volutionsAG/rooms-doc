Use Cases : 103.001 Standorte verwalten

Ziel & Beschreibung

Verwaltung und Administration von Standorten

Rechte

  • Darf Sperrzeiten editieren (aus 105.005 Sperrzeiten verwalten )
  • Darf Feiertage editieren (aus 105.006 Feiertage importieren und verwalten )
  • Darf Stellvertreter editieren.
  • Darf Standorte anzeigen/editieren
  • Darf Standorte erstellen/löschen

Bedingungen

–

Hauptszenario in Schritten

  1. Nutzer navigiert unter Einstellungen >Basisdaten für Ressourcen >Standorte
  2. Nutzer (mit entsprechendem Recht) kann durch den Standort-Tree navigieren und die weiteren Aktionen festlegen - und einen Standort löschen
  3. View eines Standorts wird angezeigt
  4. Nutzer kann (bei entsprechender Berechtigung) neue Standorte erfassen und vorhandene Standorte editieren

Beziehungen von/zu UC


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Schritt 2 - Nutzer kann durch den Standort-Tree navigieren und die weiteren Aktionen festlegen – und einen Standort löschen

Controls

Inhaltsbereich

  • Standort-Baum
    • Hinzufügen Icon
    • View (Anzeigen) Icon
    • Bearbeiten Icon
    • Löschen Icon

Regeln

Hinzufügen Icon

  • Wechsel ins Detail und Möglichkeit zur Erfassung eines neuen Standort (siehe Schritt 4)

View (Anzeigen) Icon

  • Details werden angezeigt - nicht editierbar (siehe Schritt 3)

Bearbeiten Icon

  • falls entsprechende lokale Berechtigungen vorhanden sind
  • Recht: Darf Standorte erstellen/löschen übergeht die loakelen Rechte und gibt einen globalen Zugriff auf die Bearbeitung der Standorte
  • Wechsel ins Detail und Möglichkeit zur Bearbeitung eines bestehenden Standorts (siehe Schritt 4)

Löschen Icon

  • Warnhinweis erscheint
  • Standorte können nur dann gelöscht werden, wenn keine Abhängigkeiten zu Ressourcen mehr bestehen, resp. keiner der Sub-Standorte noch eine Abhängigkeit zu einem Ressourcen mehr aufweist
  • Standorte können nur gelöscht werden, wenn der Benutzer das „Standort Bearbeiten“ Recht hat
  • Falls einem Standort (z.B. Bern) diverse Substandorte (z.B. Laupenstrasse\ 1.Stock) zugewiesen sind, aber kein Standort mehr über Ressourcen verfügt, wird der Standort inkl. allen Substandorten gelöscht (sofern man auf dem ganzen Standortbaum die nötigen Berechtigungen hat)
  • Beim Löschen werden die Abhängigkeiten zu den Berechtigungen gelöscht
  • Alle anderen Abhängigkeiten (Sperrzeiten, Feiertage, etc.) werden gelöscht
  • Bestätigungsmeldung im Actionpanel

Schritt 4 - Nutzer kann (bei entsprechender Berechtigung) neue Standorte erfassen und vorhandene Standorte editieren

Controls

Tab „Stammdaten“

  • Feld: Name
  • Feld: Bezeichnung
  • Standortpicker: Übergeordneter Standort
  • Intelibox: Verantwortliche Person
  • Dropdown: Zeitzone
  • Datei-/Verzeichnis für Plan (aus 103.002 Pläne verwalten )
  • Zugewiesener Plan (aus 103.002 Pläne verwalten )

Tab „Plan“

  • nur sichtbar mit Recht: Darf Standorte mit Plänen verwalten.
  • siehe UC 103.002 Pläne verwalten

Tab "Berechtigungen"

  • siehe auch UC 109.005 Berechtigungen erteilen und entfernen)

Tab "Sperrzeiten"
siehe UC 105.005 Sperrzeiten verwalten

Tab "Feiertage"
siehe UC 105.006 Feiertage importieren und verwalten

Tab "Notifikationen
siehe UC 107.001 Notifikationen verwalten

Tab "Buchungsanfrage"
siehe UC 120.001 Anlasstyp verwalten

Button Bereich

  • Button: Speichern
  • Button: Abbrechen

Regeln

Name
Name oder Alias des Standorts – nicht übersetzbar

Bezeichnung
Kann mehrsprachig erfasst werden und wird in der Benutzersprache angezeigt

Übergeordneter Standort

  • Durch Änderung des übergeordneten Standorts kann der Standort verschoben werden.
  • Das Verhalten der Berechtigungen ist in 109.002 Rechte verwalten definiert

Verantwortliche Person
Pro Standort kann eine verantwortliche Person selektiert werden. Dieser Wert wird beim Speichern vererbt bis auf die Ressourcen. Dieses Feld ist an keine weiteren Funktionen in ROOMS gebunden.

Zeitzone
Der Standort muss einer Zeitzone zugeordnet werden. Dieser Wert wird beim Speichern vererbt bis auf die Ressourcen

Speichern
Bestätigungsmeldung im Actionpanel

Wechsel auf Standort Baum

Abbrechen
Wechselt zurück auf den Standortbaum
