Use Cases : 104.002 Projektgruppen verwalten

Created by Former user, last modified on Jun 19, 2011

Ziel & Beschreibung

  • Die vollständige Verwaltung von Projektgruppen

Datenrechte

–

Funktionsrechte

–

Bedingungen

  • Funktionsrechte müssen implementiert werden:
    • Darf (Menü) Projektgruppen lesen
  • Die Projektgruppen werden in jeweils nur zwei Hierarchiestufen aufgeteilt (Haupt-Projektgruppen und Sub-Projektgruppen) sowie die Initialstufe (welche auch als „Projektgruppen" bezeichnet wird)
  • Es werden keine Projektgruppen standardmässig von GARAIO zur Verfügung gestellt
  • Der Kunde kann selber kann ein Set von Projektgruppen erstellen und diese als Filter- und/oder Berechtigungs-Kriterien verwenden.

Hauptszenario in Schritten

  1. Nutzer navigiert unter EINSTELLUNGEN auf GRUPPIERUNGEN\PROJEKTGRUPPEN VERWALTEN
  2. Nutzer (mit entsprechendem Funktionsrecht) kann durch den Projektgruppen-Tree navigieren und die weiteren Aktionen festlegen
  3. Nutzer kann neue Haupt-Projektgruppen erfassen und diese editieren
  4. Nutzer kann neue Sub-Projektgruppen erfassen und diese editieren
  5. Liste mit allen der Projektgruppen zugeordneten Ressourcen wird angezeigt
  6. Ressourcen können von der Projektgruppen entfernt werden
  7. Ressourcen können zu der Projektgruppen hinzugefügt werden
  8. Liste mit allen der Projektgruppen zugeordneten Personen wird angezeigt
  9. Personen können von der Projektgruppen entfernt werden
  10. Personen können zu der Projektgruppen hinzugefügt werden
  11. Nutzer kann Projektgruppen löschen

Schritt 2 - Nutzer kann durch den Projektgruppen-Tree navigieren und die weiteren Aktionen festlegen

Controls

Inhaltsbereich

  • Projektgruppen-Tree-Control
    • Edit Icon
    • View Icon
    • Neuerfassen Icon
    • Delete Icon

Regeln

Je nachdem wo das man sich im Tree befindet, kann man auf den jeweiligen Nodes mehr oder weniger Aktionen durchführen (auch an der Anzahl Icons ersichtlich).

  • Auf der Initialstufe „Projektgruppen" steht nur der Button „Neuerfassen" zur Verfügung
    • Klick auf „Neuerfassen"-Button möglich (falls entsprechende Berechtigungen vorhanden)
      • Man gelangt ins Detail und kann einen neue Haupt-Projektgruppen erfassen (siehe Schritt 3)
  • Auf der Hierarchiestufe „Haupt-Projektgruppen" stehen die Buttons „Neuerfassen", „Anzeigen", „Editieren" und „Löschen" zur Verfügung.
    • Klick auf „Neuerfassen"-Button möglich (falls entsprechende Berechtigungen vorhanden)
      • Man gelangt ins Detail und kann einen neue Sub-Projektgruppen erfassen (siehe Schritt 4)
    • Klick auf „View" Button möglich
      • Details der Haupt-Projektgruppen werden nicht editierbar angezeigt
      • Es stehen die Buttons „Editieren", „Drucken" und „Zurück" zur Verfügung
    • Klick auf „Edit Button möglich (falls entsprechende Berechtigungen vorhanden)
      • Man gelangt ins Detail der Haupt-Projektgruppen und kann diese editieren (siehe Schritt 3)
    • Klick auf „Delete" Button möglich (falls entsprechende Berechtigungen vorhanden)
      • Löscht die entsprechende Haupt-Projektgruppen (Siehe Schritt 11)
  • Auf der Hierarchiestufe „Sub-Projektgruppen" stehen die Buttons „Anzeigen", „Editieren" und „Löschen" zur Verfügung.
    • Klick auf „View" Button möglich
      • Details der Haupt-Projektgruppen werden nicht editierbar angezeigt
      • Es stehen die Buttons „Editieren", „Drucken" und „Zurück" zur Verfügung
    • Klick auf „Edit Button möglich (falls entsprechende Berechtigungen vorhanden)
      • Man gelangt ins Detail der Sub-Projektgruppen und kann diese editieren (siehe Schritt 4)
    • Klick auf „Delete" Button möglich (falls entsprechende Berechtigungen vorhanden)
      • Löscht die entsprechende Sub-Projektgruppen (Siehe Schritt 11)

Schritt 3 - Nutzer kann (bei entsprechender Berechtigung) neue Haupt-Projektgruppen erfassen und diese editieren

Controls

Inhaltsbereich TAB/Register „Stammdaten"

EINGABEBEREICH
Bei Neuerfassung (über Neu) - analog Screenshot:

  • Label: Haupt-Projektgruppen neu erfassen
  • Textfeld: Haupt-Projektgruppen-ID
  • Textfeld: Bezeichnugn
  • Rich-Intelibox (Multiselektdropdown): Ressourcenart
  • Checkbox: sichtbar (ja/nein)
  • Button: Hinzufügen
  • Button: Abbrechen

Bei Editierung (über Edit) - kein Screenshot:

  • Label: Hauptgliederung editieren
  • Textfeld: Hauptgliederungs-ID
  • Textfeld: Bezeichnung
  • Rich-Intelibox (Multiselektdropdown): Ressourcenart
  • Checkbox: Feld sichtbar (ja/nein)
  • Button: Aktualisieren
  • Button: Abbrechen

Regeln

Rot = Muss-Felder

Bezeichnung wird in der Benutzersprache des angemeldeten Benutzers geladen (über T Icon) kann das Feld übersetzt werden.

  • Mittels Haupt-Projektgruppen -ID kann für jede Liste resp. jede Darstellung (inkl. dieser) definiert werden, an welcher Stelle resp. in welcher Reihenfolge diese erstellte Haupt-Projektgruppen im Verhältnis zu den anderen Haupt-Projektgruppen auftaucht (Vorschlag: muss nicht unbedingt eine Nummer sein, ich schlage drei Buchstaben vor und es wird dann nach abc sortiert)
  • Es können 1-n Ressourcenarten (Arbeitsplatz, Equipment, etc.) ausgewählt werden, womit definiert wird, bei welchen Ressourcenarten das Feld dargestellt werden und als Filterkriterium dienen soll
  • Eine zusätzliche Checkbox „sichtbar" definiert, ob das Feld dann effektiv als Filterkriterium-Feld in den Listen sichtbar dargestellt wird oder nicht sichtbar (nur für Berechtigungszwecke) ist (über alle ausgewählten Ressourcenarten)
  • Info: Das neu erstellte (als sichtbar gekennzeichnete) Feld (Hauptgliederung), taucht in folgenden Masken wie folgt auf (in diesem Beispiel gehen wir davon aus, dass nur die Ressourcenart „Sitzungszimmer" bei der Erstellung der Hauptgliederung „Raumtyp" definiert wurde):
    • Sidepanel: Generell sofern in einem Sidepanel die Ressourcenart „Sitzungszimmer" gewählt wurde, muss das Feld / Hauptgliederung (z.B. „Raumtyp") als Dropdown innerhalb des „Ressourcenspezifischen Suchfilters" dargestellt werden und wenn es dies nicht geben sollte, dann unterhalb des Ressourcenart-Feldes.
    • Sidepanel: Bei mehreren selbst erstellten Hauptgliederungen für die Ressourcenart „Sitzungszimmer", werden diese in der Reihenfolge gemäss Hauptgliederungs-ID unterhalb der Ressourcenart angezeigt.
    • Sidepanel: Die neu erstellten Felder / Hauptgliederungen müssen (sofern die bereits erwähnten Bedingungen erfüllt sind) unter den Menüpunkten „Reservieren", „Listen" und bei „Einstellungen" im Sidepanel als Filterkriterium dargestellt werden
    • Inhaltsbereich: Hauptgliederung wird im Inhaltsbereich nicht dargestellt.
    • Info: Zugewiesene Ressourcen können immer mehreren Hauptgliederungen angehören

Schritt 11 - Nutzer kann (bei entsprechender Berechtigung) Projektgruppen löschen

Color

Inhaltsbereich

  • Projektgruppen-Tree-Control
  • Delete Icon

Regeln

  • Klickt man auf das Delete-Item, dann erscheint zuerst eine Bestätigungsmeldung, welche man quittieren muss.
  • Sub-Projektgruppen können nur dann gelöscht werden, wenn keine Abhängigkeiten zu Ressourcen/Personen mehr bestehen
  • Haupt-Projektgruppen können nur gelöscht werden, wenn keine Sub-Projektgruppen bestehen, Sub-Projektgruppen ohne Ressourcen/Personen vorhanden sind oder Sub-Projektgruppen bestehen, welche aber noch für keine Reservation verwendet wurden.
  • Beim löschen einer Sub-Projektgruppen werden die verknüpften Berechtigungen gelöscht
