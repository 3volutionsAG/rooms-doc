Ziel & Beschreibung

Systemlog kann eingesehen werden

Rechte

  • Darf das Eventlog verwalten

Bedingungen

–

Hauptszenario in Schritten

  1. Nutzer navigiert auf Einstellungen >System >Ereignisanzeige
  2. Nutzer gibt Suchkriterien ein (optional) und klickt Button „Finden"
  3. Ein Suchresultat über Logeinträge wird angezeigt - ein oder mehrere Logeintragungen können gelöscht werden
  4. Nutzer kann einen Eintrag anklicken, um die Details anzuzeigen (View Modus)

Use Case Details

Schritt 1 - Nutzer navigiert auf Einstellungen >System>Ereignisanzeige

Controls

Sidepanel

Navigation öffnet sich entsprechend

Regeln

Rechte müssen implementiert werden:

  • Darf das Eventlog verwalten
 

Schritt 2 - Nutzer gibt Suchkriterien ein (optional) und klickt Button „Suchen“

Controls

Sidepanel

  • Feld: Quelle
  • Multipicker: Ereignistyp
  • Datumpicker: Zeitstempel
  • Feld: Applikationskontext
  • Feld: Meldung
  • Feld: Benutzername
  • Feld: Servername
  • Button „Finden“
  • Button „Zurücksetzen

Regeln

Quelle

  • sucht über einen bestimmten Bereich - z.B. nur den ROOMS Service

Ereignistyp

  • Verfügbare Werte
    • Critical => Fehler auf übergeordneter Ebene, zB konnten die einzelnen Prozesse des WindowsServices gar nicht angestossen werden.
    • Error => Fehler
    • Information => Wie der Name sagt, zB Export gestartet oder Export erfolgreich.
    • Warning => Fehler ohne Auswirkungen oder „verdächtiges“ Ereignis, zB konnte bei der BackSync ein Exchange-Appointment nicht gefunden werden (könnte sein, dass es noch gar nicht auf dem Server ist, muss also kein Fehler sein) oder eine Person hat mehrere zugeteilte OEs (ist zwar ein Fehler, wir können die Ausführung aber fortsetzen).
    • Verbose => Hier kommen zT auch Dev-Informationen wie zB „DataContext initialisiert“, die fürs Debugging hilfreich sein können.
  • Jeder Typ schliesst die übergeordneten immer ein, dh bei Verbose kommt ALLES.
  • Mehrfachselektion erlaubt

Zeitstempel

  • Standardwert: nicht selektiert => alle werden angezeigt
  • Standardwert bei Selektion: der aktuelle Tage

Applikationskontext

  • sucht in einer bestimmten Code Basis

Meldung

  • schränkt sie Suche auf einen bestimmten Fehlertext ein

Benutzername

  • sucht in einer bestimmten Benutzerdomäne

Servername

  • sucht über einen bestimmten Server

Bilder

 

Schritt 3 - Ein Suchresultat über alle erfassten Logeinträge wird angezeigt – ein oder mehrere Logeintragungen können gelöscht werden

Controls

Smartlist "Themenliste"
Standard Spalten:

  • Checkbox (für Mehrfachselektion)
  • Zeitstempel (mit View Link) - siehe Schritt 4
  • Ereignistyp
  • Meldung
  • Quelle
  • Applikationskontext

Optionale Spalten:

  • Benutzername
  • Servername
  • Prozess ID
  • Thread ID
  • Stack Trace
  • Exception
  • Id

Button Bereich:

  • Button: Entfernen

Regeln

Entfernen

  1. Mehrfachselektionen möglich
  2. Klick auf Löschen
  3. Warnhinweis
    1. Bestätigungsmeldung im Actionpanel (inkl. Anzahl der gelöschten Einträge) ODER
    2. Fehlermeldung im Actionpanel

Bilder

 

Schritt 4 - Nutzer kann einen Eintrag anklicken, um die Details anzuzeigen (View Modus)

Controls

Inhaltsbereich – Tab „Logging“

  • Quelle
  • Ereignistyp
  • Zeitstempel
  • Applikationskontext
  • Meldung
  • Benutzername
  • Servername
  • Prozess ID
  • Thread ID
  • Textarea: Stack Trace (gesamter Fehlerverlauf - wo ist er entstanden)
  • Textarea: Exeption (fehlerhafte Sequenz)
  • Textarea: Daten (bewusste Datenausgabe, Zusatzinformationen)
  • Button: Zurück
  • Button: Drucken

Regeln

Der View Modus wird in der Sprache des angemeldeten Benutzers angezeigt

Button Zurück

  • wechselt zurück zum Ursprungsort

Button Drucken

  • druckt die aktuelle Seite (siehe 000.001 Globale Funktionen)
