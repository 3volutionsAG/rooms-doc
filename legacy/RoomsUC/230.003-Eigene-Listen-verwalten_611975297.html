Ziel & Beschreibung

Eigene Listen können Dritten zur Verfügung gestellt werden
Liste von Dritten können abonniert werden

Rechte

-

Bedingungen

Ein Personenprofil wurde im Edit Mode geöffnet
Die entsprechenden Berechtigungen wurden dabei berücksichtigt
Eigene Listen wurden erstellt (siehe UC 230.001 Liste erstellen und (als Favorit) speichern) und/oder
Globale Listen wurden erstellt (siehe UC 230.002 globale und öffentliche Listen verwalten)

Hauptszenario in Schritten

  1. Nutzer navigiert auf Tab "Eigene Listen"
  2. System zeigt alle eigenen oder abonnierten Listen - eine oder mehrere Listen können gelöscht werden - ein  oder mehre Publikationen können gestoppt werden
  3. Liste kann hinzugefügt oder abonniert werden
  4. Persönliche Liste kann publiziert werden

Schritt 2 - System zeigt alle eigenen oder abonnierten Listen - eine oder mehrere Listen können gelöscht werden - ein  oder mehre Publikationen können gestoppt werden

Controls

Smarlist

  • Bezeichnung
  • Listenart
  • Besitzer
  • Abos

Button Bereich

  • Button „Hinzufügen oder Abonnieren“
  • Button „Publizieren"
  • Button "Publikation stoppen"
  • Button "Entfernen"

Regeln

Bezeichnung

  • Bezeichnung der Liste - wird durch den Ersteller festgelegt (in UC 217.002 globale und öffentliche Listen verwalten (Draft) und 220.001 Reservationen in Liste anzeigen)
  • Nicht übersetztbar

Listenart

  • Es wird unterschieden zwischen folgenden Arten:
    • privat => Liste ist nur für den privaten Gebrauch - Dritte können diese nicht einsehen
    • public => Liste wurde durch angemeldeten zur Publikation publiziert (Dritte können Liste abonnieren oder sich eine Kopie erstellen lassen - siehe Schritt 4)
    • shared => Liste wurde durch einen Dritten publiziert und der angemeldete Benutzer hat sich diese abonniert (siehe Schritt 3)

Besitzer

  • Eigentümer der Liste
  • Vorname und Nachname werden dargestellt

Abos

  • Ganzzahl
  • zeigt wieviele Personen eine Shared oder Public Liste im Abonmenet haben

Hinzufügen oder Abonnieren

  • Modales Popup öffnet sich
  • siehe Schritt 3

Publizieren

  • siehe Schritt 4

Publikation stoppen

  • Mehrfachselektion möglich
  • Warnhinweis: Alle bestehenden Abonnente werden gelöscht! Soll die Publikation wirklich gestoppt werden?
  • Aktualisierung der Smartlist
    • Listenart wird zu "privat"
  • Bestättigungsmeldung im Actionpanel, inkl. der Anzahl der gestoppten Publikationen
  • Alle Abonemente zu dieser Liste werden entfernt - Liste steht den Abonementen ab sofort nicht mehr zur Verfügung

Entfernen

  • Mehrfachselektion möglich
  • Sicherheitshinweis: Soll die Liste wirklich entfernt werden
  • Aktualisierung der Smartlist
  • Bestätigungsmeldung im Actionpanel, inkl. der Anzahl der gelöschten Listen
  • Liste steht auch nicht mehr im Bereich der Listenansicht (Dropdown "Gespeicherte Listen") zur Verfügung - siehe UC 220.001 Reservationen in Liste anzeigen
  • Alle Abonemente zu dieser Liste werden entfernt - Liste steht den Abonementen ab sofort nicht mehr zur Verfügung

Schritt 3 - Liste kann hinzugefügt oder abonniert werden

Controls

Modales Popup

  • Bezeichnung
  • Listenart
  • Besitzer
  • Abos

Button Bereich

  • Option: Liste abonnieren (Listen können genutzt, aber nicht verändert werden)
  • Option: Liste kopieren (Listen werden als neue private Listen dupliziert)
  • Button „Hinzufügen“
  • Button „Abbrechen"

Regeln

Bezeichnung

  • Bezeichnung der Liste - wird durch den Ersteller festgelegt (in UC 217.002 globale und öffentliche Listen verwalten (Draft) und 220.001 Reservationen in Liste anzeigen)
  • Nicht übersetztbar

Listenart

  • Es erden nur Listen der Art "public" angezeigt => Liste wurde durch eine andere Person publiziert und somit zur Abonnierung freigegeben

Besitzer

  • Eigentümer der Liste
  • Vorname und Nachname werden dargestellt

Abos

  • Ganzzahl
  • zeigt wieviele Personen die Liste bereits im Abonmenet haben

Liste abonnieren

  • publizierte Liste wird abonniert und erscheint ab sofort im Bereich der Listen als "Shared" Liste (sihe UC 220.001 Reservationen in Liste anzeigen)
  • Liste kann nicht verändert werden

Liste kopieren

  • eine Kopie der publizierten Liste wird erstellt
  • Liste gehört nun dem angemeldeten Benutzer und kann beliebig verändert werden
  • Liste erscheint als privat im Bereich der Listen (siehe UC 220.001 Reservationen in Liste anzeigen)

Hinzufügen

  • Mehrfachselektion möglich
  • Hinzufügen gemäss Option "Liste abonnieren" oder "Liste kopieren"
  • Modales Popup schliesst sich
  • Aktualisierung der Smartlist
  • Bestättigungsmeldung im Actionpanel, inkl. der Anzahl der hinzufgefügten Einträge

Abbrechen

  • Modales Popup schliesst sich

Bilder

Schritt 4 - Persönliche Liste kann publiziert werden

Controls

Button Bereich der Smartlist

  • Button „Publizieren"

Regeln

Publizieren

  • Mehrfachselektion möglich
  • Es werden nur private Listen berücksichtigt
  • Klick auf Button "Publizieren"
  • Smartlist wird aktualisiert:
    • Listenart wechselt auf Public
  • Bestättigungsmeldung im Actionpanel, inkl. der Angabe der aktualsierten (publizierten) Einträge
