Ziel & Beschreibung

Ein oder mehrere Buchungen können automatisiert aktualisiert werden

Rechte

  • DarfMassenmutationenVornehmen

Bedingungen

  • Es bestehen Reservationen 
  • Benutzer befindet sich im Plan, Kalender oder unter Listen

Hauptszenario in Schritten

  1. Nutzer selektiert zwei oder mehr Buchungen
  2. Nutzer klickt Button „Aktualisieren"
  3. Massenaktualisierung Popup wird angezeigt

Beziehungen zu UC


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Use Case Details

Schritt 1 - Benutzer selektiert zwei oder mehr Buchungen

Controls

Inhaltsbereich

  • Listenansicht: Benutzer hat bei zwei oder mehreren Buchungen die Checkbox in der Smartlist selektiert
  • Planansicht: Benutzer selektiert mittels Mauslkick mehrere Hotspots. Wird ein Hotspot selektiert, bleibt es eingeblendet; ausserdem wird die Transparenz der Polygon-Füllung entfernt, d.h. der Polygon wird hervorgehoben.
  • Kalender: Benutzer hat bei zwei oder mehreren Reservataionen die Checkbox selektiert. Box erscheint beim Mouse-Over, sofern folgendes Recht vorhanden ist: DarfMassenmutationenVornehmen

Regeln

Rechte müssen implementiert werden:

  • DarfMassenmutationenVornehmen

Button wird im Buttonbereich des Inhaltsbereich angezeigt (Höhe "auto-refresh" icon)

Schritt 2 - Nutzer klickt Button „Aktualisieren“

Controls

Popup

  • Button "Aktualisieren"

Regeln

Aktualisieren

  • erscheint nur mit dem Recht "DarfMassenmutationenVornehmen"
  • öffnet Popup "Massenaktualisierung"

Bilder

 

Schritt 3 - Massenaktualisierung Popup wird angezeigt

Controls

Sidepanel

    • Text: Wollen Sie die selektierten (Anzahl) Reservationen wirklich aktualisieren
    • Button: Aktualisiere Selektion
    • Button: Aktualisiere gesamte Liste
    • Button: Abbrechen

Regeln

Wollen Sie die selektierten (Anzahl) Reservationen wirklich aktualisieren

  • Text, falls keine gültigen Buchungen gefunden: Sie können zur Zeit keine der selektierte(n) Reservation(en) aktualisieren.
  • Text, falls gültige Buchungen vorhanden: Wollen Sie die selektierten (Anzahl) Reservationen wirklich aktualisieren?
  • (Anzahl) = Platzhalter für die effektive Anzahl der selektierten Buchungen

Aktualisiere Selektion + Aktualisiere gesamte Liste

  • Button: "Aktualisiere Selektion" nur sichtbar wenn auch gültige Reservationen vorhanden (z.B. nicht in Vergangenheit, genügend Rechte etc.)
  • Button: "Aktualisiere gesamte Liste" nur sichtbar in der Listenansicht
  • Ausgabe im Actionpanel: "Die Änderungen werden nun übernommen. Sie werden  wieder informiert, sobald der Vorgang abgeschlossen ist."
  • Aktionseinträge für Bearbeitung werden in Action Tabelle geschrieben
  • Asynchrone Abarbeitung (durch den Service) des Auftrags. Dabei werden

    • Zeiten aktualisiert (Beispiel: Vorlaufzeiten neu berechnet, Nachlaufzeiten neu berechnet, etc.)

    • Preise aktualisiert (Beispiel: Bestellungen neu berechnet, Umstuhlungskosten neu berechnet, Tarife der Ressourcen neu berechnet, etc.)
  • Ausgabe im Actionpanel: "Die Änderung von X  Einträgen vom Typ Reservation sind abgeschlossen.
  • Zusätzlich Ausgabe im Actionpanel im Fehlerfall: (y wurden  angepasst; z konnten nicht angepasst werden)."
  • Zusätzlich Ausgabe im Actionpanel im Fehlerfall (zweite Meldung): Massen-Bearbeitung: für y Einträge vom Typ Reservation konnte die Änderung nicht validiert werden (Ids 4530,4531,4532,4533)
  • Nicht aktualisierte Buchungen werden im Log Bereich dokumentiert

Abbrechen

  • Abbrechen entfernt die Locks
  • Zurück auf die vorherige Ansicht
