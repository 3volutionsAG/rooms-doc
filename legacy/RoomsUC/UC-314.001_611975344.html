Name

Unterstützung von pessimistischem Locking

Ziel

Datensätze müssen zum Editieren ausgecheckt werden

Abhängigkeiten zu anderen UC

–



Grobbeschrieb

Zum Vorbeugen von Concurrency-Problemen soll die Applikation vor dem Editieren von Daten einen Locking-Mechanismus anbieten, der den betroffenen Datensatz für den entsprechenden Benutzer sperrt. Dazu wird eine Tabelle (Locking) erstellt, welche die entsprechenden Informationen enthält. Während einer konfigurierbaren Zeitspanne oder bis zum Unlock kann der Datensatz von keinem anderen Benutzer editiert werden. Die Funktion soll als Teil des OR-Mappers zur Verfügung stehen und kann auf alle Tabellen angewendet werden. Dieser Mechanismus behandelt nur bestehende Datensätze.
