Ziel & Beschreibung

Benutzer kann aus dem Lotus Notes eine Ressource buchen

Rechte

  • Darf Reservationen für Ressourcetyp Arbeitsplatz buchen
  • Darf Reservationen für Ressourcetyp Equipment buchen   
  • Darf Reservationen für Ressourcetyp Fahrzeug buchen
  • Darf Reservationen für Ressourcetyp Parkplatz buchen
  • Darf Reservationen für Ressourcetyp Raum buchen
  • Darf Reservationen mit Vor-/Nachlaufzeit lesen (20) 
  • Darf Reservationen mit Vor-/Nachlaufzeit editieren (21)

Bedingungen

Die Übersetzungen müssen lokal gespeichert werden. Dabei werden beliebige Sprachen unterstützt werden. Initial wird das Plugin zweisprachig (DE/EN) ausgeliefert

Hauptszenario in Schritten

  1. Benutzer erstellt Kalendereintrag (Appointement oder Meeting Request)
  2. Outlook Plugin wird geladen (inkl. Sprache)
  3. Benutzer erfasst (optional) Kalenderdaten
  4. Benutzer klickt ROOMS Sidepanel - verfügbare Ressourcenarten werden angezeigt
  5. System liefert Suchresultat gemäss Standardwerte
  6. Benutzer erfasst Suchkriterien (optional) und klickt Finden
  7. System liefert Suchresultat über freie Ressourcen gemäss den Filterkriterien
  8. Spezialfall Serie
  9. Benutzer klickt auf "Abschliessen"
  10. Benutzer klickt „Save & Close" oder „Senden"
  11. System aktualisiert über Exchange die ROOMS Buchung
  12. Benutzer editiert Kalendereintrag
  13. Benutzer löscht Kalendereintrag

Gespeicherte Listen

  • Vorgabe aus Anzahl der erfassten Teilnehmer plus Organisator

Maximale Kapazität berücksichtigen

  • Berücksichtigt alle Bestuhlungsvarianten

Dienstleistungen

  1. Es kann über den Angebotspicker (+ Zeichen) oder über Intelibox nach einem Angebot gesucht werden (Intelibox Funktion ist in unter 000.001 Globale Funktionen / 106.002 Angebotszonen verwalten ) gemäss Angebotszone angezeigt.
  2. Der Angebotspicker öffnet auf Klick des Plus Zeichen (+) ein Popup, welches die Angebote gemäss deren Angebotskategorie auflistet (siehe UC 106.001 Dienstleister verwalten)
  3. Jedes Angebot hat ein Zahlen-Eingabefeld  vorangestellt. 
    1. der Standardwert des Feldes ist 0
    2. Wird eine Zahl grösser 0 eingegeben, wird automatisch das Angebot der Liste (Bestellungen) zugefügt inkl. der definierten Menge
    3. Die Menge kann jederzeit geändert werden - die Änderungen werden umgehend in die Liste übertragen => Bedingung: der Picker bleibt die ganze Zeit geöffnet
    4. Wird der Picker einmal geschlossen, kann die Menge nicht mehr aktualisiert werden. Das Angebot hat wieder den Standardwert 0 und fügt bei einer Änderung >0 ein zweites Angebot der Liste hinzu
  4. Es werden alle Angebote von allen Dienstleistern aufgelistet, welche in Ihrer Angebotszone die aktuelle Ressource zugeordnet haben (bzw. mit dem Standort verknüpft sind)
  5. Es werden nur Angebote angezeigt, welche einer gültigen Angebotszone angehören (Gültig = kein von/bis Datum erfasst oder der aktuelle Zeitpunkt der Bestellung liegt innerhalb der erfassten Gültigkeitsdauer) - siehe UC 106.002 Angebotszonen verwalten
  6. Es werden nur Angebote angezeigt, bei welchen keine Gültigkeitsdauer definiert wurde oder der aktuelle Zeitpunkt der Bestellung liegt innerhalb der erfassten Gültigkeitsdauer
  7. Es werden nur Angebote angezeigt die Gültig sind (obige Regeln gelten), Berechtigungen vergeben sind, in der Angebotszeit des Dienstleisters liegen und den Vorlaufregeln entsprechen (siehe unten - Vorlaufregeln).
    1. Ausnahme: mit dem Recht: "Darf Bestellungen zeitlich unabhänig editieren" werden alle Angebote angezeigt, welche den Regeln 3-5 entsprechen, aber nicht mehr zwingend gültig sein müssen, aufgrund Punkt 6 (Angebotszeiten/Vorlaufzeiten des Dienstleisters)
  8. Zu Beginn wird nur die Top-Ebene der Angebotskategorien dargestellt. Der Benutzer kann mittels klick auf das + durch den Angebotsbaum navigieren
  9. Auf jeder Hierarchiestufe wird in Klammern angezeigt, wieviele Angebote darunter vorhanden sind (z.B. „Abendkarte (28/40)“ → „Vorspeisen (9/9)“ → „Salate (3/3)“)
    • Die erste Zahl zeigt, die Menge der Angebote, welche effektiv zur Verfügung stehen unter Berücksichtigung der Vorlaufzeiten und der Arbeitszeit des Angebotszone (resp. Dienstleisters) - siehe dazu das "Regelwerk für Berechnung der Vorlaufzeiten eines Angebots" weiter unten
    • Die zweite Zahl zeigt, die Anzahl der Angebote, welche maximal zur Verfügung stehen würden)
    • Angebote werden auch nur dargestellt, wenn diese den Kriterien von Vorlaufzeiten und Arbeitszeiten entsprechen angezeigt - siehe dazu das "Regelwerk für Berechnung der Vorlaufzeiten eines Angebots" weiter unten
    • Angebote oder Pakete werden identifiziert in dem kein weiteres + mehr vorhanden ist (unterste Hierarchie des Baumes)
    • Die Optionen eines Angebotpakets werden nicht angezeigt. Der Benutzer muss immer das Paket selektieren. Pakete unterscheiden sich visuell nicht von Einzelangeboten.
  10. Es ist auch möglich mittels IntelliSense direkt ein Angebot zu suchen.
    • durch Eingabe der Artikelnummer oder
    • durch Eingabe der Artikelbezeichnung (in der Sprache des angemeldeten Benutzers)
  11. Nach Auswahl des gewünschten Angebots wird automatisch validiert ob es sich um eine offene Menge handelt oder nicht.
    • Bei NICHT offener Menge wird ein Eingabefeld für die Erfassung der Bestellmenge eingeblendet
    •  im Fall einer "offenen Menge" wird das Feld nicht eingeblendet.
  12. Die mehrmalige Erfassung des gleichen Angebots ist möglich und erzeugt mehrere Einträge (kein Zusammenzug der Menge). Ausnahme: bei der Erfassung über den PIcker kann die Anzahl beliebig geändert werden, solange der Picker nicht geschlossen wird

Menge

  • Feld wird nur eingeblendet bei NICHT "offener Menge" und gleichzeitiger Eingabe über Intelibox Modus. Feld wird nicht eingeblendet bei Navigation im Angebotspicker
  • Standardwert entspricht dem Feld "Anzahl Teilnehmer" aus dem Tab "Allgemein" (siehe UC 201.001 Raum reservieren)
  • Es muss ein Wert (Ganzzahl > 0) eingegeben werden, ausser bei offener Menge - da ist keine Eingabe möglich
  • Wurden bereits bei einer vorgängigen Angebotserfassung ein Wert eingegeben, bleibt dieser bestehen.

Notiz an Dienstleister

  • Klick auf @-Icon öffnet Popup für die Erfassung einer Notiz an den Dienstleister
  • Eingabe eines Textes (max. 255 Zeichen) möglich
  • Bei Reservationsabschluss wird die Notiz an den Dienstleister geschickt (sofern entsprechendes E-Mail Template konfiguriert wurde - siehe UC Notifikationsmatrix,Notifikationsplatzhalter und 107.001 Notifikationen verwalten)

Hinzufügen

  • Durch Klick auf „Bestellen“ wird die Bestellung in den Warenkorb hinzugefügt.

Warenkorb

  • Der Warenkorb zeigt alle Bestellungen
  • Folgende Spalten stehen zur Verfügung
    • Checkbox für Selektion zur Angebots Entfernung
    • Angebotsbezeichnung

 

Regelwerk für Berechnung der Vorlaufzeiten eines Angebots

  • Diese Berechnung bestimmt ob ein Angebot im Picker erscheint oder nicht.
  • Die Berechnung bestimmt den Wert der ersten Klammerzahl (bsp: 14/24), welcher die Anzahl der buchbaren Angebote zeigt
  • Das Angebot ist nur verfügbar, wenn dieses innerhalb der Arbeitszeiten des Dienstleisters liegt (Arbeitszeiten werden auf der Angebotszone definiert - siehe UC 106.002 Angebotszonen verwalten)
  • Die Vorlaufdauer darf ausserhalb der Reservation liegen.
  • Die Nachlaufdauer darf ausserhalb der Reservation liegen
  • Das Angebot inkl. Vorlaufdauer und Nachlaufdauer, darf nicht ausserhalb der Arbeitszeiten des Dienstleisters liegen
  • Vorlaufdauer kann nicht gesplittet werden über mehrere Tage.Beispiele: Arbeitzeit Dienstleister: täglich von 08:00 bis 18:00Vorlauf des Angebot "Sandwich": 60 MinutenAktueller Zeitpunkt: Dienstag, 17:30Reservationen:
  1. Reservation Dienstag 18:00: Angebot "Sandwich" kann NICHT bestellt werden - Grund: Vorlaufzeit nicht eingehalten
  2. Reservation Mittwoch 08:30: Angebot "Sandwich" kann NICHT bestellt werden - Grund: Mit Vorlaufzeit ausserhab der Arbeitszeiten des Dienstleister (Vorlaufzeit kann nicht gespittet werden, also zur Hälfte auf Dienstag und zur Hälfte auf Mittwoch)
  3. Reservation Mittwoch 09:00: Angebot "Sandwich" kann bestellt werden - Grund: Liegt inkl. Vorlaufzeiten innerhalb der Arbeitszeit des Dienstleisters

 

Gliederungen

  • Nur Ressourcenart Raum
  • Nur wenn Flag „sichtbar“ anzeigen
  • Gemäss interner Reihenfolge wird die erste Gliederung angezeigt
  • die restlichen Gliederungen können über den Link "mehr" aufgerufen werden

Buchungsklassifikationstypen (siehe UC 104.002 Buchungsklassifikationen verwalten)

  • werden in der Listenansicht angezeigt, sofern
    • der angemeldete Benutzer in einer der Buchungsklassifikationstypen zugeordneten Benutzergruppe zugeteilt ist
    • die Ressourcenart gewählt wurde, welche der Buchungsklassifikationstyp zugeordnet wurde - bei keiner Auswahl werden alle Buchungsklassifikationstypen
    • die Checkbox "sichtbar in Listen" auf dem Buchungsklassifikationstypen selektiert worden ist
  • Die Buchungsklassifikationstypen werden in Form von Dropdown oder Multiselektoren (sofern Checkbox "Mehrfachselektionen erlaubt" angeklickt worden ist)
  • Bei Mehrfachselektionen werden die angeklickten Buchungsklassifikationen bei der Suche mit einer AND Verknüpfung berücksichtigt.

Genehmigung durch

  • Global kann ein Schwellwert für die Genehmigung von Catering definiert werden (siehe Parameter: "Schwellwerte für Kosten auf Bestellungen" aus 000.004 Konfigurationsparameter)
  • Wird dieser Schwellwert überschritten, muss ein Genehmiger erfasst werden, ansonsten kann die Reservation nicht abgeschlossen werden
    • Ausgabe Fehlermeldung im Actionpanel und beim Control
  • Bei Abschluss der Reservation/Anlasses wird eine Mittelung an den "Genehmiger" verschickt.
  • Folgende Templates werden versendet (siehe auch UC 107.001 Notifikationen verwalten und 000.006 Notifikationsmatrix)
    • 26: Reservation - Freigabe Bestellungen für Reservationen
    • 28: Anlass - Freigabe Bestellungen für Events

Ressourcen Name/Bezeichnung

  • Suche nach über die ID der Ressourcen und über die Bezeichnung in der Benutzersprache des angemeldten Benutzers

Suche ohne Substandorte

  • Begrenzt die Suche auf den  aktuell gewählten Standort oder die aktuelle gewählten Standorte. Die jeweiligen Substandorte werden nicht berücksichtigt.

meine favorisierten Ressourcen

  • zeigt nur Ressourcen auf welchen die angemeldete Person jemals als Owner mindestens eine Buchung hatte (angemeldete Person = Organisator)
  • Berücksichtigt werden alle Ressourcenarten
  • Berückscihtigt werden Vergangenheit, Gegenwart und Zukunft

 

Neue Vorlaufzeit

  • Wird nur eingeblendet mit Recht:  "Darf Reservationen mit Vor-/Nachlaufzeit editieren" (21)"
  • Standardwert = 0
  • Eingabe erfolgt in Minuten
  • Die eingegebene Zeiten übersteuern die errechnete Zeiten und werden in die Suche übernommen
  • 0 oder Minuswerte werden nicht berücksichtigt
  • Eingabe von Buchstaben führen zu einem Validierungsfehler

Neue Nachlaufzeit

  • Wird nur eingeblendet mit Recht:  "Darf Reservationen mit Vor-/Nachlaufzeit editieren" (21)"
  • Standardwert = 0
  • Eingabe erfolgt in Minuten0 oder Minuswerte werden nicht berücksichtigt
  • Die eingegebene Zeiten übersteuern die errechnete Zeiten und werden in die Suche übernommen
  • 0 oder Minuswerte werden nicht berücksichtigt
  • Eingabe von Buchstaben führen zu einem Validierungsfehler

Schritt 8 - Spezialfall Serie

Controls

Button "Prüfen"

Modales Popup "Verfügbarkeit Übersicht" mit folgenden Spalten

  • Prüfstatus
  • Ressourcen Bezeichnung
  • Datum/Zeit von
  • Datum/Zeit bis

Modales Popup "Umbuchen"

  • Datumpicker
  • Ressourcenpicker

Regeln

Prüfen

  1. Die Verfügbarkeit der ganzen Serie wird geprüft
  2. Hat der Benutzer eine zeitliche Serien Begrenzung erfolgt die Prüfung bis zur zeitlichen Begrenzung. Es wird eine Warnung ausgegeben
  3. Pro Eintrag, welcher erfolgreich geprüft wurde, wird ein Blocker gesetzt
  4. Sind alle Einträge der Serie OK, erscheint neben dem Prüfen Button ein grünes OK Icon => die Buchung kann abgeschlossen werden (s. Schritt 9)
  5. Ist min. ein Eintrag nicht OK (bereits verbucht), erscheint eine Fehlermeldung, welche pro Datum die fehlerhaften Einträge auflistet.
  6. Automatisch wird der modale Übersichtsdialog angezeigt mit allen Einträgen
  7. alle Einträge können bearbeitet werden (Zeit oder Ressourcenänderung)
  8. die roten Einträge (Fehler) müssen bearbeitet werden, Orange (Warnings) und grüne (OK) Einträge können optional geändert werden.
  9. Klick auf Edit Icon öffnet den modalen Dialog zur Umbuchung der Ressource
    1. Änderung von Datum/Zeit über Datumpicker ODER
    2. Änderung der Ressource über Ressourcenpicker (es werden nur die noch verfügbaren Ressourcen angezeigt)
  10. Klick auf OK übernimmt die Änderung.
  11. Sobald alle "roten" Einträge geändert wurden, kann die Buchung abgeschlossen werden (siehe Schritt 9)

Schritt 10 - Benutzer klickt „Save & Close“ oder „Senden“

Controls

Plugin geschlossen

  • Outlook Standard Controls

Regeln

 

  1. Benutzer klickt Save & Close oder Senden
  2. Prüfung ob Titel eingegeben wurde, ansonsten Abruch
  3. Prüfung ob Private Checkbox selektiert wurde. Falls ja,

    1. Prüfung, ob der Benutzer überhaupt in ROOMS private Buchungen machen darf. 
    2. Wenn nicht soll eine Meldung an den Benutzer ausgegeben werden ("Sie sind nicht berechtigt private Buchungen zu erstellen" / "You are not allowed to make private bookings"). 
    3. Der Speicherprozess muss unterbrochen werden.
  4. Outlook Appointement wird geschlossen
  5. Location von Outlook wird zusätzlich mit Hierarchiepfad des Standorts versehen: Ressourcen.Bezeichnung - Standort.Hierarchie.Bezeichnung

Schritt 11 – System aktualisiert über Exchange die ROOMS Buchung

Controls

Kein Inhalt

Regeln

Über die (ehem.) EventSink Komponente (Exchange) wird die Systemreservation in eine Definitive Reservation gewandelt.

Dabei werden folgende Aktualisierungen und Aktionen vorgenommen:

  1. System Reservation in Definitive Reservation
  2. System Reservation von mitreservierten Ressourcen (Equipment) in definitive Reservation
  3. Weitere Aktualisierungen:
    1. Teilnehmer
    2. Private Flag
    3. Anzahl Personen
    4. Titel
    5. Catering
    6. Klassifikationen
    7. Kostenträger
    8. Serieninformationen
  4. Meeting Request wird an Teilnehmer verschickt
    1. Verlinkung der Outlook Eintrag mit ROOMS Buchung
  5. Bestätigungsmail von ROOMS wird gemäss Notifikationseinstellungen der Benutzer, Dienstleister, etc. verschickt.

Fehlerfall
Sync. Error Mail an Benutzer

    • Bearbeiten: öffnet die Webapplikation und zeigt die aktuelle Raumbuchung
    • Annullieren: storniert die aktuelle Raumbuchung - es kann eine neue Raumbuchung zum aktuellen Kalendereintrag gebucht werden - siehe UC 201.019 Reservation stornieren (annullieren)
    • Löschen: löscht die aktuelle Raumbuchung (nur verfügbar mit entsprechenden Rechten) - siehe UC 201.010 Buchungen löschen

Outlook Controls

  • Änderungen mit Relevanz auf die Synchronisation: werden Datum/Zeit, Titel, Privateflag oder Teilnehmer geändert wird alles nach Speichern über Outlook/Exchange => EventSink synchronisiert
  • Änderungen ohne Relevanz auf die Synchronisation: werden Location oder andere Werte geändert wird nichts Synchronisiert.
  • Serienbuchung: diese kann nicht geändert werden. System informiert, dass in diesem Fall alle Reservation annulliert und neu erstellt werden müssen.

Bilder

 

Mittels Drag & Drop + Maus-Ziehen

Regeln

 

Chached Mode

  • die Änderungen werden in Outlook vorgenommen
  • Exchange gibt die Informationen an ROOMS weiter
  • kann die Buchung zeitlich nicht verschoben werden (Konflikt) wird die Buchung über Exchange wieder zurück gesetzt
  • der Benutzer erhält ein Synchronisationsfehler Mail
  • im Erfolgsfall werden die Änderungsnotifikationen und Meeting Request Updates verschickt

Online Mode

  • die Änderungen werden in Outlook vorgenommen und direkt an den ROOMS Webservice weitergegeben
  • kann die Buchung zeitlich nicht verschoben werden (Konflikt) wird die Buchung direkt durch das Outlook Plugin zurückgesetzt
  • im Erfolgsfall werden die Änderungsnotifikationen und Meeting Request Updates verschickt
