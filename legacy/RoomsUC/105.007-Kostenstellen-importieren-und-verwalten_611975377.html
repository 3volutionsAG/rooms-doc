Use Cases : 105.007 Kostenstellen importieren und verwalten

Ziel & Beschreibung

Um während des Reservationsprozesses vordefinierte (alternative) Kostenstellen zur Auswahl zur Verfügung zu stellen, solle es möglich sein eine Liste von Kostenstellen in das System zu laden. Dazu muss eine Kostenstellen Datei erstellt werden (manuell oder durch Drittsystem).

Datenrechte

–

Funktionsrechte

  • Darf Alternative Kostenstelle( n ) selektieren
  • Darf Kostenstellensplitt anpassen

Bedingungen

  1. Ist die Tabelle “Kostenstellen” leer (NICHT abgefüllt), ist eine freie Ein-gabe der Kostenstellen möglich (+ Button wird ausgeblendet und Feld hat Schreibrechte)
  2. Die selektierte Kostenstelle wird immer auf die Reservation niederge-schrieben (keine Referenz)

Hauptszenario in Schritten

  1. Service schreibt Daten aus Kostenstellen Datei in Tabelle
  2. Reservationsprozess: Benutzer kann eine Kostenstelle einer Reservation zufügen
  3. Reservationsprozess: Benutzer kann eine zugewiesene Kostenstelle entfernen
  4. Reservationsprozess: Benutzer kann Kostensplitt der Kostenstellen verändern

Beziehungen zu UC

Regeln

Eine CSV/TXT Datei wird über einen generischen Service in die Tabelle „Kostenstelle“ importiert. Dies geschieht periodisch. Die Periodizität kann in der Konfigurationsdatei des Service festgelegt werden (analog des heutigen Service für Benut-zerdaten Import).

Import Datei

  • es können txt oder csv Dateien importiert werden
  • Alle Einträge werden überschrieben (Ein-deutiger Schlüssel ist das Feld Kosten-stellennummer). Nicht mehr gelieferte Einträge werden gelöscht.
  • Verfügbarkeit der Kostenstelle (im Reser-vationsprozess) wird über Datum von/bis gesteuert
  • In Release 4.0 besteht keine Möglichkeit Kostenstellen anderweitig zu administrie-ren.

Schritt 2 - Reservationsprozess: Benutzer kann eine Kostenstelle der Reservation zufügen

Controls

  • Intelibox „Alternative Kostenstelle“
  • Button „Hinzufügen“

Regeln

Funktionsrecht „Darf alternative Kostenstelle se-lektieren“ muss gewährt sein

Alternative Kostenstelle

  • Feld kann leer bleiben
  • eine Kostenstellen kann durch (Teil-)-Eingabe eines Textes gefunden werden. Dabei werden folgende Felder durch-sucht:
    • Geschäftsbereich
    • Kostenstellennummer
    • Kostenstellenbezeichnung
  • Es werden nur Kostenstellen angezeigt, welche zur vollständigen Laufdauer der Reservation auch Gültigkeit haben

Hinzufügen

  • Durch Klick auf den Button kann eine Kostenstelle hinzugefügt werden
  • Zugefügte Kostenstellen werden unterei-nander aufgelistet mit Kostenstellennum-mer, Kostenstellenbezeichnung und Ge-schäftsbereich

Speicherung erfolgt bei Reservationsabschluss

Schritt 3 - Reservationsprozess: Benutzer kann eine Kostenstelle entfernen

Controls

  • Icon „Papierkorb“ (Entfernen) – hinter jeder zugefügten Kostenstelle

Regeln

Funktionsrecht „Darf alternative Kostenstelle se-lektieren“ muss gewährt sein

Papierkorb

  • Klick auf Papierkorbsymbol entfernt alter-native Kostenstelle von der Reservation
  • Keine Sicherheitsabfrage
  • Keine Bestätigung im Actionpanel

Schritt 4 - Reservationsprozess: Benutzer kann Kostensplitt der Kostenstellen verändern

Regeln

Grundsätzlich werden die Kosten bei Abrechnung (anderer UC) auf Grund der Anzahl der gewählten alternativen Kostenstellen prozentual aufgesplittet.

Diese Splittung kann von berechtigten Usern ge-ändert werden

Die Splittung kann über den Kosten Button ( + ) und dem Tab „Kostensplitt“ erfolgen.
* Das System errechnet initial den Splitt (in nachfolgendem Beispiel wären dies pro Feld 33.33 Prozent)
* Prozentualen können geändert werden.
* Wenn Summe < > 100 ist kann der Button „Übernehmen“ nicht geklickt werden (die Errechnung erfolgt auf zwei Kommastel-len genau – heisst, dass die Summe (auf zwei Kommastellen genau über 99.5 und unter 100 ergeben muss)
* Als zusätzliche Kostenstelle steht auch die Kostenstelle das Organisators zur Verfügung (hat aber Standardmässig den Wert 0)
