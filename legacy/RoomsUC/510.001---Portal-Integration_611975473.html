Ziel & Beschreibung

ROOMS soll über eine Portal Integration verfügen, um den Kalender in Intranet Portals, wie Sharepoint, zur Verfügung zu stellen. Dabei soll über Kalender direkt im Internet gebucht werden können.

 

Rechte

 

Bedingungen

Die Hauptprozesse sind dem untenstehenden Ablauf zu entnehmen. Die erforderliche Funktionalität kann den Mockups und dazugehörigen Beschrieben entnommen werden.

Folgende Abbildung zeigt die Möglichen Integrationsvarianten für eine Interne ROOMS Instanz mit Portalintegration

  • ROOMS Portalintegration auf Webserver in der Extranet- (DMZ) Zone
    • Verwendung für externen Zugriff von Internet-Usern
  • ROOMS Portalintegration auf Webserver in der Intranet-Zone
    • Verwendung für den Intranet-Zugriff in einer Portalumgebung
    • Verwendung für den externen Zugriff von Internet Usern mit entsprechendem Proxy, NET, o.ä. in der Extranet-Zone


Prozess Ablauf (Request Formular und Liste (blauer Teil) wurde nicht umgesetzt)

 

Hauptszenario in Schritten

  1. Benutzer navigiert auf Portal
  2. Kalender wird angezeigt
  3. Benutzer muss sich Anmelden
  4. Benutzer kann sich registrieren
  5. Benutzer macht eine Schnellbuchung (Klick in Kalender)

Use Case Details

Schritt 1 - Benutzer navigiert auf Portal

Controls

keine

Regeln

Integration mit Windows Authentication

  • Benutzer wird automatisch authentisiert

 

Der Kalender wird mittels Webpart, App, iFrame oder ähnlicher Technologie in ein Intranet integriert. Dabei muss sichergestellt werden, dass die Windows Credentials des Host (z.B. Sharepoint) an ROOMS übergeben werden. Danach wird der persönliche Kalender des angemeldeten Benutzers (Default View) geladen.

 

 

 

Schritt 2 - Kalender wird angezeigt

Controls

 

Kalender beschrieben unter 213.001 Reservationen im Kalender suchen und buchen

 

Regeln

Authentisiert

  1. Navigationsbar gemäss Webapplikation (Tabs der Kalender nicht notwendig)
  2. Ressourcen werden gemäss Berechtigungen und Filtereinstellungen geladen. Tooltipp sichtbar. Konfiguration wie im Web
  3. Persönliche Filter werden in Dropdown unten angezeigt. Dies erlaubt den Wechsel zwischen verschiedene Kalender/Filtereinstellungen
  4. Absprung Button auf Web Applikation. wird in neuem Browser geöffnet. Startseite gemäss Einstellungen Benutzer
  5. Absprung auf Profil in Webapplikation. ROOMS wird in einem neuen Browser geöffnet und es wird direkt das Profil des Benutzers angezeigt.


 

Nicht authentisiert

  1. Navigationsbar gemäss Webapplikation (Tabs der Kalender nicht notwendig)
  2. Ressourcen werden gemäss Berechtigungen und Filtereinstellungen des Vorlagenbenutzers  geladen. Tooltipp sichtbar. Konfiguration wie im Web
  3. Filter des Vorlagen Benutzers werden in Dropdown angezeigt. Dies erlaubt den Wechsel zwischen verschiedene Kalender/Filtereinstellungen 
  4. Button: Anmelden: siehe Schritt 3



 


 

 

Es sind nur Schnellbuchungen im Kalender möglich (keine Detailbuchung und Massenbuchungen; für die Detailbuchung ist ein Absprung nach ROOMS vorzusehen. Nach Abschluss der Detailbuchung (Reservation bestätigen), wird das Fenster geschlossen)

Regeln

 

Benutzername

  • Benutzername
  • Standard = leer

Passwort

  • Passwort Eingabe
  • Standard = leer

 

Passwort

  • Passwort Eingabe
  • Standard = leer
  • Zeichen werden anonymisiert

 

Firma, Anrede, Vorname, Nachname, E-Mail, Telefon, Adresse 1, Adresse 2, PLZ, Ort, Land

  • Eingabe freier Werte
  • Standard leer

 

Registrieren

  • Prüft alle MUSS Felder
  • Prüft Übereinstimmung der Passwörter
  • Schliesst Registrationsformular
  • Speichert den neuen Benutzer und Firma
  • Nach Registration müssen alle Settings des Vorlagenbenutzers kopiert werden, inkl. den Favoriten Einstellungen (Persönliche Listen) des Kalender und der Ressourcen
  • Zurück auf Kalender 
  • Kalender wird im Kontext des angemeldeten Benutzers neu geladen.

Abbrechen

  • Zurück auf Kalender 
  • Benutzer ist nicht am System angemeldet (Anonymous View)
