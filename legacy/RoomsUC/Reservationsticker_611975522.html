Use Cases : Reservationsticker

Created by Former user, last modified on Jan 09, 2012

Beschreibung

Ein Reservationsticker wird als zusätzlicher Container zur Verfügung gestellt, der je nach Layout irgendwo im Header platziert werden kann. Der Reservationsticker erlaubt einige wichtige Verwaltungsfunktionen für die laufenden Reservationen des aktuellen Benutzers. Der Reservationsticker kann pro Mandant global ausgeschaltet werden, indem keine Platzierung im Header erfolgt; wird er platziert, ist er für alle Benutzer auf allen Ansichten verfügbar.

  • Weist der aktuelle Benutzer keine laufende Reservation auf, entspricht die Darstellung derjenigen eines nicht selektierten Sub-Panels und enthält einen Standard-Text. Es ist keine Interaktion möglich.
  • Der Reservationsticker enthält beim Laden einer Seite die aktuellen Reservationsdaten. Zusätzlich führt der Ticker automatisch einen regelmässigen Refresh zusammen mit dem Action-Panel durch.
  • Der Reservationsticker zeigt nur definitiv, gebuchte Reservationen

Funktionen

Weist der aktuelle Benutzer laufende Reservationen auf, entspricht die Darstellung derjenigen eines selektierten Sub-Panels. Die Reservationen werden nach folgenden Kriterien in der angegebenen Reihenfolge sortiert:

  • Ressourcentyp: Deskshare / Raum / Parkplatz / Fahrzeug / Equipment / Person
  • Reservationsdauer: kürzer / länger
  • Reservationsbeginn: früher / später
  • Reservations-ID: kleiner / grösser

Der Ressourcenname der ersten Reservation gemäss dieser Sortierung wird direkt im Reservationsticker angezeigt; hinter dem Namen werden einige Icons für Aktionen mit dieser Reservation aufgelistet. Mit einem Klick auf den Ticker irgendwo neben den Icons wird ein Popup mit den anderen Ressourcennamen inkl. Icons eingeblendet. (Es wird bewusst auf Mouse-Over Funktionalität verzichtet, da sich diese bisher immer negativ auf die Usability ausgewirkt hat.) Ein Klick auf eines der Icons führt die entsprechende Aktion aus und schliesst das Popup; jeder Klick ausserhalb des Popups schliesst das Popup ebenfalls.

Folgende Icons werden angezeigt:

  • View: Absprung in den View-Modus der Reservation.
  • Edit (wenn es sich um keine zugeordnete Ressource handelt): Absprung in den Edit-Modus der Reservation.
  • Freigaben bearbeiten (wenn es sich um eine zugeordnete Ressource handelt): Öffnen des Bearbeitungsfensters für die Freigaben (siehe vorangehendes Kapitel).
  • Checkout (wenn das Funktionsrecht vorhanden ist): Sofortiges Beenden der Reservation (exkl. Nachlaufzeit).
  • Freigabe (wenn das Funktionsrecht vorhanden ist): Sofortiges Beenden der Reservation (inkl. Nachlaufzeit).

Formatierung

Die Breite des Reservationstickers richtet sich nach dem breitesten angezeigten Text (Ressourcennamen) und ist daher flexibel.
