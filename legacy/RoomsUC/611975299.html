Use Cases : Smartlist (Grid)

Created by Former user, last modified by Former user on May 22, 2012

Jede Smartlist beinhaltet immer das Personalisieren von Spalten, Sortierung und Paging sowie optionale Checkboxen für Mehrfachselektion.

  1. Sortierung:
    1. Spalten mit Icons sind nicht sortierbar und haben ein schwarzen Titel
    2. Es ist nur über eine Spalte eine Sortierung möglich
    3. Klick auf eine Spalte (link oder Pfeil) sortiert Absteigend (A-Z), klick das zweite Mal auf dieselbe Spalte sortiert Aufsteigend (Z-A)
    4. Die aktuelle Sortierung wird mit einem Zeichen (Pfeil) visualisiert (siehe Layout)
  2. Grösse: Die Liste hat eine fixe minimal Grösse und verfügt über eine Scrollbar – nur Vertikal
  3. Paging: Das Paging (Zeilen Pro Seite) kann durch den Benutzer pro Ansicht eingestellt werden.
    1. Klick auf Button
    2. Auswahlliste öffnet sich nach oben (genaue Grafik – inkl. selektierter wird noch geliefert)
    3. Die Auswahlmöglichkeiten werden global (pro Kunde konfiguriert)
    4. Der selektierte Punkt muss ersichtlich sein (z.B. durch Ausblenden des Links oder Icon)
  4. Anzahl Spalten und Reihenfolge
    1. Pro Listenansicht wird die Anzahl der Spalten durch das Produkt vorgegeben
    2. Der Benutzer kann jede Ansicht gemäss seinen Vorstellungen verändern (Spaltenreihenfolge und Spaltenanzahl)
    3. Klick auf Button
    4. Fenster mit allen verfügbaren Spalten öffnet sich nach oben (definitives Layout wird noch geliefert – inkl. Visualisierung der Reihenfoge)
    5. Reihenfolge: Die Spaltenreihenfolge kann mittels Drag und Drop innerhalb des Popups vorgenommen werden.
    6. Nach Schliessen wird das Smartlist aktualisiert
    7. Spaltenbreite kann durch „Maus-ziehen" im Spaltenkopf der Smartlist geändert werden.
  5. Autorefresh: Stellt dem Benutzer die möglichen Intervalls für die Aktualisierung des Smartlist zur Verfügung:
    1. Falls der Refresh eingeschalten ist muss das Smartlist immer in der aktuellen Ansicht (Page) aktualisiert werden
    2. Benutzer kann „Aktualisieren" über Button aktivieren
    3. Eine Liste öffnet sich nach oben (Layout entspricht noch nicht der definitiven Version. Der gewählte Eintrag muss selektiert sein)
    4. Ein Intervall kann gewählt werden oder der Refresh Modus ausgeschalten werden.
    5. Der selektierte Punkt muss ersichtlich sein (z.B. durch Ausblenden des Links oder Icon)
  6. Navigationsbutton: Die Navigationsbuttons sind wie folgt aufgebaut:
    1. Button mit Pfeil für eine Page vor und eine zurück
    2. Anzeigen der ersten Page mit 1
    3. Anzeigen der letzten Page in Zahlenform (z.B. 13)
    4. Anzeigen der maximal 5 mittleren Pages. Beispiel: Nutzer ist auf Page 10 von 13, also werden ihm noch die Seiten 8, 9 und 11, 12 nebst der 10 visualisiert. Details unter Gesamt Layout
    5. Navigationsbuttons (und Layout Buttons) bleiben fix positioniert (unten).

Suchfelder

Grundsätzlich verfügen die Smartlists über Suchfelder pro Spalte (oberhalb des Spaltentitels). Es kann sein, dass in bestimmten Inhaltskontexten die Suchfelder ausgeblendet werdenUm Platzoptimierter arbeiten zu können, können die Suchfelder der Smartlist ausgeblendet werden, sofern alle Resultate auf einer Seite dargestellt werden können. Dies benötigt eine globale Einstellung: Suchfelder in Smartlist automatisch ausblenden = True

Inhalt laden und keine Inhalte gefunden

1. Seriereservation: Sobald Button "Erstellen geklickt" wechselt die Beschriftung auf "Verfügbarkeit wird geprüft"

2. Bei Tabs mit Smartlist (z.B. Ressourcen im Reservationsdetail oder Benutzergruppen im Personendetail) erscheint während der Ladezeit das Label: Liste wird geladen.

3. Falls dann wirklich keine Einträge vorhanden sind, erscheint sinnesgemäss: keine Einträge gefunden

- Ist eine SmartList "ungeladen" (d.h. ist die SmartList noch nie abgefüllt worden), hat sie KEIN Label.

- Beim asynchronen Laden kann ein zusätzliches Label mitgegeben werden. Wird kein Label mitgegeben, wird als Default "Liste wird geladen..." ausgegeben (bei Serienbuchungen anpassen, s. oben).

- Ist eine SmartList "geladen" (d.h. es hat ein asynchrones Laden stattgefunden oder die SmartList ist beim Generieren der Seite abgefüllt worden), hat aber keine Einträge, wird "Es wurden keine Einträge gefunden." ausgegeben.
