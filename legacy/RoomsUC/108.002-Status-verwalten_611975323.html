Ziel & Beschreibung

Es gibt drei Arten von Status: Zustand einer Ressource, Systemstatus und frei definierbare Status auf Reservationen:

  1. Benutzer soll Status, inkl. Farbe und Symbol definieren können.
  2. Benutzer soll erfasste Status (UC 105.001 Ressourcen verwalten ) pro Ressource aktivieren können und dabei auch die Reihenfolge definieren.

Rechte

  • Darf Status verwalten

Bedingungen

Folgende Zustände einer Ressource gibt es:

  1. Frei 
  2. Gesperrt 
  3. Besetzt (Besetzt wird als Reservationsstatus behandelt)

Folgende Haupt Reservation-Status gibt es (System, bzw. Workflow Status)

  1. Besetzt (System)
  2. Besetzt Provisorisch
  3. Besetzt Temporär
  4. Besetzt Definitiv
  5. Annulliert
  6. Slot definiert
  7. Slot publiziert


Ein Systemstatus ist vom System vorgegeben und kann NIEMALS gelöscht werden. 
 Neue Haupt Status können nur durch GARAIO implementiert werden


 

Beispiele von bekannte individuelle Status, welche an einem System Status angehängt werden können

 

  1. Kunde im Raum
  2. Kundenberater avisiert
  3. Kundenberater im Raum
  4. Reinigung

Die Individuellen Status übersteuern (falls nötig) visuell immer die Systemstaus

Hauptszenario in Schritten
  1. Navigation auf Einstellungen >Basisdaten für Ressourcen >Status
  2. Statusstruktur wird geladen - ein Status kann gelöscht werden
  3. Benutzer kann in View Modus des Status wechseln
  4. Benutzer kann Status editieren/hinzufügen

Schritt 2 – Status Struktur wird geladen – ein Status kann gelöscht werden

Controls

Inhaltsbereich

Baum mit allen verfügbaren Status

  • Bezeichnung
  • Icon „Hinzufügen“ (nicht bei Annullieren und System)
  • Icon "View" (Ansehen)
  • Icon „Bearbeiten“

Bei den Substatus zusätzlich verfübar:

  • Icon "Löschen"

Regeln

Hauptstatus
Folgende Haupt Reservation-Status sind verfügbar (System/Workflow Status)

  1. Besetzt (System)
  2. Besetzt Provisorisch
  3. Besetzt Temporär
  4. Besetzt Definitiv
  5. Annulliert
  6. Slot definiert => Prozessschritt nur in Zusammenhang mit Slots möglich
  7. Slot publiziert => Prozessschrit nur in Zusammenhang mit Slots möglich

Bezeichnung
Wird in der Sprache des Benutzers ausgegeben

"Ansehen (View"

  • Wechselt in den View Modus des Status (siehe Schritt 3)

„Hinzufügen (Neu)“

  • Button nur sichtbar auf Hauptstatus 2, 3 und 4

„Bearbeiten“

  • Wechselt in den Bearbeitungsmodus des Status (siehe Schritt 4)

„Löschen"

  • Nur auf Substatus möglich
  • Löscht den aktuellen Status und sämtliche Konfigurationen (Verbindung zu Ressourcentabelle)
  • Bedingungen:
    • Hat ein Status eine Abhängigkeit zu einer Ressource, werden die entsprechenden Felder auf der Ressource auf Null gesetzt (StatusId, AlarmStatuswechsel)
    • Hat ein Status eine Abhängigkeit zu einem Workflow-Ereignis, wird das Ereignis gelöscht.
    • Hat ein Status eine Abhängigkeit zu einer Reservation kann der Status nicht gelöscht werden.
  • Sicherheitsabfrage notwendig
  • Bestätigungsmeldung im Actionpanel

Schritt 4 - Benutzer kann Status editieren/hinzufügen

Controls

Detail:

  • Feld „Bezeichnung“
  • Label "Übergeordneter Status"
  • Bildupload Funktion: "Symbol"
  • Icon "Zugewiesenes Symbol"
  • Farbpicker "Farbe Status"
  • Farbpicker "Farbe Vorlauf/Nachlauf"
  • Identifikationsnummer (aus UC UBS3_UC_319.001_SpsSteuerung.docx )
  • Button „Speichern“
  • Button „Abbrechen“

Regeln

Bezeichnung
Mehrsprachige Erfassung möglich

Symbol

  • Über den „Browse…“ Button kann ein neues Symbol selektiert werden
  • Über den „Upload“ Button wird das Symbol auf den Server geladen - das bestehende Symbol wird überschrieben.

Zugewiesenes Symbol:

  • Aktuell zugeordnetes Symbol wird angezeigt.

Farbpicker

  • Wahl einer Statusfarbe
  • Wahl der Vor-/Nachlauf Farbe

Identifikationsnummer (aus UC UBS3_UC_319.001_SpsSteuerung.docx )
Ist für die SPS Steuerung notwendig (identifiziert ein Terminal)
