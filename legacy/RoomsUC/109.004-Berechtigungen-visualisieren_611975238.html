Use Cases : 109.004 Berechtigungen visualisieren

Created by Former user, last modified by Former user (Deleted) on May 31, 2012

Use Case Überblick


ID

109.004

Owner

﻿Former user (Deleted)

Rev-Number

3.0

Priorität

3

Ziel & Beschreibung

  1. Alle berechtige Ressourcen eines Benutzers oder Gruppe einsehen
  2. Auf Grund eines Standorts oder einer Ressource alle berechtigte Gruppen und Rechte einsehen
  3. Alle globalen Rechte eines Benutzers einsehen

Rechte

  • Darf Report Berechtigungen sehen
  • Darf Sektion Reports nutzen

Bedingungen

  1. Jeder Benutzer kann seine Berechtigungen auf einem Standort oder Ressource einsehen
  2. Alle anderen beschriebenen Funktionen sind nur über das Funktionsrecht „Darf Berechtigungen sehen“ verfügbar

Hauptszenario in Schritten

  1. Benutzer navigiert auf >Reports >Standortbezogene Rechte
  2. Nutzer erfasst Filterkriterien (optional) und klickt auf "Finden"
  3. Suchresultat über alle standortbezogene Rechte wird angezeigt
  4. Benutzer navigiert auf >Reports >Globale Rechte
  5. Nutzer erfasst Filterkriterien (optional) und klickt auf "Finden"
  6. Suchresultat über alle globalen Rechte wird angezeigt

Beziehungen zu UC

109.002 Rechte verwalten
109.003 Benutzergruppen verwalten
109.005 Berechtigungen erteilen und entfernen

Schritt 1 - Benutzer navigiert auf >Reports >Standortbezogene Rechte

Controls

Regeln

  • Recht "Darf Sektion Reports nutzen" muss vorhanden sein. Sonst wird Menü Reports nicht angezeigt.

Schritt 2 - Nutzer erfasst Filterkriterien (optional) und klickt auf "Finden"

Controls im Sidepanel

Generell (wird nicht als Subpanel visualisiert)

  • Dropdown: Standortbezogenes Recht
  • Standortpicker: Standort
  • Ressourcenpicker: Ressource
  • Intelibox: Benutzergruppe
  • Intelibox: Rolle
  • Intelibox: Person

Erweiterte Kritierien

  • Multipicker: Ressourcenart
  • Dropdown: Ressourcenstatus
  • Dropdown: Equipmentyp
  • Dropdown: Raumtyp
  • Button „Finden“
  • Button „Zurücksetzen“

Regeln

Generell:

Ohne das globales Recht „Darf Report Berechtigungen sehen" sind folgende Controls NICHT editierbar:

  • Benutzergruppe
  • Rolle
  • Person

Bei der Suchabfrage werden diese Felder ignoriert, ausser bei Person. Dieses Feld wird implizit mit dem angemeldeten Benutzer vorabgefüllt

Benutzergruppe

  • es sind alle erfassten Benutzergruppen verfügbar

Rolle

  • es sind alle erfassten Rollen verfügbar

Person

  • es sind alle aktiven Personen verfügbar

Standort

  • es werden alle Standorte angezeigt (unabhängig von Rechten)

Ressourcenpicker

  • Es werden Einträge angezeigt, unabhängig von den Rechten.

Ressourcenart

  • es werden alle Ressourcenarten für welche der angemeldete Benutzer Rechte hat aufgelistet

Ressourcenstatus

  • Folgende Werte sind verfügbar
    • alle
    • aktiv
    • inaktiv
  • Standardwert: alle

Equipmenttyp

  • Folgende Werte sind verfügbar
    • alle
    • mobil
    • fix
  • Standardwert: alle

Raumtyp

  • Folgende Werte sind verfügbar
    • alle
    • alle ohne virtuelle Räume
    • virtuelle Räume
    • virtuelle Räume (ohne Subräume)
  • Standardwert: alle

Finden

  • siehe Schritt 3

Schritt 3 - Suchresultat über alle standortbezogene Rechte wird angezeigt

Controls

Smartlistim Inhaltsbereich

Standard Spalten

  • Standort
  • Ressource
  • Gruppe
  • Rolle
  • Standortbezogenes Recht
  • Geerbt
  • Vererbt

Optionale Spalten

  • Standortbezeichnung
  • Standort Name
  • Ressourcen Name
  • Ressourcen Status
  • Ressourcenart

Regeln

Standort

  • Hierarchischer Standortbaum
  • Felder: Übergeordneter Standort + Bezeichnung, Tab: Stammdaten aus UC 103.001 Standorte verwalten
  • Inhalt wird in der Sprache des angemeldeten Benutzers angezeigt

Ressourcen

  • Wird in der Sprache des angemeldeten Benutzers ausgegeben

Gruppe

  • Benutzergruppe wird in der Sprache des angemeldeten Benutzers ausgegeben

Rolle

  • Rolle wird in der Sprache des angemeldeten Benutzers ausgegeben Standortbezogenes Recht

Geerbt

  • Wurde das Recht von einem überliegenden Standort geerbt

Vererbt

  • Wurde das Recht an einen unterliegenden Standort weiter vererbt

Standort Bezeichnung

  • Bezeichnung des Standorts
  • Feld: Bezeichnung, Tab: Stammdaten aus UC 103.001 Standorte verwalten
  • Inhalt wird in der Sprache des angemeldeten Benutzers angezeigt

Standort Name

  • Alias des Standorts
  • Feld: Name, Tab: Stammdaten aus UC 103.001 Standorte verwalten

Ressourcen Name

  • eindeutige Bezeichnung der Ressource
  • nicht übersetzbar
  • Feld: Name, aus Tab "Allgemein" des UC 105.001 Ressourcen verwalten

Ressourcen Status

  • Ist die Ressource aktiv/inaktiv
  • Feld: Aktiv, in Tab "Allgemein" des UC 105.001 Ressourcen verwalten

Ressourcenart

  • Art der der Buchung zu Grundeliegenden Ressource:
    • Raum
    • Arbeitsplatz
    • Fahrzeug
    • Equipment
    • Parkplatz
