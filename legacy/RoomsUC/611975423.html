Use Cases : 220.004 Catering & Services in App anzeigen

Ziel & Beschreibung

Catering & Services eines Tages werden pro Angebotszone in einer App angezeigt.

Die Auslieferung kann bestätigt werden.

Rechte

-

Bedingungen

App ist optimiert für Tablets mit Android, iOS, Windows Mobile

Limitation: aktuell steht nur eine Sprache zur Verfügung

Hauptszenario in Schritten

  1. App wird das erste Mal gestartet
  2. App wird konfiguriert durch Klick auf Einstellungsbutton
  3. Liste mit allen Dienstleistungen des aktuellen Tages wird angezeigt

Beziehungen zu UC

Use Case Details

Schritt 1 - App wird das erste Mal gestartet

Controls


Warining wird angezeigt mit dem Hinweis dass der Benutzer den Einstellung Button klicken soll


Regeln

Grundsätzlich

  • Benutzer muss zuerst korrekte Einstellungen erfassen, um arbeiten zu können (siehe Schritt 2)

 


Schritt 2 - Nutzer klickt Einstellung Button und konfiguriert App

Controls

Einstellungen

Der Einstellungsknopf zeigt die folgenden Einstellungen

  • Feld: URL zu den ROOMS web services (String)
  • Feld: Angebotszone (ID)
  • Feld: Änderungen als kurzfristig markieren (kleiner als)
  • Feld: PIN code für Benutzer Authentisierung
  • Feld: Aktualisierungsintervall

 

  • Button: OK
  • Button: Cancel

 

Regeln

Grundsätzlich

  • alle Einstellungen werden permanent gespeichert

Änderung als kurzfristig markieren

  • Kalkuliert vom aktuellen Zeitpunkt in die Vergangenheit ob eine Änderung als kurzfristig betrachtet werden soll
  • Angabe in Stunden
  • Standardwert 1
  • Prüfung auf Ganzzahlen

Pin Code für Benutzer Authentisierung

  • nur Ganzzahlen zulässig

Aktualisierungsintervall

  • nur Ganzzahlen zulässig
  • Angabe Minuten

OK

  • Einstellungen werden gespeichert
  • Request auf Server erfolgt
  • Ist alles richtig konfiguriert erfolgt der Wechsel auf Schritt 3

 

 

 

Schritt 3 - Liste mit allen Dienstleistungen des aktuellen Tages wird angezeigt

Controls

 

Spalten

Es werden die folgenden Spalten angezeigt (Spaltentitel vor >)

  • Zeit der Bestellung > Service Startzeit und Service Endzeit (Endzeit wird in Klammer dargestellt)
  • Meeting Startzeit > Startzeitpunkt der Buchung (ohne Vorlauf) - diese kann zum "Zeitpunkt der Bestellung" differenzieren
  • Raum & Organisator > Wo wird die Bestellung hin geliefert UND Organisator der Buchung (wird auf einer zweiten Zeile angezeigt)
  • Liste der Bestellungen > Menge + Beschreibung (deutsch) 
  • Anzahl Personen > Anzahl Personen auf Buchung. Diese kann differenzieren zur bestellten Menge
  • Aktion: > Button Erledigt als Icon (vgl. Abbildung)

 

Fussbereich

Es werden 5 Filter in Form von Tabs angezeigt

  • Alle => zeigt alle (ausser die ausgelieferten)
  • New => zeigt nur die neuen
  • Storniert => zeigt die kurzfristig stornierten
  • Geändert => zeigt kurzfristige Änderungen
  • Erledigt => zeigt alle ausgelieferten

 

Regeln

Generell

Folgende Regeln gelten für die angezeigte Dienstleistungen:

  • nur für Bestellungen der in den Einstellungen gewählten Angebotszone
  • Bestellungen von Anlässen und Buchungen
  • nur Bestellungen des aktuellen Tages
  • nur Bestellungen von Buchungen die am laufen sind oder in Zukunft liegen (Bis Datum der Buchung > Jetzt)
  • die Bestellungen werden nach Lieferzeit und Lieferort gruppiert
  • die Bestellungen werden nach Lieferzeit (absteigend) sortiert
  • die Sortierung kann geändert werden - klick auf Spaltentitel

 

Highlighting Regeln

  • Kurzfristige Stornierungen werden mit einem roten vertikalen Balken rechts der Zeile dargestellt (vgl. Abbildung).
    • Kurzfristig = Stornierung erfolgte innerhalb der letzten x Minuten (x = Wert aus Einstellungen (Änderungen als kurzfristig markieren) )
  • Kurzfristige neue Bestellungen und Änderungen werden mit einem orangen vertikalen Balken rechts der Zeile dargestellt (vgl. Abbildung)
    • Kurzfristig = Neue Bestellungen oder Änderungen erfolgten innerhalb der letzten x Minuten (x = Wert aus Einstellungen (Änderungen als kurzfristig markieren) )

 

Zeit der Bestellung 

  • Service Startzeit und Service Endzeit 
  • Endzeit wird in Klammer dargestellt
  • Format: HH:MM

Meeting Startzeit

  • Startzeitpunkt der Buchung (ohne Vorlauf)  oder Event
  • dieser Wert kann zum "Zeitpunkt der Bestellung" differenzieren
  • Format: HH:MM

Raum & Organisator

  • Zeile 1: Wo wird die Bestellung hin geliefert
  • Zeile 2: Organisator der Buchung/Event

Liste der Bestellungen

  • Bezeichnung des Angebots in deutsch oder Titel der Bestellung (falls vorhanden)
  • Menge wird nur angezeigt, wenn nicht offene Menge. Ansonsten nur die Bezeichnung anzeigen
  • für jedes bestellte Angebot wird eine neue Zeile geschrieben

Anzahl Personen

  • Anzahl Personen auf Buchung
  • Diese kann differenzieren zur bestellten Menge
  • Bei Event wird kein Wert ausgegeben

Aktion

  • Die Auslieferung des Services kann bestätigt werden
  • Der Service wird nun nur noch im Tab "erledigt" angezeigt
  • Der Auslieferzeitpunkt wird auf die Bestellung geschrieben, damit diese auch in ROOMS sichtbar is
