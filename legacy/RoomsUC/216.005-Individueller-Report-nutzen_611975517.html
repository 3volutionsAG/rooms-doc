Use Cases : 216.005 Individueller Report nutzen

Created by Former user, last modified by Former user (Deleted) on Sep 19, 2012

Use Case Überblick


ID

216.004

Owner

Tim Bänziger

Rev-Number

1.0

Priorität

3

Ziel & Beschreibung

Die individuellen Reports (Word Dokumente) werden in ROOMS verwaltet und bei Reservationsabschluss, Eventabschluss oder aus einer Listenansicht mit Daten abgefüllt.
Der Report steht dann zum Download bereit oder wird gleich mit der Reservationsbestätigung verschickt.

Rechte

  •   "Darf individuelle Dokumente erstellen"

Bedingungen

-

Hauptszenario in Schritten

  • Schritt 1: Dokumente selektieren
    • Variante: Nutzung der Dokumente bei Abschluss einer Reservation
    • Variante: Nutzung der Dokumente im View Mode einer Reservation und eines Events
    • Variante: Nutzung der Dokumente unter Reports
  • Schritt 2: Actionpanel bestätigt die Erstellung
  • Schritt 3: Actionpanel liefert Dokumente zum Download
  • Schritt 4: E-Mails werden verschickt

Use Case Details

Schritt 1: Dokument selektieren

Variante - Nutzung der Dokumente bei Abschluss einer Reservation

Controls

Button Bereich im Reservationsdetail:

  • Button: Dokumente
  • Modales Popup
    • Titel: Verfügbare Dokumente
    • Text: Die Dokumente werden erstellt, sobald die Reservation abgenschlossen ist
    • Reports aphabetisch aufgelistet
    • Checkbox: Download
    • Checkbox: E-Mail
    • Button: Bestätigen

Regeln

Generell

  • Button wird nur angezeigt bei Recht "Darf individuelle Dokumente erstellen".

Klick auf „Dokumente“

  1. alle Dokumente werden angezeigt, welche in diesen Bereich publiziert wurden
  2. Die Dokumente werden alphabetisch aufgelistet
  3. Die Dokument Sprache wird in Klammer gemäss Kultur-Kürzel ausgegeben – z.B. (CH-DE für Deutsch)
  4. Selektion der Checkbox "Download" generiert das Dokument und stellt es zum Download bereit
  5. Selektion der Checkbox "E-Mail" schickt das Dokument mit der E-Mail Bestätigung mit
  6. Die Erstellung erfolgt als Word oder PDF - siehe Feld "Export Format" in UC 216.004 Individueller Report verwalten)
  7. Standardwert wird bei Erfassung des Dokuments definiert (selektiert, nicht selektiert, pflicht-selektiert) - siehe Feld "Selektion für Download" und "Selektion für E-Mail" in UC216.004 Individueller Report verwalten)
  8. Mehrfachselektion erlaubt
  9. *Klick auf „Bestätigen“
    1. Reservation muss zuerst abgeschlossen werden – dabei werden die Dokumente generiert
    2. siehe Schritt 2

Schritt 3 - Nutzung der Dokumente im Bereich "Reports"

Controls

Inhaltsbereich - Sidepanel

  • Dokument
  • Datum/Zeit
  • Organisator
  • Standort Picker
  • Ressourcen Picker
  • Anlass Titel
  • Anlass Nummer
  • Reservationstitel

Button Bereich

  • Button „Entfernen“
  • Button „Zurücksetzen“

Regeln

Generell

  • Bereich wird nur angezeigt bei Recht "Darf individuelle Dokumente erstellen".

Wahl des Dokuments

  1. Alle Dokumente werden angezeigt, welche in diesen Bereich publiziert wurden
  2. Die Dokumente werden alphabetisch aufgelistet
  3. Die Dokument Sprache wird in Klammer ausgegeben – z.B. (CH-DE)

Eingabe der Filterkriterien

  • Datum/Zeit
    • Standard = aktueller Tag von 00:00 bis 23:59
    • Veränderbar durch Klick der Checkbox und Eingabe der Werte
  • Organisator (Intelibox)
  • Standort (Intelibox mit Picker)
  • Ressource (Picker)
  • Reservationstitel

Klick auf „Erstellen“

  • Dokument wird auf Grund der Filterkriterien generiert.
  • Die Erstellung erfolgt als Word oder PDF - siehe Feld "Export Format" in UC 216.004 Individueller Report verwalten)
  • Siehe Schritt 2 + 3

Schritt 3 – Actionpanel liefert Dokumente zum Download

Controls

Actionpanel

  • Erstellungsmeldung mit Download Link des erstellten Dokuments

Regeln

Generell

  • Die aktuellste Meldung wird als erstes angezeigt (gilt immer für das Actionpanel)
  • Pro Dokument wird eine Meldung erstellt

Meldung

  • Text: Das Dokument xy wurde erstellt und kann über den Link heruntergeladen werden.

Link

  • Die Ausgabe des Links erfolgt in der Sprache des Benutzers (Bezeichnung)
