Use Cases : 201.030 Massenmutation auf Buchung

Ziel & Beschreibung

Zwei oder mehrere Buchungen können in den gleichen Prozessschritten bearbeitet werden

Rechte

  • DarfMassenmutationenVornehmen

Bedingungen

  • Es bestehen Reservationen
  • Benutzer befindet sich im Plan, Kalender oder unter Listen
  • NICHT verfügbar bei VC Events

Hauptszenario in Schritten

  1. Nutzer selektiert zwei oder mehr Reservationen
  2. Nutzer klickt Button „Bearbeiten"
  3. Eingaben im Fenster für Massenmutation

Beziehungen zu UC

 

 


Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Use Case Details

Schritt 1 - Nutzer selektiert zwei oder mehr Reservationen

Controls

  • Listenansicht: Benutzer hat bei zwei oder mehreren Reservationen die Checkbox in der Smartlist selektiert
  • Planansicht: Benutzer selektiert mittels Mauslkick mehrere Hotspots. Wird ein Hotspot selektiert, bleibt es eingeblendet; ausserdem wird die Transparenz der Polygon-Füllung entfernt, d.h. der Polygon wird hervorgehoben.
  • Kalender: Benutzer hat bei zwei oder mehreren Reservataionen die Checkbox selektiert. Box erscheint beim Mouse-Over, sofern folgendes Recht vorhanden ist: DarfMassenmutationenVornehmen
  • Event: Benutzer hat eine oder mehrere Buchungen auf dem Event selektiert und den Button "Bearbeiten" geklickt ODER Benutzer hat eine oder mehrere bestehende Buchungen zu Event hinzugefügt.

Regeln

Rechte müssen implementiert werden:

  • DarfMassenmutationenVornehmen

 

Schritt 2 - Nutzer klickt Button „Bearbeiten“

Controls

Inhaltsbereich

  • Button „Bearbeiten“

Regeln

Bearbeiten

  • erscheint nur mit dem Recht "DarfMassenmutationenVornehmen"
  • Button wird im Buttonbereich des Inhaltsbereich angezeigt (Höhe "auto-refresh" icon)
  • Klick auf Button prüft für jede selektierte Reservation, ob der Benutzer Editier-Rechte besitzt (schliesst auch Prüfung auf Vergangenheit usw. mit ein -> RechtRules) und ob der Eintrag noch nicht gelockt ist:
    • Trifft dies auf keinen Eintrag zu, wird im ActionPanel die Meldung "Sie können zur Zeit keine der selektierte(n) Reservation(en) bearbeiten." ausgegeben; der Benutzer bleibt auf der aktuellen Ansicht, die Checkboxen sind wieder deaktiviert.
    • Trifft dies auf genau einen Eintrag zu, wird er auf die Detailansicht dieser Reservation geleitet (ohne zusätzliche Meldung); wurden mehrere Einträge selektiert, steht im ActionPanel "Sie bearbeiten 1 Reservation; die andere(n) X Reservation(en) können zur Zeit von Ihnen nicht bearbeitet werden.".
    • Hat der Benutzer für mehr als eine Reservation die benötigten Rechte, wird er auf die neue Massenmutationsmaske weitergeleitet; dort wird im ActionPanel immer "Sie bearbeiten X Reservationen." ausgegeben; bei Bedarf steht zusätzlich "Sie bearbeiten X Reservationen; die andere(n) Reservation(en) können zur Zeit von Ihnen nicht bearbeitet werden." Gleichzeitig wird analog bei einer Einzelmutation ein Lock für jede zu bearbeitende Reservation vergeben; es wird aber kein Blocker erstellt.

Schritt 3 - Eingaben im Fenster für Massenmutation

Controls

Inhaltsbereich

  • Checkbox UND Datumpicker: Datum/Zeit
  • Checkbox UND Intelibox: Organisator
  • Checkbox UND Intelibox: Verantwortlicher
  • Checkbox UND Feld: Titel
  • Button „auf alle Übernehmen“
  • Button „Abbrechen"

Regeln

Checkboxen

  • Aktiviert ein Feld für eine Eingabe
  • Wird das Feld nicht aktiviert, bleiben die Werte der einzelnen Reservationen erhalten
  • Wird das Feld aktiviert, wird der neue Wert auf alle Reservationen übernommen

Datum/Zeit

  • Datumpicker (siehe auch 000.001 Globale Funktionen)
  • Standardwert bei Selektion: aktuelles Datum und aktuelle Zeit (gemäss Voreinstellung Schnellreservation im Profil - siehe UC 201.201 Schnellreservation)

Titel

  • Eingabefeld
  • Standardwert generell: leer
  • Standardwert Event: Titel des Events wird ausgegraut angezeigt
  • Feldvalidierung: Pflichtfeld

Bemerkungen

  • Eingabefeld
  • Standardwert generell: leer
  • Standardwert Event: Bemerkungen des Events werden ausgegraut angezeigt
  • Feldvalidierung: keine

Organisator

  • Intelibox Person (siehe auch 000.001 Globale Funktionen)
  • Standardwert generell: leer
  • Standardwert Event: Organisator des Events wird ausgegraut angezeigt
  • Feldvalidierung: Pflichtfeld und gültige Person

Verantwortlicher

  • Intelibox Person (siehe auch 000.001 Globale Funktionen)
  • Standardwert generell: leer
  • Standardwert Event: Verantwortlicher des Events wird ausgegraut angezeigt
  • Feldvalidierung: Pflichtfeld und gültige Person

Kostenträger

  • Intelbox Kostenträger (siehe auch 201.006 Kostenstellen bei Reservation zufügen und entfernen
  • Standardwert generell: leer
  • Standardwert Event: Kostenträger des Events werden ausgegraut angezeigt
  • Feldvalidierung: gültiger Kostenträger

Klassifikationen

  • Alle Klassifikationstpyen (siehe auch 104.002 Buchungsklassifikationen verwalten
  • Standardwerte generell: leer
  • Standardwert Event: Klassifikationen des Events werden pro Typ ausgegraut angezeigt
  • Standardwert Massenbuchung: Es werden alle Klassifikationen angezeigt, die alle involvierten Buchungen zusammen aufweisen könnten (also auch die nicht gesetzten)
  • Feldvalidierung: gültiger Eintrag aus Liste oder Multiselektion

auf alle Übernehmen

  • Zurück auf die vorherige Ansicht
  • Ausgabe im Actionpanel: "Die Änderungen werden nun übernommen. Sie werden  wieder informiert, sobald der Vorgang abgeschlossen ist."
  • Aktionseinträge für Bearbeitung werden in Action Tabelle geschrieben
  • Asynchrone Abarbeitung (durch den Service) des Auftrags, ausser bei Event-Massenmutationen
  • Ausgabe im Actionpanel: "Die Änderungen von X  Einträgen vom Typ Reservation sind abgeschlossen (a / b / c wurden  angepasst; d / e / f konnten nicht angepasst werden)."

Abbrechen

  • Abbrechen entfernt die Locks
  • Zurück auf die vorherige Ansicht
