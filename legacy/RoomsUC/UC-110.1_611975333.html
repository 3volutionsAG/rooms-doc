Wichtige Anmerkungen

Jede Liste (Grid) beinhaltet immer das Personalisieren von Spalten, Sortierung und Paging sowie optionale Checkboxen für Mehrfachselektion.

Jede Detailansicht beinhaltet immer einen Readonly-Modus, einen Editier-Modus für Einzelmutationen und einen Editier-Modus für Mehrfachmutationen.

Diese Funktionalitäten müssen deshalb beim Beschreiben der Use Cases nicht aufgeführt werden.

Zudem müssen Validierungsregeln zum aktuellen Zeitpunkt nicht beschrieben werden.

Grobbeschrieb

Bitte hier den Use Case im groben Beschreiben – auch Abhängigkeiten zu anderen Use Cases sind zu erwähnen:

Das Infoterminal zeigt aktuelle Reservationen eines oder mehrerer Räume über einen bestimmten Zeitraum:

  • Anzeige der Reservationen in tabellarischer Form
  • Kopfbereich zeigt die aktuelle Zone, Zeitraum und Logo (variabel)
  • Die angezeigten Spalten sind variabel
  • Die Anzeige kann farblich (und Logomässig) verschieden sein
  • Die Anzeige erfolgt in einer bestimmten Sprache
  • Die Anzeige muss immer aktuell sein
  • Die Anzeige muss innerhalb 0.5 Sekunden aufgebaut sein und darf nicht wahrgenommen werden (kein Flackern)
  • Vergangene Reservationen werden nicht angezeigt nur künftige.
  • Die Anzeige von Newsmeldungen in Form eines Tickers werden unterstützt. Der Ticker wird am unteren Rand der Terminalanzeige visualisiert.

Innerhalb der Administrationsoberfläche ergibt sich für dieses Terminal folgende Spezialität:

  • Aus einem Set an Spalten kann selektiert werden, welche am Terminal angezeigt werden
  • Pro Terminal stehen verschiedene Skins zur Verfügung (Logo und Farbänderung)
  • Der gewünschte Anzeige-Zeitrange kann pro Terminalgruppierung definiert werden.
  • Die Anzeigesprache kann pro Terminalgruppierung gewählt werden.
  • Pro Terminalgruppierung können ein oder mehrere Newsticker konfiguriert werden. Jeder Ticker hat eine eigene Laufzeit und einen eigenen Text.
  • Ein Terminal kann in mehreren Terminalgruppierungen vorkommen.

Subcases und Features

Bitte auflisten und im groben beschreiben:

  • Die Installation eines Terminals entfällt oder muss sehr einfach sein (z.b. Drag & Drop)
  • Die Konfiguration des Terminals sollte global vorgenommen werden können
  • Die Kommunikation der Terminals erfolgt über standardisierte Protokolle (TCP/IP, http, etc.)
