Use Cases : 230.002 globale und öffentliche Listen verwalten

Ziel & Beschreibung

Eine "public" Liste "global" machen oder eine "globale" Liste "public" machen.
Entfernen von "globalen" Listen

Unterschied zwischen "globalen" und "public" Listen:

  • public Liste gehört einer Person. Diese Person ist der Owner und kann die Liste jederzeit verändern. Jeder Benutzer kann über das Abonnementen System selber entscheiden ober er eine "public" Liste nutzen will oder nicht.
  • globale Listen haben keine Owner. Diese können nur innerhalb von "Globale und Öffentliche Listen" bearbeitet werden. "Globale" Listen sind für alle Benutzer MUSS-Listen und erscheinen in den Dropdowns im Bereich der Listenansicht

Rechte

  • Darf globale Filter editieren

Bedingungen

Liste wurde bereits erstellt und hat den Status "global" oder "public"

Weg um eine Liste zu erstellen:

  1. Listenansicht eine Liste nach Wunsch zusammenstellen (inkl. Filterkrierien) - siehe UC 230.001 Liste erstellen und (als Favorit) speichern
  2. Liste unter beliebigem Namen abspeichern
  3. Liste im Profil als "Public" publizieren - siehe UC 230.003 Eigene Listen verwalten)

Hauptszenario in Schritten

  1. Nutzer navigiert auf >Einstellungen >Stammdaten >Globale und Öffentliche Listen
  2. Nutzer gibt Suchkriterien ein (optional) und klickt Button „Finden“
  3. Ein Suchresultat über "public" und "globalen Listen" wird angezeigt – ein oder mehrere Listen können gelöscht werden
  4. Nutzer kann einen Eintrag anklicken, um die Details anzuzeigen (View Modus)
  5. Nutzer editiert eine Liste - setzt sie auf "global" oder "public"

Schritt 3 - Ein Suchresultat über "public" und "globalen Listen" wird angezeigt – ein oder mehrere Listen können gelöscht werden

Controls

Smarlist

  • Bezeichnung
  • Listenart
  • Besitzer
  • Abos

Button Bereich

  • Button "Löschen"

Regeln

Bezeichnung

  • Bezeichnung der Liste - wird durch den Ersteller festgelegt (in 220.001 Reservationen in Liste anzeigen)
  • Hat einen Link auf Viewmode (Detail)
  • Mehrsprachig, wird in der Sprache des angemeldeten Benutzers angezeigt

Listenart

  • Es wird unterschieden zwischen folgenden Arten:
    • public => Liste wurde durch einen Besitzer zur Publikation publiziert
    • global => Liste wurde durch einen Dritten als public publiziert und in diesem Bereich der Ersteller entfernt (siehe Schritt 5)

Besitzer

  • Eigentümer der Liste
  • Vorname und Nachname werden dargestellt
  • Bei globaler Liste ist der Eigentümer leer (Identifikationsmerkmal einer globalen Liste)

Abos

  • Ganzzahl
  • zeigt wieviele Personen eine Public Liste im Abonmenet haben

Löschen

  • Mehrfachselektion möglich
  • Sicherheitshinweis: Sollen die selektierten Listen wirklich gelöscht werden?
  • Listen werden gelöscht und stehen den Benutzer in den Listenansichten (siehe UC 220.001 Reservationen in Liste anzeigen) nicht mehr zur Verfügung
  • Aktualisierung der Smartliste
  • Bestätigungsmeldung im Actionpanel, inkl. Ausgabe der Anzahl der entfernten Listen

Schritt 5 - Nutzer editiert eine Liste - setzt sie auf "global" oder "public

Controls

Inhaltsbereich

  • Feld: Bezeichnung
  • Intelibox: Besitzer
  • Button „Speichern“
  • Button „Abbrechen

Regeln

Bezeichnung

  • Mehrsprachige Erfassung möglich

Besitzer

  • Personen Intelibox (siehe 000.001 Globale Funktionen)
  • ist eine Person erfasst, handelt es sich um eine "Public" Liste der erfassten Person
  • Ist Keine Person erfasst, handelt es sich um eine "Globale" Liste
  • Validierung auf eine Person aus ROOMS

Speichern

  • Speichert die Änderung
  • Rücksprung zum Ursprungspunkt (Liste oder View Mode)
  • Ausgabe einer Bestättigung im Actionpanel
