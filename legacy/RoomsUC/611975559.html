Use Cases : 230.001 Liste erstellen und (als Favorit) speichern

Created by Former user (Deleted), last modified on Mar 29, 2012

Use Case Überblick


ID

217.001

Owner

Tim Bänziger

Rev-Number

2.0

Priorität

1

Ziel & Beschreibung

  • Reservationen können gesucht und gefunden werden. Darstellung erfolgt im Listenformat

Rechte

Bedingungen

  • Benutzer befindet sich unter/im
    • Outlook Plugin
    • Reservieren
    • Plan
    • Kalender
    • Listen - Reservationen
    • Listen - Anlässe
    • Listen - Personen
    • Listen - Ressourcen
    • Listen - Teilnehmer
    • Listen - Catering & Services
  • Benutzer hat vorgängig seine Kriterien erfasst und die Suche gestartet (Klick auf "Finden")

Hauptszenario in Schritten

  1. Nutzer klickt "Speichern Icon" von "Gespeicherte Listen"
  2. Speichern-Popup öffnet sich - Benutzer klickt speichern
  3. Benutzer wählt gewünschte Liste in Dropdown

Beziehungen zu UC


Legende

  • Rote Schrift = MUSS Felder
  •  

Schritt 1 - Nutzer klickt "Speichern Icon" von "Gespeicherte Listen"

Controls

Sidepanel

  • Dropdown: Gespeicherte Liste
  • Icon: Speichern (Disketten Symbol)

Regeln

Speichern

  • Speichern Popup öffnet sich
  • siehe Schritt 2

Bilder

Schritt 2 - Speichern-Popup öffnet sich - Benutzer klickt speichern

Controls

Sidepanel

  • Feld: Bezeichnung
  • Button: Speichern
  • Checkbox: Ist Favorit

Regeln

Bezeichung

  • freie Texteingabe für die sprechende Bezeichnung

Speichern

  • Speichert die Konfiguration (Filtereinstellungen und Suchresultat (Spalten))
  • Datum wird bei Outlook, Reservieren, Plan und Kalender NICHT gespeichert
  • Popup schliesst sich
  • Dropdown wird aktualisiert

Ist Favorit

  • Checkbox kann selektiert oder deselektiert werden
  • Liste mit selektierter Checkbox wird beim Einstieg in den entsprechenden Menüpunkt Initial geladen (Favoritenliste, Favorit-Kalender oder Favoritplan) - siehe auch Als Favorit speichern
  • Ist die Checkbox bereits auf einer anderen Liste selektiert, wird diese dort entfernt

Bilder

Schritt 3 - Benutzer wählt gewünschte Liste in Dropdown

Controls

Sidepanel

  • Dropdown: Gespeicherte Liste

Regeln

Gespeicherte Liste

  • jede gespeicherte Liste hat pro Ansicht ein eigenes Label - kann individuell übersetzt werden
  • Die Dropdown beinhaltet Werte aus folgenden Bereichen und sind entsprechend gekennzeichnet
    • private => eigene erstellte
    • public = Erstellung durch einen Dritten und wurde abonniert
    • global = Erstellung durch einen Dritten als MUSS-Liste
  • nach Wechsel eines Dropdown Eintrages werden automatisch Sidepanel und Inhaltsbereich gemäss gespeicherter Konfiguration geladen
