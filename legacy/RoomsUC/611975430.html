Ziel & Beschreibung

Ressource „Raum“ kann als teilbarer Raum konfiguriert werden. Use Case 105.001 Ressourcen verwalten wurde in den Schritten 1, 2, 3 und 5 ausgeführt.
Notwendige Rechte aus UC 105.001 sind vorhanden
Das Tab virtuelle Teilräume erscheint an 6. Position

Datenrechte

–

Funktionsrechte

–

Bedingungen

  • Es handelt sich um die Ressourcenart „Raum“

Hauptszenario in Schritten

  1. Benutzer klickt auf Tab „virtuelle Räume“
  2. Liste mit Teilräume erscheint – ein oder mehrere Teilräume können gelöscht werden
  3. Administrator kann durch Button Hinzufügen einen oder mehrere Teilräume hinzufügen
  4. Ein oder mehrere Teilräume können hinzugefügt werden.

Use Case Details

Schritt 1 - Benutzer klickt auf Tab „virtuelle Räume

Controls

Sidepanel

Keine Veränderung

Regeln

Tab nur verfügbar, wenn Ressourcenart „Raum“

Bilder

 

Schritt 2 - Liste mit zugewiesenen Teilräumen Teilräumen werden angezeigt – ein oder mehrere Teilräume können gelöscht werden

Controls

Liste zugewiesene Teilräume

  • Checkbox (zur Selektion)
  • Name
  • Bezeichnung
  • Ressourcenart
  • Aktiv/inaktiv
  • Button „Virtuellen Raum Hinzufügen“
  • Button „Entfernen“

Regeln

Alle zugewiesenen Teilräume werden angezeigt.

Smartlist
Jede Spalte ein Suchfeld für eine Wildcard suche (siehe 000.001 Globale Funktionen), falls nicht ausgeblendet

Bezeichnung

wird in der Sprache des angmeldeten Benutzers ausgegeben

Entfernen

  1. Mehrfachselektion ist erlaubt.
  2. Button „Entfernen  klicken
  3. Warnhinweis
  4. Fehler oder Bestätigungsmeldung im Actionpanel
  5. Wird ein Teilraum entfernt hat dies keinen Einfluss auf die Reservationen – diese bleiben bestehen – das heisst es wird nur die Verbindung zwischen Teilraum und Hauptraum gelöscht

Virtuellen Raum Hinzufügen
Button Hinzufügen öffnet Modales Fenster (Schritt 3)

Bilder

 

Schritt 3 - Ein oder mehrere Teilräume können hinzugefügt werden.

Controls

Modaler Dialog mit Smartlist

Standard Spalten

  • Checkbox zur Mehrfachselektion
  • Name (ID)
  • Bezeichnung (in der Sprache des angemeldeten Benutzers)
  • Standort

Optionale Spalten

  • Ressourcenart
  • Aktiv/Inaktiv

Button Bereich

  • Button „Hinzufügen“
  • Button „Abbrechen“

Regeln

Smartliste

  1. Verfügbare Teilräume werden nur aufgelistet, wenn diese dem Standort des aktuellen Raumes entsprechen
  2. Verfügbare Teilräume werden nur aufgelistet, wenn diese selber noch keine Teilräume (keine Childs) beinhalten.
  3. Verfügbare Teilräume erscheinen nur, wenn diese noch keine Reservationen in der Zukunft haben.
  4. Der aktuelle Raum erscheint auch nicht unter den verfügbaren Teilräumen

Button Abbrechen
Wechselt in die Liste der zugefügten Teilräume (modales Popup wird geschlossen)

Hinzufügen:

  1. Mehrfachselektion ist erlaubt.
  2. Das modale Popup schliesst automatisch
  3. Das Actionpanel bestätigt die Menge der zugefügten Teilräume
  4. Prüfung: Wird ein Teil-Raum hinzugefügt, müssen bei sämtlichen bestehenden Reservationen in der Zukunft die Reservation des Hauptraumes (Parent) auf das Child (hinzugefügter Teilraum) übertragen werden)

Bilder

Controls

Siehe Schritt 1

Regeln

Der Detailbereich wird komplett neu geladen mit den Informationen des Teilraums. Fokus hat Tab „Stammdaten“
