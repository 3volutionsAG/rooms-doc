Use Cases : 201.032 Massenfreigabe & Massenterminierung

Created by Former user (Deleted), last modified on May 09, 2012

Ziel & Beschreibung

Zwei oder mehrere Buchungen können im gleichen Prozessschritt Freigegeben (sofortige Beendigung ohne Nachlaufzeit) oder Terminiert (sofortige Beendigung, Nachlaufzeit bleibt bestehen) werden

Rechte

  • DarfMassenmutationenVornehmen
  • DarfReservationenTerminierenMitDatum
  • DarfReservationenFreigebenMitDatum

Bedingungen

  • Es bestehen Reservationen
  • Benutzer befindet sich im Plan, Kalender oder unter Listen

Hauptszenario in Schritten

  1. Nutzer selektiert zwei oder mehr Reservationen
  2. Nutzer klickt Button "Freigabe“ oder "Terminieren"

Beziehungen zu UC




Legende

  • Rote Schrift = MUSS Felder
  • Grüne Schrift = wird in einem Folgerelease umgesetzt

Use Case Details

Schritt 1 - Nutzer selektiert zwei oder mehr Reservationen

Controls

  • Listenansicht: Benutzer hat bei zwei oder mehreren Reservationen die Checkbox in der Smartlist selektiert
  • Planansicht: Benutzer selektiert mittels Mauslkick mehrere Hotspots. Wird ein Hotspot selektiert, bleibt es eingeblendet; ausserdem wird die Transparenz der Polygon-Füllung entfernt, d.h. der Polygon wird hervorgehoben.
  • Kalender: Benutzer hat bei zwei oder mehreren Reservataionen die Checkbox selektiert. Box erscheint beim Mouse-Over, sofern folgendes Recht vorhanden ist: DarfMassenmutationenVornehmen

Regeln

Rechte müssen implementiert werden:

  • DarfMassenmutationenVornehmen

Bilder

Schritt 2 - Nutzer klickt Button "Freigabe“ oder "Terminieren"

Controls

Inhaltsbereich

  • Button „Freigabe“
  • Button "Terminieren"

Regeln

Freigabe

  • erscheint nur mit dem Recht "DarfMassenmutationenVornehmen"
  • Button wird im Buttonbereich des Inhaltsbereich angezeigt (Höhe "auto-refresh" Icon)
  • Benutzer benötigt auf einem beliebigen Standort  über ein Freigaberecht (egal ob eigene/fremde/private/öffentliche)

Ablauf:

  1. Klick auf Button prüft für jede selektierte Reservation ob der Eintrag noch nicht gelockt ist:
    • Trifft dies auf keinen Eintrag zu, wird im ActionPanel die Meldung "Sie können zur Zeit keine der selektierte(n) Reservation(en) freigeben"; der Benutzer bleibt auf der aktuellen Ansicht, die Checkboxen sind wieder deaktiviert.
    • Trifft dies auf genau einen Eintrag zu, wird der Einzel-Freigabeprozess gestartet (ohne zusätzliche Meldung); wurden mehrere Einträge selektiert, wird vorgängig ein Popup mit der Meldung "Sie geben 1 Reservation frei; die andere(n) X Reservation(en) können zur Zeit von Ihnen nicht freigegeben werden.".
  2. Hat der Benutzer das Recht "DarfReservationenFreigebenMitDatum" erscheint ein Eingabedialog mit Datum/Zeitpicker und Button "Freigeben"
    • Standardwert: Datum = heute(), Zeit: = jetzt() - was einer sofortigen Freigabe entspricht
    • Nach Klick auf den Button "Freigabe" folgt Schritt 3
  3. Hat der Benutzer für mehr als eine Reservation die benötigten Rechte, wird ein Ja/Nein-Dialog angezeigt. Die Frage lautet:
    • Wenn alle selektierten Reservationen freigegeben werden können: "Wollen Sie die selektierten X Reservationen wirklich freigeben?"
    • Wenn nicht alle selektierten Reservationen freigegeben werden können: "Sie können zur Zeit Y der selektierten Reservationen nicht freigeben. Wollen Sie die selektierten X Reservationen wirklich freigeben?"
  4. Klickt der Benutzer Nein, passiert nichts (Checkboxen bleiben selektiert).
  5. Klickt der Benutzer Ja, wird die Seite neu geladen (Checkboxen wieder deselektiert)
  6. die selektierten Reservationen werden gelockt und ein neuer Aktionseintrag mit allen relevanten Angaben für die Freigaben wird erstellt
  7. Ausgabe im ActionPanel: "Die Änderungen werden nun übernommen. Sie werden wieder informiert, sobald der Vorgang abgeschlossen ist."
  8. Asynchrone Abarbeitung (durch den Service) des Auftrags
  9. Ausgabe im Actionpanel: "Die Änderungen von X Einträgen vom Typ Reservation sind abgeschlossen (a / b / c wurden angepasst; d / e / f konnten nicht angepasst werden)."

Terminieren

  • erscheint nur mit dem Recht "DarfMassenmutationenVornehmen"
  • Button wird im Buttonbereich des Inhaltsbereich angezeigt (Höhe "auto-refresh" Icon)
  • Benutzer benötigt auf einem beliebigen Standort  über ein Terminierenrecht (egal ob eigene/fremde/private/öffentliche)

Ablauf:

  1. Klick auf Button prüft für jede selektierte Reservation ob der Eintrag noch nicht gelockt ist:
    • Trifft dies auf keinen Eintrag zu, wird im ActionPanel die Meldung "Sie können zur Zeit keine der selektierte(n) Reservation(en) terminieren."; der Benutzer bleibt auf der aktuellen Ansicht, die Checkboxen sind wieder deaktiviert.
    • Trifft dies auf genau einen Eintrag zu, wird der Einzel-Terminierungsprozess gestartet (ohne zusätzliche Meldung); wurden mehrere Einträge selektiert, wird vorgängig ein Popup mit der Meldung "Sie terminieren 1 Reservation; die andere(n) X Reservation(en) können zur Zeit von Ihnen nicht terminiert werden.".
  2. Hat der Benutzer das Recht "DarfReservationenTerminierenMitDatum" erscheint ein Eingabedialog mit Datum/Zeitpicker und Button "Terninieren"
    • Standardwert: Datum = heute(), Zeit: = jetzt() - was einer sofortigen Freigabe entspricht
    • Nach Klick auf den Button "Freigabe" folgt Schritt 3
  3. Hat der Benutzer für mehr als eine Reservation die benötigten Rechte, wird ein Ja/Nein-Dialog angezeigt. Die Frage lautet:
    • Wenn alle selektierten Reservationen terminiert werden können: "Wollen Sie die selektierten X Reservationen wirklich terminieren?"
    • Wenn nicht alle selektierten Reservationen terminiert werden können: "Sie können zur Zeit Y der selektierten Reservationen nicht terminieren. Wollen Sie die selektierten X Reservationen wirklich terminieren?"
  4. Klickt der Benutzer Nein, passiert nichts (Checkboxen bleiben selektiert).
  5. Klickt der Benutzer Ja, wird die Seite neu geladen (Checkboxen wieder deselektiert)
  6. die selektierten Reservationen werden gelockt und ein neuer Aktionseintrag mit allen relevanten Angaben für die Terminierungen wird erstellt
  7. Ausgabe im ActionPanel: "Die Änderungen werden nun übernommen. Sie werden wieder informiert, sobald der Vorgang abgeschlossen ist."
  8. Asynchrone Abarbeitung (durch den Service) des Auftrags
  9. Ausgabe im Actionpanel: "Die Änderungen von X Einträgen vom Typ Reservation sind abgeschlossen (a / b / c wurden angepasst; d / e / f konnten nicht angepasst werden)."
