Use Cases : 211.003 Anlass kopieren

Ziel & Beschreibung

Ein bestehender Anlass kann kopiert werden und mit wenigen klicks auch kopien der Reservationen erstellt werden

Rechte

  • Darf Anlässe sehen
  • Darf Anlässe buchen

Bedingungen

  • Anlass muss vorhanden sein
  • UC 211.002 Anlass suchen und finden oder 211.001 Anlass erstellen wurde zuerst ausgeführt

Hauptszenario in Schritten

  1. Nutzer navigiert auf den View Mode des Anlasses und klickt Button "Kopieren"
  2. System erstellt Kopie des Anlass, welche verändert werden kann - Nutzer speichert Kopie
  3. Nutzer klickt Button "Reservationen aus Originalanlass kopieren"
  4. System zeigt alle Reservationen des Orignal Anlass - einzelne Reservation kann übernommen werden

Beziehungen zu UC

211.001 Anlass erstellen
211.002 Anlass suchen und finden

Use Case Details

Schritt 1 - Nutzer navigiert navigiert auf den View Mode des Anlasses und klickt Button "Kopieren"

Controls

Button Bereich

  • Button: Kopieren

Regeln

Rechte muss vorhanden sein

  • Darf Anlässe buchen

Kopieren

  • siehe Schritt 2

Bilder

Schritt 2 - System erstellt Kopie des Anlass, welche verändert werden kann - Nutzer speichert Kopie

Controls

Inhaltsbereich
gemäss UC 211.001 Anlass erstellen Schritte 2, 3 und 4

Button Bereich

  • Button: Speichern
  • Button: Abbrechen

Regeln

Generell

  • die Kopie enthalt die Stammdaten aus dem Originalanlass
  • die Kopie enthält die Teilnehmerdaten aus dem Originalanlass
  • die Kopie enthält die Daten Catering&Services aus dem Originalanlass
  • die Kopie enthält KEINE Reservationen aus deim Originalanlass - diese können nachträglich hinzugefügt werden (siehe Schritt 3)
  • die Kopie übernimmt kein Request Formular

Speichern

  • Validierung der Dienstleistungen
  • Speichert den neuen Anlass ab
  • Warnmeldung im Actionpanel: Reservationen aus dem Originalanlass werden nicht automatisch kopiert. Sie können diese auf der Ansichtsseite der Kopie aus dem Original übernehmen.
  • Rücksprung auf Anlassliste

Abbrechen

  • Kopie wird nicht gespeichert
  • Rücksprung auf Anlassliste

Bilder

Schritt 4 - System zeigt alle Reservationen des Orignal Anlass - einzelne Reservation kann übernommen werden

Controls

Inhaltsbereich

  • Listenview der Anlässe (siehe UC 211.002 Anlass suchen und finden)
  • Checkbox für die Selektion einer Reservation
  •  

Button Bereich

  • Button: Als Kopie hinzufügen

Regeln

Checkbox für die Selektion einer Reservation

  • es kann nur eine Checkbox selektiert werden (nur eine Reservation kann gleichzeitig bearbeitet werden)

Als Kopie hinzufügen

  • Kopie der Reservation wird erstellt
  • Kopie öffnet sich im Bearbeiten Modus
  • Datum/Zeit oder Ressource muss geändert werden, damit die Kopie gespeichert werden kann (siehe auch UC 201.016 Reservation kopieren)
  • Kopie wird gespeichert
  • Verknüpfung zu neuem Anlass wurde erstellt
  • Rücksprung auf Anlass
  • Dieser Prozess kann beliebig wiederholt werden
